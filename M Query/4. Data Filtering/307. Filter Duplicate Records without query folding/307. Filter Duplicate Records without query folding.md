# Filter Duplicate Records without Query Folding

**Description:**
This transformation removes duplicate records from a table without relying on query folding. It ensures data integrity by eliminating redundant entries, which is crucial for accurate analysis and reporting. This process is particularly useful when dealing with data sources that do not support query folding.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when ingesting data from sources like log files or transactional databases where duplicate records may exist due to data import errors or system glitches. For instance, when loading customer data, duplicates might occur due to multiple submissions of the same information. Removing these duplicates ensures that analytics and reporting are based on unique, accurate records.
