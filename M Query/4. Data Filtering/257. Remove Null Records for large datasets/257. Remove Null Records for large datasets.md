# Remove Null Records for large datasets

**Description:**
This transformation removes rows with null values from large datasets, ensuring data cleanliness and efficiency in processing. It is particularly useful when handling big data to prevent unnecessary computations on empty or null records.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when ingesting data from various sources that might contain null or empty records. For instance, when loading customer data from a database, some rows might not have been populated correctly, resulting in null values. Removing these null records helps in optimizing data processing and analysis by focusing only on valid data.

### Core M Functions Used
`Table.SelectRows`

    3. **Data Transformation**: The result is a new table (`RemovedNulls`) that includes only the rows where none of the fields are null, effectively removing any records with null values.
    4. **Optimization**: By removing null records, subsequent data processing steps are optimized as they operate on a cleaned dataset, improving performance, especially with large datasets.
