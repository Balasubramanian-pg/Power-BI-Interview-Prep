# Keep Top N Rows with Error Handling and Retries

**Description:**
This transformation allows you to limit the number of rows in a table while incorporating error handling and retries, ensuring data reliability and consistency. It is particularly useful when dealing with large datasets and unreliable data sources. By implementing retries, the transformation can handle temporary errors, making it more robust. The goal is to provide a stable and efficient way to process data.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when ingesting data from an external API that has rate limits or unstable connections. For instance, if an API request fetches a large dataset but only allows a limited number of requests per hour, you might want to keep only the top N rows of data returned from the API while ensuring that any transient errors during data fetching are handled with retries.
