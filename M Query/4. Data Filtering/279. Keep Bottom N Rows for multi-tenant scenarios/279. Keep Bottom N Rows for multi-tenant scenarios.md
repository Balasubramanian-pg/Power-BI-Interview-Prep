# Keep Bottom N Rows for Multi-Tenant Scenarios

**Description:**
This transformation is used to retain a specified number of rows from the bottom of a table, which is particularly useful in multi-tenant scenarios where data is segregated by tenant and the most recent records need to be kept.

### Typical Scenario
In a multi-tenant data processing pipeline, each tenant's data is processed separately. For instance, in an e-commerce platform, the latest transactions for each tenant (e.g., store) need to be analyzed. The 'Keep Bottom N Rows' transformation ensures that only the most recent transactions (e.g., last 10 transactions) are kept for each tenant.

### Core M Functions Used
`Table.SelectRows`, `Table.Skip`, `Table.FirstN`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", 1},
        {"Tenant1", 2},
        {"Tenant1", 3},
        {"Tenant2", 4},
        {"Tenant2", 5},
        {"Tenant2", 6}
    }, {"Tenant", "Value"}),
    KeepBottomN = (SourceTable as table, n as number) =>
        let
            SortedTable = Table.Sort(SourceTable, {{"Value", Order.Descending}}),
            GroupedTable = Table.Group(SortedTable, {"Tenant"}, {{"Group", each _, type table}}),
            BottomNRows = Table.TransformColumns(GroupedTable, {{"Group", each Table.FirstN(_, n)}}),
            ExpandedTable = Table.ExpandTableColumn(BottomNRows, "Group", {"Tenant", "Value"}, {"Tenant", "Value"})
        in
            ExpandedTable,
    Result = KeepBottomN(Source, 2)
    in
        Result
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. Sort the table in descending order based on the 'Value' column.
    2. Group the sorted table by the 'Tenant' column.
    3. For each group, select the bottom N rows.
    4. Combine the results from each group into a single table.

For instance, if you have a table with tenants and their corresponding values, and you want to keep the bottom 2 rows for each tenant:
    - Sort the table by value in descending order.
    - Group by tenant.
    - For each tenant group, take the first 2 rows (which are the bottom 2 due to the initial sort).
    - The resulting table will contain the bottom 2 rows for each tenant.
