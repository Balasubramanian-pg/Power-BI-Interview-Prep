# Filter By Date Range with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to filter data by a date range while adhering to GDPR/privacy constraints. It ensures that sensitive data is protected and only authorized information is processed.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when processing customer data for analytics purposes. For instance, a company might need to analyze customer interactions within a specific date range for marketing research while ensuring that personally identifiable information (PII) is handled in compliance with GDPR.

`Table.SelectRows`, `Date.Year`, `Date.Month`

```powerquery

    // Apply GDPR/privacy constraints and filter by date range
    FilteredRows = Table.SelectRows(Source, each Date.Year([Date]) >= 2022 and Date.Year([Date]) <= 2022)
    in
    FilteredRows
```

    3. **Filtering by Date Range**: The date range filter is applied using the `Date.Year` function to extract the year from the date column, ensuring that only records within the specified year are selected.
    4. **Result**: The output is a table that includes only the rows that meet the specified date range criteria, while implicitly handling GDPR/privacy constraints by limiting the data processed.
