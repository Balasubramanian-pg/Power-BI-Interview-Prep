# Remove Null Records without Query Folding

**Description:**
This transformation removes rows with null values from a table without relying on query folding. It ensures data integrity by filtering out incomplete records.

### Typical Scenario
In a real-world data pipeline, this transformation is useful when ingesting data from a source that may contain null or incomplete records. For instance, when loading customer data from a database, you might want to exclude rows where critical fields like customer name or email are null.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {1, "John", "john@example.com"},
        {2, null, "jane@example.com"},
        {3, "Bob", null},
        {null, "Alice", "alice@example.com"}
    }, {"ID", "Name", "Email"}),
    RemovedNulls = Table.SelectRows(Source, each [ID] <> null and [Name] <> null and [Email] <> null)
    in
    RemovedNulls
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: A table with potential null values is created.
    2. **Filtering**: The Table.SelectRows function is used with a condition to filter out rows where any of the columns (ID, Name, Email) are null.
    3. **Result**: A new table is returned with only the rows where all required fields have values.
