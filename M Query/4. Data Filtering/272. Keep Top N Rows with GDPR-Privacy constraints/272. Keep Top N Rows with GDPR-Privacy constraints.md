# Keep Top N Rows with GDPR/Privacy constraints

**Description:**
This transformation is used to limit the number of rows in a table while considering GDPR and privacy constraints. It helps in data anonymization and pseudonymization by restricting the data to a certain number of rows, thereby protecting sensitive information. This is particularly useful when sharing data for analysis or testing purposes.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a data analyst needs to share a sample dataset with external parties while ensuring that the data does not reveal sensitive information about individuals. For instance, a company might want to share customer data with a third-party vendor for marketing purposes but needs to limit the number of rows to prevent identification of individual customers.
