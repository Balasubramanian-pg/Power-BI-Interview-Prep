# Keep Top N Rows without Query Folding

**Description:**
This transformation allows you to keep only the top N rows of a table without relying on query folding. It's useful when you need to limit the number of rows in a table, but the source data doesn't support query folding.

### Typical Scenario
In a real-world data pipeline, you might need to process a large dataset but only require a subset of rows for analysis or reporting. For instance, you might want to keep only the top 10 sales records from a large sales dataset.

### Core M Functions Used
`Table.SelectRows`, `List.FirstN`, `Table.FromList`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25},
        {"Mary", 31},
        {"David", 42},
        {"Jane", 28}
    }, {"Name", "Age"}),
    TopRows = List.FirstN(Table.ToRows(Source), 2),
    Result = Table.FromRows(TopRows, Table.ColumnNames(Source))
    in
    Result
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. Load the source data into a table.
    2. Convert the table into a list of rows using Table.ToRows.
    3. Use List.FirstN to select the top N rows from the list.
    4. Convert the list of rows back into a table using Table.FromRows, specifying the column names from the original table.
