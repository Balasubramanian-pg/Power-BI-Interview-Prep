# Exclude Specific Values with Query Folding Considerations

**Description:**
This transformation excludes specific values from a data set while considering query folding for efficient data processing. It is useful for filtering out unwanted data points. This is particularly helpful in data cleansing and preparation for analysis.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when a dataset contains unwanted or erroneous values that need to be excluded before further analysis or processing. For instance, in a sales dataset, you might want to exclude sales data from a specific region or product category.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Product", "Category", "Sales"},
        {"Product A", "Electronics", 100},
        {"Product B", "Fashion", 200},
        {"Product C", "Electronics", 300},
        {"Product D", "Fashion", 400}
    }),
    // Excluding 'Electronics' category
    ExcludedCategories = {"Electronics"},
    FilteredTable = Table.SelectRows(Source, each [Category] <> ExcludedCategories{0})
    in
    FilteredTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: Start with a table that contains the data you want to filter.
    2. **Define Exclusion Criteria**: Identify the specific values you want to exclude.
    3. **Apply Filter**: Use the `Table.SelectRows` function to create a new table that includes only the rows where the specified condition (not equal to the excluded value) is met.
    4. **Query Folding**: When possible, Power Query will fold the filtering operation back to the source, which can improve performance by reducing the amount of data transferred and processed.
