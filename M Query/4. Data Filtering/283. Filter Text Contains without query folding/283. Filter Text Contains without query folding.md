# Filter Text Contains without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to filter rows based on text contains without query folding. It allows for more flexible and complex filtering logic that isn't supported by query folding.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to filter data based on complex text conditions that cannot be achieved through query folding, such as filtering rows where a column contains a specific substring that is not known in advance.

### Core M Functions Used
`Table.SelectRows`, `Text.Contains`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "Doe", "johndoe@example.com"},
        {"Jane", "Doe", "janedoe@example.com"},
        {"Bob", "Smith", "bobsmith@example.com"}
    }, {"First Name", "Last Name", "Email"}),
    FilteredRows = Table.SelectRows(Source, each Text.Contains([Email], "example"))
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 1) The Source table is created with a list of rows containing first name, last name, and email. 2) The Table.SelectRows function is used to filter the rows where the email column contains the text 'example'. 3) The Text.Contains function is used to achieve the 'contains' logic, allowing for case-sensitive matching of the substring 'example' within the email addresses. 4) The resulting table, FilteredRows, contains only the rows where the email addresses contain 'example'.
