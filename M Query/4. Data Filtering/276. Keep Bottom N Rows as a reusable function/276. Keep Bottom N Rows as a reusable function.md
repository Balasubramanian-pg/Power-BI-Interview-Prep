# Keep Bottom N Rows as a reusable function

**Description:**
This transformation allows you to retain a specified number of rows from the bottom of a table. It is useful for keeping the most recent records or data points in a dataset. For instance, if you have a dataset with daily sales data and you want to analyze the recent sales trends, you can use this function to keep only the last n days of data.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when analyzing time-series data, such as stock prices or website traffic. For example, a business analyst might want to focus on the most recent sales data to identify current trends and make informed decisions. By keeping only the bottom n rows of a dataset sorted by date, the analyst can easily extract the relevant information.

The transformation works as follows:

    2. Then, the top n rows are selected from the sorted table using the 'Table.FirstN' function, effectively keeping the bottom n rows from the original unsorted table.
    3. The selected rows are returned as a new table.
    This approach allows for a flexible and reusable function that can be applied to various datasets and use cases.
