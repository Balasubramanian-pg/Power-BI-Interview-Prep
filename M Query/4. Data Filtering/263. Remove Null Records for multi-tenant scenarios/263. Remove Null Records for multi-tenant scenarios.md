# Remove Null Records for Multi-Tenant Scenarios

**Description:**
This transformation removes null or empty records from a table, which is particularly useful in multi-tenant scenarios where data from different sources may contain null values. By filtering out these records, the data becomes cleaner and more consistent, making it easier to analyze and process further. This is especially important when dealing with large datasets from multiple tenants, as null records can skew analysis and lead to incorrect insights.

### Typical Scenario
In a multi-tenant data environment, such as a SaaS application, different tenants may upload data with varying levels of completeness. When aggregating this data for analysis, null records can significantly impact the accuracy of the results. For instance, if a data pipeline is designed to analyze customer engagement metrics across multiple tenants, removing null records ensures that the analysis is based on complete and reliable data. This is particularly relevant in scenarios where data is sourced from various APIs, user inputs, or integrations, and null values are common due to inconsistencies or missing information.

`Table.SelectRows`


The transformation works as follows:
    1. **Data Source**: The process starts with a table that contains data from multiple tenants. This table may have null values in various columns.
    2. **Filtering Null Records**: The `Table.SelectRows` function is used with a condition (`each [Tenant] <> null and [Date] <> null and [Data] <> ""`) to filter out rows where any of the columns (Tenant, Date, Data) are null or empty.
    3. **Cleaned Data**: The resulting table, `RemovedNulls`, contains only the rows where all columns have valid, non-null values.
    4. **Further Processing**: This cleaned data can then be used for further analysis, reporting, or processing, ensuring that the insights derived are accurate and reliable.
