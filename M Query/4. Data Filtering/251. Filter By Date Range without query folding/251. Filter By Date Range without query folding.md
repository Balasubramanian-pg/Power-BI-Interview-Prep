# Filter By Date Range without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used for filtering data by a date range without relying on query folding. It is useful when the data source does not support query folding or when specific filtering logic needs to be applied post-data retrieval.

### Typical Scenario
In a real data pipeline, this transformation would be used when retrieving sales data for a specific quarter. For instance, a business analyst needs to analyze sales performance for Q2 2022. The data source may not support query folding for date ranges, or the analyst might need to apply custom filtering logic.

### Core M Functions Used
`Table.SelectRows`, `Date.FromText`, `Date.Year`, `Date.Month`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"2022-01-01", "John", 100},
        {"2022-06-01", "Jane", 200},
        {"2022-12-01", "John", 300}
    }, {"Date", "Name", "Amount"}),
    DateColumn = Date.FromText([Date]),
    StartDate = #"2022-04-01",
    EndDate = #"2022-06-30"
    in
    Table.SelectRows(Source, each DateColumn >= StartDate and DateColumn <= EndDate)
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: A table with sales data, including a date column, is retrieved.
    2. **Date Conversion**: The date column is converted to a proper date format using `Date.FromText`.
    3. **Define Date Range**: The start and end dates for the filter are defined.
    4. **Filtering**: `Table.SelectRows` is used with a condition to filter the table to only include rows where the date falls within the specified range.
    5. **Result**: The filtered table is returned, showing only the sales data for the specified date range.
