# Filter Duplicate Records Optimized for Incremental Refresh

**Description:**
This transformation removes duplicate records from a dataset, optimizing it for incremental refresh. It ensures data consistency and reduces data redundancy, making it ideal for datasets that are frequently updated.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with data sources that are constantly being updated, such as transactional databases or social media feeds. For instance, when fetching customer data from a CRM, duplicates may arise due to data inconsistencies or multiple entries. Removing these duplicates ensures that the data remains clean and reliable for analysis.

### Core M Functions Used
`Table.SelectRows`, `Table.Distinct`


    This approach ensures that the dataset remains clean and efficient for incremental refresh operations.
