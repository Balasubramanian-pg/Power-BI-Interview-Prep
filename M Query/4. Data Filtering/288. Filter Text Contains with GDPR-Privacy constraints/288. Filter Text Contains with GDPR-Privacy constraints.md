# Filter Text Contains with GDPR/Privacy constraints

**Description:**
This transformation filters text data based on GDPR/privacy constraints. It ensures that sensitive information is protected and only authorized data is processed. The use case demonstrates how to implement this filter using Power Query (M).

### Typical Scenario
In a real-world data pipeline, this transformation would be used when processing customer data that contains sensitive information such as personal identifiable information (PII). For instance, a company may want to filter out customer records that contain specific keywords related to sensitive information, such as 'health condition' or 'financial data', to ensure compliance with GDPR regulations.

### Core M Functions Used
`Table.SelectRows`, `Text.Contains`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "Doe", "johndoe@example.com"},
        {"Jane", "Doe", "janedoe@example.com"},
        {"Bob", "Smith", "bobsmith@example.com"}
    }, {"First Name", "Last Name", "Email"}),
    FilteredRows = Table.SelectRows(Source, each not Text.Contains([Email], "example"))
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The source data is a table with columns 'First Name', 'Last Name', and 'Email'.
    2. The Table.SelectRows function is used to filter the rows.
    3. The filter condition uses the Text.Contains function to check if the 'Email' column contains the text 'example'.
    4. The 'not' operator is used to select rows where the 'Email' column does not contain 'example'.
    5. The resulting table only includes rows that meet the filter condition.
