# Keep Bottom N Rows for Large Datasets

**Description:**
This transformation allows you to retain a specified number of rows from the bottom of a large dataset. It is particularly useful for data analysis and processing tasks where the most recent or last set of records are of interest. By keeping only the bottom N rows, you can efficiently reduce the size of your dataset for further analysis or processing.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when analyzing log data from a web server. For instance, you might want to focus on the most recent errors or access attempts. By keeping only the bottom N rows of the log data, you can quickly identify and investigate recent issues without being overwhelmed by the entire dataset.
