# Filter By Date Range with Query Folding Considerations

**Description:**
This transformation filters data by a specified date range. It is commonly used to limit data to a specific period, improving query performance and data relevance. By applying this filter, users can focus on a particular timeframe, making it easier to analyze and process data.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when loading sales data for analysis. For instance, a business might want to analyze sales performance over a specific quarter. By filtering the data to only include records within that date range, the pipeline can reduce the amount of data processed, leading to faster query execution and more focused insights.

### Core M Functions Used
`Table.SelectRows`, `Date.Year`, `Date.Month`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Date"},
        {1, "John", #date(2022, 1, 1)},
        {2, "Jane", #date(2022, 6, 1)},
        {3, "Bob", #date(2023, 1, 1)}
    }),
    FilteredRows = Table.SelectRows(Source, each Date.Year([Date]) = 2022 and Date.Month([Date]) >= 1 and Date.Month([Date]) <= 6)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: The example starts with a simple table containing an ID, Name, and Date.
    2. **Filtering**: The `Table.SelectRows` function is used to filter the table. It takes a condition where the year of the 'Date' column must be 2022, and the month of the 'Date' column must be between 1 and 6 (inclusive).
    3. **Query Folding**: When possible, Power Query can fold this filter operation back to the source, which means the filtering happens at the data source level before the data is retrieved. This can significantly improve performance, especially when dealing with large datasets.
