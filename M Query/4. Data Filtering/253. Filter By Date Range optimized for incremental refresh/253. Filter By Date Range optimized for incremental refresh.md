# Filter By Date Range optimized for incremental refresh

**Description:**
This transformation optimizes data filtering by date range, enabling efficient incremental data refresh. It allows for selecting specific date ranges, reducing data volume and enhancing query performance. This is particularly useful for large datasets where only recent data needs to be processed or analyzed.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial for applications like sales data analysis, where only recent sales data needs to be updated and analyzed. For instance, a company might want to analyze sales data from the last quarter to make informed decisions about future sales strategies. The transformation enables efficient loading and refreshing of data by filtering it based on a specific date range.

### Core M Functions Used
`Table.SelectRows`, `Date.Year`, `Date.Month`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"2022-01-01", 100},
        {"2022-02-01", 200},
        {"2022-03-01", 300}
    }, {"Date", "Value"}),
    DateColumn = "Date",
    StartDate = #"2022-01-01",
    EndDate = #"2022-02-28",
    FilteredRows = Table.SelectRows(Source, each DateColumn >= StartDate and DateColumn <= EndDate)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Define the Date Column**: Identify the column in your data table that contains the dates.
2. **Specify the Date Range**: Determine the start and end dates for the range you want to filter by.
3. **Apply the Filter**: Use the `Table.SelectRows` function in Power Query (M) to filter the table. This function takes a condition as an argument, which in this case is that the date in the specified date column must fall within the defined date range.
4. **Load Filtered Data**: The resulting table will contain only the rows where the date falls within the specified range, which can then be loaded into your data model or further processed as needed.
