# Filter Rows By Value for Multi-Tenant Scenarios

**Description:**
This transformation filters rows in a Power Query table based on specific values, which is particularly useful in multi-tenant scenarios where data needs to be segregated or analyzed by tenant.

### Typical Scenario
In a multi-tenant data warehouse, a data analyst needs to generate reports for each tenant separately. The data is stored in a single table with a column identifying the tenant. The analyst uses the 'Filter Rows By Value' transformation to create separate reports for each tenant.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "Data1"},
        {"Tenant2", "Data2"},
        {"Tenant1", "Data3"}
    }, {"Tenant", "Data"}),
    FilteredRows = Table.SelectRows(Source, each [Tenant] = "Tenant1")
in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The source table is created with a tenant identifier and some data.
    2. The Table.SelectRows function is used to filter the rows where the 'Tenant' column matches 'Tenant1'.
    3. The resulting table, 'FilteredRows', contains only the rows for 'Tenant1'.
