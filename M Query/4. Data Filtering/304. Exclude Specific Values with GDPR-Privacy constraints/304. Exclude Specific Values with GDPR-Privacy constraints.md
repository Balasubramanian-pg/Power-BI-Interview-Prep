# Exclude Specific Values with GDPR/Privacy constraints

**Description:**
This transformation excludes specific values from a dataset to adhere to GDPR/privacy constraints, ensuring sensitive information is not processed or shared. It is commonly used to remove or mask personal identifiable information (PII) from datasets.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to anonymize customer data for analytics purposes. For instance, a marketing team wants to analyze customer demographics without exposing individual identities. They would use this transformation to exclude specific values such as names, IDs, or other sensitive information from the dataset.

`Table.SelectRows`

```powerquery
    ExcludedEmails = {"john@example.com", "jane@example.com"},

```

    4. **Result**: The resulting table will contain only the rows where the email addresses are not in the excluded list, thus protecting sensitive information.
