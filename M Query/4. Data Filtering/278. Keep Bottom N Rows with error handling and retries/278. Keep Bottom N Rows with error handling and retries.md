# Keep Bottom N Rows with Error Handling and Retries

**Description:**
This transformation allows you to retain a specified number of rows from the bottom of a table, handling potential errors and implementing retries for robustness in data processing.

### Typical Scenario
In a real-world data pipeline, this transformation is useful when you need to prioritize recent data, such as analyzing the latest sales transactions or monitoring recent system logs. For instance, you might want to keep only the last 100 sales records for daily performance reviews.

### Core M Functions Used
`Table.SelectRows`, `Table.Skip`, `Table.FirstN`

### Example Execution Logic Step-By-Step
The transformation works as follows: 
    1. Load your data into a table within Power Query.
    2. Determine the number of rows (N) you wish to keep from the bottom.
    3. Use the Table.RowCount function to find out how many rows are in your table.
    4. Apply Table.Skip to your table, skipping the number of rows that is the difference between the total row count and N.
    This results in keeping only the bottom N rows of your original table.
