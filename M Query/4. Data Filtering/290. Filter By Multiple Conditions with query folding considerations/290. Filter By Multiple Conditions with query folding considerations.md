# Filter By Multiple Conditions with Query Folding Considerations

**Description:**
This transformation filters data based on multiple conditions while considering query folding for efficient data processing. It allows for precise data selection, enhancing performance by minimizing the data retrieved from the source. This approach is particularly useful when dealing with large datasets and complex filtering criteria.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when integrating data from various sources, such as combining customer data from a CRM and sales data from an e-commerce platform. For instance, you might want to analyze sales data for customers in a specific region who have made purchases above a certain amount within a given timeframe. By applying multiple filter conditions efficiently, you can significantly reduce the volume of data to process, leading to faster and more efficient data analysis.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"David", 42, "USA"},
        {"Jane", 28, "UK"}
    }, {"Name", "Age", "Country"}),
    FilteredRows = Table.SelectRows(Source, each [Age] > 30 and [Country] = "USA")
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: A table with customer data is created, including columns for Name, Age, and Country.
    2. **Applying Filters**: The Table.SelectRows function is used to filter the table. It takes two arguments: the table to filter and a condition.
    3. **Condition Logic**: The condition [Age] > 30 and [Country] = "USA" ensures that only rows where the age is greater than 30 and the country is "USA" are selected.
    4. **Query Folding**: When possible, Power Query folds the filter conditions back to the source, which means the filtering happens at the data source level rather than after all the data has been retrieved. This can significantly improve performance, especially with large datasets and when working with databases or other data sources that support query folding.
