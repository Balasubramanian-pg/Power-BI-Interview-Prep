# Keep Bottom N Rows optimized for incremental refresh

**Description:**
This transformation allows you to retain a specified number of rows from the bottom of a table, optimized for incremental refresh scenarios. It is particularly useful in data pipelines where new data is constantly being appended, and you need to maintain a fixed number of recent records.

### Typical Scenario
In a sales data pipeline, you might want to keep only the bottom 10 rows of daily sales data to track recent trends. As new daily sales data is added, the pipeline automatically removes older data, keeping only the most recent 10 records.

### Core M Functions Used
`Table.SelectRows`, `Table.Skip`, `Table.FirstN`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 1},
        {"Alice", 2},
        {"Bob", 3},
        {"Mary", 4},
        {"David", 5}
    }),
    BottomN = 3,
    KeepBottomNRows = Table.Skip(Source, Table.RowCount(Source) - BottomN)
    in
    KeepBottomNRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. Start with a table of data.
    2. Determine the number of rows (N) you want to keep from the bottom.
    3. Calculate the number of rows to skip from the top: Table.RowCount(Source) - N.
    4. Use Table.Skip to remove the calculated number of rows from the top of the table, effectively keeping the bottom N rows.
    5. The resulting table contains only the bottom N rows of the original table.
