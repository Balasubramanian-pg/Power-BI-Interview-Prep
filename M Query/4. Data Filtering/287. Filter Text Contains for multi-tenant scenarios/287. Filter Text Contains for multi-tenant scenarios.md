# Filter Text Contains for Multi-Tenant Scenarios

**Description:**
This transformation filters rows in a Power Query table based on text contains logic, useful for multi-tenant data scenarios where data needs to be filtered by specific text criteria.

### Typical Scenario
In a multi-tenant data environment, a company may have a large dataset containing customer information from various clients. The company needs to filter this data to only include rows where a specific text column (e.g., customer name or description) contains certain keywords unique to each client. This transformation enables efficient filtering of data for each tenant based on text contains conditions.

### Core M Functions Used
`Table.SelectRows`, `Text.Contains`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "john@example.com", "Client A"},
        {"Mary", "mary@example.com", "Client B"},
        {"Jane", "jane@example.com", "Client A"}
    }, {"Name", "Email", "Client"}),
    FilteredRows = Table.SelectRows(Source, each Text.Contains([Client], "Client A", [IgnoreCase = true]))
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: Start with a table containing customer data, including names, emails, and client names.
    2. **Filtering Logic**: Use the `Table.SelectRows` function to filter the table.
    3. **Text.Contains Function**: Within the `Table.SelectRows` function, use `Text.Contains` to check if the client name column contains the specific tenant's name (e.g., "Client A").
    4. **Ignore Case**: Optionally, set `IgnoreCase` to `true` to make the search case-insensitive.
    5. **Result**: The resulting table will only include rows where the client name column contains the specified text, effectively filtering the data for the specific tenant.
