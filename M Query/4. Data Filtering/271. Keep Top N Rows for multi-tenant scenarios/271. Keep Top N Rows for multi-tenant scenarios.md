# Keep Top N Rows for Multi-Tenant Scenarios

**Description:**
This transformation is used to limit the number of rows returned for each tenant in a multi-tenant data scenario. It helps in data filtering and optimization by only keeping the top N rows based on a specific criteria. This is particularly useful in managing and analyzing large datasets across multiple tenants.

### Typical Scenario
In a multi-tenant data environment, a company may have different clients (tenants) with large volumes of data. For reporting or analysis purposes, the company might want to only show the top 10 records for each client based on a certain metric, such as sales or engagement. This transformation helps in filtering the data to only include these top records for each tenant.
