# Exclude Specific Values as a reusable function

**Description:**
This transformation achieves the exclusion of specific values from a dataset, making it a reusable function in Power Query (M). It allows for the filtering out of unwanted data points, enhancing data quality and relevance. By encapsulating this logic into a function, users can easily apply it across different data sources and scenarios.

### Typical Scenario
In a real-world data pipeline, this transformation is particularly useful when dealing with datasets that contain irrelevant or erroneous entries. For instance, when analyzing customer feedback, you might want to exclude responses from a specific group of users or remove entries with nonsensical values. By creating a reusable function to exclude these specific values, data analysts can streamline their workflow and ensure consistency in data preprocessing.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"David", 42, "Mexico"}
    }, {"Name", "Age", "Country"}),
    ExcludedCountries = {"USA", "Canada"},
    FilteredTable = Table.SelectRows(Source, each not List.Contains(ExcludedCountries, [Country]))
    in
    FilteredTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Define the Source Table**: Start with a sample table that includes columns for 'Name', 'Age', and 'Country'.
    2. **Specify Values to Exclude**: Identify the specific values to exclude, in this case, countries 'USA' and 'Canada'.
    3. **Apply the Exclusion Logic**: Use the `Table.SelectRows` function in combination with `List.Contains` to filter out rows where the 'Country' column matches any of the excluded values.
    4. **Result**: The resulting table includes only the rows where the country is not in the excluded list, effectively excluding specific values from the dataset.
