# Filter Rows By Value without Query Folding

**Description:**
This transformation filters rows in a Power Query table based on specific values without relying on query folding. It is useful when the data source does not support query folding or when complex filtering logic is required. This approach ensures that filtering is applied after data retrieval, allowing for more flexible data manipulation.

### Typical Scenario
In a real-world data pipeline, this transformation is used when dealing with data sources that do not support query folding, such as certain web APIs or databases with limited query optimization capabilities. For instance, if you're fetching data from a web service that returns a large dataset and you need to filter the results based on complex conditions that cannot be pushed down to the source.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"David", 42, "USA"}
    }, {"Name", "Age", "Country"}),
    FilteredRows = Table.SelectRows(Source, each [Country] = "USA")
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: A table is created with columns 'Name', 'Age', and 'Country'.
    2. **Filtering Logic**: The `Table.SelectRows` function is used to filter the rows. It takes a table and a condition as input. The condition is specified as an `each` expression that evaluates to `true` or `false` for each row.
    3. **Execution**: In this example, the condition `[Country] = "USA"` is applied to each row. Rows where the condition is `true` are included in the new table.
    4. **Result**: The resulting table contains only the rows where the 'Country' is 'USA'.
