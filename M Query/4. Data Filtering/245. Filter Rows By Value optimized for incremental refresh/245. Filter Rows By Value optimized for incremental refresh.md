# Filter Rows By Value optimized for incremental refresh

**Description:**
This transformation filters rows in a Power Query table based on a specific value, optimized for incremental refresh. It allows for efficient data processing by only refreshing the necessary data. This is particularly useful when dealing with large datasets where a full refresh is not required.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when there is a need to regularly update a dataset based on new or changed data. For example, in a sales data pipeline, you might want to filter sales data to only include records from the previous day or week, rather than reprocessing the entire sales history.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Age"},
        {1, "John", 25},
        {2, "Jane", 30},
        {3, "Bob", 35}
    }),
    FilteredRows = Table.SelectRows(Source, each Age > 30)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The source data is defined as a table with columns 'ID', 'Name', and 'Age'.
    2. The Table.SelectRows function is used to filter the rows in the table.
    3. The 'each' keyword is used to specify a condition for each row, in this case, 'Age > 30'.
    4. The resulting table, FilteredRows, contains only the rows where the 'Age' is greater than 30.
