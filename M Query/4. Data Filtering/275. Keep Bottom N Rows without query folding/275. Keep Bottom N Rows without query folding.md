# Keep Bottom N Rows without Query Folding

**Description:**
This transformation allows you to retain a specified number of rows from the bottom of a table, which can be particularly useful when working with large datasets where query folding is not possible or desirable.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when you need to analyze the most recent data entries, such as the latest transactions or logs, without relying on the database to fold the query.

### Core M Functions Used
`Table.SelectRows`, `List.Last`, `List.Length`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 20},
        {"Mary", 30},
        {"Bob", 40},
        {"Alice", 50},
        {"Emma", 60}
    }, {"Name", "Age"}),
    BottomN = 3,
    RowCount = Table.RowCount(Source),
    IndexList = List.Range(0, RowCount),
    BottomIndexList = List.LastN(IndexList, BottomN),
    SelectedRows = Table.SelectRows(Source, each List.Contains(BottomIndexList, Table.PositionOf(Source, [_])))
    in
    SelectedRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. Load your data into a table.
2. Determine the number of rows (N) you want to keep from the bottom.
3. Generate a list of row indices.
4. Use List.LastN to get the indices of the bottom N rows.
5. Use Table.SelectRows with a custom condition to select rows based on their position matching the bottom N indices.
