# Keep Bottom N Rows with Query Folding Considerations

**Description:**
This transformation allows you to retain a specified number of rows from the bottom of a table, which is particularly useful for data analysis and reporting purposes where recent data is prioritized.

### Typical Scenario
In a real data pipeline, this transformation can be used when analyzing sales data to always focus on the most recent transactions, such as keeping the last 10 sales records for daily performance assessments.

### Core M Functions Used
`Table.SelectRows`, `Table.Skip`, `Table.FirstN`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25},
        {"Paul", 26},
        {"George", 27},
        {"Ringo", 28}
    }, {"Name", "Age"}),
    KeepBottomNRows = Table.Skip(Source, Table.RowCount(Source) - 2)
    in
    KeepBottomNRows
```

### Example Execution Logic Step-By-Step
The transformation works by first determining the total number of rows in the table. It then skips the appropriate number of rows from the top to reach the desired bottom N rows. For instance, to keep the bottom 2 rows, it calculates the total rows, subtracts 2, and skips that many rows from the top, effectively keeping the last 2 rows.
