# Select Specific Columns without Query Folding

**Description:**
This transformation allows you to select specific columns from a table without relying on query folding, which can be useful when working with data sources that do not support query folding. It enables more flexible data retrieval and processing. This is particularly useful when you need to ensure that only specific columns are retrieved from a data source.

### Typical Scenario
In a real-world data pipeline, you might need to retrieve data from a database that doesn't support query folding. For instance, when fetching data from an API or a web service that doesn't natively support column-level filtering. You can use this transformation to select only the required columns, thus optimizing data transfer and processing.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"Peter", 42, "Mexico"}
    }, {"Name", "Age", "Country"}),
    SelectedColumns = Table.SelectRows(Source, each [Name = "Name", Age = "Age"])
    // Alternatively, use Table.SelectColumns for column selection
    // SelectedColumns = Table.SelectColumns(Source, {"Name", "Age"})
in
    SelectedColumns
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: You start with a table or data source. In the example, a table named `Source` is created with columns `Name`, `Age`, and `Country`.
2. **Selecting Specific Columns**: You use the `Table.SelectRows` function to select specific columns. However, note that `Table.SelectRows` is typically used for filtering rows. For selecting columns, `Table.SelectColumns` is more appropriate.
3. **Execution**: When you execute this transformation, Power Query will retrieve the data source and then apply the selection. If query folding is not supported by the data source, the entire dataset is retrieved, and then the selection is applied in-memory.
4. **Result**: The resulting table, `SelectedColumns`, contains only the specified columns.
