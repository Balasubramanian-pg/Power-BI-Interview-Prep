# Filter By Multiple Conditions without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used for filtering data based on multiple conditions without relying on query folding. It is useful when dealing with complex filtering logic that cannot be optimized by the query folding mechanism. This transformation allows for more flexible and powerful data filtering.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to filter data based on multiple conditions that are not supported by query folding. For example, you might have a dataset of sales transactions and want to filter it to include only transactions from a specific region and above a certain sales amount.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "North"},
        {"Mary", 31, "South"},
        {"David", 42, "North"},
        {"Jane", 28, "South"}
    }, {"Name", "Age", "Region"}),
    FilteredRows = Table.SelectRows(Source, each [Region] = "North" and [Age] > 30)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The `Table.FromRows` function is used to create a sample table.
    2. The `Table.SelectRows` function is then used to filter the rows based on multiple conditions: `[Region] = "North"` and `[Age] > 30`.
    3. The `each` keyword is used to apply the filtering condition to each row in the table.
    4. The result is a new table that includes only the rows that meet both conditions.
