# Remove Null Records with Error Handling and Retries

**Description:**
This transformation removes null records from a table while incorporating error handling and retries. It ensures data quality by filtering out null values and provides robustness through error handling mechanisms. This is particularly useful in data pipelines where data sources may be unreliable or intermittent.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when ingesting data from multiple sources that may have intermittent connectivity issues or unreliable data feeds. For instance, when fetching data from an API that occasionally returns null or incomplete records, this transformation helps in cleaning the data before further processing.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25},
        {"Jane", null},
        {null, 30},
        {"Bob", 35}
    }, {"Name", "Age"}),
    RemovedNulls = Table.SelectRows(Source, each [Name] <> null and [Age] <> null),
    RetryAttempt = 3,
    ErrorHandling = (attemptCount) =>
        try
            RemovedNulls
        catch
            (e) as Error =>
                if attemptCount > 0 then
                    ErrorHandling(attemptCount - 1)
                else
                    Error.Record("Max retries exceeded", e.Message, e.StackTrace)
    RetryResult = ErrorHandling(RetryAttempt)
    in
    RetryResult
```

### Example Execution Logic Step-By-Step
This transformation works as follows:
1. **Data Source**: A table with potential null records is created.
2. **Remove Nulls**: The `Table.SelectRows` function is used to filter out rows where either the 'Name' or 'Age' is null.
3. **Error Handling and Retries**: A custom error handling function is defined to catch any errors during the transformation. If an error occurs, it retries the operation a specified number of times (RetryAttempt). If all retries fail, it logs the error.
