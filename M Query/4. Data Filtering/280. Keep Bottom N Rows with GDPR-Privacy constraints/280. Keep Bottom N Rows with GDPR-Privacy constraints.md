# Keep Bottom N Rows with GDPR/Privacy constraints

**Description:**
This transformation allows you to retain a specified number of rows from the bottom of a dataset while adhering to GDPR/privacy constraints. It is particularly useful when dealing with sensitive data and you need to limit the amount of data exposed. By keeping only the bottom N rows, you can ensure that the most recent or critical data points are preserved for analysis or reporting purposes.

### Typical Scenario
In a real-world data pipeline, this transformation could be used when analyzing customer data for a retail company. For instance, if the company wants to perform analytics on the most recent purchases while ensuring that only a limited number of records are exposed due to privacy concerns, they could use this transformation to keep only the bottom N rows of their dataset, such as the 10 most recent transactions.
