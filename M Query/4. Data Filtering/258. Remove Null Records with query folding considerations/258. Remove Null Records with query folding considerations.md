# Remove Null Records with Query Folding Considerations

**Description:**
This transformation removes rows with null values from a table, considering query folding for efficient data processing. It ensures that only complete records are included in the final output, improving data quality and reducing the risk of errors in downstream analysis.

### Typical Scenario
In a real-world data pipeline, this transformation is used when ingesting data from a source that may contain incomplete records. For instance, when loading customer data from a database, you might want to exclude any rows where critical fields like name, email, or address are null.

### Core M Functions Used
`Table.SelectRows`

```powerquery
let
    Source = Table.FromRows({
        {1, "John", "john@example.com"},
        {2, null, "jane@example.com"},
        {3, "Bob", null}
    }, {"ID", "Name", "Email"}),
    RemovedNulls = Table.SelectRows(Source, each [Name] <> null and [Email] <> null)
    in
    RemovedNulls
```

The transformation works as follows:
    1. **Data Source**: A table with potential null values is created.
    2. **Selection**: The Table.SelectRows function is used with a condition to filter out rows where 'Name' or 'Email' is null.
    3. **Result**: A new table is generated with only the rows where both 'Name' and 'Email' have values, effectively removing any records with null values in these critical fields.
