# Exclude Specific Values optimized for incremental refresh

**Description:**
This transformation excludes specific values from a dataset, optimizing it for incremental refresh. It allows for efficient data processing by filtering out unwanted data points. This is particularly useful in scenarios where data is constantly being updated.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when a data source contains a list of values that should be ignored, such as excluding discontinued products from sales data. For instance, a company may want to analyze sales data but exclude any transactions related to a specific product category that is no longer active.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Product", "Category"}, 
        {"Product A", "Category 1"}, 
        {"Product B", "Category 2"}, 
        {"Product C", "Category 1"}
    }),
    ExcludedCategories = {"Category 2"},
    FilteredTable = Table.SelectRows(Source, each [Category] <> ExcludedCategories{0})
    in
    FilteredTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. The source data is defined as a table with columns 'Product' and 'Category'.
2. A list of excluded categories is specified.
3. The Table.SelectRows function is used to filter the table, selecting only the rows where the 'Category' does not match any of the excluded categories.
4. The resulting filtered table is then output, containing only the relevant data.
