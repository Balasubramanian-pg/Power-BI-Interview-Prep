# Select Specific Columns for Multi-Tenant Scenarios

**Description:**
This transformation allows for selecting specific columns from a table, which is particularly useful in multi-tenant scenarios where different tenants may require access to different sets of data. By filtering the columns, data can be efficiently secured and exposed according to tenant-specific requirements. This transformation enhances data security and access control.

### Typical Scenario
In a multi-tenant data environment, a company may have different clients who need access to different sets of data. For instance, a SaaS provider may have multiple clients, each requiring access to specific data fields that are relevant to their business. The transformation can be used to dynamically select and expose only the required columns to each tenant, ensuring data privacy and compliance with access controls.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "Data1", "Data2"},
        {"Tenant2", "Data3", "Data4"}
    }, {"Tenant", "Column1", "Column2"}),
    SelectedColumns = Table.SelectRows(Source, each {
        "Tenant" = [Tenant],
        "Column1" = [Column1]
    })
in
    SelectedColumns
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Source Data**: Start with a table that contains data for multiple tenants. This table has columns for tenant identification and various data fields.
2. **Table.SelectRows**: Use the `Table.SelectRows` function to create a new table that includes only the specified columns. This function takes a table and a condition as arguments. The condition specifies which columns to include in the new table.
3. **Dynamic Selection**: For each row in the source table, the function dynamically selects the specified columns (e.g., "Tenant" and "Column1").
4. **Result**: The resulting table contains only the selected columns, effectively filtering out sensitive or irrelevant data for each tenant.
5. **Multi-Tenant Support**: This transformation can be easily adapted for different tenants by modifying the column selection criteria based on tenant-specific requirements.
