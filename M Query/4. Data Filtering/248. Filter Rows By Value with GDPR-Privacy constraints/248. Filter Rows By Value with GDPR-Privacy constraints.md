# Filter Rows By Value with GDPR/Privacy constraints

**Description:**
This transformation filters rows in a Power Query table based on specific values while adhering to GDPR/privacy constraints. It ensures that sensitive data is protected and only authorized information is processed. This is particularly useful when handling large datasets with varying levels of sensitivity.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with customer data that is subject to GDPR regulations. For instance, a company may need to filter out personal identifiable information (PII) such as names, addresses, or phone numbers from a dataset before sharing it with third-party vendors for analytics purposes.
