# Self-documenting Query Generator optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to generate self-documenting queries optimized for incremental refresh. It allows for efficient data loading and updating by leveraging Power Query's capabilities. The goal is to provide a clear and maintainable query structure.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to regularly update a dataset with new information while ensuring that the query remains understandable and easy to maintain. For example, in a daily sales report, you might want to fetch only the new sales data from the previous day, rather than reloading the entire dataset.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`, `List.DifferenceDates`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Date", "Sales"},
        {"2022-01-01", 100},
        {"2022-01-02", 200}
    }),
    PreviousDate = Date.AddDays(Date(), -1),
    NewData = Table.SelectRows(Source, each [Date] >= PreviousDate)
    in
    NewData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Table**: Start with a table that contains a list of data, including dates and corresponding values (e.g., sales figures).
    2. **Determine Previous Date**: Calculate the date from which new data should be fetched (e.g., yesterday's date).
    3. **Filter New Data**: Use `Table.SelectRows` to filter the source table and select only the rows where the date is on or after the calculated previous date.
    4. **Output**: The resulting table contains only the new or updated data, which can then be used for incremental refresh.
