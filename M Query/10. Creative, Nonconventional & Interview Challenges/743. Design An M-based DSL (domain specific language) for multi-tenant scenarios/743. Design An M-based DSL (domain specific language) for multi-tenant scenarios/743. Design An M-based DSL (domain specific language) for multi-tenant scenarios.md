# Design An M-based DSL for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to design a domain-specific language (DSL) for multi-tenant scenarios, enabling flexible and dynamic data transformations across multiple tenants.

### Typical Scenario
In a real data pipeline, this transformation would be used when integrating data from multiple tenants, each with their own schema and data structure, into a unified data model. For instance, a data platform serving multiple clients, each with their own database schema, would use this approach to dynamically map and transform data for each tenant.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformColumns`, `Text.Combine`

### Code Source Execution
```powerquery
let
    // Define a sample table
    Source = Table.FromList({
        {"Tenant1", "Data1"},
        {"Tenant2", "Data2"},
        {"Tenant3", "Data3"}
    }, Splitter.SplitByNothing(), {"Tenant", "Data"}),
    // Define a dynamic selector for tenant-based data transformation
    DynamicSelector = (tenant as text) =>
        let
            // Apply transformation based on tenant
            TransformedData = if tenant = "Tenant1" then
                Table.TransformColumns(Source, {{"Data", each Text.Upper(_)}})
            else if tenant = "Tenant2" then
                Table.TransformColumns(Source, {{"Data", each Text.Lower(_)}})
            else
                Source
        in
            TransformedData,
    // Apply transformation for a specific tenant
    Result = DynamicSelector("Tenant1")
in
    Result
```

### Example Execution Logic Step-By-Step
The M-based DSL for multi-tenant scenarios works as follows:
    1. **Define a base table**: The example starts with a simple table containing tenant data.
    2. **Create a dynamic selector function**: The `DynamicSelector` function takes a tenant identifier as input and applies transformations based on that tenant. This function uses M's conditional logic and transformation functions like `Table.TransformColumns`.
    3. **Apply transformations**: Depending on the tenant specified, the function applies different transformations to the data. In this example, it converts data to upper case for 'Tenant1', to lower case for 'Tenant2', and leaves it unchanged for other tenants.
    4. **Invoke the dynamic selector**: Finally, the transformation is applied for a specific tenant by invoking `DynamicSelector` with the desired tenant identifier.
