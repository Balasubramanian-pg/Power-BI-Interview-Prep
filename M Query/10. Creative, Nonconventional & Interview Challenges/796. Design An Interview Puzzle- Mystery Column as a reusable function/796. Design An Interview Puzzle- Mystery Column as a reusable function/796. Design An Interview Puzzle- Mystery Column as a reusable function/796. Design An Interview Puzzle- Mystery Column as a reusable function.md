# Design An Interview Puzzle: Mystery Column as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used to solve an interview puzzle by designing a mystery column as a reusable function. It allows for the transformation of data based on a specific condition or logic, making it a versatile tool in data manipulation and analysis.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to create a new column based on complex logic or conditions that are not easily met by standard data transformation tools. For instance, in a dataset containing customer information, a mystery column could be created to categorize customers based on their purchase history and demographic data.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"Peter", 42, "USA"}
    }, {"Name", "Age", "Country"}),
    AddMysteryColumn = Table.AddColumn(Source, "Mystery Column", each if [Age] > 30 then "Older" else "Younger")
    in
    AddMysteryColumn
```

### Example Execution Logic Step-By-Step
The transformation works by first creating a table from a set of rows. Then, it adds a new column called 'Mystery Column' based on the condition that if the age is greater than 30, it labels the person as 'Older', otherwise 'Younger'. This step-by-step process allows for the creation of a new column based on specific logic, making it a powerful tool for data analysis and manipulation.
