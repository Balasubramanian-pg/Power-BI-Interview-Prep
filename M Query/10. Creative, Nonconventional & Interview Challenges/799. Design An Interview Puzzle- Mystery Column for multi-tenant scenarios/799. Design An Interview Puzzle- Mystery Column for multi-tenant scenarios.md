# Design An Interview Puzzle: Mystery Column for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to solve an interview puzzle involving a mystery column in multi-tenant scenarios. It helps in identifying and isolating specific data for different tenants.

### Typical Scenario
In a multi-tenant database, there is a table that contains data for multiple clients (tenants). The table has a column that indicates which tenant each row of data belongs to. However, one of the columns (let's call it the 'mystery column') has values that are not clearly associated with any tenant. The goal is to write a Power Query transformation that can identify and separate the data for each tenant, handling the mystery column appropriately.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "Data1", "Mystery"},
        {"Tenant1", "Data2", "Mystery"},
        {"Tenant2", "Data3", "Mystery"},
        {"Tenant2", "Data4", "Mystery"}
    }, {"Tenant", "Data", "Mystery"}),
    SelectTenant1 = Table.SelectRows(Source, each [Tenant] = "Tenant1"),
    SelectTenant2 = Table.SelectRows(Source, each [Tenant] = "Tenant2")
in
    SelectTenant1
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Table Creation**: A table named 'Source' is created with sample data. This table represents the multi-tenant data with a mystery column.
    2. **Selection of Tenant Data**: Using the `Table.SelectRows` function, new tables are created for each tenant's data. For example, 'SelectTenant1' and 'SelectTenant2' are created by selecting rows from the 'Source' table where the 'Tenant' column matches 'Tenant1' or 'Tenant2', respectively.
    3. **Handling the Mystery Column**: The mystery column is included in the selection process. Depending on the requirements, further transformations can be applied to handle or resolve the mystery column's values.
    4. **Result**: The result is a set of tables, each containing data for a specific tenant, with the mystery column handled according to the tenant's context.
