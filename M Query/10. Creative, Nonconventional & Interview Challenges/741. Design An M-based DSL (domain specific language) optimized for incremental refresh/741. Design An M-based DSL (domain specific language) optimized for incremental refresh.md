# Design An M-based DSL (domain specific language) optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to design a domain-specific language (DSL) optimized for incremental refresh. The goal is to create a flexible and efficient way to handle data refreshes, minimizing the amount of data that needs to be reloaded. By leveraging M functions, users can build a customized DSL that meets their specific data pipeline requirements. This approach enables more efficient data processing and reduces the load on data sources.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when building a data warehouse or business intelligence system that requires frequent updates to data. For instance, a company might have a daily sales report that needs to be updated every hour. By using an M-based DSL optimized for incremental refresh, the data pipeline can quickly identify new or changed data and update the report accordingly, rather than reloading the entire dataset.
