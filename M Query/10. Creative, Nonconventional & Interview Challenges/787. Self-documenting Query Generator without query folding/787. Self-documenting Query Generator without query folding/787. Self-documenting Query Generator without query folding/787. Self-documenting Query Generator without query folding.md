# Self-documenting Query Generator without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used for generating self-documenting queries without relying on query folding. It allows for the creation of dynamic and understandable queries that can be easily maintained and audited. The transformation enables users to generate queries that are both flexible and transparent, making it easier to manage complex data pipelines.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to dynamically generate queries based on changing data sources or business requirements. For instance, when integrating data from multiple sources with varying structures, a self-documenting query generator can help create queries that are adaptable and provide clear documentation of the data transformation process.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Name", "Age"},
        {"John", 25},
        {"Jane", 30}
    }, {"Name", "Age"}),
    SelfDocumentingQuery = Table.SelectRows(Source, each true)
    in
    SelfDocumentingQuery
```

### Example Execution Logic Step-By-Step
The transformation typically works as follows inside a Power Query pipeline:
    1. **Define the Source Data**: Start with a table or a data source that contains the raw data.
    2. **Apply Transformation**: Use the `Table.SelectRows` function to create a new table that includes only the rows that meet a specified condition (in this case, `each true` which effectively selects all rows).
    3. **Generate Self-Documenting Query**: The query itself acts as documentation because it explicitly states what is being selected and from what source.
    4. **Dynamic Query Generation**: Depending on the requirements, parameters can be introduced to dynamically generate queries based on user inputs or configurations.
