# Create A Procedural Fake Dataset optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to create a procedural fake dataset optimized for incremental refresh. It allows for efficient data generation and updating, making it suitable for scenarios requiring frequent data refreshes.

### Typical Scenario
In a real data pipeline, this transformation would be used when a data source is not available or during testing phases. For instance, a business might need to simulate sales data for a new product launch to test their analytics system without waiting for actual sales data to be collected.

### Core M Functions Used
`List.Generate`, `Table.FromList`

### Code Source Execution
```powerquery
let
    // Define the number of rows to generate
    numberOfRows = 100,
    // Define the date range
    startDate = #date(2022, 1, 1),
    endDate = #date(2022, 12, 31),
    // Generate dates
    dates = List.Dates(startDate, Number.From(endDate) - Number.From(startDate) + 1, #day(1)),
    // Generate fake data
    data = List.Generate(() => [Date = List.First(dates), Value = Number.RandomBetween(1, 100)],
                         each [Date] < endDate,
                         each [Date = #date(Date.Year([Date]), Date.Month([Date]) + 1, 1), Value = Number.RandomBetween(1, 100)]),
    // Convert to table
    table = Table.FromList(data, Splitter.SplitByNothing(), {"Date", "Value"})
in
    table
```

### Example Execution Logic Step-By-Step
This script works as follows:
1. It first defines the number of rows to generate and a date range.
2. It generates a list of dates within the specified range.
3. It uses List.Generate to create fake data. Each iteration generates a new date (incremented by one month) and a random value between 1 and 100.
4. The generated data is then converted into a table using Table.FromList.

The resulting table can be used for testing or simulation purposes, and can be easily refreshed by re-running the query.
