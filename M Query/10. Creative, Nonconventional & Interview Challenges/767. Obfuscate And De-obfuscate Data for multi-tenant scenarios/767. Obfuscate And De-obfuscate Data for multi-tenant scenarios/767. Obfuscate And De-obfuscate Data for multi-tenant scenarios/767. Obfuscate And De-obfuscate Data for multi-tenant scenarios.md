# Obfuscate And De-obfuscate Data for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to obfuscate and de-obfuscate data, ensuring data privacy and security in multi-tenant environments. It enables data to be masked or encrypted for non-authorized users and decrypted for authorized users.

### Typical Scenario
In a multi-tenant data pipeline, sensitive data needs to be protected from unauthorized access. For instance, a data engineer can use obfuscation to mask sensitive customer information when it's being processed or stored, and then de-obfuscate it for specific business users who need access to the original data.

### Core M Functions Used
`Text.Replace`, `Text.Mask`, `Binary.ToText`, `Text.ToBinary`

### Code Source Execution
```powerquery
let
    ObfuscateData = (data as text) =>
        let
            // Obfuscate
            obfuscatedData = Text.Replace(data, "SensitiveData", "XXXXXX")
        in
            obfuscatedData,
    DeobfuscateData = (obfuscatedData as text) =>
        let
            // Deobfuscate
            deobfuscatedData = Text.Replace(obfuscatedData, "XXXXXX", "SensitiveData")
        in
            deobfuscatedData,
    data = "This is SensitiveData"

in
    ObfuscateData(data)
```

### Example Execution Logic Step-By-Step
The transformation works by first defining two functions: ObfuscateData and DeobfuscateData. The ObfuscateData function takes sensitive data as input and replaces it with a masked version (e.g., 'XXXXXX'). The DeobfuscateData function then takes the obfuscated data and replaces the masked version with the original sensitive data. In a Power Query pipeline, these functions can be applied to data as it's being processed, ensuring that sensitive information is protected from unauthorized access.
