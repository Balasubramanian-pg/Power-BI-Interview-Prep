# Design An Interview Puzzle: Mystery Column for large datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to solve an interview puzzle involving a mystery column for large datasets. It helps in understanding how to approach and solve complex data transformations.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large datasets where one column's values depend on complex conditions or puzzles that need to be solved to reveal a hidden or 'mystery' column. For instance, a data engineer might encounter a dataset where a specific column needs to be populated based on a logic that isn't straightforward, requiring a step-by-step breakdown and solution.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"A", 1, "x"},
        {"B", 2, "y"},
        {"C", 3, "z"}
    }, {"Column1", "Column2", "Column3"}),
    // Example transformation: Select rows where Column2 is even
    EvenRows = Table.SelectRows(Source, each [Column2] mod 2 = 0)
    in
    EvenRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Table Creation**: A sample table is created with three columns: 'Column1', 'Column2', and 'Column3'.
    2. **Transformation Application**: The Table.SelectRows function is applied to filter rows where 'Column2' has even values.
    3. **Result**: The resulting table contains only the rows from the source where 'Column2' was even, effectively solving the 'mystery' of selecting specific rows based on a condition.
