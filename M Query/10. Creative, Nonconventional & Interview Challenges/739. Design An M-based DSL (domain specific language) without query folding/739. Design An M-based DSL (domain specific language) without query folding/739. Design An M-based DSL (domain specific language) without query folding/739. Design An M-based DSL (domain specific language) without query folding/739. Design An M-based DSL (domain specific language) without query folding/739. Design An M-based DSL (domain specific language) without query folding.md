# Designing an M-based DSL without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be utilized to design a domain-specific language (DSL) without query folding. It allows for the creation of custom data transformation scripts using M, providing full control over data processing. This approach is particularly useful when specific data transformations are required that are not supported by standard Power Query functions.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a data engineer needs to implement a custom data processing logic that cannot be achieved through the standard Power Query interface. For instance, a company might need to process IoT sensor data in a very specific way that requires a custom DSL. The data engineer can design an M-based DSL to handle such unique processing needs.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"Peter", 42, "UK"}
    }, {"Name", "Age", "Country"}),
    SelectRows = Table.SelectRows(Source, each Age > 30)
    in
    SelectRows
```

The transformation works as follows:
    1. **Define the Source Data**: The process begins with defining a table that contains the raw data. In this example, a simple table with columns 'Name', 'Age', and 'Country' is created.
    2. **Apply Custom Logic**: The `Table.SelectRows` function is then used to filter rows where the 'Age' is greater than 30. This demonstrates how a custom DSL can be implemented in M to achieve specific data transformation requirements.
    3. **Execution**: When this script is executed within a Power Query pipeline, it selectively returns rows from the source table where the condition (Age > 30) is met, effectively applying the custom transformation logic defined in the M-based DSL.
