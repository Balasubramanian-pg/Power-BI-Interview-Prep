# Create A Procedural Fake Dataset without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to create a procedural fake dataset without query folding. It allows for the generation of sample data for testing or demonstration purposes.

### Typical Scenario
In a real data pipeline, this transformation can be used when a team needs to test a new data model or ETL process without having access to the actual data. For instance, a data engineer might use this approach to create fake sales data for a retail company to test data processing and analytics tools.

### Core M Functions Used
`List.Generate`, `Table.FromList`

### Code Source Execution
```powerquery
let
    // Define the number of rows to generate
    rowsToGenerate = 100,

    // Generate a list of random numbers
    randomNumbers = List.Generate(() => 0, each _ < rowsToGenerate, each _ + 1),

    // Create a table from the list
    fakeData = Table.FromList(randomNumbers, Splitter.SplitByNothing(), null, null, ExtraValues.Error),

    // Rename the column

    // Add more columns as needed
    fakeDataWithMoreColumns = Table.AddColumn(fakeDataRenamed, "FakeData", each "FakeData"),

    // Return the fake data
    fakeDataset = fakeDataWithMoreColumns
in
    fakeDataset
```
