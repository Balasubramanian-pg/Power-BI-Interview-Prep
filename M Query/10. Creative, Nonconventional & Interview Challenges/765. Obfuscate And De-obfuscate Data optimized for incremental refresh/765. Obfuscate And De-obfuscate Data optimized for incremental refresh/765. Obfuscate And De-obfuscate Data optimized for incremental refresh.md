# Obfuscate And De-obfuscate Data optimized for incremental refresh

**Description:**
This transformation achieves data protection through obfuscation and allows for efficient data retrieval and updates via de-obfuscation, particularly useful in scenarios requiring incremental data refresh.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial for protecting sensitive information during data transfer or storage, such as in customer data processing where details like names or IDs need to be secured but still accessible for authorized operations.

### Core M Functions Used
`Table.SelectRows`, `List.Transform`, `Text.Replace`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "Doe", 25},
        {"Jane", "Doe", 30}
    }, {"FirstName", "LastName", "Age"}),
    ObfuscateData = Table.TransformColumns(Source, {{"FirstName", each Text.Replace(_, "", "*")}, {"LastName", each Text.Replace(_, "", "*")}}),
    DeobfuscateData = Table.TransformColumns(ObfuscateData, {{"FirstName", each Text.Replace(_, "*", "")}, {"LastName", each Text.Replace(_, "*", "")}})
    in
    DeobfuscateData
```

### Example Execution Logic Step-By-Step
The transformation works by first obfuscating sensitive data (like names) with asterisks (*) to protect it. This is achieved through the 'ObfuscateData' step using 'Text.Replace'. Then, for de-obfuscation, it replaces the asterisks back with the original characters in the 'DeobfuscateData' step. This process ensures data can be securely processed or stored and then restored for use.
