# Design An M-based DSL as a Reusable Function

**Description:**
This use case demonstrates how Power Query (M) can be used to design a domain-specific language (DSL) as a reusable function, enabling efficient data transformations and simplifying complex data processing tasks.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to standardize data processing tasks across multiple datasets, such as data cleansing, filtering, and aggregation, making it easier to reuse and maintain these tasks.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"A", 1, "a"},
        {"B", 2, "b"},
        {"C", 3, "c"}
    }, {"Column1", "Column2", "Column3"}),
    FilteredRows = Table.SelectRows(Source, each [Column2] > 1)
    in
        FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows: first, a sample table is created with three columns and three rows. Then, the Table.SelectRows function is used to filter the rows where the value in Column2 is greater than 1. This results in a new table containing only the rows that meet the specified condition, demonstrating how an M-based DSL can be used to create reusable functions for data transformation.
