# Time-travel Debugging Simulation for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used for time-travel debugging simulation in multi-tenant scenarios, allowing for the analysis of data as it existed at different points in time.

### Typical Scenario
In a real data pipeline, this transformation would be used when a multi-tenant application requires data to be analyzed or debugged at a specific point in time, such as auditing or troubleshooting historical data issues.

### Core M Functions Used
`Table.SelectRows`, `Table.AddColumn`, `Table.TransformRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "2022-01-01", "Data1"},
        {"Tenant2", "2022-01-02", "Data2"},
        {"Tenant1", "2022-01-03", "Data3"}
    }, {"Tenant", "Date", "Data"}),
    SelectRows = Table.SelectRows(Source, each [Date] = "2022-01-01")
    in
    SelectRows
```

### Example Execution Logic Step-By-Step
The transformation works by first loading a table with tenant data, including a date column. It then uses the Table.SelectRows function to filter the data to only include rows where the date matches a specific point in time, effectively 'traveling back in time' to analyze the data as it existed at that point.
