# Obfuscate And De-obfuscate Data with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used for obfuscating and de-obfuscating data while adhering to GDPR/privacy constraints. It ensures that sensitive information is protected during data processing and analysis.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when handling sensitive customer data. For instance, a company may need to share customer data with a third-party service provider for analytics purposes while ensuring that the data does not reveal personal identifiable information. The data can be obfuscated (e.g., replacing names with pseudonyms) before sharing, and then de-obfuscated when needed for specific business operations.

### Core M Functions Used
`Table.SelectRows`, `List.Transform`, `Text.Replace`

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Obfuscation**: Sensitive data (e.g., names) are replaced with pseudonyms using `Table.TransformColumns` and string manipulation functions like `Text.Replace` or simple concatenation.
2. **De-obfuscation**: A mapping table is created to link pseudonyms back to real names. This is achieved through `Table.JoinColumns` with appropriate join kinds to match pseudonyms with real names.
The result is a dataset where sensitive information is protected during processing but can be accurately restored when required.
