# Dynamic API Pagination for Large Datasets

**Description:**
This transformation enables the retrieval of large datasets from APIs that use pagination, allowing for dynamic handling of page sizes and tokens. It is particularly useful when dealing with APIs that limit the number of records returned per page. By implementing dynamic pagination, users can fetch all records without manual intervention.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when integrating data from APIs that paginate results, such as fetching a large number of customer records from a CRM system. For instance, an e-commerce company needs to fetch daily sales data from a third-party API that returns 100 records per page. The company can use dynamic pagination to automatically fetch all pages of data without manual intervention, ensuring that the data pipeline remains efficient and scalable.

### Code Source Execution
```powerquery
let
    url = "https://api.example.com/data",
    pageSize = 100,
    initialPage = 1,
    maxPages = 10,
    GetPage = (page) =>
        let
            params = [#"page" = page, #"pageSize" = pageSize],
            response = Web.Contents(url, [Query = params]),
            json = Json.Document(response),
            data = json["data"],
            nextPageToken = json["nextPageToken"]
        in
            data,
    pages = List.Generate(
        () => initialPage,
        (page) => page < maxPages,
        (page) => if page = initialPage then GetPage(page) else GetPage(page)
    ),
    combinedData = Table.CombineColumns(pages, "data", type any, "data")
in
    combinedData
```

The transformation works as follows: 1) Initialize the API URL, page size, and initial page number. 2) Define a function `GetPage` that takes a page number, constructs the API request with pagination parameters, and extracts the data and next page token from the response. 3) Use `List.Generate` to iterate through pages, fetching data from each page using the `GetPage` function. 4) Combine the data from all pages into a single table using `Table.CombineColumns`. This approach allows for dynamic handling of pagination tokens and page sizes.
