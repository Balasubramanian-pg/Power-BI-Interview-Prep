# Parameterizing Data Source for Multi-Tenant Scenarios

**Description:**
This use case demonstrates how to parameterize a data source in Power Query (M) for multi-tenant scenarios, allowing for dynamic data retrieval based on tenant-specific parameters.

### Typical Scenario
In a multi-tenant data pipeline, different tenants may have their own databases or data sources. By parameterizing the data source, you can dynamically switch between tenants' data sources based on a parameter, such as a tenant ID.

### Core M Functions Used
`Table.SelectRows`, `Fn.Invoke`

### Code Source Execution
```powerquery
let
    // Define the parameter for the tenant ID
    tenantId = "tenant1";

    // Define the base URL for the data source
    baseUrl = "https://data-source.com/";

    // Construct the URL for the specific tenant
    url = baseUrl & tenantId & "/data";

    // Retrieve data from the constructed URL
    data = Web.Contents(url);

    // Transform the data
    transformedData = Table.SelectRows(data, each [_"Tenant ID"" = tenantId])

in
    transformedData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. Define a parameter for the tenant ID.
2. Construct the URL for the data source based on the tenant ID.
3. Retrieve data from the constructed URL.
4. Transform the data using Table.SelectRows to filter data specific to the tenant.
