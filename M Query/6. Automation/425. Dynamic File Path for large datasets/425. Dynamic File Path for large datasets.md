# Dynamic File Path for Large Datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to dynamically handle file paths for large datasets, making it easier to manage and automate data imports.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with large datasets that are constantly updated or when the file path structure changes frequently. For instance, a company receives daily sales data from various regions, and the file paths are determined by the date and region. A dynamic file path approach allows for easy adaptation to these changing file structures without manual intervention.

### Core M Functions Used
`Table.SelectRows`, `File.Contents`, `Folder.Contents`

### Code Source Execution
```powerquery
let
    Source = Folder.Contents("C:\Data\Source"),
    Files = Table.SelectRows(Source, each [Kind = "File"]),
    FileContents = Files => List.Transform(Files, each File.Contents([Name])),
    CombinedData = Table.CombineColumns(FileContents(Files), "Wide", "Text", "Name")
in CombinedData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Folder.Contents**: This function is used to get a list of files and folders in a specified directory.
2. **Table.SelectRows**: This function filters the list to only include files (not folders).
3. **File.Contents**: This function reads the contents of each file.
4. **List.Transform**: This function applies the File.Contents function to each file in the list.
5. **Table.CombineColumns**: This function combines the data from all files into a single table.
