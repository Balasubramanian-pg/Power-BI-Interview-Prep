# Build Reusable Function optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to build a reusable function optimized for incremental refresh. It allows for efficient data refresh by only loading new or updated data.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large datasets that are updated periodically. For instance, a daily sales report that only needs to reflect new sales data from the previous day.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Age"},
        {1, "John", 25},
        {2, "Jane", 30},
        {3, "Bob", 35}
    }),
    IncrementalRefresh = (startDate, endDate) =>
        Table.SelectRows(Source, each DateTime.Date([Date]) >= startDate and DateTime.Date([Date]) <= endDate)
    in
        IncrementalRefresh(#date(2022,1,1), #date(2022,12,31))
```

### Example Execution Logic Step-By-Step
The transformation works by defining a reusable function that takes a start and end date as parameters. This function uses Table.SelectRows to filter the data based on a date column, allowing for incremental refresh by only loading data within the specified date range.
