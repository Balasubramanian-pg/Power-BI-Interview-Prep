# Dynamic Date Filter Optimized for Incremental Refresh

**Description:**
This transformation achieves dynamic filtering of data based on dates, optimized for incremental refresh scenarios. It allows for efficient data loading and updating by focusing on specific date ranges. This approach is particularly useful in data pipelines where data is constantly being added or updated.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with large datasets that are regularly updated, such as sales data or log data. For instance, a company might want to analyze sales data for the last quarter, but only needs to update the data on a daily basis. The dynamic date filter would allow the company to focus on the new data added since the last update, rather than reprocessing the entire dataset.

The transformation works as follows:
    1. **Data Source**: The data source is defined, which in this case is a simple table with dates, names, and values.
    2. **Dynamic Date Calculation**: The `DateTime.FixedLocalNow()` function is used to get the current date and time.
    3. **Filtering Data**: The `Table.SelectRows` function is used with a condition that filters the data to include only rows where the date is within the last 7 days from the current date.
    4. **Incremental Refresh**: By adjusting the date range dynamically, the transformation enables incremental refresh, which means only new or updated data within the specified date range is loaded or refreshed, rather than the entire dataset.
