# Loop Through File List with Error Handling and Retries in Power Query

**Description:**
This transformation enables looping through a list of files while incorporating error handling and retries. It is particularly useful for data pipelines that involve fetching or processing files from potentially unreliable sources. By implementing retries, it helps ensure that transient errors do not cause the entire process to fail.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when ingesting data from a list of files hosted on a network share or cloud storage that may occasionally be unavailable or return temporary errors. For instance, a daily data ingestion process that fetches files from an external partner might use this approach to handle intermittent connectivity issues or file locking errors.

### Code Source Execution
```powerquery
let
    // Assume 'fileList' is a list of file paths
    fileList = {"path/to/file1.csv", "path/to/file2.csv"},
    // Define a function to read a file with error handling
    readFileWithRetry = (filePath) =>
        let
            maxRetries = 3,
            retryCount = 0,
            readFile = () =>
                try
                    Csv.Document(File.Contents(filePath), [Delimiter=",", Encoding=1252])
                catch
                    Error.Record as e
                    let
                        retry = if retryCount < maxRetries then
                            readFileWithRetry(filePath)
                        else
                            error "Failed to read file after " & Text.FromNumber(maxRetries) & " retries"
                    in
                        retry
            in
                readFile(),
    // Loop through each file and apply the read function
    results = List.Transform(fileList, readFileWithRetry)
    in
        results
```
