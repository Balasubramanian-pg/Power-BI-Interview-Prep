# Auto Load New Files for Large Datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to automatically load new files for large datasets. It allows for efficient data pipeline management by dynamically incorporating new data as it becomes available.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with continuously updated data sources, such as daily or weekly reports that are appended to a folder. For instance, a company might receive new sales data files every day, and using Power Query, they can automate the process of loading these new files into their analysis system without manual intervention.

### Core M Functions Used
`Folder.Files`, `DateTime.ToText`, `Table.Combine`

### Code Source Execution
```powerquery
let
    Source = Folder.Files("C:\Path\To\Folder"),
    GetFiles = Table.SelectRows(Source, each [Kind = "File"]),
    CombineFiles = Table.Group(GetFiles, "Name", {{"Files", each [Content], type table}}),
    TransformFiles = Table.TransformColumns(CombineFiles, {{"Files", each Table.Combine(_)}}),
    LoadData = TransformFiles[Files]
 in
    LoadData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Folder.Files**: This function is used to get a list of files from a specified folder.
2. **Table.SelectRows**: This is used to filter the files based on their kind (e.g., only files, not folders).
3. **Table.Group**: This groups the files by name and combines their contents into a single table.
4. **Table.TransformColumns**: This transforms the grouped files into a combined table format.
5. **Table.Combine**: This function combines the individual tables (files) into a single table.

The result is a dynamic query that automatically loads new files from the specified folder, making it highly useful for managing large datasets that are frequently updated.
