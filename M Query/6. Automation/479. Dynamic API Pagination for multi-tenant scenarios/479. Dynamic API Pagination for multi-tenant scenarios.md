# Dynamic API Pagination for Multi-Tenant Scenarios

**Description:**
This transformation enables dynamic API pagination for multi-tenant scenarios using Power Query (M). It allows for efficient data retrieval from APIs with pagination, supporting multiple tenants. This approach ensures scalable and flexible data ingestion.

### Typical Scenario
In a real data pipeline, this transformation would be used when integrating data from a multi-tenant API that requires pagination. For instance, a company using a CRM system with multiple tenants might need to fetch data from each tenant's API, which returns results in pages. This transformation helps automate the process of fetching data from each page across all tenants.

### Core M Functions Used
`Web.Contents`, `Json.Document`, `Table.FromList`, `List.Transform`

### Code Source Execution
```powerquery
let
    apiBaseUrl = "https://api.example.com/",
    tenantIds = {"tenant1", "tenant2"},
    pageSize = 100,
    fetchData = (tenantId as text, page as number) =>
        let
            url = apiBaseUrl & "data?tenantId=" & tenantId & "&pageSize=" & Number.ToText(pageSize) & "&page=" & Number.ToText(page),
            response = Web.Contents(url),
            json = Json.Document(response),
            data = json[data]
        in
            data,
    getPages = (tenantId as text) =>
        let
            firstPageData = fetchData(tenantId, 1),
            totalCount = firstPageData[totalCount],
            totalPages = Number.RoundUp(totalCount / pageSize),
            pagesData = List.Transform(List.Numbers(1, totalPages), each fetchData(tenantId, _))
        in
            pagesData,
    combinedData = List.Transform(tenantIds, each getPages(_)),
    flattenedData = List.Transform(combinedData, each List.Combine(_))
    in
        Table.FromList(flattenedData, Splitter.SplitByNothing(), {"Data"})
```

### Example Execution Logic Step-By-Step
This transformation works as follows:
1. **Define API Base URL and Tenant IDs**: Specify the base URL of the API and a list of tenant IDs.
2. **Define Page Size**: Determine the number of records per page.
3. **Fetch Data Function**: Create a function that takes a tenant ID and a page number, constructs the API URL, fetches data using `Web.Contents`, and parses the JSON response.
4. **Get Pages Function**: For each tenant, fetch the first page to determine the total count of records and calculate the total number of pages. Then, fetch data from all pages.
5. **Combine and Flatten Data**: Combine data from all pages across all tenants into a single list and convert it into a table.
