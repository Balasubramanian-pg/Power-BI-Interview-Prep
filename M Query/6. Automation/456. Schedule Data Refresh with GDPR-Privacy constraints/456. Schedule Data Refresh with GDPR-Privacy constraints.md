# Schedule Data Refresh with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used for scheduling data refresh while adhering to GDPR/privacy constraints. It ensures that data is updated regularly while protecting sensitive information.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to regularly refresh data from a source while ensuring that any personal or sensitive data is handled in compliance with GDPR and other privacy regulations. For instance, a company may need to daily update customer data for analytics purposes but must ensure that all personally identifiable information is properly masked or anonymized.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"CustomerID", "Name", "Email"},
        {1, "John Doe", "john.doe@example.com"},
        {2, "Jane Doe", "jane.doe@example.com"}
    }),
    // Apply GDPR/Privacy constraints
    ProtectedData = Table.SelectRows(Source, each "PrivacyLevel" = "Protected"),
    // Anonymize or mask sensitive data
    AnonymizedData = Table.TransformColumns(ProtectedData, {{"Email", each "anonymized"}})
    in
    AnonymizedData
```
