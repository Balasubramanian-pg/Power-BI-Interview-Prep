# Schedule Data Refresh for large datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to schedule data refresh for large datasets, ensuring that the data remains up-to-date and relevant for analysis and reporting.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with large datasets that need to be refreshed periodically, such as daily or weekly, to reflect changes in the underlying data. For instance, a company may have a large dataset of sales data that needs to be updated daily to reflect new sales transactions.

### Core M Functions Used
`Table.SelectRows`, `Table.Refresh`

### Code Source Execution
```powerquery
let
    Source = Table.SelectRows(TableA, each [_refresh] = "daily"),
    Refresh = Table.Refresh(Source, [RefreshMode = "Full"])
in
    Refresh
```

### Example Execution Logic Step-By-Step
The transformation works by first selecting the rows in the table that need to be refreshed, and then using the Table.Refresh function to refresh the data. The Table.SelectRows function is used to filter the table based on a condition, such as a column indicating whether the data needs to be refreshed. The Table.Refresh function is then used to refresh the data, specifying the refresh mode as "Full" to refresh all data.
