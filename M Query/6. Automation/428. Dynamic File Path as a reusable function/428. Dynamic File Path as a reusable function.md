# Dynamic File Path as a Reusable Function

**Description:**
This use case demonstrates how Power Query (M) can be used to create a dynamic file path as a reusable function, allowing for flexible and dynamic data imports.

### Typical Scenario
In a real data pipeline, this transformation would be used when the file path of the data source changes frequently, such as when dealing with daily or weekly data dumps with varying file names or paths.

### Core M Functions Used
`Table.SelectRows`, `File.Contents`, `Text.Combine`

### Code Source Execution
```powerquery
let
    dynamicFilePath = (filePath as text) =>
    let
        fileContents = File.Contents(filePath),
        data = Csv.Document(fileContents, [Delimiter=",", Encoding=1252]),
        selectData = Table.SelectRows(data, each _[Column1] <> null)
    in
        selectData,
    filePath = "C:\Path\To\Your\File.csv"
in
    dynamicFilePath(filePath)
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. Define a reusable function `dynamicFilePath` that takes a file path as an input parameter.
2. Inside the function, use `File.Contents` to read the file contents from the provided file path.
3. Use `Csv.Document` to parse the file contents into a table.
4. Apply a filter using `Table.SelectRows` to select rows where the first column is not null.
5. Call the `dynamicFilePath` function with a specific file path to get the data.
