# Dynamic Date Filter with Query Folding Considerations

**Description:**
This transformation enables dynamic filtering of data based on date criteria, allowing for efficient query folding. It facilitates adaptable data pipelines that can adjust to varying date ranges. By leveraging query folding, it optimizes performance by pushing filters down to the source.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when analyzing sales data. For instance, a business might want to dynamically analyze sales performance based on a rolling date range (e.g., last 30 days) that changes with each report generation. This allows for timely insights without manually adjusting date filters.

### Core M Functions Used
`Table.SelectRows`, `Date.AddDays`, `Date.FromText`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", #date(2022,1,1), 100},
        {"Mary", #date(2022,1,15), 200},
        {"Peter", #date(2022,2,1), 300}
    }, {"Name", "Date", "Sales"}),
    CurrentDate = Date.FromText("2022-01-20"),
    FilterDate = Date.AddDays(CurrentDate, -30),
    FilteredTable = Table.SelectRows(Source, each [Date] >= FilterDate)
    in
    FilteredTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Data**: A table with columns 'Name', 'Date', and 'Sales' is created.
    2. **Current Date**: The current date is defined.
    3. **Filter Date Calculation**: A date 30 days prior to the current date is calculated.
    4. **Dynamic Filtering**: The Table.SelectRows function is used to filter the table, selecting only rows where the 'Date' column is on or after the calculated filter date.
    5. **Result**: A filtered table containing only the relevant data for the dynamic date range is returned.
