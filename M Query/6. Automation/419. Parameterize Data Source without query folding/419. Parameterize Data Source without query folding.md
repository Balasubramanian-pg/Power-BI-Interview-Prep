# Parameterizing Data Source without Query Folding in Power Query

**Description:**
This use case demonstrates how to parameterize a data source in Power Query without relying on query folding. It allows for dynamic data source configuration, enabling more flexible and reusable queries. This approach is particularly useful when working with data sources that do not support query folding or when complex filtering is required.

### Typical Scenario
In a real-world data pipeline, this transformation is useful when you need to dynamically change the data source based on user input or configuration. For example, in a sales data analysis application, you might want to analyze sales data from different regions. By parameterizing the data source, you can easily switch between regions without modifying the query.

### Core M Functions Used
`Table.SelectRows`, `let`, `in`

### Code Source Execution
```powerquery
let
    // Define a parameter
    parameterValue = "YourParameterValue",
    // Define the data source
    dataSource = Table.FromRows({
        {"Column1", "Column2"},
        {"Value1", "Value2"}
    }),
    // Apply the parameter to the data source
    filteredData = Table.SelectRows(dataSource, each [Column1] = parameterValue)
in
    filteredData
```
