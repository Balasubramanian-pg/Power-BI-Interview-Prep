# Dynamic File Path with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used for dynamic file path handling while considering query folding. It enables flexible data ingestion from various file locations. The transformation allows for efficient data retrieval by optimizing query performance.

### Typical Scenario
In a real-world data pipeline, this transformation is useful when dealing with files stored in dynamic locations, such as date-based directories (e.g., /data/2022/01, /data/2022/02). By using dynamic file paths, the pipeline can adapt to changing data sources without manual intervention.

### Core M Functions Used
`Table.SelectRows`, `Folder.Files`, `File.Contents`

### Code Source Execution
```powerquery
let
    // Define dynamic file path parameters
    basePath = "data/",
    year = 2022,
    month = 1,

    // Construct dynamic file path
    filePath = Folder.Combine({basePath, Text.FromNumber(year), Text.PadLeft(Text.FromNumber(month), 2, "0")}),

    // Read files from dynamic path
    files = Folder.Files(filePath),

    // Transform file contents
    data = Table.TransformColumns(files, {{"Content", each File.Contents(_)}})
in
    data
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. Define the base path and dynamic parameters (e.g., year and month).
2. Construct the dynamic file path by combining the base path with the dynamic parameters.
3. Use the Folder.Files function to read files from the dynamic path.
4. Transform the file contents using Table.TransformColumns and File.Contents.
