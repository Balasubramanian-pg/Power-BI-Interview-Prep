# Loop Through File List without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to loop through a file list without query folding, allowing for dynamic processing of files based on certain conditions or transformations that cannot be folded back to the source.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to process files based on dynamic conditions that cannot be handled by the source query. For example, processing files from a folder where the file names or structures change frequently, and you need to apply transformations that depend on the file's metadata or content.

### Core M Functions Used
`Table.SelectRows`, `List.Transform`, `List.Contains`

### Code Source Execution
```powerquery
let
    // Assume 'files' is a list of file paths
    files = {"file1.csv", "file2.csv", "file3.csv"},
    // Loop through each file and apply transformation
    transformedFiles = List.Transform(files, each
        let
            fileContent = Csv.Document(File.Contents(_), [Delimiter=",", Encoding=1252]),
            // Apply your transformation here
            transformedContent = Table.SelectRows(fileContent, "Column1 > 0")
        in
            transformedContent
    )
in
    transformedFiles
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **List of Files**: Start with a list of file paths.
2. **Loop Through Files**: Use `List.Transform` to loop through each file in the list.
3. **Read File Content**: For each file, read its content using `File.Contents` and parse it as a CSV document with `Csv.Document`.
4. **Apply Transformation**: Apply your required transformation, for example, selecting rows where a condition is met using `Table.SelectRows`.
5. **Result**: The transformed tables for each file are collected in a list.
