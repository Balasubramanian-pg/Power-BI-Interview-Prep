# Generate Parameter Table without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to generate a parameter table without query folding. It allows for the creation of a table that can be used as parameters in a data pipeline.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to create a table of parameters that can be used to filter or query data without relying on query folding. For example, in a data warehouse, a parameter table might be used to store filter values for reports.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    // Define a list of parameters
    parameters = {
        [Name = "Parameter1", Value = 10],
        [Name = "Parameter2", Value = 20],
        [Name = "Parameter3", Value = 30]
    },
    // Create a table from the list of parameters
    parameterTable = Table.FromList(parameters, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    // Rename columns
    parameterTable = Table.RenameColumns(parameterTable, {{"Column1", "Name"}, {"Column2", "Value"}})
in
    parameterTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. Define a list of parameters with their names and values.
2. Create a table from the list of parameters using the `Table.FromList` function.
3. Rename the columns of the table to meaningful names using the `Table.RenameColumns` function.
4. The resulting table can be used as a parameter table in a data pipeline.
