# Dynamic File Path without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used for dynamic file path resolution without query folding. It allows for flexible file path construction based on parameters or other data. This transformation is useful when the file path is not fixed and needs to be determined at runtime.

### Typical Scenario
In a real data pipeline, this transformation would be used when ingesting data from files whose locations are determined dynamically, such as files generated by a date or a specific process. For example, loading log files from a specific date range or loading data files generated by different departments.

### Core M Functions Used
`Table.SelectRows`, `File.Contents`, `Text.Combine`

### Code Source Execution
```powerquery
let
    // Define the dynamic path parameters
    dateRangeStart = "2023-01-01",
    dateRangeEnd = "2023-01-31",
    filePath = "C:/Data/Logs/",
    fileExtension = ".log",
    // Construct the file path dynamically
    dynamicFilePath = Text.Combine({filePath, dateRangeStart, "_", dateRangeEnd, fileExtension}),
    // Read the file contents
    fileContents = File.Contents(dynamicFilePath),
    // Convert file contents to table (example assumes a simple text file)
    fileData = Table.FromRows( {
        {"data", Text.FromBinary(fileContents)} 
    }, {"data"})
    // Example transformation: Select specific rows
    in
    Table.SelectRows(fileData, "data", each [data] <> "")
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Define Dynamic Path Parameters**: Parameters such as date ranges, directories, and file extensions are defined.
2. **Construct Dynamic File Path**: The file path is constructed by combining the directory, date range, and file extension using `Text.Combine`.
3. **Read File Contents**: The file is read using `File.Contents`.
4. **Convert to Table**: The file contents are converted into a table.
5. **Apply Transformations**: Finally, transformations such as selecting specific rows can be applied using `Table.SelectRows`.
