# Parameterize Data Source optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to parameterize a data source for incremental refresh, allowing for efficient and flexible data updates.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a data source needs to be refreshed incrementally, such as when new data is added daily and only the new data needs to be loaded.

### Core M Functions Used
`Table.SelectRows`, `Query.Parameters`

### Code Source Execution
```powerquery
let
    startDate = Query.Parameters["startDate"]{0}[#"Value"],
    endDate = Query.Parameters["endDate"]{0}[#"Value"],
    Source = Table.SelectRows(Table.FromRows(YourDataSource), each [_date] >= startDate and [_date] <= endDate)
    in
    Source
```

### Example Execution Logic Step-By-Step
The transformation works by first defining parameters for the start and end dates of the incremental refresh. These parameters are then used in a Table.SelectRows function to filter the data source and only load data within the specified date range. The Query.Parameters syntax is used to define and access these parameters.
