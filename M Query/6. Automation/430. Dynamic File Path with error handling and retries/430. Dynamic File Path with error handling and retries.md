# Dynamic File Path with Error Handling and Retries

**Description:**
This transformation enables dynamic file path handling in Power Query with robust error handling and retry mechanisms. It allows for flexible data ingestion from various sources, handling potential errors and retries seamlessly. This approach ensures reliable data pipelines by managing dynamic file paths efficiently.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with frequently changing file locations or when files are intermittently available. For instance, a data pipeline fetching sales data from different regional servers daily might use dynamic file paths to adapt to changing file locations or to handle temporary server unavailability.

### Core M Functions Used
`File.Contents`, `try`, `catch`, `Number.Retries`

### Code Source Execution
```powerquery
let
    // Define dynamic file path
    filePath = "C:\Data\SalesData_" & Date.ToText(DateTime.LocalNow(), "yyyy-MM-dd") & ".csv",

    // Define retry policy
    maxRetries = 3,
    retryCount = 0,

    // Function to get file contents with retries
    getFileContents = () =>
        try
            File.Contents(filePath)
        catch
            [Error] =>
                if retryCount <= maxRetries then
                    (retryCount + 1, getFileContents())
                else
                    Error.Record("Failed to retrieve file after " & Text.FromNumber(maxRetries) & " retries.")
    in
    getFileContents()

```

### Example Execution Logic Step-By-Step
This transformation works by defining a dynamic file path based on the current date. It attempts to retrieve the file contents using the File.Contents function within a try-catch block. If an error occurs, it retries the operation up to a specified number of times (maxRetries). If all retries fail, it records an error. This approach ensures that the data pipeline can adapt to changing file locations and handle temporary file or server unavailability.
