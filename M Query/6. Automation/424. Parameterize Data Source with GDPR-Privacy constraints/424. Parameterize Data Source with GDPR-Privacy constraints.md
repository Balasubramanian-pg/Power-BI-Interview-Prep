# Parameterize Data Source with GDPR/Privacy Constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to parameterize a data source while applying GDPR/privacy constraints. It allows for flexible data retrieval while ensuring sensitive information is handled appropriately. By parameterizing the data source, users can easily modify the query to retrieve different subsets of data, all while maintaining the necessary privacy constraints.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to regularly extract specific data from a large dataset while adhering to GDPR and other privacy regulations. For instance, a marketing team might need to analyze customer data for targeted campaigns, but only with the consent of those customers and within the limits set by data protection laws.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    // Define the data source
    data = Table.FromList({
        {"John", 25, "USA"},
        {"Mary", 31, "UK"},
        {"David", 42, "Australia"}
    }, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    dataWithHeaders = Table.AddColumn(data, "Name", each _[Column1]),
    dataWithHeaders = Table.RenameColumns(dataWithHeaders, {{"Column1", "Name"}, {"Column2", "Age"}, {"Column3", "Country"}}),
    // Apply GDPR/privacy constraints (e.g., filter for specific countries)
    filteredData = Table.SelectRows(dataWithHeaders, each [Country] = "USA")
in
    filteredData
```
