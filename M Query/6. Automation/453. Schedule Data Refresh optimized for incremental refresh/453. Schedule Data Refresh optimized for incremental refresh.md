# Schedule Data Refresh optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to optimize schedule data refresh for incremental data loading, improving performance and reducing data redundancy.

### Typical Scenario
In a real-world data pipeline, this transformation is useful when dealing with large datasets that are updated periodically, such as daily sales data. By implementing incremental refresh, the data pipeline can focus on loading only the new or updated records, rather than reloading the entire dataset.

### Core M Functions Used
`Table.SelectRows`, `Table.AddColumn`, `Table.Group`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Date", "Sales"}, 
        {"2022-01-01", 100}, 
        {"2022-01-02", 200}, 
        {"2022-01-03", 300}
    }),
    IncrementalRefresh = Table.SelectRows(Source, each [Date] >= Date.AddDays(DateTime.LocalNow(), -1)) 
in
    IncrementalRefresh
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. The source data is loaded into a table. 
2. The IncrementalRefresh variable uses Table.SelectRows to filter the data and only include rows where the date is within the last day. 
3. The resulting table contains only the new or updated records, which can then be loaded into the data pipeline for further processing.
