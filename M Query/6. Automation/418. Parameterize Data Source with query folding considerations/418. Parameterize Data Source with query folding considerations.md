# Parameterizing Data Source with Query Folding Considerations

**Description:**
This use case demonstrates how to parameterize a data source in Power Query (M) while considering query folding. Parameterizing a data source allows for more flexible and reusable queries, and query folding enables efficient data retrieval by pushing filtering and other operations down to the source.

### Typical Scenario
In a real data pipeline, this transformation would be used when connecting to a database or data warehouse where the data needs to be filtered or processed based on dynamic parameters. For instance, a data analyst might want to analyze sales data for different regions or time periods without creating separate queries for each case.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`, `Table.FilterRows`

### Code Source Execution
```powerquery
let
    // Define a parameter
    startDate = #date(2022, 1, 1),
    endDate = #date(2022, 12, 31),

    // Define a data source
    dataSource = Sql.Database("server", "database", [Query="SELECT * FROM sales"]),

    // Apply filtering based on parameters
    filteredData = Table.SelectRows(dataSource, each [date] >= startDate and [date] <= endDate)
in
    filteredData
```

### Example Execution Logic Step-By-Step
This transformation typically works as follows inside a Power Query pipeline:
1. **Define Parameters**: First, parameters such as startDate and endDate are defined.
2. **Connect to Data Source**: A data source, such as a SQL database, is connected using a function like Sql.Database.
3. **Apply Filtering**: The data is then filtered using Table.SelectRows, applying conditions based on the defined parameters.
4. **Query Folding**: If the data source supports query folding, operations like filtering are 'pushed down' to the source, improving performance by reducing the amount of data transferred.
