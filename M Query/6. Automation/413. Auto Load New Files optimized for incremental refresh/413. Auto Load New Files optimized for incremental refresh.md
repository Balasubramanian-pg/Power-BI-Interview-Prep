# Auto Load New Files Optimized for Incremental Refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to automatically load new files in a folder, optimized for incremental refresh. It allows for efficient and automated data loading, reducing manual intervention. This transformation is particularly useful for data pipelines that require frequent updates.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a business needs to regularly ingest new data files from a folder, such as daily or weekly sales reports. For instance, a company receives a new CSV file every day with sales data, and they want to automatically load this new data into their analytics system without manually refreshing the data source.

### Core M Functions Used
`Folder.Files`, `Table.SelectRows`, `DateTime.ToText`

### Code Source Execution
```powerquery
let
    Source = Folder.Files("C:\\Path\\To\\Folder"),
    FilteredFiles = Table.SelectRows(Source, each DateTime.ToText([Date], "yyyy-MM-dd") = DateTime.ToText(DateTime.LocalNow(), "yyyy-MM-dd"))
    in
    FilteredFiles
```

### Example Execution Logic Step-By-Step
This transformation typically works as follows:
1. **Identify the Folder**: Specify the folder path containing the new files.
2. **List Files in Folder**: Use `Folder.Files` to get a list of files in the specified folder.
3. **Filter New Files**: Use `Table.SelectRows` to filter files based on their date, for example, to only include files from today.
4. **Load Files**: Load the filtered files into the Power Query pipeline for further processing and analysis.

By automating the detection and loading of new files, this transformation streamlines data ingestion and supports incremental data refresh, making it easier to keep analytics systems up-to-date.
