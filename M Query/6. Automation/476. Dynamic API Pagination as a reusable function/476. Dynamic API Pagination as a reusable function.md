# Dynamic API Pagination as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used for dynamic API pagination, making it easy to fetch data from APIs that require pagination. It provides a reusable function to handle pagination dynamically.

### Typical Scenario
In a real data pipeline, this transformation would be used when fetching large datasets from APIs that are paginated, such as fetching sales data from an e-commerce platform where data is split across multiple pages.

### Core M Functions Used
`Web.Contents`, `Json.Document`

### Code Source Execution
```powerquery
let
    apiUrl = "https://api.example.com/data",
    pageSize = 100,
    getPage = (page as number) =>
        let
            url = apiUrl & "?page=" & Number.ToText(page) & "&pageSize=" & Number.ToText(pageSize),
            response = Web.Contents(url),
            json = Json.Document(response)
        in
            json,
    getPages = (page as number, data as any) =>
        let
            totalPages = data[totalPages],
            nextPage = page + 1
        in
            if page < totalPages then
                getPages(nextPage, getPage(nextPage))
            else
                {}
    getData = () =>
        let
            firstPage = getPage(1)
        in
            getPages(1, firstPage)
    data = getData()
    in
    data
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 1) Define the API URL and pagination parameters. 2) Create a function `getPage` to fetch a specific page from the API. 3) Create a recursive function `getPages` to fetch all pages until there are no more pages left. 4) Call `getData` to initiate the fetching of data from the first page.
