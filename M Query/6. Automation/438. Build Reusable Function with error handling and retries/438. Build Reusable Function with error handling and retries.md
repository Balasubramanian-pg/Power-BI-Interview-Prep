# Build Reusable Function with Error Handling and Retries

**Description:**
This use case demonstrates how Power Query (M) can be used to build a reusable function that handles errors and implements retries, enhancing data pipeline reliability.

### Typical Scenario
In a real data pipeline, this transformation is crucial when fetching data from external APIs or services that may intermittently fail or return errors. By implementing retries with error handling, the pipeline can automatically recover from temporary issues, ensuring data ingestion is more robust and resilient.

### Core M Functions Used
`Table.SelectRows`, `try`, `catch`, `Number.Retries`

### Code Source Execution
```powerquery
let
    // Define a function with error handling and retries
    FetchDataWithRetry = (url as text, optional maxRetries as number) =>
    let
        retries = if maxRetries = null then 3 else maxRetries,
        fetchData = () =>
        try
            Web.Contents(url)
        catch
            (e) =>
            if retries > 0 then
                FetchDataWithRetry(url, retries - 1)
            else
                error "Failed to fetch data after retries."
    in
        fetchData()
    // Usage example
    data = FetchDataWithRetry("https://example.com/api/data", 2)
in
    data
```

### Example Execution Logic Step-By-Step
This transformation works by defining a reusable function, `FetchDataWithRetry`, which takes a URL and an optional maximum number of retries. It attempts to fetch data from the specified URL using `Web.Contents`. If an error occurs, it catches the error and, if the number of retries is greater than 0, calls itself with the same URL and decrements the retry count. If all retries fail, it throws an error. This approach ensures that temporary failures do not halt the data pipeline.
