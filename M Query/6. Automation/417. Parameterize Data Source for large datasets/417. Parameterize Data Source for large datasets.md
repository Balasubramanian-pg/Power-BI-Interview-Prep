# Parameterizing Data Source for Large Datasets

**Description:**
This use case demonstrates how to parameterize a data source in Power Query (M) to efficiently handle large datasets. By parameterizing the data source, users can easily switch between different data sources or filter large datasets without having to rewrite the query. This transformation is useful when working with big data, where filtering or switching data sources is a common requirement. It allows for more flexible and reusable queries.

### Typical Scenario
In a real-world data pipeline, this transformation is useful when a business intelligence analyst needs to regularly analyze sales data from different regions. By parameterizing the data source, the analyst can easily switch between different regional datasets without modifying the query. For instance, a query can be set up to fetch sales data from a database, and by changing a parameter, the query can fetch data from different regions (e.g., North, South, East, West) without having to rewrite or duplicate the query for each region.

### Core M Functions Used
`Table.SelectRows`, `Query.Parameters`

### Code Source Execution
```powerquery
let
    // Define a parameter
    startDate = Query.Parameters{"StartDate", type date, #date(2022, 1, 1)},
    endDate = Query.Parameters{"EndDate", type date, #date(2022, 12, 31)},
    // Assume this is your data source
    dataSource = Table.FromRows({
        {"John", #date(2022, 1, 15), 100},
        {"Jane", #date(2022, 6, 15), 200}
    }, {"Name", "Date", "Sales"}),
    // Filter data based on parameters
    filteredData = Table.SelectRows(dataSource, each [Date] >= startDate and [Date] <= endDate)
    in
    filteredData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Define Parameters**: First, parameters are defined using `Query.Parameters`. These parameters can be edited by the user through the Power Query interface, allowing for interactive filtering of the data.
    2. **Data Source Setup**: A data source is set up, which could be a table, a database query, or any other type of data source that Power Query supports.
    3. **Apply Filter**: The `Table.SelectRows` function is used to filter the data based on the defined parameters. In this example, the data is filtered to only include rows where the 'Date' column falls within the range defined by the `startDate` and `endDate` parameters.
    4. **Execute Query**: When the query is executed, Power Query applies the filter based on the current values of the parameters, returning only the relevant data.
    5. **Reuse Query**: The query can be reused for different datasets or with different parameters without needing to modify the query itself, making it highly flexible and efficient for handling large datasets.
