# Auto Load New Files for Multi-Tenant Scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to automatically load new files in multi-tenant scenarios, ensuring seamless data integration and updating. It allows for dynamic data ingestion from various sources, making it ideal for environments with multiple tenants. The transformation helps in keeping the data up-to-date by automatically detecting and loading new files.

### Typical Scenario
In a real-world data pipeline, this transformation would be particularly useful in a cloud-based data warehousing scenario where multiple clients (tenants) are uploading new files regularly. For instance, a data service provider managing sales data for various retailers might use this approach to automatically ingest new sales data files uploaded by each retailer.

### Core M Functions Used
`File.Contents`, `Folder.Files`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Folder.Files("C:\Source\Files"),
    FilesToLoad = Table.SelectRows(Source, each [Attributes][&IsFolder] = false),
    CombinedData = Table.Combine(FilesToLoad[Content])
    in
    CombinedData
```

### Example Execution Logic Step-By-Step
The transformation typically works as follows inside a Power Query pipeline:
1. **Source Identification**: Identify the folder or directory where new files are being uploaded. This could be a local directory or a cloud storage location.
2. **Folder.Files Function**: Use the `Folder.Files` function to get a list of files in the specified directory. This function returns a table with file metadata, including file names, sizes, and attributes.
3. **Filtering Files**: Apply a filter to exclude folders and only select files. This can be done using `Table.SelectRows` to filter out rows where the `IsFolder` attribute is true.
4. **Content Retrieval**: For each file listed, use the `File.Contents` function to read the file contents.
5. **Data Combination**: Combine the data from all files into a single table using `Table.Combine`.
6. **Data Transformation**: Apply any necessary data transformations to the combined data.
7. **Loading Data**: Finally, load the transformed data into the desired data model or database.
