# Dynamic Date Filter for Multi-Tenant Scenarios

**Description:**
This transformation enables dynamic filtering of data based on dates for multi-tenant environments, ensuring that each tenant only accesses their relevant data.

### Typical Scenario
In a multi-tenant data warehouse, a business intelligence analyst needs to ensure that sales reports are filtered by date dynamically for each tenant, without manually adjusting the date range for every query.

### Core M Functions Used
`Table.SelectRows`, `Date.Year`, `Date.Month`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant", "Date", "Sales"},
        {"Tenant1", #date(2022, 1, 1), 100},
        {"Tenant1", #date(2022, 1, 2), 200},
        {"Tenant2", #date(2022, 1, 1), 50},
        {"Tenant2", #date(2022, 1, 3), 300}
    }),
    CurrentTenant = "Tenant1",
    FilteredData = Table.SelectRows(Source, each [Tenant] = CurrentTenant)
    in
    FilteredData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: A table containing tenant information, dates, and sales data.
    2. **Current Tenant Variable**: A variable is set to the current tenant of interest.
    3. **Filtering Data**: The `Table.SelectRows` function is used to filter the table, selecting only rows where the tenant matches the current tenant.
    4. **Result**: The filtered table is returned, showing only the data relevant to the specified tenant.
