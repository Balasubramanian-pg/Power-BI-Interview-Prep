# Building Reusable Functions with Query Folding in Power Query

**Description:**
This use case demonstrates how to create reusable functions in Power Query (M) that support query folding, enhancing performance by leveraging optimizations on the source data systems.

### Typical Scenario
In a real data pipeline, this transformation is crucial when building data ingestion processes that require dynamic filtering or data transformation based on parameters that can be folded back to the source, such as when dealing with large datasets stored in databases or data warehouses.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"Peter", 28, "UK"}
    }, {"Name", "Age", "Country"}),
    FilteredRows = Table.SelectRows(Source, each [Age] > 28)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works by defining a reusable function that takes a table and a condition as inputs. Inside the function, Table.SelectRows is used to filter the table based on the given condition. When the condition can be 'folded' back to the source (like in databases), Power Query optimizes the query execution by applying the filter at the source level, reducing the amount of data transferred and processed.
