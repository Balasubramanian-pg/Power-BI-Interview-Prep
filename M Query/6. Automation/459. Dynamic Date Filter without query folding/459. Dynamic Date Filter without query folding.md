# Dynamic Date Filter without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used for dynamic date filtering without relying on query folding. It allows for flexible date range selection, enabling dynamic updates to the data based on specified criteria. This transformation is particularly useful when dealing with large datasets where query folding may not be efficient or possible.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when you need to analyze sales data for a dynamically changing date range, such as filtering sales data for the last 30 days, a specific quarter, or a custom date range provided by the user. For instance, a sales manager might want to view sales reports for the current month, and the report needs to automatically update as the month progresses.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"2022-01-01", "John", 100},
        {"2022-01-02", "Jane", 200},
        {"2022-01-03", "John", 300},
        {"2022-01-04", "Jane", 400}
    }, {"Date", "Name", "Sales"}),
    DynamicDateFilter = (startDate, endDate) =>
        Table.SelectRows(Source, each Date >= startDate and Date <= endDate),
    FilteredTable = DynamicDateFilter(#"2022-01-01", #"2022-01-31")
    in
    FilteredTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: The process begins with a data source, for example, a table with columns for 'Date', 'Name', and 'Sales'.
    2. **Dynamic Date Filter Function**: A custom function `DynamicDateFilter` is defined. This function takes two parameters, `startDate` and `endDate`, representing the dynamic date range.
    3. **Table.SelectRows**: Inside the `DynamicDateFilter` function, `Table.SelectRows` is used to filter the table rows based on the condition that the 'Date' column falls within the specified range (`startDate` to `endDate`).
    4. **Applying the Filter**: The `DynamicDateFilter` function is then applied with specific dates (e.g., from "2022-01-01" to "2022-01-31") to produce a filtered table.
    5. **Result**: The result is a table that includes only the rows where the date falls within the specified range, demonstrating a dynamic date filter without query folding.
