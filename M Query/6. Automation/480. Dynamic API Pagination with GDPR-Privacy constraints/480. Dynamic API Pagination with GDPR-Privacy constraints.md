# Dynamic API Pagination with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used for dynamic API pagination while adhering to GDPR/privacy constraints. It enables fetching data from APIs that paginate results, ensuring compliance with privacy regulations.

### Typical Scenario
In a real data pipeline, this transformation is crucial when integrating data from external APIs that limit the number of results returned per page. For instance, fetching customer data from a CRM API that paginates responses, while ensuring that personal data handling meets GDPR standards.

### Core M Functions Used
`Web.Contents`, `Json.Document`, `Table.SelectRows`, `List.Skip`, `List.First`

### Code Source Execution
```powerquery
let
    apiUrl = "https://api.example.com/data",
    apiKey = "your_api_key",
    pageSize = 100,
    getPage = (page) =>
        Web.Contents(apiUrl & "?page=" & Number.ToText(page) & "&size=" & Number.ToText(pageSize),
            [
                Authorization = "Bearer " & apiKey
            ]),
    getJson = (page) => Json.Document(getPage(page)),
    getData = (page) => getJson(page)["data"],
    combinePages = {1..10},
    data = List.Transform(combinePages, getData),
    combinedData = List.Combine(data)
    in
        Table.FromList(combinedData, Splitter.SplitByNothing(), null)
```

### Example Execution Logic Step-By-Step
This transformation works by:
1. Defining the API URL, API key, and page size.
2. Creating a function `getPage` to fetch data from the API for a given page.
3. Using `Json.Document` to parse the JSON response.
4. Extracting the data from the JSON response.
5. Combining data from multiple pages into a single list.
6. Finally, converting the combined list into a table.
