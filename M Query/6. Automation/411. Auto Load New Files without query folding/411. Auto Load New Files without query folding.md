# Auto Load New Files without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to automatically load new files without relying on query folding. It allows for the incremental loading of new data files, which is particularly useful for data pipelines that require up-to-date information.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with continuously generated data files, such as daily or hourly logs from applications or IoT devices. For instance, a business intelligence system might need to load new sales data files daily to update dashboards and reports. By automating the loading of these new files without manual intervention, businesses can ensure their analytics and reporting are always based on the most current data.

### Core M Functions Used
`Folder.Files`, `DateTime.ToText`

### Code Source Execution
```powerquery
let
    Source = Folder.Files("C:\\Path\\To\\Files"),
    NewFiles = Table.SelectRows(Source, each DateTime.ToText([Date], "yyyy-MM-dd") = DateTime.ToText(DateTime.LocalNow(), "yyyy-MM-dd"))
    in
    NewFiles
```

### Example Execution Logic Step-By-Step
The transformation works by first accessing a folder containing files to be loaded using the Folder.Files function. It then filters these files to only include those that were created on the current day by comparing their date with the current date. This is achieved through the DateTime.ToText function, which converts dates to text for comparison. The result is a table containing only the files created on the current day, allowing for the automatic loading of new files into the data pipeline.
