# Dynamic File Path optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used for dynamic file path optimization, particularly for incremental data refresh. It allows for efficient data loading by dynamically adjusting file paths based on specific conditions or parameters.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with daily or weekly data dumps that need to be incrementally loaded into a data warehouse. For instance, a company receiving daily sales data in CSV files named by date (e.g., sales_20230401.csv) can use this approach to automatically load only new data since the last refresh.

### Core M Functions Used
`Table.SelectRows`, `File.Contents`, `Folder.Files`

### Code Source Execution
```powerquery
let
    Source = Folder.Files("C:\Data\Files"),
    // Filter files based on dynamic criteria
    FilteredFiles = Table.SelectRows(Source, each [Name] = "dynamic_file_name.csv"),
    // Read the file contents
    FileContent = [Transform = each File.Contents(CombinePaths({"C:\Data\Files", [Name]}))]
    // Further transformations...
in FilteredFiles
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Identify the Folder**: The process starts by specifying the folder path containing the files.
2. **List Files**: Use `Folder.Files` to get a table of files in the folder.
3. **Dynamic Filtering**: Apply a dynamic filter using `Table.SelectRows` to select files based on certain criteria (e.g., name, date).
4. **Read File Contents**: For each selected file, use `File.Contents` to read its contents.
5. **Transform Data**: Apply any necessary transformations to the data.
6. **Load Data**: Finally, load the transformed data into the desired destination.
