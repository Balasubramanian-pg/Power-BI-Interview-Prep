# Schedule Data Refresh for Multi-Tenant Scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to schedule data refresh for multi-tenant scenarios, enabling regular updates to tenant-specific data.

### Typical Scenario
In a multi-tenant data pipeline, each tenant requires regular data refreshes. This transformation is used to automate the refresh process, ensuring that each tenant's data is up-to-date.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformRows`, `DateTimeZone.UtcNow`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "2022-01-01"},
        {"Tenant2", "2022-01-02"}
    }, {"Tenant", "LastRefresh"}),
    RefreshDate = DateTimeZone.UtcNow(),
    UpdateLastRefresh = Table.TransformRows(Source, each {_.Tenant, RefreshDate}),
    UpdateData = Table.SelectRows(UpdateLastRefresh, each _.Tenant = "Tenant1")
    in
    UpdateData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. The Source table contains tenant data with their last refresh dates.
2. The RefreshDate variable captures the current UTC date and time.
3. The UpdateLastRefresh step updates the LastRefresh column for each tenant with the current date and time.
4. The UpdateData step filters the updated table to refresh data for a specific tenant.
