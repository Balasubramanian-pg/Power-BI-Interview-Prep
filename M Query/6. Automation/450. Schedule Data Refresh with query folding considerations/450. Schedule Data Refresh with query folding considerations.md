# Schedule Data Refresh with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to schedule data refresh with query folding considerations. It allows for efficient data refresh by leveraging query folding, which pushes data processing operations to the source database, reducing the amount of data transferred and processed.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when there is a need to regularly refresh data from a source database. For example, a company may need to daily refresh customer data from a SQL database to ensure that their analytics platform has the most up-to-date information.

### Core M Functions Used
`Table.SelectRows`, `Table.Folder`

### Code Source Execution
```powerquery
let
    Source = Sql.Database("server", "database"),
    RefreshData = Table.SelectRows(Source, each [Date] = Date.FromToday(1))
    in
    RefreshData
```

### Example Execution Logic Step-By-Step
The transformation works by first connecting to a source database using the Sql.Database function. Then, it uses Table.SelectRows to select rows from the source data where the date column matches today's date. The query folding occurs when the Table.SelectRows operation is pushed to the source database, allowing the database to filter the data before it is transferred to the Power Query pipeline.
