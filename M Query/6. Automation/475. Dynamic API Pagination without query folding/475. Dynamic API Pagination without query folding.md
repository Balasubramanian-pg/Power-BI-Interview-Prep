# Dynamic API Pagination without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used for dynamic API pagination without query folding, enabling efficient data retrieval from APIs that use pagination.

### Typical Scenario
In a real data pipeline, this transformation would be used when connecting to APIs that paginate their results, such as fetching large datasets from web services where data is split across multiple pages.

### Core M Functions Used
`Web.Contents`, `Json.Document`

### Code Source Execution
```powerquery
let
    url = "https://api.example.com/data",
    params = [pageSize = 100, page = 1],
    GetJson = (url, params) =>
        Web.Contents(url, [Query = params]),
    json = Json.Document(GetJson(url, params)),
    data = json[data],
    nextPage = (json) =>
        if json[nextPageLink] <> null then
            Json.Document(Web.Contents(json[nextPageLink]))
        else
            null,
    allData = {data},
    next = nextPage(json);
    while next <> null do
    begin
        allData &= {next[data]},
        next = nextPage(next)
    end;
    Table.FromList(allData, Splitter.SplitByNothing(), null, null, ExtraValues.Error)
```

### Example Execution Logic Step-By-Step
The transformation works by first making an initial request to the API with pagination parameters. It then checks for a next page link in the response. If a next page link exists, it makes another request to that link and continues this process until there are no more pages. All the data from each page is combined into a single table.
