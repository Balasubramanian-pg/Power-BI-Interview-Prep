# Append Multiple Tables Optimized for Incremental Refresh

**Description:**
This transformation allows for the efficient appending of multiple tables, optimized for incremental refresh, enabling real-time data updates and reducing data redundancy.

### Typical Scenario
In a sales data pipeline, daily sales data from multiple regions are appended to a central table. With incremental refresh, only new sales data from each region is added daily, reducing processing time and storage needs.

### Core M Functions Used
`Table.Combine`, `Table.SelectRows`, `Table.AddIndexColumn`

### Code Source Execution
```powerquery
let
    Source1 = Table.FromRows({
        {"ID", "Name", "Value"},

    }),
    Source2 = Table.FromRows({
        {"ID", "Name", "Value"},
        {3, "Bob", 30},
        {4, "Alice", 40}
    }),
    Combined = Table.Combine({Source1, Source2}),
    IncrementalRefresh = Table.AddIndexColumn(Combined, "Index", 1, 1)
    in
    IncrementalRefresh
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Source Tables**: Define multiple tables (e.g., `Source1`, `Source2`) containing data to be appended.
2. **Combine Tables**: Use `Table.Combine` to merge these tables into a single table (`Combined`).
3. **Incremental Refresh**: Add an index column (`Index`) to the combined table using `Table.AddIndexColumn`, which facilitates incremental refresh by tracking new records.
