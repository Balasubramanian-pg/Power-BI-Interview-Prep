# Left Join Datasets without Query Folding in Power Query

**Description:**
This transformation achieves a left join between two datasets in Power Query without relying on query folding. It is useful when the join operation cannot be optimized by the data source, ensuring that all required data is processed within Power Query.

### Typical Scenario
In a real data pipeline, this transformation is used when combining data from different sources where one dataset must be joined with another, but the data source does not support efficient query folding. For example, joining customer information from a CRM system with transaction data from a database, where the CRM system does not optimize such joins.

### Core M Functions Used
`Table.LeftJoin`, `Table.SelectRows`

```powerquery
let
    // Define the left table
    LeftTable = Table.FromList({
        {"ID", "Name"},
        {1, "John"},

    }, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    // Define the right table
    RightTable = Table.FromList({
        {"ID", "Age"},
        {1, 25},
        {2, 30}
    }, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    // Perform left join
    JoinedTable = Table.LeftJoin(LeftTable, RightTable, {{"ID", "ID"}}, JoinKind.LeftOuter)
    in
    JoinedTable
```

### Example Execution Logic Step-By-Step
This transformation works by first loading both datasets into Power Query. Then, it uses the Table.LeftJoin function to perform a left outer join on a common column (ID in this case). The result is a new table that includes all rows from the left table and matching rows from the right table. If there are no matches, the result will contain null values for the right table columns.
