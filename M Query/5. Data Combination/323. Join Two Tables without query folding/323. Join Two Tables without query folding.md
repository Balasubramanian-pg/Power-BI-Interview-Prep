# Join Two Tables without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to join two tables without query folding. It is useful when the join operation cannot be pushed to the source database, often due to limitations of the data source or the complexity of the transformation.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with data from sources that do not support query folding, such as certain web APIs or data lakes. For instance, when combining data from an external API with data from a local database, the join operation might need to be performed in-memory.

### Core M Functions Used
`Table.SelectRows`, `Table.Join`

### Code Source Execution

```powerquery
    Table1 = Table.FromRows({
        {"ID", "Name"}, {1, "John"}, {2, "Jane"}
    }),
    Table2 = Table.FromRows({
        {"ID", "Age"}, {1, 25}, {2, 30}
    }),
    JoinedTable = Table.Join(Table1, Table2, "ID")
    in
    JoinedTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows: First, two tables are created in Power Query using the Table.FromRows function. Then, the Table.Join function is used to join these tables on the 'ID' column. The result is a new table that combines columns from both tables where the 'ID' matches.
