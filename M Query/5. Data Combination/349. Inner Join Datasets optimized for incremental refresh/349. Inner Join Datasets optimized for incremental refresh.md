# Inner Join Datasets Optimized for Incremental Refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to perform an inner join on datasets optimized for incremental refresh. It enables efficient data merging while supporting incremental data updates.

### Typical Scenario
In a real data pipeline, this transformation is useful when combining data from multiple sources, such as sales transactions and customer information, where only matching records are required, and data is frequently updated.

### Core M Functions Used
`Table.Join`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name"},
        {1, "John"},
        {2, "Jane"}
    }),
    IncrementalSource = Table.FromRows({
        {"ID", "Sales"},
        {1, 100},
        {2, 200}
    }),
    JoinedTable = Table.Join(Source, IncrementalSource, "ID", "ID", JoinKind.Inner)
    in
    JoinedTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. **Source Table Creation**: A source table is created with customer data.
2. **Incremental Source Table Creation**: An incremental source table with sales data is created.
3. **Inner Join Operation**: The `Table.Join` function is used to perform an inner join on the source and incremental source tables based on the 'ID' column.
4. **Result**: The resulting joined table contains only the rows where 'ID' matches in both tables, optimized for incremental refresh.
