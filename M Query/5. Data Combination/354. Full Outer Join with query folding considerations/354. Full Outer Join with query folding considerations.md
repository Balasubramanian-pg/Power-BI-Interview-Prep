# Full Outer Join with Query Folding Considerations in Power Query

**Description:**
This transformation achieves a full outer join between two tables while considering query folding for optimization. It returns all rows from both tables, with nulls in the columns where there are no matches. This is useful for combining data from different sources while preserving all information.

### Typical Scenario
In a real data pipeline, this transformation is used when combining customer data from a CRM system with order data from an ERP system. For instance, to analyze customer interactions and orders comprehensively, including customers without orders and orders without matching customer details.
