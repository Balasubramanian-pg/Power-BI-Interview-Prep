# Inner Join Datasets with Error Handling and Retries

**Description:**
This transformation achieves combining two datasets based on a common column while handling potential errors and implementing retries for robust data processing.

### Typical Scenario
In a real data pipeline, this transformation would be used when integrating data from multiple sources, such as customer information and order details, where an inner join is required to correlate the data accurately.

### Core M Functions Used
`Table.SelectRows`, `Table.Join`, `try`, `catch`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name"},
        {1, "John"},
        {2, "Jane"},
        {3, "Joe"}
    }),
    JoinTable = Table.FromRows({
        {"ID", "Order"},
        {1, "Order1"},
        {1, "Order2"},
        {2, "Order3"}
    }),
    InnerJoin = try
        Table.Join(Source, JoinTable, "ID", "ID", JoinKind.Inner)
    catch
        (e) =>
            let
                errorMessage = "Error during join: " & e.Message,
                retryCount = 3
            in
                if retryCount > 0 then
                    InnerJoin
                else
                    Error.Record(errorMessage, "Inner Join")
    in
    InnerJoin
```

### Example Execution Logic Step-By-Step
The transformation works as follows: First, two tables are created from sample data. Then, an inner join is attempted on these tables based on the 'ID' column. If an error occurs during the join, the code catches the error, logs an error message, and retries the join up to a specified number of times (in this case, 3 retries). If all retries fail, it records the error.
