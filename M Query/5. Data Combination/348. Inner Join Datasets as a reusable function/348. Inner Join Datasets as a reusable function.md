# Inner Join Datasets as a Reusable Function

**Description:**
This transformation achieves combining two datasets based on a common column, allowing for the creation of a reusable function in Power Query (M) for efficient data merging.

### Typical Scenario
In a real data pipeline, this transformation is used when integrating customer information from a sales dataset with their order details from an orders dataset, ensuring that only customers with matching orders are included in the final analysis.

### Core M Functions Used
`Table.Join`

### Code Source Execution
```powerquery
let
    // Define the two tables
    Table1 = Table.FromRows({
        {"ID", "Name"}, 
        {1, "John"}, 
        {2, "Jane"}, 
        {3, "Bob"}
    }),
    Table2 = Table.FromRows({
        {"ID", "Order"}, 
        {1, "Order1"}, 
        {1, "Order2"}, 
        {2, "Order3"}
    }),
    // Perform inner join on ID column
    JoinedTable = Table.Join(Table1, Table2, "ID")
in
    JoinedTable
```

### Example Execution Logic Step-By-Step
The transformation works by first defining two tables, Table1 and Table2, which contain data to be merged. An inner join is then performed on these tables based on the 'ID' column, which is common to both. The result, JoinedTable, includes only the rows where the 'ID' exists in both Table1 and Table2, effectively merging the data from both tables for matching IDs.
