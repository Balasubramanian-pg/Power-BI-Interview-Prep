# Inner Join Datasets without Query Folding

**Description:**
This transformation achieves an inner join between two datasets without relying on query folding. It is useful when working with data sources that do not support query folding or when the join needs to be performed manually for specific logic. This approach allows for more control over the join process.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when integrating customer data from two separate systems: a CRM system and a marketing database. For instance, you might want to join customer information from the CRM (e.g., customer ID, name, email) with marketing data (e.g., campaign ID, interaction date) to analyze customer interactions without having the data in a single source.

### Core M Functions Used
`Table.SelectRows`, `Table.Join`

```powerquery
let

    Table2 = Table.FromList({

    // Inner join on ID
    JoinedTable = Table.Join(Table1, Table2, "ID", "ID", JoinKind.Inner)

```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Define Tables**: First, two sample tables, `Table1` and `Table2`, are created using `Table.FromList`. Each table has an 'ID' column that will be used for joining.
    2. **Perform Inner Join**: The `Table.Join` function is then used to perform an inner join on `Table1` and `Table2` based on the 'ID' column. The `JoinKind.Inner` parameter specifies that an inner join should be performed.
    3. **Result**: The resulting `JoinedTable` will contain only the rows where there are matching 'ID' values in both `Table1` and `Table2`, effectively combining the 'Name' and 'Age' information for matching IDs.
