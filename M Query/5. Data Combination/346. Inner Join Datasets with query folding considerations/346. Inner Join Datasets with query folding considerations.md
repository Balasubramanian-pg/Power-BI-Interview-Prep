# Inner Join Datasets with Query Folding Considerations

**Description:**
This transformation achieves combining two datasets based on a common column while optimizing performance through query folding. It allows for efficient data merging, enabling subsequent operations to be pushed back to the source, reducing data processing load. This is particularly useful in large-scale data integration tasks.

### Typical Scenario
In a real-world data pipeline, this transformation is used when integrating customer information from a CRM system with order details from an e-commerce platform. For instance, an organization wants to analyze customer purchase behavior by joining customer demographics with order history based on a common customer ID, optimizing the query to fold back to the source databases.

`Table.Join`, `Table.SelectRows`

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Define Source Tables**: Two tables, `Table1` and `Table2`, are created, each with a common column `ID`.
2. **Inner Join Operation**: The `Table.Join` function is used to perform an inner join on `Table1` and `Table2` based on the `ID` column.
3. **Query Folding**: The query is optimized so that operations are 'pushed back' to the source, minimizing data transfer and processing load.
4. **Result**: The resulting `JoinedTable` contains columns from both source tables, matched on the `ID` column.
