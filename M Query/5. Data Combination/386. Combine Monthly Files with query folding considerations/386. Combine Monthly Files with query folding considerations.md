# Combine Monthly Files with Query Folding Considerations

**Description:**
This transformation combines data from multiple monthly files into a single dataset while optimizing performance through query folding. It enables efficient data aggregation and analysis by reducing the amount of data that needs to be processed. This approach is particularly useful when dealing with large datasets that are updated periodically.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company receives monthly sales data in separate files and wants to combine these files into a single dataset for analysis and reporting. For instance, a sales analyst might need to aggregate sales data from different regions, and combining these monthly files enables them to perform year-to-date analysis or compare sales trends across different months.

### Core M Functions Used
`Table.Combine`, `Table.SelectRows`, `Folder.Files`

### Code Source Execution
```powerquery

    // Source folder path
    sourceFolder = "C:\Source\MonthlyFiles",

    // Get list of files in the folder
    files = Folder.Files(sourceFolder),

    combinedData = Table.Combine(
        List.Transform(files, each Table.SelectRows(File.Contents(_), {"Column1", "Column2"}))),

    transformedData = Table.TransformColumns(combinedData, {{"Column1", each _}})

in
    transformedData
```

4. **Combine Tables**: The `Table.Combine` function then merges these transformed tables into a single table.
