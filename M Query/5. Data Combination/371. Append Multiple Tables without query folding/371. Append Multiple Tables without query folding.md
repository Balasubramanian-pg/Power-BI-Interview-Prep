# Append Multiple Tables without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to append multiple tables without query folding, which is useful when dealing with tables that don't support query folding.

### Typical Scenario
In a real data pipeline, this transformation would be used when combining data from multiple sources that don't support query folding, such as appending data from different Excel files or SQL tables.

### Core M Functions Used
`Table.Combine`

### Code Source Execution
```powerquery
let
    Table1 = Table.FromRows({
        {"John", 25},
        {"Jane", 30}
    }, {"Name", "Age"}),
    Table2 = Table.FromRows({
        {"Bob", 35},
        {"Alice", 20}
    }, {"Name", "Age"}),
    CombinedTable = Table.Combine({Table1, Table2})
    in
    CombinedTable
```

### Example Execution Logic Step-By-Step
The transformation works by first creating two tables, Table1 and Table2. Then, it uses the Table.Combine function to append these tables together. The result is a new table that contains all the rows from both tables.
