# Merge Fact And Dimension for multi-tenant scenarios

**Description:**
This transformation merges fact and dimension tables for multi-tenant scenarios, enabling efficient data analysis across multiple tenants. It allows for the combination of fact data with dimensional data, providing a comprehensive view of the data. This is particularly useful in data warehousing and business intelligence applications.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with multi-tenant data sources, such as a SaaS application with multiple clients. For instance, a sales data pipeline might need to merge sales facts with customer and product dimensions, while ensuring data is properly segregated and analyzed per tenant.

`Table.SelectRows`, `Table.Join`

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Define Fact and Dimension Tables**: Identify the fact table (e.g., sales data) and dimension tables (e.g., tenant and product information).
2. **Merge Fact with Dimension**: Use `Table.Join` to merge the fact table with the first dimension table on a common key (e.g., TenantID).
3. **Merge with Additional Dimensions**: Continue merging the result with additional dimension tables on their common keys (e.g., ProductID).
4. **Result**: The final merged table contains the fact data with all relevant dimensions, ready for analysis.
