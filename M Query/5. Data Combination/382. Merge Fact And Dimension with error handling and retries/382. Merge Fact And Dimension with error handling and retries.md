# Merge Fact And Dimension with Error Handling and Retries

**Description:**
This transformation merges fact and dimension tables in Power Query while incorporating error handling and retries. It ensures data consistency and reliability by managing potential errors during the merge process. This approach is useful for handling large datasets where merge operations might fail due to temporary issues.

### Typical Scenario
In a real data pipeline, this transformation is crucial when integrating data from multiple sources. For instance, in a sales data pipeline, you might have a fact table containing sales transactions and a dimension table with product details. The transformation merges these tables based on a common key, such as product ID, while handling potential errors like missing values or network issues during data fetch.

### Core M Functions Used
`Table.Join`, `try`, `catch`, `Number.Retries`

### Code Source Execution
```powerquery
let
    FactTable = Table.FromRows({
        {1, "Product A", 100},
        {2, "Product B", 200},
        {3, "Product C", 300}
    }, {"ProductID", "ProductName", "Sales"}),
    DimensionTable = Table.FromRows({
        {1, "Product A", "Electronics"},
        {2, "Product B", "Fashion"},
        {3, "Product C", "Home Goods"}
    }, {"ProductID", "ProductName", "Category"}),
    MergedTable = try
        Table.Join(FactTable, DimensionTable, "ProductID", "ProductID", JoinKind.LeftOuter)
    catch
        [Error] =>
            let
                RetryCount = Number.Retries(3),
                RetryAttempt = if RetryCount > 0 then
                    try
                        Table.Join(FactTable, DimensionTable, "ProductID", "ProductID", JoinKind.LeftOuter)
                    catch
                        [Error] =>
                            if RetryCount = 1 then
                                error [Message = "Failed to merge tables after retries."]
                            else
                                MergedTable
                else
                    error [Message = "Exceeded maximum retries."]
            in
                RetryAttempt
in
    MergedTable
```

### Example Execution Logic Step-By-Step
This transformation works by first defining two tables: FactTable and DimensionTable. It then attempts to merge these tables on the 'ProductID' column using a left outer join. If the merge fails due to an error, it initiates a retry mechanism. The retry function attempts to re-merge the tables up to a specified number of times (in this case, 3 retries). If all retries fail, it throws an error indicating that the merge operation failed after multiple attempts.
