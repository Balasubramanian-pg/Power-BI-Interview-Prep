# Expand Nested Table with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to expand a nested table while adhering to GDPR/privacy constraints. It enables the transformation of nested data structures into a more usable format while ensuring sensitive information is handled appropriately.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with customer data that contains nested structures, such as addresses or contact information, and the organization needs to ensure that personally identifiable information (PII) is protected according to GDPR guidelines.

### Core M Functions Used
`Table.SelectRows`, `Table.ExpandRecordColumn`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", {"123 Main St", "Anytown", "USA"}}, 
        {"Jane", {"456 Elm St", "Othertown", "USA"}}
    }, {"Name", "Address"}),
    ExpandedAddress = Table.ExpandRecordColumn(Source, "Address", {"Address"}, {"Street", "City", "Country"})
    in
    ExpandedAddress
```

### Example Execution Logic Step-By-Step
The transformation works by first creating a table with a nested structure representing customer data. The Table.ExpandRecordColumn function is then used to expand the nested 'Address' column into separate columns for 'Street', 'City', and 'Country'. This process allows for easier analysis and reporting of the data while maintaining control over sensitive information.
