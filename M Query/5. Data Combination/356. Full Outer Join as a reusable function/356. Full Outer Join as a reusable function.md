# Full Outer Join as a Reusable Function in Power Query

**Description:**
This transformation achieves a full outer join between two tables, returning all rows from both tables where the join condition matches, and nulls where there are no matches.

### Typical Scenario
In a real data pipeline, a full outer join can be used to combine customer data from two different sources, such as a CRM system and a marketing database, to get a complete view of customer interactions.

`Table.Join`, `Table.SelectRows`

```powerquery
        {"B", 20},
        {"C", 30},
        {"D", 40}
    }, {"Key", "Value"}),
    FullOuterJoin = Table.Join(Table1, Table2, "Key", "Key", JoinKind.FullOuter)
    in
    FullOuterJoin
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. Two tables, Table1 and Table2, are created with sample data.
    2. The Table.Join function is used to perform a full outer join on the two tables, matching on the 'Key' column.
    3. The JoinKind.FullOuter parameter specifies that a full outer join should be performed.
    4. The resulting table contains all rows from both tables, with nulls where there are no matches.
