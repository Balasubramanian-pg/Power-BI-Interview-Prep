# Right Join Datasets for Multi-Tenant Scenarios

**Description:**
This transformation achieves combining data from two datasets based on a common key, specifically useful in multi-tenant environments where data needs to be aggregated or analyzed across different tenants.

### Typical Scenario
In a SaaS application, you have two datasets: one containing tenant information and another containing usage statistics. A right join allows you to combine these datasets to analyze usage by tenant, including tenants without any usage data.

### Core M Functions Used
`Table.Join`

### Code Source Execution
```powerquery
let
    Table1 = Table.FromRows({
        {"TenantID", "TenantName"},
        {"T1", "Tenant 1"},
        {"T2", "Tenant 2"},
        {"T3", "Tenant 3"}
    }),
    Table2 = Table.FromRows({
        {"TenantID", "Usage"},
        {"T1", 100},
        {"T2", 200}
    }),
    CombinedTable = Table.Join(Table1, Table2, "TenantID", "right outer")
in
    CombinedTable
```

### Example Execution Logic Step-By-Step
The transformation works by first loading the two datasets (Table1 and Table2) into Power Query. Table1 contains tenant information, and Table2 contains usage statistics. The Table.Join function is then used to perform a right outer join on these tables based on the 'TenantID' column. This means that all rows from Table2 (usage statistics) will be included in the resulting table, and matching rows from Table1 (tenant information) will be added. If there's no match (i.e., for tenants without usage data), the result will contain null values for the tenant information.
