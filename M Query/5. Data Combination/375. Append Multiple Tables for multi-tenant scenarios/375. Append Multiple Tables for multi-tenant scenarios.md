# Append Multiple Tables for Multi-Tenant Scenarios

**Description:**
This transformation allows for the combination of data from multiple tables, each representing a different tenant, into a single table. This is particularly useful in multi-tenant data environments where data needs to be aggregated or analyzed across multiple tenants. By appending multiple tables, users can easily integrate and analyze data from different sources.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with a multi-tenant database where each tenant has its own table of data. For instance, a service provider manages data for multiple clients, each client's data residing in a separate table. To generate a unified report or perform cross-client analysis, the data from these separate tables needs to be combined.
