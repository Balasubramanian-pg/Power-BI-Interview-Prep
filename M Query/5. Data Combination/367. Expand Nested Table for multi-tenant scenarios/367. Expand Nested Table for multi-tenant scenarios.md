# Expand Nested Table for Multi-Tenant Scenarios

**Description:**
This transformation is used to expand nested tables in a multi-tenant data environment, allowing for more granular data analysis and reporting. It enables the flattening of complex data structures, making it easier to work with data from multiple tenants.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with multi-tenant data sources, such as a database containing customer data from various tenants, each with their own nested table structures. For instance, a SaaS application may store data for multiple clients, each having their own set of orders, and these orders are nested under their respective client records.
