# Left Join Datasets for Multi-Tenant Scenarios

**Description:**
This transformation achieves combining data from multiple sources in a multi-tenant environment, ensuring all records from one dataset are included while matching records from another.

### Typical Scenario
In a SaaS application, you have a master tenant database and multiple client databases. You need to combine client-specific data with the master tenant data for reporting purposes, ensuring all client data is included, even if there are no matches in the master tenant database.

### Core M Functions Used
`Table.LeftJoin`

### Code Source Execution
```powerquery
let
    MasterTable = Table.FromRows({
        {"TenantID", "Name"},
        {"T1", "Master Data 1"},
        {"T2", "Master Data 2"},
        {"T3", "Master Data 3"}
    }),
    ClientTable = Table.FromRows({
        {"TenantID", "ClientData"},
        {"T1", "Client Data 1"},
        {"T2", "Client Data 2"},
        {"T4", "Client Data 4"}
    }),
    CombinedData = Table.LeftJoin(MasterTable, ClientTable, {"TenantID", "TenantID"})
    in
    CombinedData
```

### Example Execution Logic Step-By-Step
The transformation works by first defining two tables: MasterTable and ClientTable. A left join is then performed using Table.LeftJoin, matching rows based on the 'TenantID' column. The result includes all rows from MasterTable and matching rows from ClientTable. If there are no matches, the result set will contain null values for ClientTable columns.
