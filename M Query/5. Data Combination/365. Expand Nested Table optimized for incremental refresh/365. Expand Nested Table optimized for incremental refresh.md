# Expand Nested Table optimized for incremental refresh

**Description:**
This transformation achieves expanding a nested table in a data model to optimize it for incremental refresh, allowing for more efficient data updates. It enables the expansion of nested tables while preserving the benefits of incremental data refresh, making it ideal for handling large datasets. By doing so, it improves data processing efficiency and reduces the need for full data reloads.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with data sources that contain nested tables and require frequent updates. For instance, when integrating data from APIs that return nested JSON structures, or when working with databases that store aggregated data in nested formats. By optimizing for incremental refresh, data pipelines can focus on changed data, reducing processing time and resource usage.
