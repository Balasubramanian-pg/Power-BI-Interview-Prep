# Join Two Tables as a Reusable Function

**Description:**
This transformation allows you to combine two tables based on a common column, enabling you to merge data from different sources into a unified view. It is particularly useful for integrating data from various systems or tables. By creating a reusable function, you can easily apply this transformation across multiple queries or data pipelines.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to integrate customer information from a CRM system with sales data from an ERP system. For instance, you might have a 'Customers' table with details like customer ID, name, and address, and a 'Sales' table with customer ID, product, and sales amount. By joining these two tables on the customer ID, you can analyze sales by customer, location, or product.

In the example, we create two sample tables, 'Customers' and 'Sales', and then apply the 'JoinTables' function to join them on the 'CustomerID' column. The output will be a table that includes customer details along with their sales data.
