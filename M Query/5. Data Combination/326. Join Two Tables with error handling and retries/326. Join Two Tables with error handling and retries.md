# Join Two Tables with Error Handling and Retries in Power Query

**Description:**
This transformation joins two tables using Power Query (M) language while incorporating error handling and retries. It ensures robustness in data pipelines by managing potential errors during the join operation. This approach is useful for handling large datasets or unstable data sources.

### Typical Scenario
In a real data pipeline, this transformation is crucial when integrating data from multiple sources that may have intermittent connectivity issues or when dealing with large volumes of data that could cause timeouts. For instance, joining customer information from a CRM with sales data from an ERP system where either system might experience temporary outages.

`Table.Join`, `try`, `catch`, `Number.Retries`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({{"ID", "Name"}, {1, "John"}, {2, "Jane"}}),
    AttemptJoin = try
        Table.Join(Source, Table.FromRows({{"ID", "Sales"}, {1, "100"}, {2, "200"}}), "ID", "ID", JoinKind.Inner),
    CatchError = catch AttemptJoin with (Exception as Exception) =>
        let
            RetryCount = Number.Retries(0),
            Result = if RetryCount < 3 then
                AttemptJoin
            else
                Error.Record("Max retries exceeded", "Error occurred while joining tables.")
        in
            Result
in
    CatchError
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Source Table Creation**: A source table is created with customer IDs and names.
2. **Attempt Join**: The code attempts to join this table with another table (sales data) on the ID column using an inner join.
3. **Error Handling**: If an error occurs during the join (e.g., due to a connectivity issue), the code catches the error.
4. **Retry Mechanism**: It retries the join operation up to a specified number of times (in this case, 3 retries) if an error is encountered.
5. **Max Retries Exceeded**: If all retries fail, it logs an error message indicating that the maximum number of retries was exceeded.
