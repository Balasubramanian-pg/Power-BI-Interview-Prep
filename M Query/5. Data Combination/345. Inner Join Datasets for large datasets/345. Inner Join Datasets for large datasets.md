# Inner Join Datasets for Large Datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to perform an inner join on large datasets. The transformation achieves this by combining rows from two tables where the join condition is met, resulting in a new table with combined columns.

### Typical Scenario
In a real data pipeline, this transformation would be used when integrating data from multiple sources, such as customer information and order details, where only matching records need to be combined for analysis or reporting.

`Table.Join`


### Example Execution Logic Step-By-Step
The transformation works by first loading the two datasets (Table1 and Table2) into Power Query. Then, it uses the Table.Join function to perform an inner join on the 'ID' column, which is common to both tables. The result is a new table (InnerJoin) that contains columns from both Table1 and Table2, but only for the rows where the 'ID' matches in both tables.
