# Right Join Datasets as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used to create a reusable function for performing right joins on datasets. The transformation allows for combining data from two tables based on a common column, retaining all records from the right table and matching records from the left table.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to integrate data from two sources, ensuring that all records from the primary dataset (e.g., customer information) are included, along with matching data from a secondary dataset (e.g., order history). For instance, in a customer analytics project, you might want to right join customer demographics with their transaction history to analyze customer behavior while ensuring all customers are represented, even if they have no transaction history.

### Core M Functions Used
`Table.Join`

### Code Source Execution
```powerquery
let
    Table1 = Table.FromRows({
        {"ID", "Name"}, 
        {1, "John"}, 
        {2, "Jane"}, 
        {3, "Bob"}
    }),
    Table2 = Table.FromRows({
        {"ID", "Age"}, 
        {1, 25}, 
        {2, 30}
    }),
    RightJoin = Table.Join(Table1, Table2, "ID", "ID", JoinKind.RightOuter)
    in
    RightJoin
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Define the Datasets**: First, you define two tables, `Table1` and `Table2`, which you want to join.
2. **Specify the Join Key**: Identify the common column (`ID`) that will be used to match rows between the two tables.
3. **Apply the Right Join**: Use the `Table.Join` function with `JoinKind.RightOuter` to perform the right join. This ensures all records from `Table2` (the right table) are included, and matching records from `Table1` (the left table) are added.
4. **Result**: The resulting table contains all columns from both tables, with null values in the left table columns where there are no matches.
