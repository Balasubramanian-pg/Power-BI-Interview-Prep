# Merge Fact And Dimension without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used for merging fact and dimension tables without relying on query folding. It is useful when dealing with data sources that do not support query folding, ensuring that the merge operation is performed within the Power Query environment.

### Typical Scenario
In a real data pipeline, this transformation is typically used when integrating data from multiple sources that do not support query folding. For instance, when merging customer information (dimension) with sales data (fact) from different databases that do not optimize query performance through folding.

### Core M Functions Used
`Table.SelectRows`, `Table.Join`

### Code Source Execution
```powerquery
let
    FactTable = Table.FromRows({
        {1, 100, "Product A"},
        {2, 200, "Product B"},
        {3, 300, "Product C"}
    }, {"FactID", "SalesAmount", "ProductID"},
    "FactTable"),
    DimensionTable = Table.FromRows({
        {"Product A", "Description A"},
        {"Product B", "Description B"},
        {"Product C", "Description C"}
    }, {"ProductID", "ProductDescription"},
    "DimensionTable"),
    MergedTable = Table.Join(FactTable, DimensionTable, {"ProductID"}, {"ProductID"}, JoinKind.LeftOuter)
    in
    MergedTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. Two tables are created: FactTable and DimensionTable.
    2. The FactTable contains sales data with a reference to the product (ProductID).
    3. The DimensionTable contains product details.
    4. The Table.Join function is used to merge these tables on the ProductID column.
    5. The result is a new table that combines data from both sources based on the ProductID.
