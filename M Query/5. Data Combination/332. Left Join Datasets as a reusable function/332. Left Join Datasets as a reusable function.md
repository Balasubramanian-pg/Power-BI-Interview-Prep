# Left Join Datasets as a reusable function

**Description:**
This transformation achieves combining two datasets based on a common column, retaining all records from the left dataset and matching records from the right dataset.

### Typical Scenario
In a real data pipeline, this transformation would be used when integrating customer information from a CRM system with their order history from an e-commerce platform, ensuring all customers are included even if they have no orders.

### Core M Functions Used
`Table.LeftJoin`

### Code Source Execution
```powerquery
let
    LeftTable = Table.FromRows({
        {"ID", "Name"},
        {1, "John"},
        {2, "Jane"},
        {3, "Bob"}
    }),
    RightTable = Table.FromRows({
        {"ID", "Order"},
        {1, "Order1"},
        {1, "Order2"},
        {2, "Order3"}
    }),
    CombinedTable = Table.LeftJoin(LeftTable, RightTable, {"ID", "ID"})
    in
    CombinedTable
```

### Example Execution Logic Step-By-Step
The transformation works by first defining two tables, LeftTable and RightTable. It then uses the Table.LeftJoin function to combine these tables on the 'ID' column. The result is a new table that includes all columns from both tables, with matching records from RightTable appended to LeftTable. If there are no matches, the result will contain null values for the right table columns.
