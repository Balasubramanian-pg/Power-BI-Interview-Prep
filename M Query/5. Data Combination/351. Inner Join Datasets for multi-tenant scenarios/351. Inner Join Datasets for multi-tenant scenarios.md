# Inner Join Datasets for Multi-Tenant Scenarios

**Description:**
This transformation achieves combining data from multiple sources based on a common key, specifically designed for multi-tenant environments where data needs to be joined across different datasets.

### Typical Scenario
In a SaaS application, each customer (tenant) has their own database or dataset. When generating reports that require data from multiple tenants, an inner join across these datasets is necessary to aggregate the required information.

### Core M Functions Used
`Table.Join`

### Example Execution Logic Step-By-Step
The transformation works by first loading the datasets (tables) that need to be joined. Then, it uses the Table.Join function to perform an inner join on a common column, in this case, the 'Tenant' column. The result is a new table that contains columns from both joined tables where the tenant matches.
