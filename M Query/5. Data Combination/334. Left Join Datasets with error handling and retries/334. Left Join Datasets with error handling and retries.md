# Left Join Datasets with Error Handling and Retries

**Description:**
This transformation achieves a left join between two datasets while incorporating error handling and retries. It ensures data consistency and reliability by managing potential errors during the join operation. This approach is particularly useful when dealing with large datasets or unstable data sources.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when integrating data from multiple sources, such as customer information and transaction records. For instance, a company might want to analyze customer transactions while ensuring that all customer data is accurately linked, even if some transaction records are missing or contain errors.

        {101, 1, 100.00},

    }),
    RetryCount = 3,
    AttemptLeftJoin = (retryCount) =>
        try
            Table.LeftJoin(Source, Transactions, {{"CustomerID", "CustomerID"}}, "left.anti")
        catch
            (e) =>
                if retryCount > 0 then
                    AttemptLeftJoin(retryCount - 1)
                else
                    Error.Record("Left Join Failed", e.Message, e.StackTrace)
    LeftJoinResult = AttemptLeftJoin(RetryCount)
    in
    LeftJoinResult
