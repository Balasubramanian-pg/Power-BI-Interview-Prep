# Full Outer Join without Query Folding in Power Query

**Description:**
This transformation achieves a full outer join between two tables without relying on query folding. It is useful when the join cannot be pushed to the source and must be performed in the Power Query engine. This approach ensures that all records from both tables are included in the results, with nulls in the columns where there are no matches.

### Typical Scenario
In a real data pipeline, this transformation is used when combining data from two disparate sources that do not have a common key or when the join operation cannot be optimized by the query folding mechanism. For instance, joining customer information from a CRM system with order details from an e-commerce platform, where the join cannot be performed at the source level due to data privacy restrictions or technical limitations.

`Table.Join`, `Table.SelectRows`

```powerquery
    FullOuterJoin = Table.Join(Table1, Table2, "Key", "Key", JoinKind.FullOuter)
    in
    FullOuterJoin
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. Two tables, Table1 and Table2, are created with sample data.
    2. The Table.Join function is used with the JoinKind.FullOuter parameter to perform a full outer join on the 'Key' column.
    3. The result is a new table that includes all rows from both Table1 and Table2. Where there are matches, the values from both tables are included; where there are no matches, the result set contains null values for the respective columns.
