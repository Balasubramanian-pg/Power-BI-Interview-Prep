# Merge Sales With Customers without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to merge sales data with customer data without relying on query folding. It is useful when dealing with data sources that do not support query folding, ensuring that the merge operation is performed directly on the data retrieved from the source.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when integrating sales transactions with customer information from an external database or data warehouse that doesn't optimize query performance through folding. For instance, if a company wants to analyze sales by customer region, but the sales and customer data reside in separate systems, this approach allows for the combination of these datasets for meaningful analysis.

### Core M Functions Used
`Table.SelectRows`, `Table.Join`

### Code Source Execution
```powerquery
let
    // Sample sales table
    salesTable = Table.FromList({
        {1, "Product A", 100},
        {2, "Product B", 200},
        {3, "Product C", 300}
    }, {"SalesID", "Product", "Amount"}),
    // Sample customers table
    customersTable = Table.FromList({
        {1, "John Doe", "johndoe@example.com"},
        {2, "Jane Doe", "janedoe@example.com"}
    }, {"CustomerID", "Name", "Email"}),
    // Merge sales with customers without query folding
    mergedTable = Table.Join(salesTable, customersTable, {{"SalesID", "CustomerID"}}, {"Left.Outer"})
in
    mergedTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Define Tables**: First, two tables are defined: `salesTable` containing sales data and `customersTable` containing customer information.
2. **Merge Tables**: The `Table.Join` function is used to merge these tables based on a common key (`SalesID` and `CustomerID`). The join type is specified as `Left.Outer`, meaning all records from the `salesTable` will be included, and matching records from `customersTable` will be added if available.
3. **Result**: The resulting `mergedTable` contains sales data enriched with customer information where available, demonstrating how to effectively merge datasets without relying on the data source's query folding capabilities.
