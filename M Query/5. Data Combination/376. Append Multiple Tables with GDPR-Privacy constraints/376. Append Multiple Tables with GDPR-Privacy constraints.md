# Append Multiple Tables with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to append multiple tables while adhering to GDPR/privacy constraints. It ensures that sensitive data is handled appropriately during the data transformation process.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when combining data from multiple sources, such as customer information from different regions, while ensuring that personally identifiable information (PII) is protected according to GDPR guidelines.

### Core M Functions Used
`Table.SelectRows`, `Table.Combine`

### Code Source Execution
```powerquery
let
    // Sample tables
    Table1 = #"Table 1"({"Name", "Age"}, {{"John", 25}, {"Alice", 30}}),
    Table2 = #"Table 2"({"Name", "Age"}, {{"Bob", 35}, {"Eve", 40}}),
    // Apply GDPR/Privacy constraints
    ProtectedTable1 = Table.SelectRows(Table1, each Age > 25),
    ProtectedTable2 = Table.SelectRows(Table2, each Age < 40),
    // Append tables
    CombinedTable = Table.Combine({ProtectedTable1, ProtectedTable2})
    in
    CombinedTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Define Sample Tables**: First, two sample tables, `Table1` and `Table2`, are created for demonstration purposes.
2. **Apply GDPR/Privacy Constraints**: The `Table.SelectRows` function is used to filter rows in both tables based on conditions that mimic GDPR/privacy requirements. For instance, only rows where `Age` is greater than 25 from `Table1` and less than 40 from `Table2` are selected.
3. **Append Tables**: The `Table.Combine` function is then used to append the protected tables, `ProtectedTable1` and `ProtectedTable2`, into a single table, `CombinedTable`.
4. **Result**: The final combined table contains data from both sources, filtered according to the specified privacy constraints.
