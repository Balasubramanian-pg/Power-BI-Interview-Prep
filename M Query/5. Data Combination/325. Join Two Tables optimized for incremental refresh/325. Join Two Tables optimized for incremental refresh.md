# Join Two Tables Optimized for Incremental Refresh

**Description:**
This transformation joins two tables in a way that optimizes incremental refresh, reducing the amount of data that needs to be reloaded. It is particularly useful for large datasets where only new or updated records need to be fetched. By leveraging Power Query's M language, this approach ensures efficient data loading and refresh.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when integrating data from multiple sources, such as sales and customer information. For instance, a company might have a daily sales table and a master customer table. The daily sales table is constantly updated with new sales data, while the customer table is updated less frequently. By joining these tables optimized for incremental refresh, the company can efficiently load only the new sales data along with the relevant customer information, saving processing time and resources.
