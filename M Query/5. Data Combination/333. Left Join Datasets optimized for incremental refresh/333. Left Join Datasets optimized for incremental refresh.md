# Left Join Datasets Optimized for Incremental Refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to perform a left join on datasets optimized for incremental refresh. It allows for efficient merging of data from two sources while preserving data from the left table. This transformation is useful for combining data from different sources while maintaining data integrity.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when combining customer information from a CRM system with sales data from an ERP system. For instance, a company wants to analyze sales performance for each customer, but the CRM and ERP systems store data separately. By performing a left join, the company can create a unified view of customer data and sales performance, even if some customers do not have corresponding sales data.

### Core M Functions Used
`Table.LeftJoin`, `Table.SelectRows`

```powerquery
let

    LeftTable = Table.FromRows({
        {"Customer ID", "Customer Name"},

    RightTable = Table.FromRows({

    // Perform left join
    JoinedTable = Table.LeftJoin(LeftTable, RightTable, {{"Customer ID", "Customer ID"}})

    // Select specific columns
    Result = Table.SelectRows(JoinedTable, {"Customer ID", "Customer Name", "Sales Amount"})

in
    Result
```

    3. The result is a new table that includes all rows from LeftTable and matching rows from RightTable. If there are no matches, the result will contain null values for the right table columns.
    4. Finally, use Table.SelectRows to select specific columns from the joined table, such as 'Customer ID', 'Customer Name', and 'Sales Amount'.
