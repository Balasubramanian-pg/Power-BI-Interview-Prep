# Join Two Tables with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to join two tables while adhering to GDPR/privacy constraints. It ensures that sensitive data is handled appropriately during the join operation.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when combining customer data from different sources while ensuring compliance with GDPR. For instance, joining customer demographics with transactional data requires careful handling of personal identifiable information.

### Core M Functions Used
`Table.SelectRows`, `Table.Join`

### Code Source Execution
```powerquery
let
    Table1 = Table.FromRows({
        {"CustomerID", "Name", "Email"},
        {1, "John Doe", "john.doe@example.com"},
        {2, "Jane Doe", "jane.doe@example.com"}
    }),
    Table2 = Table.FromRows({
        {"OrderID", "CustomerID", "OrderDate"},
        {101, 1, "2022-01-01"},
        {102, 1, "2022-01-15"},
        {103, 2, "2022-02-01"}
    }),
    // Apply GDPR/privacy constraints by selecting specific columns
    ConstrainedTable1 = Table.SelectRows(Table1, each {"CustomerID", "Name"}),
    // Perform the join
    JoinedTable = Table.Join(ConstrainedTable1, Table2, "CustomerID")
in
    JoinedTable
```

### Example Execution Logic Step-By-Step
This transformation works by first loading the two tables, Table1 and Table2, into the Power Query environment. To adhere to GDPR/privacy constraints, it uses Table.SelectRows to select only non-sensitive columns from Table1. Then, it performs an inner join on the constrained Table1 and Table2 using the common 'CustomerID' column. The result is a joined table that contains the required information while minimizing the exposure of sensitive data.
