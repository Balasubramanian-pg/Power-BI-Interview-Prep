# Left Join Datasets for Large Datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to perform a left join on large datasets. The transformation achieves this by combining two tables based on a common column, preserving all records from the left table and matching records from the right table.

### Typical Scenario
In a real data pipeline, this transformation would be used when you have two large datasets and you want to combine them based on a common key, while preserving all records from the primary dataset.

### Core M Functions Used
`Table.Join`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Table1 = Table.FromRows({
        {"ID", "Name"}, 
        {1, "John"}, 
        {2, "Jane"}, 
        {3, "Bob"}
    }),
    Table2 = Table.FromRows({
        {"ID", "Age"}, 
        {1, 25}, 
        {2, 30}
    }),
    JoinedTable = Table.Join(Table1, Table2, "ID", "ID", JoinKind.LeftOuter)
    in
    JoinedTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. Two tables, Table1 and Table2, are created from sample data.
2. The Table.Join function is used to perform a left outer join on Table1 and Table2 based on the 'ID' column.
3. The result is a new table, JoinedTable, which contains all records from Table1 and matching records from Table2. If there are no matches, the result will contain null values for the right table columns.
