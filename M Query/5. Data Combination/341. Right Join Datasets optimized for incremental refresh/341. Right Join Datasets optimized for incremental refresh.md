# Right Join Datasets Optimized for Incremental Refresh

**Description:**
This transformation achieves a right join between two datasets, optimized for incremental refresh. It allows for efficient data merging while minimizing the amount of data that needs to be reloaded. This is particularly useful when dealing with large datasets that are updated periodically.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when combining customer information from a CRM system with order data from an e-commerce platform. The CRM system data is updated daily, while the e-commerce platform data is updated hourly. By using a right join optimized for incremental refresh, the pipeline can efficiently merge these datasets, ensuring that all customer information is up-to-date while minimizing the amount of data that needs to be reloaded.
