# Right Join Datasets without Query Folding

**Description:**
This transformation achieves a right join between two datasets without relying on query folding. It is useful when the join operation cannot be pushed down to the source, and the full datasets need to be retrieved before joining.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with data sources that do not support query folding, such as joining data from multiple heterogeneous sources. For instance, combining customer information from a CRM system with order details from an e-commerce platform, where the join cannot be optimized at the source level.

### Core M Functions Used
`Table.Join`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Table1 = Table.FromList({{"A", 1}, {"B", 2}, {"C", 3}}, {"Key", "Value1"}),
    Table2 = Table.FromList({{"A", 10}, {"B", 20}, {"D", 40}}, {"Key", "Value2"}),
    RightJoin = Table.Join(Table2, Table1, {"Key"}, {"Key"}, JoinKind.RightOuter)
    in
    RightJoin
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. Two tables, Table1 and Table2, are created from lists.
    2. The Table.Join function is used to perform a right outer join on Table2 and Table1 based on the 'Key' column.
    3. The result is a new table that includes all records from Table2 and matching records from Table1. If there are no matches, the result will contain null values for Table1 columns.
