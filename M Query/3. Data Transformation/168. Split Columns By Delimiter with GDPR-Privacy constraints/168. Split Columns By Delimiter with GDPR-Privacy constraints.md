# Split Columns By Delimiter with GDPR/Privacy constraints

**Description:**
This transformation splits columns by a specified delimiter while adhering to GDPR/privacy constraints, ensuring sensitive data is handled appropriately. It is crucial for anonymizing data and protecting personal identifiable information (PII).

### Typical Scenario
In a real-world data pipeline, this transformation is used when processing customer data that contains personal identifiable information (PII) like names, addresses, or phone numbers. For instance, a company might need to split a full name column into first and last names for further analysis while ensuring that the data is anonymized and privacy constraints are met.
