# Pivot Table with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to create a pivot table while considering query folding. Query folding allows Power BI and other data processing tools to push data transformations back to the source, improving performance. This transformation is useful for rotating data from a long format to a wide format, aggregating values in the process.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with large datasets that need to be aggregated and pivoted for analysis, such as sales data by region and product. For instance, a company might have a table with sales data that includes product names, regions, and sales amounts, and they want to analyze this data in a pivot table format to see total sales by product and region.

### Core M Functions Used
`Table.Pivot`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Product A", "North", 100},
        {"Product B", "North", 200},
        {"Product A", "South", 300},
        {"Product B", "South", 400}
    }, {"Product", "Region", "Sales"}),
    PivotedTable = Table.Pivot(Source, List.Distinct(Source[Region]), "Region", "Sales")
    in
    PivotedTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The source data is defined in a table format within the M query.
    2. The Table.Pivot function is used to pivot the 'Region' column, creating new columns for each unique region.
    3. The 'Sales' column values are aggregated (by default, using the Sum function) for each product and region combination.
    4. The result is a pivoted table where the data is aggregated and presented in a wide format, suitable for analysis.
