# Create Index Column without Query Folding

**Description:**
This transformation creates an index column in a table without relying on query folding. It is useful when the data source does not support query folding or when the index needs to be generated on the client-side. This approach can be particularly handy when working with data sources that do not natively support indexing or when the indexing needs to be dynamic.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when ingesting data from a source that doesn't support indexing, such as certain web APIs or file systems. For instance, when fetching data from a web API that paginates results, an index column can help track the original order of records across paginated responses.
