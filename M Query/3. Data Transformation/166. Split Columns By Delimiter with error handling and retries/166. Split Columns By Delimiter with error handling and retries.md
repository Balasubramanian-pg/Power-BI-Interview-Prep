# Split Columns By Delimiter with Error Handling and Retries

**Description:**
This transformation splits a column into multiple columns based on a specified delimiter, handling potential errors and allowing for retries. It ensures data integrity by managing exceptions during the splitting process. This is particularly useful when dealing with data that may have inconsistent formatting or missing values.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when ingesting data from various sources where a column might contain multiple values separated by a delimiter. For instance, a CSV file with a column for addresses where each address component (street, city, state, zip) is separated by a comma. The transformation helps in splitting such a column into separate columns for better analysis and processing.

### Core M Functions Used
`Table.SplitColumn`, `try`, `catch`
