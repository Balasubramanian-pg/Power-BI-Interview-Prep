# Pivot Table with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used for creating a pivot table while adhering to GDPR/privacy constraints. It enables data transformation and aggregation while ensuring sensitive information is protected.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when analyzing customer data for business insights while ensuring compliance with GDPR/privacy regulations. For instance, a company might want to analyze sales data by region without exposing individual customer information.

### Core M Functions Used
`Table.SelectRows`, `Table.Pivot`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Customer", "Region", "Sales"},
        {"John", "North", 100},
        {"Jane", "South", 200},
        {"Bob", "North", 300}
    }),
    // Apply GDPR/privacy constraints
    ProtectedData = Table.SelectRows(Source, each "Region" = "North"),
    // Create pivot table
    PivotTable = Table.Pivot(ProtectedData, List.Distinct(ProtectedData[Region]), "Sales", List.Sum)
    in
    PivotTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. Load the source data into a table.
    2. Apply GDPR/privacy constraints using Table.SelectRows to filter sensitive data.
    3. Create a pivot table using Table.Pivot to aggregate data based on the selected rows.
    4. The resulting pivot table provides insights while protecting sensitive information.
