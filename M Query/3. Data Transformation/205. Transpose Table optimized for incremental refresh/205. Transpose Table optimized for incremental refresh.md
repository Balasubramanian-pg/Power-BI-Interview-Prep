# Transpose Table optimized for incremental refresh

**Description:**
This transformation optimizes data transposition for incremental data refreshes, improving performance and efficiency in data pipelines.

### Typical Scenario
In a real-world data pipeline, this transformation is used when dealing with large datasets that require periodic updates, such as financial data or IoT sensor readings, where only new data is added and existing data remains unchanged.

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. **Data Ingestion**: The source table is ingested, which in this case is a simple table with one column for headers and multiple columns for values.
2. **Transpose Operation**: The `Table.Transpose` function is applied to flip the table structure, turning columns into rows and vice versa.
3. **Header Promotion**: After transposition, headers are promoted to properly label the new columns using `Table.PromoteHeaders`.
4. **Incremental Refresh Optimization**: For optimization with incremental refresh, only new or changed data is processed, reducing the load on the system and improving performance.
