# Create Calculated Column for Large Datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to create a calculated column for large datasets. The transformation allows for efficient data processing and analysis by adding new columns based on existing data.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when there is a need to derive new insights or metrics from existing data. For instance, in e-commerce data analysis, a calculated column could be created to determine the total cost of an order, including taxes and shipping fees.

### Core M Functions Used
`Table.SelectRows`, `Table.AddColumn`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, 1000},
        {"Mary", 31, 2000},
        {"David", 42, 3000}
    }, {"Name", "Age", "Sales"}),
    AddedCustom = Table.AddColumn(Source, "Bonus", each if [Sales] > 1500 then [Sales] * 0.1 else 0)
    in
    AddedCustom
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
    1. The source table is created with columns 'Name', 'Age', and 'Sales'. 
    2. The Table.AddColumn function is used to add a new column 'Bonus'. 
    3. The 'Bonus' column is calculated based on the 'Sales' column, where if sales are greater than 1500, the bonus is 10% of sales, otherwise it is 0.
