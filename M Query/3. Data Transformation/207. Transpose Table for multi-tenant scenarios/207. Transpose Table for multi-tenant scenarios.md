# Transpose Table for Multi-Tenant Scenarios

**Description:**
This transformation is used to transpose a table in Power Query for multi-tenant scenarios, allowing for efficient data restructuring and analysis across multiple tenants.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with multi-tenant data sources where each tenant's data is stored in a similar structure, but needs to be analyzed or reported on separately. For instance, a service provider may have multiple clients, each with their own set of data, and needs to transpose the data for each client into a standardized format for analysis.

### Core M Functions Used
`Table.SelectRows`, `Table.Transpose`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "Value1"},
        {"Tenant1", "Value2"},
        {"Tenant2", "Value3"},
        {"Tenant2", "Value4"}
    }, {"Tenant", "Value"}),
    GroupedRows = Table.Group(Source, {"Tenant"}, {{"Values", each _[Value], type text}}),
    Transposed = Table.Transpose(Table.SelectRows(GroupedRows, {"Tenant"}))
    in
    Transposed
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The source table is created with tenant data.
    2. The data is grouped by tenant.
    3. The values for each tenant are selected and transposed into a new table.
    This results in a transposed table where each row represents a tenant and each column represents a value.
