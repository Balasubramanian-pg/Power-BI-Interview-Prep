# Unpivot Table for Multi-Tenant Scenarios

**Description:**
This transformation is used to unpivot a table in a multi-tenant scenario, where data from multiple tenants is stored in a single table. The goal is to transform the data into a format that is easier to analyze and process. This is achieved by unpivoting the table, which involves rotating the data from a wide format to a long format.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with multi-tenant data, such as customer data from different regions or companies. For example, an e-commerce company may have a single table containing sales data from multiple regions, and they want to analyze the sales data for each region separately. The unpivot transformation would be used to transform the data into a format that can be easily analyzed and processed.
