# Create Calculated Column with Query Folding Considerations

**Description:**
This transformation creates a new calculated column in a Power Query table while considering query folding for efficient data processing. It allows for the addition of custom calculations to existing data, enhancing data analysis capabilities. Query folding ensures that operations are pushed back to the source, improving performance.

### Typical Scenario
In a real-world data pipeline, this transformation is used when analyzing sales data. For instance, a calculated column can be created to categorize sales transactions as 'High Value' or 'Low Value' based on a specific threshold. This categorization helps in generating reports that provide insights into sales performance across different regions or product categories.

### Core M Functions Used
`Table.AddColumn`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Category", "Product", "Sales"},
        {"Electronics", "Laptop", 1000},
        {"Electronics", "Tablet", 500},
        {"Fashion", "Shirt", 50}
    }),
    AddedCustom = Table.AddColumn(Source, "Custom Column", each if [Sales] > 500 then "High Value" else "Low Value")
    in
    AddedCustom
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Data**: Start with a table containing sales data, including columns for 'Category', 'Product', and 'Sales'.
    2. **Adding a Calculated Column**: Use the `Table.AddColumn` function to add a new column named 'Custom Column'.
    3. **Custom Calculation**: In the 'Custom Column', use a conditional statement to evaluate the 'Sales' column. If the sales value is greater than 500, categorize it as 'High Value'; otherwise, categorize it as 'Low Value'.
    4. **Query Folding**: Power Query's query folding capability optimizes the query by pushing the filtering and calculation operations back to the data source whenever possible, reducing the amount of data transferred and processed.
    5. **Result**: The resulting table includes the original columns plus the new 'Custom Column', enabling further analysis or visualization based on these categories.
