# Parse Date From Text with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to parse dates from text while adhering to GDPR/privacy constraints. It ensures that sensitive information is handled appropriately during data processing.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when processing customer data that includes date fields, such as birthdates or registration dates, while ensuring compliance with GDPR and other privacy regulations.

### Core M Functions Used
`Table.SelectRows`, `Date.FromText`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"2022-01-01"},
        {"2022-02-01"}
    }, {"DateText"}),
    ParsedDate = Table.TransformColumns(Source, {{"DateText", each Date.FromText(_, [Format="yyyy-MM-dd"])}})
    in
    ParsedDate
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. The source data is a table with a column of date text.
2. The Date.FromText function is used to parse the date text into a date format.
3. The Table.TransformColumns function applies this parsing to each row in the table.
4. The result is a new table with the parsed dates.
