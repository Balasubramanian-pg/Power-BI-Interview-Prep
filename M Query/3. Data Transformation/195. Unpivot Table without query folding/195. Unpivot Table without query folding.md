# Unpivot Table without Query Folding

**Description:**
This transformation is used to unpivot a table without relying on query folding, which can be useful when working with data sources that do not support query folding. It rearranges data from a pivoted format to a normalized, unpivoted format.

### Typical Scenario
In a real-world data pipeline, this transformation might be used when dealing with sales data that is aggregated by product and month. For instance, a table with columns for 'Product', 'Jan', 'Feb', ..., 'Dec' can be unpivoted to have a single column for months and corresponding sales amounts, making it easier to analyze and process.

### Core M Functions Used
`Table.Unpivot`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Product", "Jan", "Feb", "Mar"},
        {"Product A", 100, 200, 300},
        {"Product B", 400, 500, 600}
    }),
    Unpivoted = Table.Unpivot(Source, "Attribute", "Values", {"Jan", "Feb", "Mar"})
    in
    Unpivoted
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Table Creation**: A sample table is created with products and their sales figures for January, February, and March.
    2. **Unpivoting**: The Table.Unpivot function is used to unpivot the table. This function takes the source table, a new column name for the attribute (in this case, 'Attribute'), a new column name for the values (in this case, 'Values'), and a list of columns to unpivot.
    3. **Result**: The result is a new table where each row represents a product's sales figure for a specific month. The 'Attribute' column contains the original column headers ('Jan', 'Feb', 'Mar'), and the 'Values' column contains the corresponding sales figures.
