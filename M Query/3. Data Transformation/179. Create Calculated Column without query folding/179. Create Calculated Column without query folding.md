# Create Calculated Column without Query Folding

**Description:**
This transformation allows for the creation of a new column in a table based on calculations that cannot be pushed to the source database, ensuring data integrity and flexibility in data manipulation.

### Typical Scenario
In a real-world data pipeline, this transformation is useful when dealing with complex calculations that depend on data from multiple sources or require advanced logic that cannot be handled by the source database. For instance, calculating a customer's lifetime value based on their purchase history and demographic data.

### Example Execution Logic Step-By-Step
The transformation works by first loading the data into Power Query. Then, it uses the Table.AddColumn function to create a new column named 'Commission'. The 'each' keyword is used to specify that a calculation should be performed on each row. In this case, the commission is calculated as 10% of the sales for each individual. This new column is then added to the existing table, resulting in an updated table with the calculated column.
