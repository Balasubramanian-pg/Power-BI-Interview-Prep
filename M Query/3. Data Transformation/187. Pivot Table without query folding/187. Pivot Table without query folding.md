# Pivot Table without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to create a pivot table without query folding, allowing for more flexible data transformations.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to aggregate and pivot data from a source that doesn't support query folding, such as when dealing with complex data sources or specific business logic that can't be handled by the data source itself.

### Core M Functions Used
`Table.Pivot`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Category", "Subcategory", "Value"},
        {"A", "X", 10},
        {"A", "Y", 20},
        {"B", "X", 30},
        {"B", "Y", 40}
    }),
    PivotTable = Table.Pivot(Source, List.Distinct(Source[Category]), "Category")
in
    PivotTable
```

### Example Execution Logic Step-By-Step
The transformation works by first loading the data into a table. Then, it uses the Table.Pivot function to pivot the data based on the 'Category' column, creating new columns for each unique value in 'Category'. This process allows for a flexible pivot table creation directly within the Power Query pipeline.
