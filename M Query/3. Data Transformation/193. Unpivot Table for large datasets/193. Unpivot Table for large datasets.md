# Unpivot Table for Large Datasets

**Description:**
This transformation is used to unpivot large datasets from a wide format to a long format, making it easier to analyze and process. It rearranges data from columns to rows, which is particularly useful for handling large datasets with many columns.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when dealing with sales data that has been aggregated by product and month. For instance, a table with columns for each month (Jan, Feb, Mar, etc.) can be unpivoted to create a new table with a single column for all months, making it easier to analyze sales trends over time.

### Core M Functions Used
`Table.Unpivot`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Product", "Jan", "Feb", "Mar"},
        {"Product A", 100, 200, 300},
        {"Product B", 400, 500, 600}
    }),
    UnpivotedTable = Table.Unpivot(Source, "Attribute", "Value", {"Jan", "Feb", "Mar"})
    in
    UnpivotedTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The source table is defined with product sales data for each month.
    2. The Table.Unpivot function is used to unpivot the table, specifying the 'Attribute' column to hold the original column names (Jan, Feb, Mar) and the 'Value' column to hold the corresponding values.
    3. The result is a new table with the months in a single column and the sales values in another column, making it easier to analyze and process the data.
