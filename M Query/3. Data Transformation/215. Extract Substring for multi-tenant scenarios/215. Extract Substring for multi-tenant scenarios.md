# Extract Substring for Multi-Tenant Scenarios

**Description:**
This transformation extracts specific substrings from a string column, useful in multi-tenant environments where data needs to be filtered or processed based on substrings. It helps in organizing and analyzing data from multiple sources. The transformation enables efficient data processing and analysis by extracting relevant information.

### Typical Scenario
In a multi-tenant data pipeline, a company has a column containing URLs with tenant-specific identifiers. The transformation extracts these identifiers to filter or group data by tenant, enabling tenant-specific analysis and reporting.

### Core M Functions Used
`Text.Extract`, `Text.BetweenDelimiters`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"url1", "https://example.com/tenant1/data"},
        {"url2", "https://example.com/tenant2/data"}
    }, {"id", "url"}),
    ExtractedText = Table.TransformColumns(Source, {{"url", each Text.BetweenDelimiters(_, "https://example.com/", "/data")}})
    in
    ExtractedText
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 1) Load the data into Power Query. 2) Use the Text.BetweenDelimiters function to extract the substring between 'https://example.com/' and '/data' from the 'url' column. 3) The extracted substrings (tenant identifiers) are placed in a new column, enabling further filtering or analysis by tenant.
