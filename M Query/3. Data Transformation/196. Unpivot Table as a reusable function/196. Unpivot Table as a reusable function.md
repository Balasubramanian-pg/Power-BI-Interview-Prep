# Unpivot Table as a Reusable Function

**Description:**
This transformation achieves the unpivoting of a table, converting data from a pivoted format to a normalized format. It is commonly used to restructure data for easier analysis and processing. The unpivot function transforms data by rotating data from columns to rows.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when dealing with financial or sales data that is aggregated by multiple categories (e.g., months, products). For instance, a table with sales data organized by product across different months can be unpivoted to have a single column for product, another for month, and a third for sales amount, making it easier to analyze and process.

### Core M Functions Used
`Table.Unpivot`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Product A", 100, 200, 300},
        {"Product B", 400, 500, 600}
    }, {"Product", "Jan", "Feb", "Mar"}),
    Unpivoted = Table.Unpivot(Source, "Attribute", "Value", {"Jan", "Feb", "Mar"})
    in
    Unpivoted
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Table Creation**: A sample table is created with products across different months (Jan, Feb, Mar).
    2. **Unpivot Operation**: The Table.Unpivot function is applied to this table, specifying the 'Attribute' column to be created from the month headers and the 'Value' column to hold the corresponding values.
    3. **Result**: The resulting table has a new row for each product-month combination, effectively unpivoting the original data.
