# Pivot Table for Multi-Tenant Scenarios

**Description:**
This transformation achieves creating a pivot table that can handle data from multiple tenants, allowing for dynamic and flexible data analysis across different tenants. It enables users to easily switch between tenants and view summarized data specific to each tenant. This is particularly useful in multi-tenant environments where data needs to be segregated and analyzed on a per-tenant basis.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with data from multiple clients or tenants. For instance, a business provides services to various clients, each having their own set of data. The business needs to analyze this data in a summarized form for each client separately. A pivot table that can dynamically adjust to show data for different tenants based on the input data would be highly valuable.
