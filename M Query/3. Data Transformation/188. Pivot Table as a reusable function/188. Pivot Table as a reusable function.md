# Pivot Table as a Reusable Function in Power Query

**Description:**
This transformation achieves creating a pivot table that can be reused across different data sets. It allows for dynamic summarization and aggregation of data based on specified rows, columns, and values. This is particularly useful for data analysis and reporting where data needs to be summarized and viewed from different perspectives.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when there is a need to analyze sales data by different regions and product categories. For instance, a company wants to view total sales by region and product category, and have the flexibility to easily switch between viewing sales by region or by product category.

### Core M Functions Used
`Table.Pivot`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Category", "Subcategory", "Sales"},
        {"A", "X", 100},
        {"A", "Y", 200},
        {"B", "X", 300},
        {"B", "Y", 400}
    }),
    PivotTable = Table.Pivot(Source, List.Distinct(Source[Category]), "Category")
    in
    PivotTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: Start with a table that contains data you want to pivot. For example, a table with columns for 'Category', 'Subcategory', and 'Sales'.
    2. **Pivot Function**: Use the `Table.Pivot` function in Power Query (M language) to create a pivot table. This function takes the source table, a list of distinct values to pivot on (in this case, 'Category'), and the name of the column to aggregate ('Sales').
    3. **Dynamic Aggregation**: The pivot table allows for dynamic aggregation of 'Sales' based on 'Category'. This means you can easily switch between viewing total sales by category or by any other dimension you have in your data.
    4. **Reusable Function**: By wrapping the pivot logic in a function, you can reuse it across different data sets or within the same data pipeline, making it easier to perform repetitive analysis tasks.
