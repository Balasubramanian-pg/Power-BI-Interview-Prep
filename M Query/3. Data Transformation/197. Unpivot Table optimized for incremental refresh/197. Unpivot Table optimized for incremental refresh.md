# Unpivot Table optimized for incremental refresh

**Description:**
This transformation achieves optimizing a table for incremental refresh by unpivoting it, allowing for more efficient data updates. It rearranges data from a pivoted format to a normalized format, making it easier to manage and refresh. This process is particularly useful for handling large datasets where incremental updates are crucial for performance.

### Typical Scenario
In a real-world data pipeline, this transformation is used when dealing with sales data that is aggregated by month and product. For instance, a company receives new sales data every month and needs to update its database efficiently without reloading all the historical data. By unpivoting the table, the data can be stored in a more normalized form, with each row representing a single sales transaction, making it easier to add new monthly data.

### Core M Functions Used
`Table.SelectRows`, `Table.Unpivot`

```powerquery
let
    Source = Table.FromRows({

    Unpivoted = Table.Unpivot(Source, "Attribute", "Value", {"Jan", "Feb", "Mar"})
    in
    Unpivoted
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: Start with a pivoted table where columns represent different months (Jan, Feb, Mar) and rows represent products.
    2. **Unpivot Function**: Apply the `Table.Unpivot` function to the source table, specifying the months (Jan, Feb, Mar) as the columns to unpivot.
    3. **Result**: The function transforms the table so that each month's value becomes a separate row, with a new column (Attribute) indicating the original column name and another column (Value) containing the corresponding value.
    4. **Incremental Refresh**: With the data now in a normalized form, it's easier to implement incremental refresh, where only new or updated data for a given month needs to be added, rather than reloading the entire dataset.
