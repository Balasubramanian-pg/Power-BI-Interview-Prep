# Create Calculated Column with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to create a calculated column while adhering to GDPR/privacy constraints. It allows for the transformation of data in a way that sensitive information is protected. This is particularly useful when dealing with personal data that requires anonymization or pseudonymization.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to process customer data for analytics purposes while ensuring compliance with GDPR. For instance, a company might need to create a calculated column that categorizes customers based on their age group without directly exposing their dates of birth.

### Core M Functions Used
`Table.SelectRows`, `Table.AddColumn`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"CustomerID", "DateOfBirth"},
        {1, #date(1990, 1, 1)},
        {2, #date(1995, 6, 15)}
    }),
    AddedAgeGroupColumn = Table.AddColumn(Source, "AgeGroup", each if Date.Year(_[DateOfBirth]) >= 1990 and Date.Year(_[DateOfBirth]) < 2000 then "1990-1999" else "Other")
    in
    AddedAgeGroupColumn
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: The process starts with a table that contains customer data, including a 'DateOfBirth' column.
2. **Adding a Calculated Column**: The Table.AddColumn function is used to add a new column called 'AgeGroup'.
3. **Conditional Logic**: The 'AgeGroup' column is populated based on conditional logic that checks the year of birth. For example, if the year of birth falls between 1990 and 1999, the age group is categorized as '1990-1999'; otherwise, it's categorized as 'Other'.
4. **Result**: The resulting table includes the original columns plus the new 'AgeGroup' column, which can be used for analysis without exposing sensitive information like exact dates of birth.
