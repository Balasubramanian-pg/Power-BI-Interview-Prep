# Unpivot Table with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used for unpivoting tables while adhering to GDPR/privacy constraints. It enables data transformation while protecting sensitive information.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with user data that must be anonymized or pseudonymized. For instance, a company may need to unpivot customer data for analysis while ensuring that personally identifiable information (PII) is protected.

### Core M Functions Used
`Table.SelectRows`, `Table.Unpivot`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "Sales"},
        {"Mary", 31, "Marketing"},
        {"David", 42, "IT"}
    }, {"Name", "Age", "Department"}),
    Unpivoted = Table.Unpivot(Source, "Attribute", "Value"),
    ProtectedData = Table.SelectRows(Unpivoted, each [Attribute] <> "Name")
in
    ProtectedData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: Start with a table that needs to be unpivoted.
2. **Unpivot**: Use `Table.Unpivot` to transform the data from a pivoted state to a more normalized state.
3. **Apply GDPR/Privacy Constraints**: Use `Table.SelectRows` to filter out sensitive columns, such as names, to protect user privacy.
4. **Result**: The final output is a table that has been unpivoted and has sensitive information removed or protected.
