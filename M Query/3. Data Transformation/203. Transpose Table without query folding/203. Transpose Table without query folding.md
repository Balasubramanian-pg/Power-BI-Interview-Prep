# Transpose Table without Query Folding

**Description:**
This transformation is used to transpose a table without relying on query folding, which can be useful when dealing with data sources that do not support query folding. It rearranges the data from rows to columns or vice versa.

### Typical Scenario
In a real-world data pipeline, this transformation might be used when extracting data from a source that doesn't support query folding, such as certain web APIs or databases with limited query capabilities. For instance, if you're pulling sales data from an online store and need to convert it from a row-based format to a column-based format for easier analysis.

### Core M Functions Used
`Table.SelectRows`, `Table.Transpose`

```powerquery
let
    Source = Table.FromRows({

    }),
    Transposed = Table.Transpose(Source),
    PromotedHeaders = Table.PromoteHeaders(Transposed, [PromoteAllScalars=true])

```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Source Table Creation**: A sample table is created with two columns: 'Category' and 'Value'.
2. **Transpose**: The Table.Transpose function is used to swap the rows and columns of the table.
3. **Promote Headers**: After transposing, the first row becomes the header. The Table.PromoteHeaders function is used to promote these to headers, making it easier to reference the columns by name.
4. **Result**: The final transposed table has 'Category' as column headers and the original rows become the values under these headers.
