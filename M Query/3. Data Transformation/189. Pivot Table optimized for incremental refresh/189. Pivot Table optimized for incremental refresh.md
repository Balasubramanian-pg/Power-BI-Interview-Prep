# Pivot Table optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used for Pivot Table optimized for incremental refresh. It allows for efficient data processing and updating of pivot tables with new data.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to regularly update a pivot table with new data, such as daily or weekly sales data, without having to reprocess the entire dataset.

### Core M Functions Used
`Table.SelectRows`, `Table.Pivot`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Category", "Subcategory", "Value"},
        {"A", "X", 10},
        {"A", "Y", 20},
        {"B", "X", 30},
        {"B", "Y", 40}
    }),
    PivotTable = Table.Pivot(Source, List.Distinct(Source[Category]), "Category"),
    IncrementalRefresh = Table.SelectRows(PivotTable, each [Value] > 0)
    in
    IncrementalRefresh
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. The source data is loaded into a table.
2. The Table.Pivot function is used to create a pivot table from the source data.
3. The Table.SelectRows function is used to filter the pivot table and only include rows with values greater than 0.
4. The resulting table is then used for incremental refresh, allowing for efficient updating of the pivot table with new data.
