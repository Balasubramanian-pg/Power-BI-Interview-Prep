# Unpivot Table with Query Folding Considerations

**Description:**
This transformation is used to unpivot a table, converting columns into rows while considering query folding for efficient data processing. It is particularly useful when dealing with data that has multiple columns representing similar data points over time or across categories. By unpivoting, the data becomes more normalized and easier to analyze. This process involves selecting the appropriate columns to unpivot and then applying the transformation.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when ingesting data from a source system where metrics are captured across multiple columns. For instance, a sales report might have columns for sales amounts in different regions (North, South, East, West). By unpivoting this table, you can transform it into a more normalized form where each row represents a single sales transaction with a region identifier, making it easier to aggregate and analyze sales data across different regions.

`Table.Unpivot`, `Table.SelectRows`

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Selecting the Source Table**: The process begins with a table that needs to be unpivoted. This table typically has a structure where some columns represent identifiers or keys, and others represent values that are currently structured in a pivoted format.
2. **Applying Table.Unpivot**: The `Table.Unpivot` function is then applied to this table. This function takes three main arguments: the table to unpivot, the name of the new column that will contain the original column headers (often referred to as the 'Attribute' column), and the name of the new column that will contain the values from the original columns (often referred to as the 'Value' column). Optionally, you can specify which columns to unpivot.
3. **Result**: The result is a new table where each row represents a single observation or data point from the original pivoted structure. For example, if the original table had columns for sales in different regions, the unpivoted table would have rows for each region's sales, with identifiers for the region and the sales amount.
4. **Query Folding Considerations**: When working with large datasets or databases that support query folding, it's crucial to structure the transformation in a way that allows the data source to perform as much of the computation as possible. This can significantly improve performance by reducing the amount of data that needs to be transferred and processed in-memory.
