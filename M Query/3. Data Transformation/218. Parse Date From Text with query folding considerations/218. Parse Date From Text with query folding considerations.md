# Parse Date From Text with Query Folding Considerations

**Description:**
This transformation extracts date information from text data, considering query folding for efficient data processing. It enables the conversion of text representations of dates into a standard date format, facilitating date-based analysis and filtering. This process is crucial for handling data from various sources where dates might be represented differently.

### Typical Scenario
In a real-world data pipeline, this transformation is used when ingesting data from multiple sources, such as customer feedback forms, sales invoices, or social media posts, where date information is provided in text format and needs to be standardized for analysis. For instance, a company might receive sales data from different regions, with each region's data containing date information in a different text format. This transformation would be applied to parse these text dates into a uniform date format for further analysis.

### Core M Functions Used
`Date.FromText`, `Table.TransformColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"2022-01-01"},
        {"2022/02/02"},
        {"2022.03.03"}
    }, {"DateText"}),
    ParsedDate = Table.TransformColumns(Source, {{"DateText", each Date.FromText(_, [Format="yyyy-MM-dd"], [Style=DateTimeFormat.StyleEurope])}})
in
    ParsedDate
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Ingestion**: The process begins with ingesting data, typically in a table format, where one of the columns contains date information in text format.
    2. **Application of Date.FromText**: The `Date.FromText` function is applied to the text date column. This function converts text representations of dates into actual date formats that can be used in date-based calculations.
    3. **Specifying Date Format**: The format of the date text is specified to ensure accurate conversion. For example, if the text date is in the format 'dd/mm/yyyy', this format needs to be specified.
    4. **Query Folding**: When possible, Power Query applies query folding, which means that the operations (in this case, parsing dates) are pushed back to the source, if the source supports it. This can significantly improve performance by reducing the amount of data that needs to be transferred and processed.
    5. **Result**: The output is a table where the date information is now in a standard date format, ready for filtering, sorting, and analysis.
