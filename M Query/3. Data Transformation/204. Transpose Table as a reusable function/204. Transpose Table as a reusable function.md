# Transpose Table as a Reusable Function in Power Query

**Description:**
This transformation achieves the conversion of a table from rows to columns or vice versa, making it easier to analyze and process data. It is particularly useful when dealing with datasets that have a matrix-like structure. By transposing a table, users can transform data into a more suitable format for analysis or reporting. This process involves swapping the rows and columns of a table.

### Typical Scenario
In a real-world data pipeline, transposing a table can be crucial when integrating data from different sources. For instance, if a database stores sales data with product IDs as rows and months as columns, but a reporting tool requires the data to be structured with months as rows and product IDs as columns, a transpose operation becomes necessary. This transformation enables seamless data integration and analysis across different systems.

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Define the Source Table**: The process begins with defining a source table that needs to be transposed. This table is created using the `Table.FromRows` function, which takes a list of lists (where each inner list represents a row in the table) and an optional list of column names.
    2. **Apply Table.Transpose**: The `Table.Transpose` function is then applied to the source table. This function swaps the rows and columns of the table. The first row of the original table becomes the column headers in the transposed table, and each subsequent row in the original table becomes a row in the transposed table.
    3. **Result**: The result is a new table where the original rows are now columns and vice versa. This transposed table can then be used for further data analysis or transformation within the Power Query pipeline.
