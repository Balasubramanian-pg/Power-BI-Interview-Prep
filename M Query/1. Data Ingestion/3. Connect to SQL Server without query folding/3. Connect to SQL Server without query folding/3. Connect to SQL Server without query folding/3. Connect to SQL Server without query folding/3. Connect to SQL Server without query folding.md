# Connect to SQL Server without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to SQL Server without query folding, allowing for more flexible data retrieval and processing.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to perform complex data processing or calculations on the client-side that cannot be pushed to the SQL Server for execution, such as data cleansing, complex aggregations, or joins that are not supported by SQL Server.

### Core M Functions Used
`Sql.Database`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    sqlServerDatabase = Sql.Database("your_server_name", "your_database_name"),
    yourTable = sqlServerDatabase{[Name="your_table_name"]}[Data]
    in
    yourTable
```

### Example Execution Logic Step-By-Step
The transformation works by first establishing a connection to the SQL Server database using the Sql.Database function. Then, it accesses a specific table from the database. The Table.SelectRows function can then be used to filter or transform the data without relying on SQL Server to perform these operations, thus avoiding query folding.
