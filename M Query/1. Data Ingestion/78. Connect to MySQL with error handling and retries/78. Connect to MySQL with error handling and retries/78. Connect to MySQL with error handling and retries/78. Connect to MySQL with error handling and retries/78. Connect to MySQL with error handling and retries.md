# Connect to MySQL with error handling and retries

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a MySQL database with error handling and retries. It provides a robust method for handling transient errors that may occur during database connections, ensuring reliable data retrieval.

### Typical Scenario
In a real data pipeline, this transformation would be used when connecting to a MySQL database that may experience intermittent connectivity issues, such as a database server that is occasionally restarted or experiences network outages.

### Core M Functions Used
`Table.SelectRows`, `Sql.Database`, `Error.Message`

### Code Source Execution
```powerquery
let
    // MySQL connection details
    server = "your_server_name",
    port = 3306,
    database = "your_database_name",
    username = "your_username",
    password = "your_password",

    // Retry policy
    maxRetries = 3,
    retryDelay = 5000, // 5 seconds

    // MySQL connection function with error handling
    ConnectToMySQL = () =>
        try
            Sql.Database(server, port, ["database" = database, "user" = username, "password" = password])
        catch
            error as ErrorRecord =>
                if Error.Message(error) = "The server has not responded within the specified timeout period." then
                    if maxRetries > 0 then
                        (
                            Thread.Sleep(retryDelay),
                            @ConnectToMySQL()
                        )
                    else
                        error
                else
                    error,

    // Execute the connection
    mysqlData = ConnectToMySQL()
    in
    mysqlData
```

### Example Execution Logic Step-By-Step
This script works by defining a function `ConnectToMySQL` that attempts to connect to a MySQL database using the `Sql.Database` function. If the connection attempt fails with a timeout error, it waits for a specified delay and retries the connection up to a maximum number of retries. If all retries fail or a different error occurs, it returns the error.
