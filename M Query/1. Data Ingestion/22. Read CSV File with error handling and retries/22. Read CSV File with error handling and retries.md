# Read CSV File with Error Handling and Retries

**Description:**
This transformation reads a CSV file while implementing error handling and retries, ensuring robustness in data pipelines. It is useful for handling transient errors such as file locking or network issues.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to reliably import data from CSV files that may occasionally be inaccessible due to temporary errors.

### Core M Functions Used
`Table.SelectRows`, `try`, `catch`, `retry`

### Code Source Execution
```powerquery
let
    Source = try
        Csv.Document(File.Contents("C:\Path\To\File.csv"), [Delimiter=",", Encoding=1252]),
    ErrorHandling = catch
        ErrorRecord as Error,
        [Message = "Error reading file: " & Error.Message]
    RetryPolicy = [RetryCount = 3, Delay = Duration.FromSeconds(10)]
    in
        if Source = null then
            Retry(RetryPolicy, () => Source)
        else
            Source
```

### Example Execution Logic Step-By-Step
The transformation works by first attempting to read the CSV file using Csv.Document. If an error occurs, it catches the error and retries the operation up to a specified number of times with a delay between retries. If all retries fail, it returns the error message.
