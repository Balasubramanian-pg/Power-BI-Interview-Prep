# Import Excel Workbook optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to import an Excel workbook optimized for incremental refresh. It allows for efficient data loading by leveraging Excel's built-in data storage and Power Query's transformation capabilities.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when there is a need to regularly update data stored in an Excel workbook. For instance, a marketing team might use an Excel workbook to track campaign performance metrics on a daily basis. By setting up an incremental refresh, the team can ensure that their data pipeline stays up-to-date without having to reload the entire workbook each time.

### Core M Functions Used
`Table.SelectRows`, `Excel.Workbook`

### Code Source Execution
```powerquery
let
    Source = Excel.Workbook(File.Contents("C:\Path\To\Workbook.xlsx"), null, true),
    Data = Source{[0]}{[Data]},
    IncrementalRefresh = Table.SelectRows(Data, each [Date] >= Date.AddDays(DateTime.FixedLocalNow(), -7))
    in
    IncrementalRefresh
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. The Excel workbook is loaded using the `Excel.Workbook` function.
2. The data table is selected from the workbook contents.
3. The `Table.SelectRows` function is used to filter the data for incremental refresh, typically selecting rows based on a date column.
4. The filtered data is then loaded into the Power Query pipeline for further transformations or analysis.
