# Connect to REST API for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a REST API in multi-tenant scenarios, enabling data retrieval from various tenants.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when integrating data from multiple tenants of a SaaS application, such as retrieving sales data from different customers' accounts.

### Core M Functions Used
`Web.Contents`, `Json.Document`

### Code Source Execution
```powerquery
let
    baseUrl = "https://api.example.com/",
    tenantId = "your_tenant_id",
    apiKey = "your_api_key",
    headers = [#"Authorization" = "Bearer " & apiKey],
    url = baseUrl & "data/" & tenantId,
    data = Web.Contents(url, [Headers=headers]),
    jsonData = Json.Document(data)
    in
    jsonData
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 1) define the base URL of the REST API and the tenant ID, 2) construct the API request with authentication headers, 3) use Web.Contents to send a GET request to the API, 4) parse the JSON response using Json.Document, and 5) return the parsed data.
