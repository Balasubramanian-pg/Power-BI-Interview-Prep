# Read CSV File optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to read a CSV file in a way that optimizes incremental refresh. It allows for efficient data loading by only refreshing the changed data.

### Typical Scenario
In a real data pipeline, this transformation would be used when you have a large CSV file that is frequently updated, and you want to minimize the amount of data that needs to be reloaded. For example, in a daily sales report, you might only need to update the sales data for the current day.

### Core M Functions Used
`Table.SelectRows`, `Csv.Document`, `Date.Year`

### Code Source Execution
```powerquery
let
    Source = Csv.Document(File.Contents("C:\Path\To\File.csv"), [Delimiter=",", Encoding=1252, QuoteStyle=QuoteStyle.None]),
    IncrementalRefresh = Table.SelectRows(Source, each Date.Year([Date]) = Date.Year(DateTime.LocalNow()))
    in
    IncrementalRefresh
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. The `Csv.Document` function reads the CSV file.
2. The `Table.SelectRows` function filters the rows to only include those where the year of the date column matches the current year.
3. This allows for incremental refresh by only reloading the data for the current year.
