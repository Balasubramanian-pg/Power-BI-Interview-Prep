# Reading CSV Files for Large Datasets using Power Query (M)

**Description:**
This use case demonstrates how to efficiently read CSV files for large datasets using Power Query (M). It enables data engineers to handle big data by leveraging the capabilities of Power Query to process and transform data.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with large datasets stored in CSV files. For instance, when a company needs to analyze customer feedback stored in a massive CSV file, they can use Power Query (M) to read the file, apply necessary transformations, and then load the data into a database or data warehouse for further analysis.

### Core M Functions Used
`Csv.Document`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Csv.Document(File.Contents("C:\Path\To\Your\File.csv"), [Delimiter=",", Encoding=1252, QuoteStyle=QuoteStyle.None]),
    // Example of selecting specific rows
    SelectedRows = Table.SelectRows(Source, "= each [ColumnName] > 0"),
    // Additional transformations can be applied here
    // For demonstration, let's assume we rename a column
    RenamedColumns = Table.RenameColumns(SelectedRows, {{ "ColumnName", "NewColumnName" }})
    in
    RenamedColumns
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Read CSV File**: The `Csv.Document` function is used to read the contents of a CSV file. This function takes the file path and several optional parameters such as delimiter, encoding, and quote style.
2. **Select Specific Rows**: The `Table.SelectRows` function can be used to filter rows based on a condition. This is useful for handling large datasets where you might only need a subset of the data.
3. **Apply Additional Transformations**: You can apply additional transformations as needed, such as renaming columns using `Table.RenameColumns`, filtering, grouping, or adding custom columns.
4. **Load Data**: Finally, the transformed data can be loaded into a database, data warehouse, or used for further analysis in Power BI or Excel.
