# Connect to MySQL with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a MySQL database while adhering to GDPR/Privacy constraints. It ensures that sensitive data is handled appropriately, and only authorized personnel have access to specific data. This transformation is crucial for organizations that need to comply with data protection regulations. By applying GDPR/Privacy constraints, businesses can protect personal data and maintain customer trust.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when an organization needs to extract data from a MySQL database for analytics or reporting purposes while ensuring compliance with GDPR/Privacy regulations. For instance, a company might want to analyze customer data stored in MySQL for marketing purposes, but it must do so in a way that anonymizes personal identifiable information (PII) and adheres to data protection laws.

`Table.SelectRows`, `Sql.Database`

```powerquery
    table_with_gdpr_data = mysql_db{[Name = "TableWithGDPRData"]}[Data],

    // Apply GDPR/Privacy constraints, e.g., filter out sensitive data or anonymize it
    filtered_data = Table.SelectRows(table_with_gdpr_data, each [SensitiveColumn] = "Public" or [AccessLevel] = "Allowed")

in
    filtered_data
```

This transformation works by first establishing a secure connection to the MySQL database using the Sql.Database function.
