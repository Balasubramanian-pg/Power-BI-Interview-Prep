# Import SharePoint File for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to import SharePoint files in multi-tenant scenarios, enabling data retrieval from SharePoint files across different tenants.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when an organization has multiple SharePoint tenants and needs to aggregate data from files stored in each tenant. For instance, a company with multiple subsidiaries, each having its own SharePoint tenant, can use this transformation to import files from each tenant and consolidate the data for reporting or analysis purposes.

### Core M Functions Used
`Table.SelectRows`, `Web.Contents`, `SharePoint.Files`

### Code Source Execution
```powerquery
let
    // Specify SharePoint site URL and file path
    siteUrl = "https://yourtenant.sharepoint.com",
    filePath = "sites/your-site/Shared Documents/your-file.csv",

    // Get access token for authentication
    accessToken = "your_access_token",

    // Construct SharePoint file URL
    fileUrl = siteUrl & "/" & filePath,

    // Use Web.Contents to get file contents
    fileContents = Web.Contents(fileUrl, [
        Headers = [#"Authorization" = "Bearer " & accessToken]
    ]),

    // Convert file contents to table
    fileTable = Csv.Document(fileContents, [Delimiter=",", Encoding=1252, QuoteStyle=QuoteStyle.None]),

    // Select specific rows
    selectedRows = Table.SelectRows(fileTable, "Column1")

    // Return the imported table
    selectedRows
```
