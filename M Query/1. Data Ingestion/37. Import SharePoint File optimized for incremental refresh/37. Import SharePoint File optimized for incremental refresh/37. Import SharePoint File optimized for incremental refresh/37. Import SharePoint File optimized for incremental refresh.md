# Import SharePoint File optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to import SharePoint files in a way that optimizes incremental data refreshes. It allows for efficient updates by only reloading data that has changed, rather than the entire dataset.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when regularly updating data from a SharePoint file. For example, a company might use SharePoint to store sales data that is updated daily. By using incremental refresh, the company can ensure that their reports and analyses are always up-to-date without having to reload the entire dataset every time.

### Core M Functions Used
`Table.SelectRows`, `SharePoint.Files`, `Incremental.Refresh`

### Code Source Execution
```powerquery
let
    Source = SharePoint.Files("https://your-sharepoint-site.sharepoint.com", [ApiVersion = "auto"]),
    Folder = Source{[Name="YourFolderName"]}[Content],
    File = Folder{[Name="YourFileName.csv"]}[Content],
    CsvContent = File{[Extension="csv"]},
    ImportedTable = Table.FromRows(CsvContent, type text),
    IncrementalRefresh = Table.SelectRows(ImportedTable, each [Date] >= Date.AddDays(DateTime.LocalNow(), -7))
    in
    IncrementalRefresh
```

### Example Execution Logic Step-By-Step
This transformation works by first connecting to the SharePoint site and selecting the specific file to import. The M code uses the SharePoint.Files function to access the SharePoint site and then navigates to the specific folder and file. The CsvContent variable holds the content of the file. The IncrementalRefresh variable uses Table.SelectRows to filter the imported table and only include rows where the date column is within the last 7 days, demonstrating an incremental refresh approach.
