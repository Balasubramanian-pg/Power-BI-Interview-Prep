# Connect to SQL Server for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to SQL Server in multi-tenant scenarios, enabling data retrieval from multiple tenants.

### Typical Scenario
In a real data pipeline, this transformation would be used when integrating data from multiple clients or tenants, each with their own SQL Server database, requiring a scalable and dynamic connection approach.

### Core M Functions Used
`Sql.Database`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    // Define your SQL Server connection details
    server = "your_server_name",
    database = "your_database_name",
    // Define a list of tenant IDs
    tenantIds = {"tenant1", "tenant2"},
    // Function to connect to SQL Server for each tenant
    getTenantData = (tenantId) =>
        Sql.Database(server, database, [Query="SELECT * FROM your_table WHERE tenant_id = '" & tenantId & "'"]),
    // Loop through each tenant and get their data
    data = List.Transform(tenantIds, each getTenantData(_)),
    // Combine the data from all tenants
    combinedData = Table.Combine(data)
    in
    combinedData
```

### Example Execution Logic Step-By-Step
This transformation works by first defining a function that connects to a SQL Server database for a given tenant ID. It then applies this function to a list of tenant IDs, retrieves data for each tenant, and finally combines the data from all tenants into a single table.
