# Connect to REST API optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a REST API and optimize it for incremental refresh, allowing for efficient data updates.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when connecting to an API that provides frequently updated data, such as IoT sensor readings or financial transactions, to minimize the amount of data that needs to be reloaded.

### Core M Functions Used
`Web.Contents`, `Json.Document`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    baseUrl = "https://api.example.com/data",
    optionalHeaders = [#"Authorization" = "Bearer your_token"]
    data = Web.Contents(baseUrl, [Headers = optionalHeaders]),
    jsonData = Json.Document(data),
    tableData = jsonData[data]
    in
    tableData
```

The transformation works by first defining the base URL of the REST API and any optional headers, such as an authorization token. It then uses the Web.Contents function to retrieve the data from the API. The Json.Document function is used to parse the JSON response, and finally, the Table.SelectRows function can be used to filter and shape the data as needed.
