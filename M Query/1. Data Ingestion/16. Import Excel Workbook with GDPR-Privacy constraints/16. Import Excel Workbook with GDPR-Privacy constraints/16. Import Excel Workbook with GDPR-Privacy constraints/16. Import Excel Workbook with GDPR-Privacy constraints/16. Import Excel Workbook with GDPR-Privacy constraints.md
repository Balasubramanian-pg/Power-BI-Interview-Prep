# Import Excel Workbook with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used for importing an Excel workbook while adhering to GDPR/privacy constraints. It ensures that sensitive data is handled appropriately during the import process.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to import data from an Excel workbook that contains sensitive information, such as personal data, financial information, or confidential business data. For instance, a company may need to import customer data from an Excel file while ensuring that the data is anonymized or pseudonymized to comply with GDPR regulations.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Excel.Workbook(File.Contents("C:\Path\To\Workbook.xlsx"), null, true),
    Data = Source{[Name="Sheet1"]}[Content],
    // Apply GDPR/privacy constraints
    AnonymizedData = Table.SelectRows(Data, each [Age] > 18 and [Country] = "France")
    in
    AnonymizedData
```
