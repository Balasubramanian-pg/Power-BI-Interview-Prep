# Connect to PostgreSQL optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a PostgreSQL database and optimize the data retrieval process for incremental refresh. This transformation enables efficient data loading by retrieving only the changed data since the last refresh.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when building a data warehouse or business intelligence dashboard that relies on up-to-date data from a PostgreSQL database. For instance, a company might use this transformation to load sales data from their PostgreSQL database into a data warehouse, ensuring that the data is updated daily without reloading the entire dataset.
