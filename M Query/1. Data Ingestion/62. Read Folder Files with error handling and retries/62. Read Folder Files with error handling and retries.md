# Read Folder Files with Error Handling and Retries

**Description:**
This use case demonstrates how to read files from a folder with error handling and retries using Power Query (M). It provides a robust method for fetching data from file systems, handling potential errors, and retrying failed attempts.

### Typical Scenario
In a real data pipeline, this transformation is crucial when ingesting data from file systems where files may be temporarily unavailable or locked by other processes. For instance, when setting up a daily data ingestion pipeline from a shared network folder, the pipeline might encounter files that are still being written to or are temporarily inaccessible. The transformation ensures that the pipeline can gracefully handle such situations by retrying the file reads.

### Core M Functions Used
`Folder.Files`, `ErrorHandling`, `RetryPolicy`

### Code Source Execution
```powerquery
let
    Source = Folder.Files("C:\\Path\\To\\Folder"),
    RetryPolicy = [MaxRetries = 3, Delay = #duration(0,0,10,0)],
    ErrorHandling = (error as Error) =>
        if error[Message] = "File not found" then
            RetryPolicy
        else
            error,
    Result = List.Transform(Source, each try
        File.Contents([Content])
    catch
        (e as Error) => ErrorHandling(e))
    in
    Result
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Folder.Files**: This function is used to get a list of files in a specified folder.
2. **Retry Policy**: A retry policy is defined with a maximum of 3 retries and a delay of 10 seconds between retries.
3. **Error Handling**: An error handling function is defined to catch specific errors (e.g., file not found) and apply the retry policy.
4. **List.Transform**: This function is used to transform each file in the list by attempting to read its contents.
5. **Try-Catch Block**: A try-catch block is used within the transformation to catch any errors that occur during file reading and apply the error handling logic.
