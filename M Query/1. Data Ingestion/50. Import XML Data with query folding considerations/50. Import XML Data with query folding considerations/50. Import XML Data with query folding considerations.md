# Import XML Data with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to import XML data while considering query folding. Query folding allows Power Query to push data processing operations back to the source, improving performance. This transformation is useful when working with large XML files and optimizing data import.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when ingesting large volumes of XML data from an external source, such as an API or file system. For instance, a company might receive daily XML updates from a partner, and they need to efficiently import this data into their data warehouse for analysis.

### Core M Functions Used
`Xml.Document`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Xml.Document(File.Contents("C:\Path\To\Example.xml")),
    Root = Source{[Root="Root"]}[Content],
    TableFromList = Table.FromList(Root, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    ExpandedTable = Table.ExpandRecordColumn(TableFromList, "TableFromList", {"Name", "Value"}, {"Name", "Value"})
    in
    ExpandedTable
```
