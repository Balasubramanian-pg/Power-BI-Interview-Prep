# Connect to PostgreSQL for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a PostgreSQL database in a multi-tenant environment, enabling data retrieval and transformation from multiple tenants.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when integrating data from multiple tenants in a PostgreSQL database, such as in a Software as a Service (SaaS) application where each customer has their own database or schema.

### Core M Functions Used
`PostgreSQL.Database`, `Table.SelectRows`

```powerquery
    // Optional: specify the port if it's not the default 5432
    port = 5432,
    // Optional: specify the tenant ID or other parameters
    tenantId = "your_tenant_id",

    // Connect to PostgreSQL database
    postgresqlDb = PostgreSQL.Database(server, database, [Username = username, Password = password, Port = port]),

in
    filteredData
```
