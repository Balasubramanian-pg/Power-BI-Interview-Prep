# Import Excel Workbook with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to import an Excel workbook while considering query folding. Query folding allows Power Query to push data processing operations back to the source, improving performance. This transformation is useful when working with large Excel workbooks and optimizing data retrieval.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a data analyst needs to import data from an Excel workbook stored on a network drive. For instance, a monthly sales report in Excel needs to be imported into a data warehouse for further analysis. By using query folding, the data processing can be optimized, reducing the amount of data transferred and processed.

### Core M Functions Used
`Table.SelectRows`, `Excel.Workbook`

### Code Source Execution
```powerquery
let
  Source = Excel.Workbook(File.Contents("C:\Path\To\Workbook.xlsx"), null, true),
  Sheet1 = Source{[0]}
  in
  Sheet1
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
  1. The Excel workbook is accessed using the `Excel.Workbook` function, which returns a table containing a row for each sheet in the workbook.
  2. The `File.Contents` function is used to read the Excel file from a specified path.
  3. The `Table.SelectRows` function can be used to filter specific rows or sheets from the workbook, demonstrating query folding.
  4. The result is a table representing the selected sheet, which can then be further processed in the Power Query pipeline.
