# Connect to REST API with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a REST API while considering query folding. Query folding allows Power Query to push filtering and other operations down to the source, reducing the amount of data that needs to be retrieved and processed. This can improve performance and efficiency when working with large datasets.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when connecting to a REST API that supports query folding, such as a data warehouse or a business intelligence platform. For example, when retrieving sales data from an e-commerce platform, the pipeline could use query folding to filter the data by date range, product category, and geographic region, reducing the amount of data that needs to be transferred and processed.

### Core M Functions Used
`Table.SelectRows`, `Web.Contents`

### Code Source Execution
```powerquery
let
    baseUrl = "https://api.example.com/data",
    query = [
        "$filter": "date gt '2022-01-01' and date lt '2022-12-31'",
        "$select": "product, sales"
    ],
    options = [Query = query],
    data = Web.Contents(baseUrl, options)
    in
        Table.SelectRows(Json.Document(data), {"product", "sales"})
```
