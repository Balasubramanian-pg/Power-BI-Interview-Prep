# Connect to SQL Server with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to SQL Server while leveraging query folding for optimized data retrieval. Query folding allows Power Query to push data processing operations down to the source database, reducing the amount of data transferred and improving performance. This approach is particularly useful when working with large datasets.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when connecting to a SQL Server database to retrieve specific data. For example, when building a data warehouse, you might use Power Query to connect to a SQL Server database, apply filters, and retrieve only the necessary data, reducing the load on the database and improving data transfer times.

### Core M Functions Used
`Sql.Database`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    sqlServerDatabase = Sql.Database("your_server_name", "your_database_name"),
    yourTable = sqlServerDatabase{[Name="your_table_name"]}[Content]

    yourTable
```

Here's a step-by-step explanation of how the script works:

    3. The database connection is established, and the list of tables in the database is retrieved.
