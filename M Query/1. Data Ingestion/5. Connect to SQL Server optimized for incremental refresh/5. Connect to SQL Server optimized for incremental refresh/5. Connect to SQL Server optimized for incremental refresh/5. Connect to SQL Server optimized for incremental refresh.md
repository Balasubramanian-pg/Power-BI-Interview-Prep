# Connect to SQL Server optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a SQL Server database and optimize the data retrieval process for incremental refresh. This approach enables efficient data loading by only retrieving new or updated records.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when building a data warehouse or business intelligence solution that relies on incremental data loads from a SQL Server database. For instance, a company might use this approach to load sales data from their SQL Server transactional database into a data warehouse for analysis, ensuring that only new sales records are loaded, rather than the entire dataset.

The transformation works as follows:
1. Connect to the SQL Server database using the Sql.Database function.
2. Define the table name and a SQL query that selects data based on a timestamp column, filtering for records newer than the last load time.
3. Use the Sql.Database.Query function to execute the SQL query with the last load time parameter.
4. The result is a table with only the new or updated records, which can then be loaded into the data warehouse or used for further analysis.
