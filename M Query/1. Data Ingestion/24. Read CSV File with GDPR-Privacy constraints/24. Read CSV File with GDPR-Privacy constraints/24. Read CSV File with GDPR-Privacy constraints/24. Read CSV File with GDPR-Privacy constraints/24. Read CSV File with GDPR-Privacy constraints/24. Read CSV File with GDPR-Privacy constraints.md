# Read CSV File with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to read a CSV file while adhering to GDPR/privacy constraints. It ensures that sensitive data is handled appropriately, and only authorized personnel have access to it.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when ingesting customer data from a CSV file, where certain columns contain sensitive information such as personal identifiable information (PII). For instance, a company may need to process customer data for marketing purposes while ensuring that sensitive information is protected and only accessible to authorized personnel.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Csv.Document(File.Contents("C:\Path\To\File.csv"), [Delimiter=",", Encoding=1252]),
    SelectRows = Table.SelectRows(Source, each [column_name] = "some_value")
    in
    SelectRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. The `Csv.Document` function is used to read the CSV file.
2. The `Table.SelectRows` function is then applied to filter rows based on specific conditions, such as selecting only rows where a certain column matches a specific value.
3. This approach allows for the selective loading of data, reducing the risk of exposing sensitive information.
