# Import Excel Workbook without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to import an Excel workbook without query folding. It allows for the direct import of data from an Excel file, providing flexibility in handling the data within the Power Query pipeline.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when you need to import data from an Excel workbook that contains complex formulas or dynamic data that doesn't support query folding. For instance, when dealing with Excel files that have multiple worksheets, or when the data needs to be processed in a specific way that isn't supported by query folding.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Excel.Workbook(File.Contents("C:\\path\\to\\your\\file.xlsx"), null, true),
    Sheet1 = Source{[0]},
    Data = Table.SelectRows(Sheet1[Data], each _ <> null)
    in
    Data
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The `Excel.Workbook` function is used to read the Excel file. The `File.Contents` function reads the file contents.
    2. The `Source` variable holds the workbook contents, with each sheet represented as a table.
    3. `Table.SelectRows` is then used to select rows from the data table where the value is not null, demonstrating a simple data transformation step.
    4. The result is a table that can be further processed within the Power Query pipeline.
