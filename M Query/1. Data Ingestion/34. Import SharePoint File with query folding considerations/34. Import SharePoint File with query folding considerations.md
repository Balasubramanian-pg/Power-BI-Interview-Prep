# Import SharePoint File with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to import a SharePoint file while considering query folding. Query folding allows Power Query to push data processing operations back to the source, improving performance. This transformation is useful when working with large files in SharePoint and needing to filter or manipulate data before loading into a data model.

### Typical Scenario
In a real data pipeline, this transformation would be used when a business analyst needs to regularly import data from a large Excel file stored in SharePoint. The file contains sales data from different regions, and the analyst only needs data for a specific region. By using query folding, the data import process can filter the data in SharePoint before loading it into the data model, reducing the amount of data transferred and processed.

### Core M Functions Used
`Table.SelectRows`, `Folder.Files`, `Csv.Document`

### Code Source Execution
```powerquery
let
    // Get the file from SharePoint
    sharePointFolder = "https://your-sharepoint-site.sharepoint.com/your-folder/",
    fileName = "your-file.csv",
    filePath = sharePointFolder & fileName,
    fileContent = Folder.Files(sharePointFolder)[Content],
    // Filter the file content
    filteredContent = Table.SelectRows(fileContent, each [Kind] = "File" and [Name] = fileName),
    // Read the CSV file
    csvContent = Csv.Document(filteredContent[Content]{0}, [Delimiter=",", Encoding=1252])
in
    csvContent
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Access SharePoint Folder**: The M script starts by defining the path to the SharePoint folder containing the file.
2. **Specify File Name**: The name of the file to import is specified.
3. **Get File Content**: The `Folder.Files` function is used to get the content of the file from SharePoint.
4. **Filter File**: The `Table.SelectRows` function filters the file content to only include the desired file.
5. **Read CSV File**: Finally, the `Csv.Document` function reads the CSV file, and the data is loaded into the Power Query pipeline.

By applying query folding, operations such as filtering and sorting can be pushed back to the SharePoint source, reducing the amount of data that needs to be transferred and processed.
