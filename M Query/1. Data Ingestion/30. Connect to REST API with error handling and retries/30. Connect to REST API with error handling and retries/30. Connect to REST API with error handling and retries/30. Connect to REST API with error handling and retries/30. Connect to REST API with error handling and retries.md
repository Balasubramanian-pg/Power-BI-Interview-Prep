# Connect to REST API with error handling and retries

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a REST API with error handling and retries, ensuring a robust data retrieval process.

### Typical Scenario
In a real data pipeline, this transformation would be used when connecting to external APIs that may experience intermittent failures or rate limiting, such as fetching data from a third-party service.

### Core M Functions Used
`Web.Contents`, `Json.Document`, `try`, `catch`

### Code Source Execution
```powerquery
let
    url = "https://api.example.com/data",
    maxRetries = 3,
    retryCount = 0,
    data = try
        Json.Document(Web.Contents(url))
    catch
        as Error
        let
            retryCount = retryCount + 1
        in
            if retryCount <= maxRetries then
                @data
            else
                Error.Record("Failed to retrieve data after " & Text.FromNumber(maxRetries) & " retries.")
    in
        data
```

### Example Execution Logic Step-By-Step
This transformation works by first defining the URL of the REST API and the maximum number of retries. It then attempts to fetch the data using Web.Contents and parse it as JSON. If an error occurs, it catches the error, increments the retry count, and retries the fetch if the count is within the limit. If all retries fail, it records an error.
