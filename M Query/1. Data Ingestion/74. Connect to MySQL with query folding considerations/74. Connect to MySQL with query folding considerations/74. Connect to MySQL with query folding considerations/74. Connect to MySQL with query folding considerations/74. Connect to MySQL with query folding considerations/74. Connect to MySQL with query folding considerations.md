# Connect to MySQL with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a MySQL database while considering query folding. Query folding allows Power Query to push data processing operations back to the source database, improving performance. This is particularly useful when working with large datasets.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when connecting to a MySQL database to retrieve specific data. For example, a business analyst might need to extract sales data from a MySQL database for a specific region. By using query folding, the data processing operations can be pushed back to the MySQL database, reducing the amount of data that needs to be transferred and processed in Power Query.

### Core M Functions Used
`MySql.Database`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    mysqlDatabase = MySql.Database("localhost", "mydatabase", [User="myuser", Password="mypassword"]),
    myTable = mysqlDatabase{[Name="mytable"]}[Content],
    selectedRows = Table.SelectRows(myTable, each [column1] > 0)
    in
    selectedRows
```

### Example Execution Logic Step-By-Step
Here's a step-by-step explanation of how the script works:
    1. The `MySql.Database` function is used to connect to the MySQL database.
    2. The `mysqlDatabase` variable holds the connection to the database.
    3. The `myTable` variable holds the content of the specified table.
    4. The `Table.SelectRows` function is used to select rows from the table where the value in column1 is greater than 0.
    5. The `selectedRows` variable holds the resulting table with the selected rows.
    6. The query folds the filter operation back to the MySQL database, so only the relevant data is retrieved.
