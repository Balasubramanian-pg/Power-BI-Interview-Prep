# Read Folder Files with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to read folder files while considering query folding. It allows for efficient data retrieval by leveraging the capabilities of the data source.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to process a large number of files stored in a folder. For instance, when loading data from a data lake or a shared file system, you can use this approach to filter files based on certain criteria before loading them into your data model.

### Core M Functions Used
`Folder.Files`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    folderPath = "C:\Your\Folder\Path",
    files = Folder.Files(folderPath),
    filteredFiles = Table.SelectRows(files, each [Kind] = "File")
    in
        filteredFiles
```

### Example Execution Logic Step-By-Step
Here's how the transformation typically works inside a Power Query pipeline:
    1. Specify the folder path you want to read files from.
    2. Use the `Folder.Files` function to get a list of files and folders in the specified folder.
    3. Apply a filter using `Table.SelectRows` to select only files (or folders) based on your criteria.
    4. The query folding occurs when the data source (in this case, the file system) can optimize the query based on the filter criteria provided, reducing the amount of data that needs to be retrieved.
