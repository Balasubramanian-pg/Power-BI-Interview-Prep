# Import Excel Workbook for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to import an Excel workbook in a multi-tenant environment, allowing for the separation and processing of data based on different tenants or users.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when there is a need to process Excel workbooks from multiple clients or tenants, where each workbook may have a similar structure but contain data specific to each tenant.

### Core M Functions Used
`Table.SelectRows`, `Excel.Workbook`

### Code Source Execution
```powerquery
let
    Source = Excel.Workbook(File.Contents("C:\Path\To\Workbook.xlsx"), null, true),
    Data = Source{[Content = "Data"]}[Data],
    SelectData = Table.SelectRows(Data, each [Tenant] = "Tenant1")
    in
    SelectData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. The Excel workbook is loaded using the `Excel.Workbook` function.
2. The workbook is then navigated to find the specific worksheet that contains the data.
3. The data is then selected based on the tenant using the `Table.SelectRows` function.
4. The selected data is then returned as a table.
