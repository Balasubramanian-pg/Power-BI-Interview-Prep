# Connect to REST API with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a REST API while adhering to GDPR/Privacy constraints. It enables secure data retrieval and processing, ensuring compliance with data protection regulations.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to fetch customer data from a REST API while ensuring that the data handling meets GDPR requirements. For instance, a marketing team might use this to retrieve customer preferences data for targeted campaigns, ensuring that only authorized personnel have access to sensitive information.

### Core M Functions Used
`Table.SelectRows`, `Web.Contents`

### Code Source Execution
```powerquery
let
    apiUrl = "https://api.example.com/data",
    apiKey = "your_api_key",
    headers = [#"Authorization" = "Bearer " & apiKey],
    data = Web.Contents(apiUrl, [Headers=headers]),
    jsonData = Json.Document(data),
    rows = jsonData["data"]
    in
    Table.FromList(rows, Splitter.SplitByNothing(), null)
```

### Example Execution Logic Step-By-Step
This transformation works by first defining the URL of the REST API and any required authentication headers. It then uses the Web.Contents function to fetch data from the API, passing in the authentication headers. The retrieved data is converted from JSON format to a table using Json.Document and Table.FromList. Finally, Table.SelectRows can be used to filter the data based on GDPR/privacy constraints, ensuring only authorized data is processed further.
