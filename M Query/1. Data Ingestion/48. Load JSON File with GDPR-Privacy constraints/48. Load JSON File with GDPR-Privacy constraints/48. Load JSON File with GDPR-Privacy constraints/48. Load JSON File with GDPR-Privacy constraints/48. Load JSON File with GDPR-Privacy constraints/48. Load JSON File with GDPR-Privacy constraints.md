# Load JSON File with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to load a JSON file while adhering to GDPR/privacy constraints. It ensures that sensitive data is handled appropriately, and only authorized data is loaded for analysis or processing.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when loading customer data from a JSON file. For instance, a company may need to load customer information for marketing analysis while ensuring that personally identifiable information (PII) is either anonymized or excluded.

### Core M Functions Used
`Table.SelectRows`, `Json.Document`

### Code Source Execution
```powerquery
let
    Source = Json.Document(File.Contents("C:\path\to\file.json")),
    // Assume 'Source' is a JSON table
    RemovedColumns = Table.SelectRows(Source, each [privacyFlag] = "public")
in
    RemovedColumns
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. Load the JSON file using `Json.Document(File.Contents("C:\path\to\file.json"))`.
2. Assume the loaded JSON data is in a table format.
3. Use `Table.SelectRows` to filter rows based on a 'privacyFlag'.
4. Only rows with 'privacyFlag' equal to "public" are selected, effectively excluding sensitive data.
5. The resulting table contains only the data that meets the GDPR/privacy constraints.
