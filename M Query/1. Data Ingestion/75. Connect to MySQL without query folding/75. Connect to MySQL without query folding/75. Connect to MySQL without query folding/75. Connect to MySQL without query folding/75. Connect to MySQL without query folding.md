# Connect to MySQL without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a MySQL database without query folding. It allows for the retrieval of data from MySQL databases while preserving the flexibility of Power Query transformations.

### Typical Scenario
In a real-world data pipeline, connecting to MySQL without query folding is useful when you need to perform complex data transformations or aggregations on the data retrieved from MySQL that cannot be folded back to the source. For example, when analyzing customer data stored in MySQL, you might need to apply filters, aggregations, or joins that are not supported by MySQL or are more efficiently processed in Power Query.

### Core M Functions Used
`MySql.Database`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    mysqlDatabase = MySql.Database("your_server", "your_database", [User="your_user", Password="your_password"]),
    yourTable = mysqlDatabase{"your_table"},
    selectRows = Table.SelectRows(yourTable, each true)
    in
    selectRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Connect to MySQL Database**: The `MySql.Database` function is used to establish a connection to the MySQL database. This function takes the server name, database name, and credentials as parameters.
2. **Retrieve Data**: The connection is then used to access a specific table from the MySQL database.
3. **Select Rows**: The `Table.SelectRows` function is applied to select rows from the retrieved table. In this example, `each true` means all rows are selected, but you can apply any condition as needed.
4. **Result**: The final result is a table with the selected rows, which can be further transformed or analyzed within Power Query.
