# Reading Folder Files for Large Datasets in Power Query

**Description:**
This use case demonstrates how Power Query (M) can be used to read files from a folder for large datasets. It allows users to efficiently load and process multiple files from a directory.

### Typical Scenario
In a real-world data pipeline, this transformation is useful when dealing with large datasets split across multiple files, such as log files from a web server or daily sales reports. For instance, a data engineer might use this to aggregate sales data from multiple CSV files stored in a folder.

### Core M Functions Used
`Folder.Files`, `File.Contents`

### Code Source Execution
```powerquery
let
    Source = Folder.Files("C:\Path\To\Folder"),
    FilesToCombine = Table.SelectRows(Source, each [Kind] = "File"),
    CombinedData = Table.CombineFiles(FilesToCombine, "Content", "Data", "Error")
in CombinedData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Folder.Files**: This function is used to get a list of files and subfolders in a specified folder.
2. **Filter Files**: Typically, you filter the list to only include files (not subfolders) using `Table.SelectRows`.
3. **Combine Files**: The `Table.CombineFiles` function is then used to combine the contents of these files into a single table.
4. **Handling Errors**: Depending on the structure of the files, you might need to handle errors during the combination process.
