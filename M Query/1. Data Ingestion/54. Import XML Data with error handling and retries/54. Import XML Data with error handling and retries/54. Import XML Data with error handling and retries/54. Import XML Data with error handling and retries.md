# Import XML Data with Error Handling and Retries

**Description:**
This use case demonstrates how Power Query (M) can be used to import XML data with robust error handling and retry mechanisms. It ensures reliable data import by handling potential errors and retrying failed attempts.

### Typical Scenario
In a real data pipeline, this transformation is crucial when integrating data from external XML sources that may experience intermittent connectivity issues or return errors. For instance, fetching data from an external service that provides XML responses.

### Core M Functions Used
`Xml.Document`, `Table.SelectRows`, `try`, `catch`, `Number.Retries`

### Code Source Execution
```powerquery
let
    Source = try Xml.Document(Web.Contents("https://example.com/data.xml")) catch Error.Record,
    RetryCount = 3,
    RetryPolicy = Number.Retries(RetryCount),
    Data = if Source[HasError] then
        error "Failed to retrieve data after retries"
    else
        Table.SelectRows(Xml.Table(Source, "//item"), {"column1", "column2"})
    in
        Data
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Web.Contents**: Fetches XML data from a specified URL.
2. **try-catch**: Wraps the XML document retrieval in a try block to catch any errors.
3. **Xml.Document**: Converts the web content into an XML document.
4. **Xml.Table**: Transforms the XML document into a table.
5. **Table.SelectRows**: Selects specific columns from the table.
6. **Number.Retries**: Implements a retry policy for failed attempts.
