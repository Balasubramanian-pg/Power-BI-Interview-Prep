# Connect to SQL Server with error handling and retries

**Description:**
This use case demonstrates how Power Query (M) can be used to connect to a SQL Server database with error handling and retries. It provides a robust method for handling transient errors that may occur during the connection process, ensuring a more reliable data retrieval process.

### Typical Scenario
In a real data pipeline, this transformation would be used when connecting to a SQL Server database that may experience intermittent connectivity issues, such as a database server that is behind a firewall or a server that may be temporarily down for maintenance.

### Core M Functions Used
`Sql.Database`, `Error.Message`, `Error.Record`

### Code Source Execution
```powerquery
let
    sqlServerName = "your_server_name",
    databaseName = "your_database_name",
    optionalParameters = [
        "Connection Timeout" = 30,
        "Query" = "SELECT * FROM your_table_name"
    ],
    retryCount = 3,
    retryDelay = 5,
    ConnectToSqlServer = () =>
        try
            Sql.Database(sqlServerName, databaseName, optionalParameters)
        catch
            error
                if Error.Message(error) = "The server did not respond in a timely fashion" then
                    if retryCount > 0 then
                        (retryCount - 1) * retryDelay
                    else
                        error
                else
                    error,
    result = ConnectToSqlServer()
    in
        result
```

### Example Execution Logic Step-By-Step
This script works by defining a function `ConnectToSqlServer` that attempts to connect to a SQL Server database using the `Sql.Database` function. If a timeout error occurs, it retries the connection after a specified delay. The retry count and delay are configurable. The script uses a try-catch block to catch any errors that occur during the connection process. If the error message indicates a timeout, it retries the connection; otherwise, it re-throws the error.
