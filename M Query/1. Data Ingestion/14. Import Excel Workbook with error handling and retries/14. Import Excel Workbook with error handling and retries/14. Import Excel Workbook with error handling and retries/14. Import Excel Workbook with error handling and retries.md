# Import Excel Workbook with Error Handling and Retries

**Description:**
This use case demonstrates how Power Query (M) can be used to import an Excel workbook with error handling and retries. It provides a robust method for handling potential errors when importing Excel files, ensuring data pipeline reliability.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to regularly import data from Excel workbooks that may occasionally be locked, corrupted, or in use by another process. For instance, a daily data import task from an Excel file shared among multiple users.

### Core M Functions Used
`Table.SelectRows`, `try`, `catch`, `retry`

### Code Source Execution
```powerquery
let
    Source = try
        Excel.Workbook(File.Contents("C:\Path\To\Workbook.xlsx"), null, true),
    catch
        (e) =>
            Error.Record("Error importing workbook", e, []),
    RetryCount = 3,
    RetryPolicy = List.Generate(
        () => 0,
        each _ < RetryCount,
        each _ + 1
    ),
    AttemptImport = (retryCount) =>
        try
            Source,
        catch
            (e) =>
                if retryCount > 0 then
                    AttemptImport(retryCount - 1)
                else
                    error e
in
    AttemptImport(RetryCount)
```

### Example Execution Logic Step-By-Step
This transformation works by first attempting to import the Excel workbook using the Excel.Workbook function within a try-catch block. If an error occurs during import, it is caught and retried up to a specified number of times (RetryCount). If all retries fail, the final error is recorded and returned. This approach ensures that transient errors (like file locks) are handled gracefully.
