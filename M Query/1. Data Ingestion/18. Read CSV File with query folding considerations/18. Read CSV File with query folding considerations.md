# Read CSV File with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to read a CSV file while considering query folding. Query folding allows Power Query to push data processing operations back to the source, improving performance. This is particularly useful when working with large datasets.

### Typical Scenario
In a real data pipeline, this transformation would be used when ingesting large CSV files from a data lake or file system. For example, when loading sales data from a CSV file into a data warehouse, you might want to filter out unnecessary columns or rows before loading the data.

### Core M Functions Used
`Table.SelectRows`, `Csv.Document`

### Code Source Execution
```powerquery
let
    Source = Csv.Document(File.Contents("C:\Path\To\File.csv"), [Delimiter=",", Encoding=1252, QuoteStyle=QuoteStyle.None]),
    RemovedColumns = Table.SelectRows(Source, "= [_]"),
    PromotedHeaders = Table.PromoteHeaders(RemovedColumns, [PromoteAllScalars=true])
    in
    PromotedHeaders
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The Csv.Document function is used to read the CSV file.
    2. The Table.SelectRows function is used to select specific rows from the CSV file, demonstrating query folding.
    3. The Table.PromoteHeaders function is used to promote the first row of the CSV file to headers.
    The result is a table with the selected rows and promoted headers.
