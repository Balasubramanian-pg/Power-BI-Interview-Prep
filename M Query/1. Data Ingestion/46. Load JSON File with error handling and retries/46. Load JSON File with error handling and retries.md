# Load JSON File with Error Handling and Retries

**Description:**
This transformation loads a JSON file while implementing error handling and retries. It ensures robustness in data pipelines by managing potential file access issues or parsing errors. This approach helps in maintaining data pipeline reliability.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when loading JSON data from potentially unreliable sources, such as external APIs or file systems that may experience intermittent failures. For instance, when fetching daily logs from a service that occasionally might be down or return corrupted data.

### Core M Functions Used
`Web.Contents`, `Json.Document`, `try`, `catch`

### Code Source Execution
```powerquery
let
    url = "https://example.com/data.json",
    retries = 3,
    delay = 5000, // 5 seconds
    loadJson = () =>
        try
            Json.Document(Web.Contents(url))
        catch
            [error] =>
                if retries > 0 then
                    (
                        Thread.Sleep(delay),
                        retries - 1,
                        loadJson()
                    )
                else
                    error [error],
    data = loadJson()
    in
        data
```

### Example Execution Logic Step-By-Step
This transformation works by defining a function `loadJson` that attempts to load and parse a JSON file from a specified URL. If the initial attempt fails (either due to network issues or parsing errors), it will retry the operation a specified number of times with a delay between attempts. If all retries fail, it will propagate the last error encountered.
