# Read Folder Files as a Reusable Function

**Description:**
This use case demonstrates how Power Query (M) can be used to create a reusable function for reading files from a folder. The function can be used to fetch files from a specified directory, which is particularly useful for data pipelines that require ingesting data from multiple files stored in a folder. By encapsulating this logic into a function, users can easily reuse it across different parts of their data processing workflow.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when data is scattered across multiple files within a folder. For instance, when daily sales reports are saved as separate CSV files in a directory, a data engineer can use this function to read all files in that folder, combine the data, and then perform further transformations or analysis.

### Core M Functions Used
`Folder.Files`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    ReadFolderFiles = (folderPath) =>
    let
        files = Folder.Files(folderPath),
        combinedData = Table.CombineColumns(files, {"Name", "Content"}, "Combine"),
        selectedColumns = Table.SelectRows(combinedData, {"Name", "Content"})
    in
        selectedColumns
in
    ReadFolderFiles
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. The function `ReadFolderFiles` takes a `folderPath` as an input parameter.
2. It uses the `Folder.Files` function to get a list of files in the specified folder.
3. The `Table.CombineColumns` function is then used to combine the columns of interest (e.g., file name and content) from each file into a single table.
4. Finally, `Table.SelectRows` is applied to select the required columns (e.g., file name and content) from the combined table.
5. The resulting table, with data from all files in the folder, can then be further processed or analyzed within the Power Query pipeline.
