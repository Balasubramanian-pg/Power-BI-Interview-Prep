# Read Folder Files without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to read files from a folder without relying on query folding. It allows for the retrieval of file metadata or contents directly, which is useful when working with files that need to be processed individually.

### Typical Scenario
In a real data pipeline, this transformation is useful when you need to automate the ingestion of files from a folder, such as processing invoices, logs, or other types of documents that are regularly dropped into a folder. For instance, a business intelligence system might use this to automatically read and process new sales data stored in a shared folder.

### Core M Functions Used
`Folder.Files`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Folder.Files("C:\\Path\\To\\Folder"),
    // Assuming you want to select specific columns
    SelectedColumns = Table.SelectColumns(Source, {"Name", "Content"})
    // Further transformations can be added here
in
    SelectedColumns
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Folder.Files**: This function is used to get a list of files in a specified folder. It returns a table with file metadata such as name, size, and date modified.
2. **Table.SelectRows or Table.SelectColumns**: Depending on the exact requirements, you might select specific rows (files) or columns (metadata fields) to work with. For example, you might want to select only files with a certain extension or only work with specific metadata fields.
3. **Further Transformations**: You can then perform additional transformations on the selected files, such as reading the file content, renaming columns, filtering files based on certain criteria, etc.
4. **Load**: Finally, the transformed data can be loaded into a data model for analysis or further processing.
