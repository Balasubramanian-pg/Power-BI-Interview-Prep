# Change Column Types with GDPR/Privacy constraints

**Description:**
This transformation achieves changing column types in a data table while adhering to GDPR/privacy constraints. It ensures that sensitive information is handled appropriately during data processing. This is particularly useful when dealing with personal data that requires anonymization or pseudonymization.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to process customer data for analytics purposes while ensuring compliance with GDPR. For instance, a company might need to change the data type of a 'date of birth' column from string to date, but first, it must mask or anonymize the data to protect customer privacy.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformColumns`

```powerquery
let
    Source = Table.FromRows({
        {"John", "1990-01-01"},
        {"Jane", "1995-06-01"}
    }, {"Name", "Date of Birth"}),
    // Anonymize date of birth
    AnonymizedDateOfBirth = Table.TransformColumns(Source, {{"Date of Birth", each Date.FromText(_, [Format="yyyy-MM-dd"])}}),
    // Apply GDPR/privacy constraints, e.g., mask year
    MaskedDateOfBirth = Table.TransformColumns(AnonymizedDateOfBirth, {{"Date of Birth", each Date.AddYears(_, -30)}})
    in
    MaskedDateOfBirth
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Ingestion**: The source data is ingested into a table.
2. **Type Conversion**: The 'Date of Birth' column is converted to a date type using `Date.FromText`.
3. **Anonymization**: To adhere to GDPR/privacy constraints, the year in the 'Date of Birth' column is masked or modified, for example, by adding 30 years to make it anonymous.
4. **Output**: The transformed data with changed column types and applied privacy constraints is outputted for further processing or analysis.
