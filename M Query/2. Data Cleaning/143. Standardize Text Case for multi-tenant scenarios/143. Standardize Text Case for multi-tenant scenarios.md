# Standardize Text Case for Multi-Tenant Scenarios

**Description:**
This transformation standardizes text case across multiple tenants, ensuring consistency in data processing. It is particularly useful in multi-tenant environments where data is sourced from various clients, each potentially having different text case conventions. By standardizing text case, organizations can improve data quality, enhance query performance, and simplify data analysis.

### Typical Scenario
In a multi-tenant data pipeline, a company aggregates customer feedback from various sources. Each customer's feedback may have different text case usage (e.g., 'Customer A' uses title case, while 'Customer B' uses all caps). To ensure uniform processing and analysis, the company applies a text case standardization transformation to convert all feedback text into a standard case, such as lowercase.

### Core M Functions Used
`Text.Lower`, `Text.Upper`, `Text.Proper`

```powerquery

    Source = Table.FromRows({

```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Data**: The pipeline starts with a table containing customer feedback from multiple tenants, where each tenant may have different text case conventions.
    2. **Text Case Standardization**: The `Text.Lower` function is applied to the 'Feedback' column to convert all text into lowercase, standardizing the text case across tenants.
    3. **Standardized Output**: The resulting table has all feedback text in a uniform lowercase format, facilitating consistent processing and analysis.
