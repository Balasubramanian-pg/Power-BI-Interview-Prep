# Change Column Types with Error Handling and Retries

**Description:**
This transformation changes the data type of specific columns in a table while incorporating error handling and retry mechanisms. It ensures data consistency and accuracy by attempting to convert column values to the desired data type, handling potential errors, and retrying failed conversions.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when ingesting data from various sources with inconsistent or incorrect data types. For instance, when loading customer data from multiple systems, some columns like age or income might be incorrectly imported as text instead of numbers. This transformation can correct these discrepancies, ensuring uniformity across the dataset.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformColumns`, `try`, `catch`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 30, "12345"},
        {"Jane", "twenty-five", "67890"}
    }, {"Name", "Age", "Zip"}),
    ChangeType = Table.TransformColumns(Source, {
        {"Age", each try Number.FromText(_, "en-US") catch Error.Record}, // Attempt to convert to number with error handling
        {"Zip", each Text.PadStart(_, 5, "0")} // Pad Zip with leading zeros
    }),
    RetryFailed = Table.SelectRows(ChangeType, each [Age] <> Error.Record),
    FinalData = Table.TransformColumns(RetryFailed, {{"Age", each Number.FromText(_, "en-US")}})
    in
    FinalData
```
