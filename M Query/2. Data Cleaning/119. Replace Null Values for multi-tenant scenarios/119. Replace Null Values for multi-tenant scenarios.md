# Replace Null Values for Multi-Tenant Scenarios

**Description:**
This transformation replaces null values in a multi-tenant data environment, ensuring data consistency and reliability. It is particularly useful when dealing with data from multiple sources, where null values may have different meanings or implications. By replacing null values, users can standardize their data processing and analysis.

### Typical Scenario
In a multi-tenant data pipeline, different tenants may have varying data quality, leading to null values in different columns. For instance, a marketing company may have customer data from multiple clients, some of whom do not provide phone numbers or addresses. When aggregating this data for analysis, replacing null values with default or placeholder values (e.g., 'Unknown') ensures that the analysis can proceed smoothly without worrying about null value errors.

### Core M Functions Used
`Table.SelectRows`, `Replacer.ReplaceValue`

```powerquery
let

        {"Tenant1", 1, "John", null},

        {"Tenant3", 3, null, null}

    ReplacedNulls = Table.SelectRows(Source, each {
        Tenant = [Tenant],
        ID = [ID],
        Name = if [Name] = null then "Unknown" else [Name],
        Address = if [Address] = null then "Not Provided" else [Address]
    })
in
    ReplacedNulls
```
