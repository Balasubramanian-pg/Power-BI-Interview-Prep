# Promote Headers with Query Folding Considerations

**Description:**
This transformation promotes the first row of a table to headers, considering query folding for optimization. It is useful when working with data sources that return data in a format that needs header promotion. The goal is to make the data more usable for analysis by properly naming the columns.

### Typical Scenario
In a real data pipeline, this transformation is used when ingesting data from a source that returns a table without proper headers, such as a CSV file without a header row or an API that returns data in a non-standard format. For example, when loading sales data from a CSV file where the first row contains the column names but they are not properly recognized as headers.

### Core M Functions Used
`Table.PromoteHeaders`

The transformation works as follows:

    3. The PromoteAllScalars option is set to true to ensure all columns are properly promoted.
    4. The resulting table has the correct column names and data types, making it ready for further analysis.
