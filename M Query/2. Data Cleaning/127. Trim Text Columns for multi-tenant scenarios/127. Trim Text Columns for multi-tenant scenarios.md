# Trim Text Columns for Multi-Tenant Scenarios

**Description:**
This transformation trims text columns to remove leading and trailing whitespaces, ensuring data consistency across a multi-tenant environment. It helps in standardizing text data, which is crucial for accurate analysis and reporting. By trimming text columns, data pipelines can handle variations in data entry and formatting.

### Typical Scenario
In a multi-tenant data warehouse, different tenants may have varying data entry practices, leading to inconsistencies in text data. For instance, a tenant's customer names might be stored with extra spaces, which can cause issues in data matching and aggregation. This transformation is used to clean and standardize text columns, ensuring that data is consistent and reliable across all tenants.

### Core M Functions Used
`Text.Trim`, `Table.TransformColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John  ", " Doe"},
        {"Jane", " Smith"}
    }, {"Name", "Surname"}),
    Trimmed = Table.TransformColumns(Source, {{"Name", each Text.Trim(_)}, {"Surname", each Text.Trim(_)}})
    in
    Trimmed
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Data**: Start with a table containing text columns that may have leading or trailing whitespaces.
    2. **Apply Trim Function**: Use `Table.TransformColumns` to apply the `Text.Trim` function to each text column.
    3. **Trim Operation**: The `Text.Trim` function removes leading and trailing whitespaces from each text value in the specified columns.
    4. **Result**: The resulting table has clean text columns with no leading or trailing whitespaces, ensuring data consistency across the multi-tenant environment.
