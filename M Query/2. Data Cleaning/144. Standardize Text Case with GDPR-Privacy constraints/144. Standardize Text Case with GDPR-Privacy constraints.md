# Standardize Text Case with GDPR/Privacy constraints

**Description:**
This transformation standardizes text case in a dataset while adhering to GDPR/privacy constraints. It ensures that personal data is handled in a way that protects individual privacy. The transformation is crucial for maintaining data consistency and compliance with data protection regulations.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when processing customer data for analysis or reporting purposes. For instance, when consolidating customer names from various sources, standardizing text case helps in matching and deduplicating records while ensuring that personal data is anonymized or pseudonymized as required by GDPR.

The transformation works by first loading the data into a Power Query table. For the 'Last Name' column, it applies the Text.Upper function to convert all letters to uppercase. This approach ensures that the text case is standardized across the dataset, enhancing data quality and consistency.
