# Fill Down Missing Values for Multi-Tenant Scenarios

**Description:**
This transformation fills down missing values in a multi-tenant data scenario, ensuring that each tenant's data has consistent and complete information. It is particularly useful when dealing with datasets where some records may not have all the necessary details, and the missing information needs to be propagated from the last known value.

### Typical Scenario
In a multi-tenant data pipeline, different tenants may have varying levels of detail in their data records. For instance, a sales dataset might have sales amounts listed for each day but only have the salesperson's name listed occasionally. When aggregating or reporting on this data, it's essential to have complete information for accurate analysis. The 'Fill Down Missing Values' transformation ensures that missing salesperson names are filled in based on the last known salesperson for each tenant.
