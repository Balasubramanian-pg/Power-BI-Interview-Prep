# Rename Columns as a Reusable Function in Power Query

**Description:**
This transformation allows users to rename columns in a table as a reusable function, making it easier to manage and standardize data transformations across different data pipelines.

### Typical Scenario
In a real-world data pipeline, renaming columns is crucial when integrating data from multiple sources with different column names but similar data types. For instance, when combining customer data from different CRM systems, renaming columns like 'Customer Name' and 'Client Name' to a standard 'Customer_Name' ensures consistency and simplifies data analysis.

### Core M Functions Used
`Table.SelectColumns`, `Table.TransformColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"}
    }, {"Name", "Age", "Country"}),
    RenameColumns = Table.TransformColumns(Source, {{"Name", each _ as text, "Full_Name"}, {"Age", each _ as number, "Years_Old"}})
    in
    RenameColumns
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Define the Source Table**: Start with a table that needs column renaming.
    2. **Apply Table.TransformColumns**: Use this function to specify the columns to rename and their new names.
    3. **Specify Data Types**: Optionally, specify or confirm the data types of the columns.
    4. **Output the Transformed Table**: The result is a new table with the columns renamed according to the specifications.
