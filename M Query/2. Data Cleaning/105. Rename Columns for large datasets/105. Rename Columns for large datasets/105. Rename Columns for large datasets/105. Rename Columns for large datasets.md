# Rename Columns for Large Datasets

**Description:**
This transformation allows for efficient renaming of columns in large datasets using Power Query (M). It enables users to map existing column names to new names, improving data readability and consistency. This is particularly useful when working with large datasets that require restructuring for analysis or reporting purposes.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when ingesting data from various sources, such as databases, files, or APIs, where column names may not be descriptive or consistent. For instance, a company may receive customer data with abbreviated column names (e.g., 'CustID', 'Name') that need to be renamed to more descriptive names (e.g., 'CustomerID', 'FullName') for better understanding and analysis.

### Core M Functions Used
`Table.SelectColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Jane", 30, "UK"}
    }, {"Name", "Age", "Country"}),
    RenamedColumns = Table.SelectColumns(Source, {"Name", "Age", "Country"}, {"Full Name", "Age in Years", "Country of Residence"})
    in
    RenamedColumns
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Table Creation**: A sample table is created with columns 'Name', 'Age', and 'Country'.
    2. **Renaming Columns**: The Table.SelectColumns function is used to rename the columns. It takes the source table, the list of columns to select, and the new names for these columns.
    3. **Result**: The resulting table has the same data but with columns renamed to 'Full Name', 'Age in Years', and 'Country of Residence', enhancing readability and usability for further analysis.
