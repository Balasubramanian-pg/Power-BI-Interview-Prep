# Change Column Types without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to change column types without query folding. It involves modifying the data type of existing columns in a table.

### Typical Scenario
In a real data pipeline, this transformation is useful when the source data types do not match the required data types for analysis or reporting. For instance, when loading data from a source system where numeric data is incorrectly represented as text, and it needs to be converted to a numeric type for calculations.

### Core M Functions Used
`Table.TransformColumns`, `Number.FromText`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"1", "John", "25"},
        {"2", "Jane", "30"}
    }, {"ID", "Name", "Age"}),
    ChangedType = Table.TransformColumns(Source, {{"Age", each Number.FromText(_, "en-US")}})
    in
    ChangedType
```

### Example Execution Logic Step-By-Step
The transformation works by first creating a table with sample data. Then, it uses the Table.TransformColumns function to change the data type of the 'Age' column from text to number. The Number.FromText function is used with the 'en-US' culture to correctly interpret the text as a number.
