# Remove Unnecessary Columns without Query Folding

**Description:**
This transformation removes unnecessary columns from a table in Power Query (M) without relying on query folding. It is useful for optimizing data pipelines by reducing the amount of data being processed. By selectively removing columns, it improves performance and streamlines data analysis.

### Typical Scenario
In a real data pipeline, this transformation is typically used when ingesting large datasets where not all columns are necessary for analysis or reporting. For instance, when loading data from a database into a data warehouse, you might only need a subset of columns for your business intelligence tools. By removing unnecessary columns early in the ETL process, you can significantly reduce the data volume, leading to faster processing times and lower storage costs.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Column1", "Column2", "Column3"}, 
        {"Value1", "Value2", "Value3"}
    }),
    RemovedColumns = Table.SelectRows(Source, each [Column1 <> "" and Column2 <> ""])
    // Alternatively, to remove specific columns:
    // RemovedColumns = Table.SelectColumns(Source, {"Column1", "Column2"})
    // Or to keep specific columns:
    // RemovedColumns = Table.SelectColumns(Source, {"Column1", "Column2"})
    in
    RemovedColumns
```

The transformation works by first creating a table with all the columns. Then, it uses the `Table.SelectRows` function to filter out rows based on conditions (in this case, ensuring that both 'Column1' and 'Column2' are not empty). This step-by-step process allows for efficient data optimization by eliminating unnecessary data early in the pipeline.
