# Remove Unnecessary Columns for Multi-Tenant Scenarios

**Description:**
This transformation removes unnecessary columns from a dataset to optimize it for multi-tenant scenarios, improving data privacy and reducing storage costs. It ensures that only relevant data is exposed to users, enhancing data security and query performance.

### Typical Scenario
In a multi-tenant data environment, such as a SaaS application, different customers share the same database but need access only to their specific data. Removing unnecessary columns helps in data anonymization and reduces the risk of data leakage. For instance, a column containing sensitive information like employee salaries can be removed to prevent unauthorized access.

### Core M Functions Used
`Table.SelectColumns`

```powerquery
    RemovedColumns = Table.SelectColumns(Source, {"TenantID", "Name", "Email"})
    in
    RemovedColumns
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: The process starts with a data source that contains multiple columns, some of which are not necessary for the multi-tenant scenario.
    2. **Identify Unnecessary Columns**: Identify columns that are not required for the specific tenant or are sensitive in nature.
    3. **Apply Transformation**: Use the `Table.SelectColumns` function in Power Query (M) to select only the necessary columns, effectively removing the unnecessary ones.
    4. **Result**: The resulting table contains only the required columns, enhancing data privacy and reducing storage costs.
