# Promote Headers without Query Folding

**Description:**
This transformation promotes the first row of a table to be the column headers without relying on query folding. It is useful when dealing with data sources that do not support query folding or when the data needs to be processed in a specific way that prevents folding.

### Typical Scenario
In a real-world data pipeline, this transformation is used when ingesting data from a source that doesn't natively support header promotion through query folding, such as certain web APIs or file systems. For instance, when fetching data from an API that returns a CSV-like structure without proper headers, you can use this method to explicitly define and promote headers.

### Core M Functions Used
`Table.SelectRows`, `Table.PromoteHeaders`

```powerquery
let
    Source = Table.FromRows({
        {"Name", "Age"},

        {"Paul", 34}
    }),
    PromotedHeaders = Table.PromoteHeaders(Source, [PromoteAllScalars=true])

```

### Example Execution Logic Step-By-Step
The transformation works by first creating a table from a list of rows. The Table.PromoteHeaders function is then used with the option PromoteAllScalars set to true, which tells Power Query to treat all scalar values in the first row as headers. This results in a table where the first row is now the column headers, and the rest of the data is correctly associated with those headers.
