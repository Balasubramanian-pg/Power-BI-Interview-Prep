# Promote Headers as a Reusable Function in Power Query

**Description:**
This transformation promotes the first row of a table to be the column headers. It is useful when working with data that has column names in the first row, but they are not recognized as headers by Power Query. This function can be reused across different data pipelines to standardize the data import process.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when extracting data from CSV or Excel files where the first row contains column names. For instance, when importing customer data from a CSV file, the first row might contain headers like 'Name', 'Email', 'Phone', but Power Query does not automatically recognize them as headers. By promoting these headers, the data can be easily transformed and used for further analysis or processing.

`Table.PromoteHeaders`

The transformation works as follows:

    2. The first row of the table contains the column names.
    3. The `Table.PromoteHeaders` function is then used to promote the first row to be the column headers.
    4. The `[PromoteAllScalars=true]` argument ensures that all scalar values in the first row are promoted to headers, not just the first column.
    5. The resulting table has the promoted headers and can be used for further data transformation and analysis.
