# Clean Invalid Characters with Error Handling and Retries

**Description:**
This transformation removes invalid characters from a dataset while incorporating error handling and retries to ensure data integrity and reliability.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when ingesting data from external sources that may contain special or invalid characters. For instance, when loading customer data from a legacy system, the data might include characters that are not compatible with modern databases or applications. This transformation ensures that such data is cleaned and processed correctly, preventing errors during further analysis or processing.

### Core M Functions Used
`Table.SelectRows`, `Text.Replace`, `try-catch blocks for error handling`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "Doe", "johndoe@example.com"},
        {"Jane", "Doe", "janedoe@example.com"}
    }, {"First Name", "Last Name", "Email"}),
    CleanInvalidCharacters = (text as text) as text =>
        try
            text
        catch
            ErrorRecord as Error =>
            try
                Text.Replace(text, Error.Message, "")
            catch
                _ as Error =>
                    "",
    CleanedData = Table.TransformColumns(Source, {{"Email", CleanInvalidCharacters}})
    in
        CleanedData
```

### Example Execution Logic Step-By-Step
This transformation works by first defining a custom function `CleanInvalidCharacters` that attempts to return the input text. If an error occurs (for example, if the text contains an invalid character), it tries to replace the invalid character with an empty string. If a second error occurs during replacement, it returns an empty string. The `Table.TransformColumns` function is then used to apply this cleaning function to the 'Email' column of the input table.
