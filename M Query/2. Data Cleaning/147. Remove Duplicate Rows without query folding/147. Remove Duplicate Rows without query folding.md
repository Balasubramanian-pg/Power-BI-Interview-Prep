# Remove Duplicate Rows without Query Folding

**Description:**
This transformation removes duplicate rows from a table without relying on query folding. It ensures data integrity by eliminating redundant entries, which is crucial for accurate analysis and reporting. This process is particularly useful when dealing with data sources that do not support query folding.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when ingesting data from sources like log files or transactional databases where duplicate records may exist due to data import errors or application-level issues. For instance, when loading customer data from a CRM system, duplicate customer records might be present, and removing them is essential for maintaining a clean and reliable dataset.
