# Change Column Types for Multi-Tenant Scenarios

**Description:**
This transformation is used to change column types in a multi-tenant data environment. It allows for flexible data type conversions based on tenant-specific requirements. This is particularly useful when dealing with data from different sources or regions, each with its own set of data type standards.

### Typical Scenario
In a multi-tenant data pipeline, different tenants may have data with varying formats. For instance, one tenant might store dates as strings in a 'YYYY-MM-DD' format, while another uses a 'DD/MM/YYYY' format. This transformation can be applied to standardize or convert these formats according to each tenant's needs or the system's requirements.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "2022-01-01"},
        {"Tenant2", "01/01/2022"}
    }, {"Tenant", "Date"}),
    ChangeType = Table.TransformColumns(Source, {{"Date", each Date.FromText(_, [Format = "yyyy-MM-dd"])}}),
    ChangeTypeForTenant2 = Table.TransformColumns(ChangeType, {{"Date", each if [Tenant] = "Tenant2" then Date.FromText(_, [Format = "dd/MM/yyyy"])) else _}})
    in
    ChangeTypeForTenant2
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Ingestion**: The pipeline starts by ingesting data from various tenants. This data is often in a raw or unprocessed state.
    2. **Identification of Tenant**: The system identifies the tenant from which the data originates. This can be done through various methods such as a tenant ID column in the data.
    3. **Application of Transformation**: Based on the tenant, the system applies the necessary transformations to the data. For example, it might convert a date column from a string type to a date type, handling different formats for different tenants.
    4. **Standardization**: The transformation standardizes the data types across tenants where necessary, ensuring consistency for further processing or analysis.
    5. **Loading Transformed Data**: The transformed data is then loaded into a target system for reporting, analysis, or further processing.
