# Remove Duplicate Rows as a Reusable Function

**Description:**
This transformation removes duplicate rows from a table, ensuring that only unique records are retained. It is commonly used for data cleansing and preparation. By eliminating duplicates, data analysis and processing become more efficient and accurate.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when integrating data from multiple sources. For instance, when merging customer databases from different departments, duplicates may arise. Removing these duplicates ensures that the unified database does not contain redundant entries, thereby improving data quality and reducing storage needs.

### Core M Functions Used
`Table.Distinct`

```powerquery
let

    RemovedDuplicates = Table.Distinct(Source)
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Define the Source Table**: A table is created with potentially duplicate rows.
    2. **Apply Table.Distinct**: The Table.Distinct function is applied to the source table. By default, it removes duplicate rows based on all columns, ensuring that only unique combinations of values across all columns are kept.
    3. **Output the Result**: The resulting table with duplicates removed is then outputted. This table can be used for further data processing or analysis.
