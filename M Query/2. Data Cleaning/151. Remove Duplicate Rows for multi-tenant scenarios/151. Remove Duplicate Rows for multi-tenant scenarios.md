# Remove Duplicate Rows for Multi-Tenant Scenarios

**Description:**
This transformation removes duplicate rows from a table, ensuring data integrity in multi-tenant environments. It helps prevent data duplication and ensures that each record is unique. This is particularly useful when dealing with data from multiple sources or tenants.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when ingesting data from multiple tenants or sources, where duplicate records may exist. For instance, when combining customer data from different subsidiaries, removing duplicates ensures a unified and accurate view of customer information.
