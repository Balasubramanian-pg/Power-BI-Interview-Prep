# Standardize Text Case with Query Folding Considerations

**Description:**
This transformation standardizes text case in a data pipeline, ensuring consistency across the dataset. It is particularly useful when dealing with data from various sources, where text may be in different cases. By standardizing text case, it facilitates easier data comparison, filtering, and analysis. This use case demonstrates how Power Query (M) can be utilized for this purpose.

### Typical Scenario
In a real-world data pipeline, standardizing text case is crucial when integrating data from multiple sources. For instance, when merging customer data from different CRM systems, standardizing the case of customer names ensures uniformity and simplifies data processing and analysis.

### Core M Functions Used
`Table.SelectRows`, `Text.Lower`, `Text.Upper`, `Text.Proper`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "Doe"},
        {"JOHN", "SMITH"},
        {"jOhN", "dOe"}
    }, {"First Name", "Last Name"}),
    StandardizeCase = Table.TransformColumns(Source, {{"First Name", each Text.Proper(_)}, {"Last Name", each Text.Upper(_)}})
    in
    StandardizeCase
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: The process begins with a table containing names in different cases.
    2. **Standardization**: The `Table.TransformColumns` function is used to apply case standardization to specific columns.
    3. **Case Functions**: `Text.Proper` is used to convert text to proper case (first letter capitalized, rest in lower case), and `Text.Upper` is used to convert text to upper case.
    4. **Result**: The resulting table has standardized text case, enhancing data consistency and facilitating further data processing.
