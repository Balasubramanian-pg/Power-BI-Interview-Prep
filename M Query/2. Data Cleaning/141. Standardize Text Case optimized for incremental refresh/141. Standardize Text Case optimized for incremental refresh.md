# Standardize Text Case optimized for incremental refresh

**Description:**
This transformation standardizes text case in a dataset, ensuring consistency and readability. It is particularly useful for datasets that require case uniformity, such as customer names or product descriptions. By optimizing for incremental refresh, this transformation enables efficient updates to large datasets.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when integrating data from multiple sources with varying text case conventions. For instance, when merging customer data from different CRM systems, standardizing text case ensures that equivalent values (e.g., 'John Smith' and 'JOHN SMITH') are treated as the same.

### Core M Functions Used
`Text.Upper`, `Text.Lower`, `Text.Proper`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John Smith", "NEW YORK"},
        {"Jane Doe", "LOS ANGELES"},
        {"Bob Johnson", "CHICAGO"}
    }, {"Name", "City"}),
    StandardizedName = Table.TransformColumns(Source, {{"Name", each Text.Proper(_)}}),
    StandardizedCity = Table.TransformColumns(StandardizedName, {{"City", each Text.Upper(_)}})
    in
    StandardizedCity
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The source data is loaded into a table.
    2. The `Text.Proper` function is applied to the 'Name' column to standardize the case of each name.
    3. The `Text.Upper` function is applied to the 'City' column to convert all city names to uppercase.
    4. The resulting table has standardized text case for both names and cities.
