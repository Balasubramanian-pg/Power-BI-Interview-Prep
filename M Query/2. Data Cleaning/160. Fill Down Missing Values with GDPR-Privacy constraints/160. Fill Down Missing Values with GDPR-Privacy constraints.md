# Fill Down Missing Values with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to fill down missing values in a dataset while adhering to GDPR/privacy constraints. It ensures that sensitive information is handled appropriately during data transformation.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with datasets that contain missing values and sensitive information. For instance, in a customer database, some records might have missing addresses or phone numbers, and it's essential to fill these gaps while ensuring that personally identifiable information is protected according to GDPR guidelines.

### Core M Functions Used
`Table.SelectRows`, `Table.FillDown`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "Doe", null, "johndoe@example.com"},
        {null, "Smith", "123 Main St", "smith@example.com"},
        {"Jane", null, "456 Elm St", "jane@example.com"}
    }, {"First Name", "Last Name", "Address", "Email"}),
    FilledDown = Table.FillDown(Source, {"First Name", "Last Name", "Address"})
    in
    FilledDown
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. The source table is created with some missing values.
2. The Table.FillDown function is applied to fill down the missing values in the specified columns (First Name, Last Name, Address).
3. The filled-down table ensures that each row has a complete set of information while respecting GDPR/privacy constraints by not exposing sensitive information improperly.
