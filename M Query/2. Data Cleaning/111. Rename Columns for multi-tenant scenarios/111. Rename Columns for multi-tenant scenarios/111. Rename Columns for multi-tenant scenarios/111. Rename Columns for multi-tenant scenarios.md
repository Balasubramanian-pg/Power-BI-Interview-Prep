# Rename Columns for Multi-Tenant Scenarios

**Description:**
This transformation achieves the renaming of columns in a Power Query table for multi-tenant scenarios, ensuring data consistency and accuracy across different tenants.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when ingesting data from multiple tenants, each with their own set of column names, and there is a need to standardize these column names for unified reporting and analysis.

### Example Execution Logic Step-By-Step
The transformation works by first ingesting data from multiple tenants into a Power Query table. Then, it utilizes the Table.TransformColumns function to rename the columns to a standardized set of names. For instance, if 'Tenant1' and 'Tenant2' have columns named 'FirstName' and 'LastName' respectively, the transformation can rename them to 'Name' and 'Age' for consistency. This is achieved through the Table.TransformColumns function, which takes the source table and a list of column transformations as input.
