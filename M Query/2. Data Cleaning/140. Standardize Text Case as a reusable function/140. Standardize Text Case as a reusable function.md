# Standardize Text Case as a reusable function

**Description:**
This transformation achieves standardizing text case in a dataset to ensure consistency. It is useful for cleaning and preprocessing data. The goal is to make all text entries uniform, which can improve data quality and make analysis more reliable.

### Typical Scenario
In a real data pipeline, standardizing text case is often used when integrating data from multiple sources. For instance, customer names might appear in different cases across various databases. Standardizing the case ensures that 'John Doe', 'JOHN DOE', and 'john doe' are treated as the same entity.

### Core M Functions Used
`Text.Upper`, `Text.Lower`, `Text.Proper`

```powerquery
let
    Source = Table.FromRows({
        {"John Doe"},
        {"JOHN DOE"},
        {"john doe"}
    }, {"Name"}),
    StandardizeCase = Table.TransformColumns(Source, {{"Name", each Text.Proper(_)}})
    in
    StandardizeCase
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
    1. The source data is defined as a table with a column named 'Name'. 
    2. The Table.TransformColumns function is used to apply a transformation to the 'Name' column. 
    3. The Text.Proper function is applied to each value in the 'Name' column, which converts the text to proper case (first letter of each word capitalized, rest in lower case).
