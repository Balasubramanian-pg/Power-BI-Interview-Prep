# Remove Duplicate Rows with Query Folding Considerations

**Description:**
This transformation removes duplicate rows from a table while considering query folding. It ensures data is unique and optimizes performance by leveraging query folding, which allows operations to be pushed back to the source.

### Typical Scenario
In a real data pipeline, this transformation is used when ingesting data from a source that may contain duplicate records, such as when fetching data from a database or file. For instance, when loading customer data from a CRM, duplicates might exist due to sync issues or data entry errors. Removing these duplicates ensures clean data for analysis.

### Core M Functions Used
`Table.Distinct`

The transformation works as follows:

    2. **Duplicate Removal**: The `Table.Distinct` function is used to remove duplicate rows based on specified columns (e.g., "Name" and "Age").
    3. **Query Folding**: If the source supports query folding, the operation is pushed back to the source, improving performance by reducing the amount of data transferred.
    4. **Result**: The resulting table contains only unique rows.
