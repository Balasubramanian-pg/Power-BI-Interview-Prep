# Promote Headers for large datasets

**Description:**
This transformation promotes the first row of a table to be the column headers. It is useful when working with large datasets where the column names are not explicitly defined. By promoting headers, you can easily analyze and process your data.

### Typical Scenario
In a real-world data pipeline, this transformation is often used when ingesting data from external sources, such as CSV files, where the first row contains the column names. For instance, when loading sales data from a CSV file, promoting headers allows for easy filtering, grouping, and calculation of sales metrics.

### Core M Functions Used
`Table.PromoteHeaders`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Header1", "Header2", "Header3"},
        {1, 2, 3},
        {4, 5, 6}
    }),
    PromotedHeaders = Table.PromoteHeaders(Source, [PromoteAllScalars=true])
    in
    PromotedHeaders
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The `Table.PromoteHeaders` function is used to promote the first row of the table to be the column headers.
    2. The `[PromoteAllScalars=true]` argument ensures that all scalar values are promoted to headers, even if they are not explicitly defined as headers.
    3. The resulting table has the first row as column headers, allowing for easy analysis and processing of the data.
