# Apply Row Level Security with Query Folding Considerations

**Description:**
This transformation applies row-level security (RLS) to a Power Query data pipeline, ensuring that users only see authorized data. It achieves this by filtering data based on user-specific criteria, which is then folded into the query to optimize performance. This approach is crucial for maintaining data privacy and compliance with security regulations.

### Typical Scenario
In a real-world data pipeline, this transformation is used when different users or groups need to access a shared data source but with restricted views based on their roles or permissions. For instance, in a sales data dashboard, regional managers should only see sales data for their respective regions.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "North", 100},
        {"Jane", "South", 200},
        {"Bob", "North", 300}
    }, {"Name", "Region", "Sales"}),
    // Simulating user parameter for RLS
    UserRegion = "North",
    // Applying RLS with Table.SelectRows
    FilteredRows = Table.SelectRows(Source, each [Region] = UserRegion)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: A table with user or sales data is created.
2. **User Parameter**: A parameter representing the user's region is defined.
3. **Applying RLS**: The Table.SelectRows function is used to filter the table, only including rows where the 'Region' matches the user's region.
4. **Query Folding**: The filter operation is 'pushed back' to the source, optimizing performance by reducing the amount of data transferred and processed.
