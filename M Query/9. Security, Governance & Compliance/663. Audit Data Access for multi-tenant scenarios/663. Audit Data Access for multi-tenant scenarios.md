# Audit Data Access for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to audit data access in multi-tenant scenarios, ensuring data privacy and security across multiple tenants.

### Typical Scenario
In a real data pipeline, this transformation would be used when an organization needs to ensure that users from different tenants can only access their own data. For instance, a cloud-based service provider wants to monitor and restrict data access for its various clients, ensuring that each client's data remains isolated and secure.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    // Sample table with tenant and data
    data = Table.FromList({
        {"Tenant1", "Data1"},
        {"Tenant1", "Data2"},
        {"Tenant2", "Data3"},
        {"Tenant2", "Data4"}
    }, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    // Filter data for current tenant
    currentTenant = "Tenant1",
    filteredData = Table.SelectRows(data, each [Column1] = currentTenant)
    in
    filteredData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: A table containing data for multiple tenants is created.
2. **Tenant Identification**: The current tenant's ID is defined.
3. **Data Filtering**: The `Table.SelectRows` function is used to filter the data table, selecting only the rows where the tenant matches the current tenant.
4. **Result**: The filtered table is returned, ensuring that only the data for the current tenant is accessible.
