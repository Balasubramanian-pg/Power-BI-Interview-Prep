# Apply Row Level Security as a reusable function

**Description:**
This transformation applies row-level security to a table by filtering rows based on specific conditions. It allows for reusable functions that can be easily integrated into a Power Query pipeline. The goal is to restrict access to certain rows of data based on predefined rules.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when there is a need to restrict access to sensitive data based on user roles or permissions. For example, in a sales database, sales representatives should only have access to their own sales data, while managers should have access to the data of all representatives under their supervision.

### Core M Functions Used
`Table.SelectRows`

```powerquery
let

    data = Table.FromList({
        {"John", 25, "Sales"},

    }, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    // Define a function to apply row-level security
    applyRLS = (table as table, role as text) =>
        Table.SelectRows(table, each [Department] = role),
    // Apply row-level security for 'Sales' role
    salesData = applyRLS(data, "Sales")
    in
    salesData
```

The transformation works as follows:
    1. A sample table 'data' is created with columns for name, age, and department.
    2. A reusable function 'applyRLS' is defined, which takes a table and a role as input.
    3. Inside the 'applyRLS' function, Table.SelectRows is used to filter the rows of the table where the 'Department' column matches the specified role.
    4. The function is then applied to the sample table with the role 'Sales', resulting in a new table 'salesData' that only includes rows where the department is 'Sales'.
