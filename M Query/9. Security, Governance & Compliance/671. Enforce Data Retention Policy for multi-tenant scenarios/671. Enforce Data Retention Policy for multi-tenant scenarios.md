# Enforce Data Retention Policy for multi-tenant scenarios

**Description:**
This transformation ensures that data is retained for a specified period, in accordance with a data retention policy, across multiple tenants. It helps organizations manage and comply with data storage regulations. By enforcing a data retention policy, organizations can automatically remove outdated data, reducing storage costs and improving data management.

### Typical Scenario
In a multi-tenant data environment, such as a cloud-based service, each tenant has its own dataset. The organization needs to enforce a data retention policy that dictates data should be stored for a maximum of 365 days. The transformation would be applied in the data pipeline to filter out data older than 365 days for each tenant, ensuring compliance with the policy.

### Core M Functions Used
`Table.SelectRows`, `Date.AddDays`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "2022-01-01"},
        {"Tenant1", "2022-01-15"},
        {"Tenant2", "2022-02-01"}
    }, {"Tenant", "Date"}),
    // Calculate date 365 days ago
    daysAgo = 365,
    cutoffDate = Date.AddDays(DateTime.LocalNow(), -daysAgo),
    // Filter data to only include rows where date is within the retention period
    FilteredData = Table.SelectRows(Source, each [Date] > cutoffDate)
    in
    FilteredData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Data**: Start with a table containing tenant data and corresponding dates.
    2. **Calculate Cutoff Date**: Determine the cutoff date by subtracting the retention period (365 days) from the current date.
    3. **Filter Data**: Use the `Table.SelectRows` function to filter the data, keeping only the rows where the date is later than the calculated cutoff date.
    4. **Result**: The resulting table contains only the data that complies with the data retention policy for each tenant.
