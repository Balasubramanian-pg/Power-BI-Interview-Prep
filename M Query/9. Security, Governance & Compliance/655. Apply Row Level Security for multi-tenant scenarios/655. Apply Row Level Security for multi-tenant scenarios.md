# Apply Row Level Security for Multi-Tenant Scenarios

**Description:**
This transformation applies row-level security to filter data based on tenant-specific criteria, ensuring that each tenant only accesses their own data. It is commonly used in multi-tenant environments where data segregation is crucial. By applying filters at the row level, organizations can enforce data privacy and security policies effectively.

### Typical Scenario
In a SaaS application that provides data analytics services to multiple clients, each client should only have access to their own data. The data is stored in a shared database, and row-level security is applied to ensure that a client's query results only include their own data.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"TenantID", "1", "1", "2", "2"},
        {"Data", "Data for Tenant 1-1", "Data for Tenant 1-2", "Data for Tenant 2-1", "Data for Tenant 2-2"}
    }, {"TenantID", "Data"}),
    CurrentTenantID = "1",
    FilteredRows = Table.SelectRows(Source, each [TenantID] = CurrentTenantID)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: Start with a table that contains data for multiple tenants. This table includes a column identifying the tenant for each row.
    2. **Specify Current Tenant**: Determine the ID of the current tenant who is accessing the data.
    3. **Apply Filter**: Use the `Table.SelectRows` function to create a new table that only includes rows where the tenant ID matches the current tenant's ID.
    4. **Result**: The resulting table only contains data accessible to the current tenant, effectively applying row-level security.
