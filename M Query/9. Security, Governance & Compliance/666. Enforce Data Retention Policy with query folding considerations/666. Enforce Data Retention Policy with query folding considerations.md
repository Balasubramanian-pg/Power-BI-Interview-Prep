# Enforce Data Retention Policy with Query Folding Considerations

**Description:**
This transformation ensures that data is retained for a specified period, and then removed. It is crucial for compliance with data retention policies. The transformation uses Power Query (M) to filter data based on a date column, ensuring that only relevant data is retained.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to enforce a data retention policy. For example, a financial institution may need to retain transaction data for 7 years, after which it must be deleted. The transformation would be applied to the transaction data table to filter out records older than 7 years.

### Core M Functions Used
`Table.SelectRows`, `Date.AddYears`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", #date(2020, 1, 1), "USA"},
        {"Mary", #date(2021, 1, 1), "UK"},
        {"Peter", #date(2019, 1, 1), "Australia"}
    }, {"Name", "Date", "Country"}),
    RetainedData = Table.SelectRows(Source, each Date.Year([Date]) >= Date.Year(Date.AddYears(Date.Now(), -7)))
    in
    RetainedData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. The source data is loaded into a table.
    2. The `Table.SelectRows` function is used to filter the rows in the table.
    3. The filter condition uses the `Date.AddYears` function to calculate the date 7 years ago from the current date.
    4. The `Table.SelectRows` function then selects only the rows where the date column is greater than or equal to the calculated date 7 years ago.
    5. The resulting table contains only the retained data.
