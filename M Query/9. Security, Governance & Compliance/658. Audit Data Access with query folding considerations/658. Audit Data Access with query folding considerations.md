# Audit Data Access with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to audit data access while considering query folding. It helps in optimizing data access and ensuring that only authorized data is accessed. The transformation achieves this by applying filters and selecting specific rows.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to monitor and control access to sensitive data. For instance, in a financial application, it can be used to ensure that only authorized personnel can view specific financial reports.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "Sales", 100},
        {"Mary", "Marketing", 200},
        {"David", "Sales", 300}
    }, {"Name", "Department", "Sales Amount"}),
    // Applying a filter to fold the query
    FilteredRows = Table.SelectRows(Source, each [Department] = "Sales")
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: A table is created with columns for 'Name', 'Department', and 'Sales Amount'.
    2. **Applying Filter**: The Table.SelectRows function is used to filter the rows where the 'Department' is 'Sales'.
    3. **Query Folding**: The filter operation is folded into the source query if possible, which means the data source itself processes the filter, reducing the amount of data transferred.
    4. **Result**: The resulting table contains only the rows where the department is 'Sales', thus ensuring that only relevant data is accessed and processed.
