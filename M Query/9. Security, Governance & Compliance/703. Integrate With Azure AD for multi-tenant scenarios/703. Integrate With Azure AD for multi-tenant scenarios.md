# Integrate With Azure AD for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used for integrating with Azure AD in multi-tenant scenarios, enabling data retrieval and manipulation across multiple Azure AD tenants.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to aggregate data from multiple Azure AD tenants, such as when a service provider manages multiple customer tenants and needs to compile data across all of them.

### Core M Functions Used
`Web.Contents`, `Json.Document`, `Table.FromList`

### Code Source Execution
```powerquery
let
    // Specify Azure AD tenant IDs and credentials
    tenantIds = {"tenant1_id", "tenant2_id"},
    clientId = "your_client_id",
    clientSecret = "your_client_secret",
    authority = "https://login.microsoftonline.com/",
    resource = "https://graph.microsoft.com/",

    // Function to get access token for a tenant
    getAccessToken = (tenantId) =>
        let
            tokenEndpoint = authority & tenantId & "/oauth2/v2.0/token",
            body = "grant_type=client_credentials&client_id=" & clientId & "&client_secret=" & clientSecret & "&scope=" & resource,
            headers = [#"Content-Type" = "application/x-www-form-urlencoded"]
        in
            Json.Document(Web.Contents(tokenEndpoint, [Headers=headers, Content=Text.ToBinary(body)])),

    // Function to get data using access token
    getData = (tenantId, accessToken) =>
        let
            graphEndpoint = resource & "/v1.0/myorganization",
            headers = [#"Authorization" = "Bearer " & accessToken["access_token"]]
        in
            Json.Document(Web.Contents(graphEndpoint, [Headers=headers]))

    // Main logic to iterate over tenants and retrieve data
    data = List.Transform(tenantIds, each
        let
            token = getAccessToken(_),
            accessToken = token["access_token"],
            result = getData(_, accessToken)
        in
            result)

in
    data
```

### Example Execution Logic Step-By-Step
This script works as follows:
1. It defines a list of Azure AD tenant IDs and credentials (client ID and client secret).
2. It defines a function `getAccessToken` to obtain an access token for each tenant using the OAuth 2.0 token endpoint.
3. It defines another function `getData` to retrieve data from the Microsoft Graph API using the obtained access token.
4. It iterates over the list of tenant IDs, obtains an access token for each, and then uses that token to retrieve data from the Graph API.
5. The results from all tenants are returned as a list of JSON objects.
