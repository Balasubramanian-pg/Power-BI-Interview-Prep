# Enforce Data Retention Policy optimized for incremental refresh

**Description:**
This transformation ensures that data is retained for a specified period, optimizing storage and compliance with data retention policies. It allows for efficient data management by filtering out data that is older than the defined retention period. This is particularly useful in scenarios where data volume is high and storage needs to be managed effectively.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to enforce a data retention policy. For example, a financial services company may only be required to keep customer transaction data for 7 years. As new data is ingested daily, the pipeline would automatically remove data older than 7 years, ensuring compliance with regulatory requirements while also optimizing storage costs.

### Core M Functions Used
`Table.SelectRows`, `Date.AddYears`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Date", "Data"}, 
        {"2020-01-01", "Sample Data 1"},
        {"2021-01-01", "Sample Data 2"},
        {"2022-01-01", "Sample Data 3"}
    }),
    RetentionPeriodYears = 2,
    CurrentDate = Date.FromText("2022-01-01"),
    MaxDate = Date.AddYears(CurrentDate, -RetentionPeriodYears),
    FilteredData = Table.SelectRows(Source, each [Date] >= MaxDate)
    in
    FilteredData
```

### Example Execution Logic Step-By-Step
This transformation works by first defining the data source and the retention period. It then calculates the maximum date for which data should be retained based on the current date and the retention period. The Table.SelectRows function is used with a conditional statement to filter the data, only keeping rows where the date is greater than or equal to the maximum date. The result is a filtered table that only includes data within the specified retention period.
