# Enforce Data Retention Policy for large datasets

**Description:**
This transformation ensures that large datasets adhere to a specified data retention policy, automatically removing or archiving data that exceeds the retention period. It helps organizations manage storage costs and comply with data regulations. By implementing this policy, businesses can efficiently handle data lifecycle management.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to manage and enforce data retention rules for large datasets. For instance, a financial institution may have a policy to retain transaction data for only 7 years. The transformation would be applied to automatically filter out data older than 7 years, ensuring compliance with regulatory requirements and optimizing data storage.

### Core M Functions Used
`Table.SelectRows`, `Date.AddYears`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Date", "Data"},
        {"2020-01-01", "Sample Data 1"},
        {"2021-01-01", "Sample Data 2"},
        {"2022-01-01", "Sample Data 3"}
    }),
    RetentionPeriod = 2,
    CurrentDate = Date.FromText("2022-01-01"),
    FilteredTable = Table.SelectRows(Source, each Date.FromText([Date]) >= Date.AddYears(CurrentDate, -RetentionPeriod))
    in
    FilteredTable
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Define the Data Source**: The process starts with a table containing data with a date column.
    2. **Specify the Retention Period**: Determine the data retention period (e.g., 2 years).
    3. **Calculate the Cut-off Date**: Calculate the cut-off date based on the current date and the retention period.
    4. **Filter the Data**: Use the `Table.SelectRows` function to filter the data, keeping only rows where the date is within the retention period.
    5. **Output the Filtered Data**: The resulting table contains only the data that complies with the retention policy.
