# Audit Data Access for Large Datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to audit data access for large datasets, ensuring data integrity and compliance with access regulations.

### Typical Scenario
In a real data pipeline, this transformation would be used when an organization needs to monitor and log access to sensitive or large datasets, ensuring that only authorized personnel can view or modify the data.

### Core M Functions Used
`Table.SelectRows`, `Table.ColumnNames`, `Table.AddColumn`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"User", "Access", "Data"},
        {"Admin", "Read", "Sales"},
        {"User1", "Write", "Marketing"}
    }),
    AccessLog = Table.AddColumn(Source, "Access Log", each [Access] & " access to " & [Data]),
    FilteredAccess = Table.SelectRows(AccessLog, each [Access] = "Read" or [Access] = "Write")
in
    FilteredAccess
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source Creation**: A table named `Source` is created from a list of rows containing user access data.
2. **Adding Access Log**: A new column `Access Log` is added to the `Source` table, which concatenates the access type with the data accessed.
3. **Filtering Access**: The `Table.SelectRows` function is used to filter the rows where the access type is either 'Read' or 'Write', resulting in `FilteredAccess`.
4. **Result**: The final output is a table containing only the rows where access was granted for reading or writing.
