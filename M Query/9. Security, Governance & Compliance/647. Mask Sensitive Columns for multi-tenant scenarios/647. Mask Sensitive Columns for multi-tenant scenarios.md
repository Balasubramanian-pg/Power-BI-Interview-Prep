# Mask Sensitive Columns for Multi-Tenant Scenarios

**Description:**
This transformation masks sensitive columns in a multi-tenant environment to protect confidential data. It ensures that only authorized users can access specific data. This is particularly useful in scenarios where data is shared across multiple tenants.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with customer data that contains sensitive information, such as credit card numbers or personal identifiable information (PII). For instance, a company may have multiple clients (tenants) who share the same database, but certain columns need to be hidden from some clients for security reasons.
