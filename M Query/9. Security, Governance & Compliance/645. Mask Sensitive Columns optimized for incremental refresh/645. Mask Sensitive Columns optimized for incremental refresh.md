# Mask Sensitive Columns Optimized for Incremental Refresh

**Description:**
This transformation masks sensitive columns in a dataset to protect sensitive information, particularly when using incremental refresh. It ensures that only authorized users can access specific data. This is crucial for maintaining data privacy and security. The transformation helps in safeguarding confidential information.

### Typical Scenario
In a real-world data pipeline, this transformation is used when dealing with customer data that includes sensitive information such as credit card numbers, social security numbers, or personal identifiable information (PII). For instance, a financial institution may use this transformation to mask credit card numbers in their dataset before sharing it with analytics teams for analysis.

### Core M Functions Used
`Table.SelectRows`, `List.Transform`, `Text.Replace`
