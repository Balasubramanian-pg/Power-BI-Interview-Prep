# Audit Data Access with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used for auditing data access while adhering to GDPR and other privacy constraints. It helps in controlling and monitoring data access, ensuring compliance with privacy regulations.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when an organization needs to ensure that sensitive data is accessed only by authorized personnel. For instance, in a healthcare setting, patient data access must be audited to comply with regulations like GDPR and HIPAA.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"User", "DataAccessed", "AccessDate"},
        {"John", "PatientData", "2022-01-01"},
        {"Jane", "FinancialData", "2022-01-02"}
    }),
    // Select rows where access date is after a certain date
    FilteredRows = Table.SelectRows(Source, each [AccessDate] > "2022-01-01"),
    // Select specific columns to limit data exposure
    SelectedColumns = Table.SelectColumns(FilteredRows, {"User", "DataAccessed"})
    in
    SelectedColumns
```

### Example Execution Logic Step-By-Step
This transformation works by first loading data into a table. Then, it filters the rows to only include data accessed after a specific date using the Table.SelectRows function. Finally, it selects only specific columns to limit the exposure of sensitive data using the Table.SelectColumns function.
