# Parameterize Credentials Securely for Multi-Tenant Scenarios

**Description:**
This use case demonstrates how to securely parameterize credentials in Power Query (M) for multi-tenant scenarios, ensuring secure data access and retrieval across multiple tenants.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when integrating data from multiple tenants of a SaaS application, such as Salesforce or Dynamics 365, where each tenant requires its own set of credentials for authentication.

### Core M Functions Used
`Table.SelectRows`, `Web.Contents`

### Code Source Execution
```powerquery
let
    // Define a table with tenant credentials
    credentials = Table.FromList({
        {"tenant1", "username1", "password1"},
        {"tenant2", "username2", "password2"}
    }, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    // Define a function to fetch data for a single tenant
    fetchData = (tenant as text, username as text, password as text) =>
        let
            url = "https://api.example.com/data",
            headers = [#"Authorization" = "Basic " & Binary.ToText(Base64.EncodeText(username & ":" & password))],
            data = Web.Contents(url, [Headers=headers])
        in
            data,
    // Apply the function to each row in the credentials table
    data = Table.AddColumn(credentials, "Data", each fetchData([Column1], [Column2], [Column3]))
in
    data
```

### Example Execution Logic Step-By-Step
This transformation works by first defining a table that stores credentials for multiple tenants. A function, `fetchData`, is then defined to fetch data for a single tenant using the provided credentials. This function utilizes `Web.Contents` to make an HTTP request to the data source with the tenant's credentials in the `Authorization` header. The `fetchData` function is then applied to each row in the credentials table using `Table.AddColumn`, resulting in a table with data fetched for each tenant.
