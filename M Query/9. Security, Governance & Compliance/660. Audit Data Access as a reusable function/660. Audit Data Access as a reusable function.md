# Audit Data Access as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used for auditing data access. It provides a reusable function to track and monitor data access.

### Typical Scenario
In a real data pipeline, this transformation would be used to monitor and track data access for auditing and compliance purposes. For example, in a financial data pipeline, it can be used to track who accessed sensitive financial data and when.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"User", "AccessDate", "DataAccessed"},
        {"John", "2022-01-01", "SalesData"},
        {"Jane", "2022-01-02", "CustomerData"}
    }),
    AccessedData = Table.SelectRows(Source, each [DataAccessed] = "SalesData")
in AccessedData
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. The data source is defined as a table with columns for User, AccessDate, and DataAccessed.
2. The Table.SelectRows function is used to filter the table and select only the rows where DataAccessed equals 'SalesData'.
3. The resulting table contains only the rows where the data accessed was 'SalesData', effectively auditing the data access for that specific data.
