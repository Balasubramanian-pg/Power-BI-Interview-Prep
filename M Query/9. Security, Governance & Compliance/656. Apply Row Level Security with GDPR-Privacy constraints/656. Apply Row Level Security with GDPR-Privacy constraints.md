# Apply Row Level Security with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to apply row-level security with GDPR/privacy constraints. It enables data privacy by filtering out sensitive information based on user access levels.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when sensitive data needs to be protected based on user roles or permissions. For instance, in a sales dataset, regional managers might only have access to sales data for their respective regions, while country managers have access to all regional data.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "North", 100},
        {"Jane", "South", 200},
        {"Bob", "East", 300},
        {"Alice", "West", 400}
    }, {"Name", "Region", "Sales"}),
    // Assume User.RegionAccess = "North"
    RegionAccess = "North",
    FilteredRows = Table.SelectRows(Source, each [Region] = RegionAccess)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: Start with a table containing sensitive data.
    2. **User Access Definition**: Define the user's access level (e.g., region).
    3. **Filtering Data**: Use `Table.SelectRows` to filter the table, keeping only rows where the region matches the user's access level.
    4. **Result**: The resulting table only shows data that the user is authorized to see, thus applying row-level security.
