# Apply Row Level Security optimized for incremental refresh

**Description:**
This transformation applies row-level security to a dataset, ensuring that users only see authorized data. It is optimized for incremental refresh, making it suitable for large datasets. The transformation filters data based on user-specific criteria, enhancing data security and access control.

### Typical Scenario
In a sales data pipeline, different sales teams should only see sales data for their respective regions. This transformation ensures that each team can only access data for their assigned areas, maintaining data confidentiality and compliance.

### Core M Functions Used
`Table.SelectRows`, `Table.FilterRows`

### Code Source Execution
```powerquery
let
    // Sample table
    data = Table.FromList({
        {"John", "North", 100},
        {"Jane", "South", 200},
        {"Bob", "North", 300},
        {"Alice", "South", 400}
    }, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    // Define user region
    userRegion = "North",
    // Apply filter
    filteredData = Table.SelectRows(data, each [Region] = userRegion)
    in
    filteredData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Define the Data Source**: Start with a table that contains the data you want to secure.
    2. **Identify User Credentials**: Determine the user's credentials or attributes (e.g., region) that will be used for filtering.
    3. **Apply Filtering Logic**: Use `Table.SelectRows` or similar functions to filter the data based on the user's credentials.
    4. **Output Filtered Data**: The resulting table will only contain rows that match the user's access criteria, effectively applying row-level security.
