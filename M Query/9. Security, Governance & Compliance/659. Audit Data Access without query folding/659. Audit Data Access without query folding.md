# Audit Data Access without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used for auditing data access without relying on query folding. It allows for the tracking and logging of data access patterns, which is crucial for data governance and security. By implementing this transformation, users can monitor and analyze data access without the need for the data source to support query folding.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a data engineer needs to track and log every access to sensitive data. For instance, in a financial institution, it's crucial to monitor who accesses customer financial data and when. This transformation can be applied at the data source layer to ensure that all data access is properly audited and logged for compliance and security purposes.
