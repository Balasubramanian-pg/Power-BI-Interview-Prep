# Detect PII Patterns for Large Datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to detect Personally Identifiable Information (PII) patterns in large datasets. It helps in identifying sensitive information such as names, addresses, and social security numbers.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when ingesting large datasets from various sources, such as customer information, financial transactions, or healthcare records. For instance, a data engineer might use this transformation to scan a dataset for potential PII patterns before storing it in a data warehouse, ensuring compliance with data protection regulations.

### Core M Functions Used
`Table.SelectRows`, `Text.Contains`, `Text.Regex`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John Doe", "123 Main St", "123-45-6789"},
        {"Jane Smith", "456 Elm St", "987-65-4321"}
    }, {"Name", "Address", "SSN"}),
    PII_Patterns = {
        {"Name", Text.Contains(_, "Doe", Occurrence.Each)},
        {"SSN", Text.Regex(_, "\d{3}-\d{2}-\d{4}")}
    },
    Detect_PII = Table.TransformColumns(Source, PII_Patterns)
    in
    Detect_PII
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Ingestion**: The M script starts by ingesting a sample dataset containing customer information.
    2. **Define PII Patterns**: It then defines the PII patterns to detect, such as names and social security numbers.
    3. **Apply PII Detection**: The script applies these patterns to the dataset using `Table.TransformColumns` and `Text.Contains` or `Text.Regex` functions.
    4. **Output**: The output is a table with an additional column indicating whether a PII pattern was detected for each row.
