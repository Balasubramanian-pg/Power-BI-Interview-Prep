# Apply Row Level Security for large datasets

**Description:**
This transformation applies row-level security to large datasets, ensuring that users only have access to authorized data. It is commonly used in data pipelines where data access needs to be restricted based on user roles or permissions. By applying row-level security, organizations can maintain data confidentiality and integrity.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when there are multiple stakeholders with different access levels to the same dataset. For instance, in a sales dataset, regional managers may only have access to sales data for their respective regions, while the CEO has access to all sales data.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"User", "Region", "Sales"},
        {"John", "North", 100},
        {"Jane", "South", 200},
        {"Bob", "East", 300},
        {"Alice", "West", 400}
    }),
    // Apply row-level security based on user roles
    Role = "North",
    FilteredRows = Table.SelectRows(Source, each [Region] = Role)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: The M query starts with a sample sales dataset containing user information, regions, and sales amounts.
2. **Role Definition**: A variable `Role` is defined to represent the current user's role or region.
3. **Row-Level Security Application**: The `Table.SelectRows` function is used to filter the dataset. It selects only the rows where the 'Region' column matches the defined `Role`.
4. **Result**: The filtered table is then returned, showing only the data that the user is authorized to see.
