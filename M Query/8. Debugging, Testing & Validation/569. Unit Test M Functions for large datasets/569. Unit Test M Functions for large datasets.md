# Unit Test M Functions for Large Datasets

**Description:**
This use case demonstrates how Power Query (M) can be utilized to unit test M functions, particularly when dealing with large datasets. It ensures that M functions behave as expected under various conditions and with different inputs. This process is crucial for validating the correctness and reliability of data transformations.

### Typical Scenario
In a real data pipeline, unit testing M functions for large datasets is essential during the development phase of a Power BI project or when migrating existing data models to a new structure. For instance, if a data engineer is tasked with creating a complex data transformation that involves cleaning, aggregating, and then analyzing large volumes of sales data, they would first test individual M functions with sample large datasets to ensure each function works correctly before integrating them into the larger pipeline.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformRows`, `List.Transform`

### Code Source Execution
```powerquery
let
    // Sample table

    // Function to test

    // Applying the test function
    result = TestFunction(data)

```

### Example Execution Logic Step-By-Step
The transformation typically works as follows inside a Power Query pipeline:
    1. **Define a Sample Dataset**: First, a sample large dataset is created or loaded. This dataset represents the data that the M function will be tested against.
    2. **Create the M Function to Test**: The M function that needs to be tested is defined. This could be a simple function like filtering a table based on a condition.
    3. **Apply the M Function**: The defined M function is then applied to the sample dataset.
    4. **Verify the Results**: The output of the M function is compared against expected results to verify that the function behaves as expected.
    5. **Iterate and Refine**: Based on the test results, the M function may need to be refined or corrected to ensure it works correctly under various conditions and with different inputs.
