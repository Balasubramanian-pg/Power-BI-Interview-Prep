# Simulate Failure Modes as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used to simulate failure modes as a reusable function. It allows for the creation of a flexible and dynamic transformation that can be applied to various data sources. The goal is to provide a reliable and efficient way to handle potential failures in a data pipeline.

### Typical Scenario
In a real-world data pipeline, simulating failure modes can be crucial for testing and validating the robustness of data transformations. For instance, a data engineer might use this transformation to mimic data quality issues, such as missing or erroneous values, to ensure that the pipeline can handle such scenarios gracefully.

### Code Source Execution
```powerquery
let
    // Sample table
    data = Table.FromList({
        {"Item1", 10},
        {"Item2", 20},
        {"Item3", 30}
    }, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    // Simulate failure mode: randomly set some values to null
    simulateFailure = (table as table) =>
        let
            rows = Table.ToRows(table),
            failedRows = List.Transform(rows, each
                let
                    random = Number.Random(),
                    value = if random < 0.5 then null else _[Value]
                in
                    {_[Item], value}
            ),
            failedTable = Table.FromRows(failedRows, {"Item", "Value"})
        in
            failedTable,
    // Apply the simulateFailure function
    failedData = simulateFailure(data)
    in
        failedData
```

    6. Apply the `simulateFailure` function to the original data table to obtain a new table with simulated failures.
