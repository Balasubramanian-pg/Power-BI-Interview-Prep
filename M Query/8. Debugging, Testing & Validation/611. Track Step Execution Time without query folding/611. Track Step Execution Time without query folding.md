# Track Step Execution Time without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to track the execution time of steps without query folding. It provides a way to measure the performance of individual steps in a Power Query pipeline.

### Typical Scenario
In a real data pipeline, tracking step execution time is crucial for optimizing performance. For instance, when dealing with large datasets, it helps identify which steps are taking the most time, allowing for targeted optimizations.

### Core M Functions Used
`Table.SelectRows`, `DateTime.ToText`, `DateTime.FixedLocalNow`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Step", "Start Time", "End Time", "Duration"},
        {"Step 1", DateTime.FixedLocalNow(), null, null},
        {"Step 2", null, null, null}
    }),
    GetStartTime = (stepName) => DateTime.FixedLocalNow(),
    GetEndTime = (stepName) => DateTime.FixedLocalNow(),
    CalculateDuration = (start, end) => Duration.ToSeconds(end - start),
    AddStartTime = Table.TransformColumns(Source, {{"Start Time", each GetStartTime(_)}},
    AddEndTime = Table.TransformColumns(AddStartTime, {{"End Time", each GetEndTime(_)}},
    AddDuration = Table.TransformColumns(AddEndTime, {{"Duration", each CalculateDuration([Start Time], [End Time])}})
    in
    AddDuration
```

### Example Execution Logic Step-By-Step
The transformation works by first creating a table with steps and their respective start and end times. It then uses M functions like DateTime.FixedLocalNow() to get the current time and calculate the duration of each step. The start and end times are added to the table using Table.TransformColumns. Finally, the duration is calculated and added to the table.
