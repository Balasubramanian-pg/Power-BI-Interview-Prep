# Assert Row Counts without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to assert row counts without query folding, ensuring data integrity in a pipeline.

### Typical Scenario
In a real data pipeline, this transformation would be used to verify that a specific data extract or transformation step has produced the expected number of rows, helping to catch any unexpected data losses or additions.

### Core M Functions Used
`Table.RowCount`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25},
        {"Jane", 30},
        {"Bob", 35}
    }, {"Name", "Age"}),
    RowCount = Table.RowCount(Source),
    ExpectedRowCount = 3,
    Assert = if RowCount = ExpectedRowCount then Source else error "Row count mismatch"
    in
    Assert
```

### Example Execution Logic Step-By-Step
The transformation works by first loading data into a table. It then counts the number of rows in the table using Table.RowCount. An expected row count is defined. The transformation asserts that the actual row count matches the expected row count. If they match, the table is returned; otherwise, an error is raised indicating a row count mismatch.
