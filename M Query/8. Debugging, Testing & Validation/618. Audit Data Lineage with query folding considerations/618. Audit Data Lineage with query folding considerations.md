# Audit Data Lineage with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to track and manage data lineage while considering query folding. It helps in understanding where data originates from and how it is transformed throughout the data pipeline.

### Typical Scenario
In a real data pipeline, this transformation is crucial for data governance and compliance. For instance, when a business analyst needs to track the origin of specific data points used in reports, this transformation helps in tracing back the data lineage.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Data Source", "Data Type", "Transformation"},
        {"SalesDB", "Integer", "Data Load"},
        {"SalesDB", "String", "Data Transformation"},
        {"MarketingDB", "String", "Data Load"}
    }),
    // Select rows where Data Source is 'SalesDB'
    FilteredRows = Table.SelectRows(Source, each ["Data Source"] = "SalesDB")
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source Table Creation**: A table is created with columns for 'Data Source', 'Data Type', and 'Transformation'.
    2. **Row Selection**: Using the `Table.SelectRows` function, rows are filtered where the 'Data Source' is 'SalesDB'.
    3. **Result**: The resulting table contains only the rows that originated from 'SalesDB', helping in tracing the lineage of data from this source.
