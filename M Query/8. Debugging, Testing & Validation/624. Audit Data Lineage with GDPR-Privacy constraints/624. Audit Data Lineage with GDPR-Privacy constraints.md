# Audit Data Lineage with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used for auditing data lineage while adhering to GDPR/Privacy constraints. It helps track data sources and transformations, ensuring compliance with data protection regulations.

### Typical Scenario
In a real data pipeline, this transformation would be used when sensitive data is being processed and GDPR/Privacy constraints need to be enforced. For instance, when personal data is being loaded from various sources into a data warehouse, this transformation can be applied to maintain a record of data lineage, ensuring that data can be traced back to its original source and that all transformations are documented.
