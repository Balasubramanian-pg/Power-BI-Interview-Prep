# Track Step Execution Time as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used to track the execution time of a step as a reusable function. It allows for monitoring and optimizing the performance of data pipelines. By tracking execution times, users can identify bottlenecks and improve overall efficiency.

### Typical Scenario
In a real data pipeline, tracking step execution time is crucial for performance optimization. For instance, when dealing with large datasets, users need to identify which steps are taking the most time to execute. This information helps in optimizing those specific steps, thereby improving the overall performance of the data pipeline.
