# Unit Test M Functions Optimized for Incremental Refresh

**Description:**
This use case demonstrates how Power Query (M) can be used for unit testing M functions optimized for incremental refresh. It allows for efficient testing of functions that are designed to work with incremental data refreshes, ensuring data consistency and accuracy.

### Typical Scenario
In a real data pipeline, this transformation would be used when you have a large dataset that is updated periodically, and you want to test specific M functions on new or changed data without reprocessing the entire dataset.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`, `List.Distinct`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Date"},
        {1, "John", #date(2022,1,1)},
        {2, "Jane", #date(2022,1,2)}
    }),
    IncrementalRefresh = Table.SelectRows(Source, each [Date] >= #date(2022,1,1)),
    KeyFunctionsTested = List.Distinct(IncrementalRefresh[Name])
    in
    KeyFunctionsTested
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Data**: Start with a sample table that includes an 'ID', 'Name', and 'Date' column.
    2. **Incremental Refresh**: Use `Table.SelectRows` to filter the table for rows where the 'Date' is on or after a specific date (e.g., #date(2022,1,1)). This simulates the incremental refresh.
    3. **Key Functions Tested**: Apply `List.Distinct` on the filtered 'Name' column to test the M functions' behavior on the incrementally refreshed data.
    4. **Output**: The result is a list of distinct names from the incrementally refreshed data.
