# Detect Null Injection without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to detect null injection in a data pipeline without relying on query folding. It helps in identifying and handling null values that could potentially disrupt the data processing flow. By detecting null injections, data pipelines can be made more robust and reliable.

### Typical Scenario
In a real-world data pipeline, detecting null injection is crucial when integrating data from multiple sources. For instance, when combining data from various APIs, databases, or files, null values might be introduced intentionally or unintentionally. This transformation ensures that such null values are identified and handled appropriately, preventing potential errors or inconsistencies in the downstream analysis or processing.

### Core M Functions Used
`Table.SelectRows`, `List.Contains`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {1, "John", "john@example.com"},
        {2, null, "jane@example.com"},
        {3, "Bob", null}
    }, {"ID", "Name", "Email"}),
    DetectNulls = Table.SelectRows(Source, each not (Name = null or Email = null))
    in
    DetectNulls
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Table Creation**: A sample table is created with columns 'ID', 'Name', and 'Email'.
    2. **Null Detection**: The Table.SelectRows function is used with a condition to filter out rows where either 'Name' or 'Email' is null.
    3. **Result**: The resulting table contains only the rows where both 'Name' and 'Email' are not null, effectively detecting and excluding rows with null injections.
