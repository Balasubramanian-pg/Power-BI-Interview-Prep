# Replay Transformations as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used for replay transformations as a reusable function. It allows for the transformation of data to be reused across different parts of a data pipeline.

### Typical Scenario
In a real data pipeline, replay transformations would be used when there is a need to apply the same data transformation logic multiple times. For instance, in a data warehousing scenario, a data transformation function can be created to convert date formats, and this function can be reused across different data tables.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Age"},
        {1, "John", 25},
        {2, "Jane", 30},
        {3, "Bob", 35}
    }),
    Transform = Table.SelectRows(Source, each "Age" > 25)
    in
    Transform
```

### Example Execution Logic Step-By-Step
The transformation works by first creating a table with sample data. Then, it uses the Table.SelectRows function to filter the rows where the 'Age' is greater than 25. This results in a new table that only includes rows where the condition is met. The transformation can be wrapped in a function to make it reusable across the data pipeline.
