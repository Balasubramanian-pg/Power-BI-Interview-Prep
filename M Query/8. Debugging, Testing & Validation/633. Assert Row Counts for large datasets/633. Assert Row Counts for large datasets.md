# Assert Row Counts for large datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to assert row counts for large datasets, ensuring data integrity and consistency. It helps in verifying that the expected number of rows is present in a dataset. This is particularly useful during data migration, ETL processes, or data validation.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when validating data extracts from source systems to ensure they match expected volumes, or during data migration to verify that all records have been successfully transferred.

### Core M Functions Used
`Table.RowCount`, `Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25},
        {"Jane", 30},
        {"Bob", 35}
    }, {"Name", "Age"}),
    ExpectedRowCount = 3,
    ActualRowCount = Table.RowCount(Source),
    Assert = if ActualRowCount = ExpectedRowCount then Source else error "Row count assertion failed"
    in
    Assert
```

### Example Execution Logic Step-By-Step
The transformation works by first defining a source table. It then specifies the expected row count. The Table.RowCount function is used to get the actual row count of the source table. An assertion is made by comparing the actual row count to the expected row count. If they match, the source table is returned; otherwise, an error is raised indicating that the row count assertion failed.
