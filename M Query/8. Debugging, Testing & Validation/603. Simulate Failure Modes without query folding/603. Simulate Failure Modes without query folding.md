# Simulate Failure Modes without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to simulate failure modes without query folding, allowing for the testing of data pipelines under various failure scenarios.

### Typical Scenario
In a real data pipeline, simulating failure modes can help ensure data resilience and robustness. For instance, a data engineer might use this transformation to test how a pipeline behaves when a data source is temporarily unavailable or when data is corrupted.

### Core M Functions Used
`Table.SelectRows`, `List.Generate`, `try`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Age"},
        {1, "John", 25},
        {2, "Jane", 30},
        {3, "Bob", 35}
    }),
    SimulateFailure = Table.SelectRows(Source, each try [Age] > 25 catch [Age] = 0),
    Result = List.Generate(() => SimulateFailure, each _)
    in
    Result
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. A sample table is created with columns 'ID', 'Name', and 'Age'.
    2. The 'SimulateFailure' step uses 'Table.SelectRows' with a try-catch block to simulate a failure mode. If the 'Age' is greater than 25, it returns the row; otherwise, it sets 'Age' to 0.
    3. The 'List.Generate' function generates a list of tables, each applying the 'SimulateFailure' step.
