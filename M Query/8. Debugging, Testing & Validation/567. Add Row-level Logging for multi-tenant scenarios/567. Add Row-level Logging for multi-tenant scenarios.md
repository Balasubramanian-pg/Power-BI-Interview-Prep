# Add Row-level Logging for multi-tenant scenarios

**Description:**
This transformation adds row-level logging to support multi-tenant scenarios, allowing for detailed tracking and auditing of data access and modifications across multiple tenants.

### Typical Scenario
In a multi-tenant data environment, such as a SaaS application, this transformation can be used to log user interactions, data changes, or access events at the row level. For instance, when a user from Tenant A updates a specific record, the transformation logs the tenant ID, user ID, timestamp, and details of the change. This log can then be used for auditing, compliance, or troubleshooting purposes.
