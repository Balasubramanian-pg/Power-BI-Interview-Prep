# Audit Data Lineage without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used for auditing data lineage without relying on query folding. It helps track data sources and transformations applied to the data.

### Typical Scenario
In a real data pipeline, this transformation is crucial for compliance and data governance. For instance, a financial institution needs to track the origin and transformations of customer data to ensure regulatory compliance.

### Core M Functions Used
`Table.SelectRows`, `Table.AddColumn`, `Table.TransformColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Data Source", "Transformation"},
        {"Database", "Query"},
        {"File", "Import"}
    }),
    AddedCustom = Table.AddColumn(Source, "Audit Log", each "Transformed"),
    SelectRows = Table.SelectRows(AddedCustom, each [Transformation] = "Query")
    in
    SelectRows
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
    1. **Source Data**: A table is created with columns for 'Data Source' and 'Transformation'.
    2. **Adding Audit Log**: A new column 'Audit Log' is added to track transformations, with a custom value 'Transformed'.
    3. **Filtering Rows**: The Table.SelectRows function filters the table to only include rows where the 'Transformation' is 'Query', demonstrating data lineage.
