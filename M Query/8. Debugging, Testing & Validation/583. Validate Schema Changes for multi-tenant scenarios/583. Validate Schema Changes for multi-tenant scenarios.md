# Validate Schema Changes for multi-tenant scenarios

**Description:**
This transformation validates schema changes in a multi-tenant environment, ensuring data consistency across different tenants. ### Typical Scenario
In a real data pipeline, this transformation would be used when integrating data from multiple tenants, each with their own schema. For instance, a company providing a SaaS solution might have different tenants, each with their own database schema. When consolidating data from these tenants, it's crucial to validate schema changes to ensure data consistency and accuracy.

        {"Tenant1", "Name", "Age"},

    }, {"Tenant", "Column1", "Column2"}),
    GetColumnNames = (table as table) => Table.ColumnNames(table),
    ColumnNames = GetColumnNames(Source),
    ValidateSchema = (columnNames as list) =>
        let
            ExpectedSchema = {"Name", "Age"},
            Diff = List.Difference(columnNames, ExpectedSchema)
        in
            if Diff <> {} then
                error "Schema mismatch: " & Text.FromList(Diff)
            else
                "Schema is valid"
    ValidateResult = ValidateSchema(ColumnNames)
    in
    ValidateResult
