# Replay Transformations without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used for replay transformations without query folding. It allows for the re-execution of transformations on already loaded data, which is particularly useful when the original data source changes or when testing different transformation scenarios.

### Typical Scenario
In a real data pipeline, this transformation would be used when you need to reapply a series of transformations to data that has already been loaded into the Power Query environment. For instance, if you're working with a large dataset and you want to test different filtering or grouping scenarios without having to reload the data from the source each time.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"David", 42, "Mexico"}
    }, {"Name", "Age", "Country"}),
    // Apply transformations here, e.g.,
    FilteredRows = Table.SelectRows(Source, each Age > 30),
    TransformedRows = Table.TransformRows(FilteredRows, each {_.Name, _.Age * 2, _.Country})
    in
    TransformedRows
```

### Example Execution Logic Step-By-Step
The transformation typically works as follows inside a Power Query pipeline:
    1. Load your data into Power Query.
    2. Apply an initial set of transformations to the data.
    3. Use functions like Table.SelectRows or Table.TransformRows to filter or modify the data.
    4. Once you have the desired transformations, you can re-execute them on the already loaded data by simply re-running the transformation steps.
    This process allows for quick testing and iteration on different transformation scenarios without the need to re-query the original data source each time.
