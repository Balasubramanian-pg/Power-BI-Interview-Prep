# Simulate Failure Modes for large datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to simulate failure modes for large datasets, allowing for the identification and analysis of potential failures in a data pipeline.

### Typical Scenario
In a real data pipeline, this transformation would be used when data engineers need to stress test their data processing workflows against potential failures, such as data corruption, system crashes, or incorrect data transformations.

### Core M Functions Used
`Table.SelectRows`, `List.Random`, `Int32.Random`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Age"},
        {1, "John", 25},
        {2, "Jane", 30},
        {3, "Bob", 35}
    }),
    SimulateFailure = Table.SelectRows(Source, each Int32.Random() > 50),
    FailureMode = if List.Count(Table.ColumnNames(SimulateFailure)) = 0 then "No failures" else "Failure"
    in
    SimulateFailure
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. Start with a sample dataset.
    2. Use Table.SelectRows to randomly select rows based on a condition that simulates a failure (e.g., Int32.Random() > 50).
    3. Analyze the resulting table to identify which rows were selected (or not) due to the simulated failure.
    4. This process can be repeated multiple times to simulate different failure modes and analyze their impact on the data pipeline.
