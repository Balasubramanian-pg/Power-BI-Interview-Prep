# Assert Row Counts for Multi-Tenant Scenarios

**Description:**
This transformation ensures that the expected number of rows is present in a table for multi-tenant data pipelines, helping to identify discrepancies in data loading or processing.

### Typical Scenario
In a multi-tenant data warehouse, each client's data is segregated by a tenant ID. When loading data for analytics, it's crucial to verify that the correct number of records is loaded for each tenant to ensure data integrity and accurate reporting.

### Core M Functions Used
`Table.SelectRows`, `Table.RowCount`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"TenantID", "Data"},
        {"Tenant1", "SampleData1"},
        {"Tenant2", "SampleData2"}
    }, {"TenantID", "Data"}),
    GroupedRows = Table.Group(Source, {"TenantID"}, {{"RowCount", each Table.RowCount(_), type number}}),
    AssertRowCount = Table.SelectRows(GroupedRows, each [RowCount] = 1)
in AssertRowCount
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: Start with a table that contains tenant-specific data.
2. **Grouping**: Group the data by the 'TenantID' column.
3. **Row Count**: Calculate the row count for each tenant group.
4. **Assertion**: Filter the groups to only include those with an expected row count (e.g., 1 row per tenant).
5. **Result**: The resulting table contains only the tenants with the expected number of rows, helping to identify any discrepancies.
