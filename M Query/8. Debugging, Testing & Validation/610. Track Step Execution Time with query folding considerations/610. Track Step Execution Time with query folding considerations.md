# Track Step Execution Time with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to track the execution time of steps while considering query folding. It helps in optimizing and understanding the performance of data transformations.

### Typical Scenario
In a real data pipeline, tracking step execution time with query folding considerations is crucial when dealing with large datasets. For instance, when extracting data from a database, it helps in identifying performance bottlenecks and optimizing the query for better performance.

### Core M Functions Used
`Table.SelectRows`, `Query.Folding`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"Peter", 42, "UK"}
    }, {"Name", "Age", "Country"}),
    // Simulating a slow step
    SlowStep = Table.AddColumn(Source, "Slow Calculation", each Number.Sqrt(Number.Power([Age], 2)) * 2),
    // Tracking execution time
    Stopwatch = (func as function) => let
        Start = DateTime.FixedLocalNow(),
        Result = func(),
        End = DateTime.FixedLocalNow()
    in {
        Result,
        Duration.Hours(End - Start)
    },
    TrackedSlowStep = Stopwatch(() => SlowStep),
    ResultTable = Table.FromList(TrackedSlowStep, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    RenamedColumns = Table.RenameColumns(ResultTable, {{0, "Result"}, {1, "Execution Time (hours)"}})
    in RenamedColumns
```

### Example Execution Logic Step-By-Step
The script works as follows:
1. A sample table 'Source' is created.
2. A slow step 'SlowStep' is simulated by adding a column that performs a computationally intensive operation.
3. A 'Stopwatch' function is defined to track the execution time of any given function.
4. The 'Stopwatch' function is applied to 'SlowStep' to measure its execution time.
5. The result and execution time are combined into a single table 'ResultTable'.
6. Finally, the columns are renamed for clarity.
