# Audit Data Lineage for Large Datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to track and manage data lineage for large datasets, ensuring data integrity and compliance with regulatory requirements.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with large datasets from multiple sources, such as financial transactions or customer data, to maintain a clear record of data origin, processing, and changes.

### Core M Functions Used
`Table.SelectRows`, `Table.AddColumn`, `Table.TransformColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Data Source", "Data Type", "Value"},
        {"Source1", "Integer", "10"},
        {"Source2", "String", "Text"}
    }),
    AddLineage = Table.AddColumn(Source, "Lineage", each "Source: " & ["Data Source"] & ", Type: " & ["Data Type"]),
    TransformData = Table.TransformColumns(AddLineage, {{"Value", each if ["Data Type"] = "Integer" then Number.FromText(_) else _}})
    in
    TransformData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Table Creation**: A table is created with columns for data source, data type, and value.
    2. **Adding Lineage**: A new column named 'Lineage' is added to the table, which concatenates the data source and data type for each row.
    3. **Data Transformation**: The 'Value' column is transformed based on the 'Data Type'. If the type is 'Integer', the value is converted to a number; otherwise, it is left as is.
    4. **Result**: The final table includes the original data along with its lineage and transformed values.
