# Assert Row Counts with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to assert row counts while adhering to GDPR/privacy constraints. It ensures data integrity and compliance with data protection regulations.

### Typical Scenario
In a real data pipeline, this transformation would be used when validating data extracts for GDPR compliance, ensuring that personally identifiable information (PII) is handled correctly and that data row counts are accurately reported without exposing sensitive data.

### Core M Functions Used
`Table.SelectRows`, `Table.RowCount`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "UK"},
        {"David", 42, "Australia"}
    }, {"Name", "Age", "Country"}),
    // Apply GDPR/Privacy constraints
    ProtectedData = Table.SelectRows(Source, each not Text.Contains(Name, "John")) ,
    // Assert Row Count
    RowCount = Table.RowCount(ProtectedData)
    in
    RowCount
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: Start with a table containing personal data.
2. **Apply GDPR/Privacy Constraints**: Use `Table.SelectRows` to filter out rows that contain sensitive information (e.g., names that could identify individuals).
3. **Assert Row Count**: Use `Table.RowCount` to count the number of rows in the filtered table, ensuring that the data extract complies with GDPR/privacy requirements.
4. **Output**: The row count of the filtered data, providing a verifiable assertion of data integrity and compliance.
