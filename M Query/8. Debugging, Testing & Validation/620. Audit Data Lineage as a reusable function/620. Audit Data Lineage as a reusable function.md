# Audit Data Lineage as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used for auditing data lineage as a reusable function. It helps track the origin and transformations applied to data throughout its lifecycle.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to monitor and document the flow of data from source systems through various transformations and loading into target systems. For instance, during data migration or integration projects, it's crucial to maintain an audit trail of data lineage to ensure data quality and compliance.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`, `Table.AddColumn`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Data Source", "Transformation"}, 
        {"Source A", "Loaded"},
        {"Source B", "Transformed"}
    }),
    AddLineage = Table.AddColumn(Source, "Lineage", each "Audit: " & [Transformation]),
    SelectRelevantColumns = Table.SelectColumns(AddLineage,{"Data Source", "Lineage"})
    in
    SelectRelevantColumns
```

### Example Execution Logic Step-By-Step
The transformation typically works as follows inside a Power Query pipeline:
    1. **Source Data**: Begin with a table that contains information about the data sources and the transformations applied.
    2. **Add Lineage Column**: Use `Table.AddColumn` to create a new column that logs the lineage by concatenating an audit prefix with the transformation details.
    3. **Select Relevant Columns**: Utilize `Table.SelectColumns` to filter the table and only include columns of interest, such as the data source and lineage.
    4. **Reusable Function**: Wrap these steps in a function that can be easily reused across different parts of the data pipeline to consistently track data lineage.
