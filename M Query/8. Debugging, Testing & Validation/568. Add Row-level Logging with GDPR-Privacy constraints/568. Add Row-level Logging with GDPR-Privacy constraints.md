# Add Row-level Logging with GDPR/Privacy constraints

**Description:**
This transformation adds row-level logging to a dataset while adhering to GDPR and privacy constraints. It enables tracking data lineage and access at the row level, ensuring compliance with data protection regulations. This is particularly useful in sensitive data environments where data access and modifications need to be meticulously recorded.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when handling sensitive customer data. For instance, in a customer relationship management (CRM) system, adding row-level logging ensures that any changes to customer data are tracked and recorded. This is crucial for maintaining transparency and compliance with GDPR and other privacy laws.
