# Audit Data Lineage for Multi-Tenant Scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to track and manage data lineage across multiple tenants, ensuring data integrity and compliance. It helps in identifying the source, processing, and destination of data for each tenant.

### Typical Scenario
In a multi-tenant data environment, such as a cloud-based data platform serving various clients, it's crucial to maintain a clear lineage of data. This transformation is used to track data from its origin, through various transformations and processes, to its final destination, ensuring that data can be traced back to its source for auditing and compliance purposes.

### Core M Functions Used
`Table.SelectRows`, `Table.AddColumn`, `Table.TransformColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "Data1", "Source1"},
        {"Tenant2", "Data2", "Source2"}
    }, {"Tenant", "Data", "Source"}),
    AddLineage = Table.AddColumn(Source, "Lineage", each "Source: " & [Source] & ", Data: " & [Data]),
    TransformLineage = Table.TransformColumns(AddLineage, {{"Lineage", each _ & ", Tenant: " & [Tenant]}})
    in
    TransformLineage
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Data**: Start with a table containing data from multiple tenants, including the tenant identifier, data, and its source.
    2. **Add Lineage Column**: Use `Table.AddColumn` to add a new column that starts tracing the lineage, initially recording the source and data.
    3. **Transform Lineage**: Use `Table.TransformColumns` to further enhance the lineage information by appending the tenant information to the lineage column.
    4. **Result**: The final table contains a comprehensive lineage for each data record, showing its source, processing steps, and associated tenant.
