# Simulate Failure Modes with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to simulate failure modes while considering query folding. It helps in understanding how data transformations can be optimized and folded back to the source, improving performance. This is particularly useful in data pipelines where data is large and complex.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large datasets where simulating failure modes is crucial for robustness testing. For instance, a data engineer might use this to test how a data pipeline behaves when certain data sources are temporarily down or return errors.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Age"},
        {1, "John", 25},
        {2, "Jane", 30},
        {3, "Bob", 35}
    }),
    SimulateFailure = Table.TransformRows(Source, each try [_] otherwise [Error = "Simulated Failure"]),
    FoldedQuery = Table.SelectRows(SimulateFailure, each [Age] > 25)
    in
    FoldedQuery
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Data**: Start with a simple table containing ID, Name, and Age columns.
    2. **Simulate Failure**: Use `Table.TransformRows` to simulate a failure mode. For demonstration, this tries to return each row as is, but if an error occurs, it returns a row with a 'Simulated Failure' error message.
    3. **Query Folding**: Apply a filter using `Table.SelectRows` to only include rows where Age is greater than 25. Power Query's query folding capabilities will attempt to push this filter back to the source if possible, optimizing performance by reducing the amount of data processed.
    4. **Result**: The final output is a table that includes only the rows where Age > 25, with any failure modes appropriately handled.
