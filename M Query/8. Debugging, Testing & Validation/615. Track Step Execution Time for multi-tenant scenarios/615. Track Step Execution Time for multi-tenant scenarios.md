# Track Step Execution Time for Multi-Tenant Scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to track the execution time of steps in a data pipeline for multi-tenant scenarios. It helps in monitoring and optimizing the performance of data transformations across different tenants.

### Typical Scenario
In a real data pipeline, this transformation would be used when there are multiple tenants sharing the same data pipeline, and the data team needs to monitor the execution time of each step for each tenant. For instance, in a SaaS application, tracking step execution time helps in identifying performance bottlenecks specific to certain tenants, enabling targeted optimizations.

### Core M Functions Used
`Table.SelectRows`, `List.Timing`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "2022-01-01"},
        {"Tenant2", "2022-01-02"}
    }, {"Tenant", "Date"}),
    GetExecutionTime = (steps as list) =>
        let
            timings = List.Timing(steps),
            result = Table.FromList(timings, {"Step", "ExecutionTime"})
        in
            result,
    Steps = {"Step1", "Step2", "Step3"},
    ExecutionTime = GetExecutionTime(Steps)
    in
        ExecutionTime
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Define the Source Data**: The pipeline starts with a table containing tenant information.
    2. **Define a Function to Get Execution Time**: A custom function `GetExecutionTime` takes a list of steps and uses `List.Timing` to measure the execution time of each step.
    3. **Specify the Steps**: A list of steps (e.g., "Step1", "Step2", "Step3") is defined for which the execution time needs to be tracked.
    4. **Calculate Execution Time**: The function `GetExecutionTime` is called with the list of steps to calculate the execution time.
    5. **Result**: The result is a table with the step names and their respective execution times, which can be used for further analysis or monitoring.
