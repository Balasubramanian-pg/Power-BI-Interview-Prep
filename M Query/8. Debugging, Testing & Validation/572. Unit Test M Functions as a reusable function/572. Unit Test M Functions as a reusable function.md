# Unit Test M Functions as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used for unit testing M functions as a reusable function. It allows for the validation of M functions in isolation, ensuring they behave as expected. This is particularly useful in complex data pipelines where functions need to be thoroughly tested.

### Typical Scenario
In a real data pipeline, unit testing M functions can be crucial when developing a complex data transformation process. For instance, if a data engineer is creating a function to clean and transform customer data, they can use this approach to test the function with sample data before applying it to the entire dataset.

### Code Source Execution

```powerquery
let
    // Sample M function to be tested
    AddOne = (x) => x + 1,
    // Reusable function for unit testing
    TestFunction = (func, input, expectedOutput) =>
        let
            result = func(input),
            pass = result = expectedOutput
        in
            [Pass = pass, Result = result, ExpectedOutput = expectedOutput],
    // Test the AddOne function
    TestAddOne = TestFunction(AddOne, 5, 6)
    
in
    TestAddOne
```

### Example Execution Logic Step-By-Step
The transformation typically works as follows inside a Power Query pipeline:
1. Define the M function to be tested, such as `AddOne`.
2. Create a reusable unit testing function, `TestFunction`, which takes another function, an input, and an expected output.
3. Apply the `TestFunction` to the function being tested, providing the necessary input and expected output.
4. The `TestFunction` executes the provided function with the given input and compares the result with the expected output.
5. The test result, including whether the test passed, the actual result, and the expected output, is returned and can be reviewed or further processed.
