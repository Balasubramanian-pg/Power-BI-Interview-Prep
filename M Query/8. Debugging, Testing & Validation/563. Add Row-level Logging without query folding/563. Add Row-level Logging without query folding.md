# Add Row-level Logging without query folding

**Description:**
This transformation adds row-level logging to a Power Query pipeline without relying on query folding. It allows for the inclusion of detailed logs at the row level, which can be useful for auditing, debugging, or data lineage purposes. By incorporating logging directly within the M code, users can ensure that each row is processed and logged accordingly.

### Typical Scenario
In a real-world data pipeline, this transformation can be used when you need to track changes or movements of specific data rows through your ETL process. For instance, when integrating data from multiple sources, you might want to log which rows are being added, updated, or deleted during the integration process.

### Example Execution Logic Step-By-Step
The transformation works by adding a new column to the existing table, where each row in this new column contains a log message specific to that row. This is achieved using the `Table.AddColumn` function in Power Query's M language. The function takes a table, a new column name, and an expression that defines the value for each row in the new column. In this case, the expression concatenates a static string with the value from the 'Name' column of each row, effectively creating a log entry for each row processed.
