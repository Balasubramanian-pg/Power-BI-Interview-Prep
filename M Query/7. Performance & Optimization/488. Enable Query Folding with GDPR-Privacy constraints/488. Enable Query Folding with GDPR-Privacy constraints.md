# Enable Query Folding with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to enable query folding while adhering to GDPR/privacy constraints. It allows for efficient data retrieval while ensuring sensitive information is protected. By applying specific transformations, users can filter and anonymize data according to privacy regulations.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when handling sensitive customer data. For instance, a company may need to provide analytics data to stakeholders while ensuring that personally identifiable information (PII) is not exposed. By enabling query folding with GDPR/privacy constraints, the company can filter data at the source, reducing the amount of data that needs to be processed and minimizing the risk of data breaches.

### Core M Functions Used
`Table.SelectRows`, `Table.FilterRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "john@example.com"},
        {"Jane", 30, "jane@example.com"},
        {"Bob", 35, "bob@example.com"}
    }, {"Name", "Age", "Email"}),
    // Apply GDPR/Privacy constraints
    ProtectedData = Table.SelectRows(Source, each not Text.Contains([Email], "example.com"))
    in
    ProtectedData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: The process begins with a data source, such as a table containing customer information.
    2. **Applying Constraints**: M functions like `Table.SelectRows` are used to filter out sensitive information. For example, you might filter out rows where the email address contains a specific domain to anonymize the data.
    3. **Query Folding**: Power Query optimizes the query by 'folding' the filter operations back to the source, if possible. This means that instead of retrieving all data and then filtering, the filtering occurs at the data source level, reducing the amount of data transferred.
    4. **Result**: The resulting table contains only the data that meets the specified criteria, ensuring that sensitive information is protected according to GDPR/privacy constraints.
