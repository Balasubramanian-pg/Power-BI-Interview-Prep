# Minimize Column Scans for large datasets

**Description:**
This transformation optimizes data processing by reducing the number of column scans, improving performance when handling large datasets. It achieves this by selectively accessing only the required columns, thus minimizing unnecessary data reads.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with massive datasets where not all columns are necessary for analysis. For instance, when loading a large customer database but only needing a subset of columns such as names, addresses, and IDs for a specific report.

### Core M Functions Used
`Table.SelectColumns`

```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Jane", 30, "UK"},
        {"Tom", 35, "Australia"}
    }, {"Name", "Age", "Country"}),
    // Select only the required columns to minimize scans
    SelectedColumns = Table.SelectColumns(Source, {"Name", "Age"})
    in
    SelectedColumns
```

### Example Execution Logic Step-By-Step
The transformation works by first loading a large dataset into Power Query. Then, instead of scanning all columns, it uses the Table.SelectColumns function to specify and select only the columns that are necessary for the analysis or report. This approach significantly reduces the amount of data that needs to be processed, leading to improved performance and efficiency.
