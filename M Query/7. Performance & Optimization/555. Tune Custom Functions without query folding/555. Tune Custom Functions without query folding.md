# Tune Custom Functions without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to optimize custom functions without relying on query folding, allowing for more flexible and efficient data transformations.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when you need to apply complex, non-foldable operations to large datasets. For instance, when you're working with a massive dataset and need to apply a custom function that doesn't support query folding, you can use this approach to optimize performance.

### Core M Functions Used
`Table.SelectRows`, `Custom.Function`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"David", 42, "UK"}
    }, {"Name", "Age", "Country"}),
    CustomFunction = (input as table) as table =>
        Table.SelectRows(input, each Age > 30),
    Result = CustomFunction(Source)
in
    Result
```

### Example Execution Logic Step-By-Step
Here's how the transformation typically works inside a Power Query pipeline:
    1. **Source Data**: You start with a table that contains the data you want to transform.
    2. **Define a Custom Function**: You create a custom function that takes a table as input and applies a transformation using `Table.SelectRows` or similar functions.
    3. **Apply the Custom Function**: You then apply this custom function to your source data.
    4. **Optimization**: Since the custom function doesn't support query folding, Power Query will execute it directly on the data, allowing for optimization of the transformation process.
    5. **Result**: The transformed data is then returned, showing only the rows where the condition (e.g., Age > 30) is met.
