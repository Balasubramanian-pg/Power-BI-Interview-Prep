# Push Transformations to Source with Error Handling and Retries

**Description:**
This use case demonstrates how Power Query (M) can be used to push transformations back to the source while handling errors and implementing retries. It ensures data consistency and reliability by managing potential issues during data processing.

### Typical Scenario
In a real data pipeline, this transformation is crucial when dealing with large datasets that need to be updated frequently. For instance, when synchronizing data between a data warehouse and a source system, pushing transformations back to the source can help maintain data consistency and reduce latency.

### Core M Functions Used
`Table.SelectRows`, `try`, `catch`, `retry`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Jane", 30, "UK"},
        {"Bob", 35, "Canada"}
    }, {"Name", "Age", "Country"}),
    Transform = Table.SelectRows(Source, each try
        // Simulating a transformation with potential errors
        [Name = .Name, Age = .Age * 2, Country = .Country]
    catch
        [Error = "Transformation failed"]),
    RetryAttempt = 3,
    RetryPolicy = (attempt as number) =>
        if attempt <= RetryAttempt then
            try
                Transform
            catch
                @RetryPolicy(attempt + 1)
        else
            error "Max retry attempts reached"
    in
        RetryPolicy(0)
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: A table with sample data is created.
2. **Transformation**: The `Table.SelectRows` function is used within a `try-catch` block to apply a transformation to each row. The transformation doubles the age and keeps the name and country.
3. **Error Handling**: If an error occurs during the transformation, it is caught and handled by retrying the transformation up to a specified number of times (`RetryAttempt`).
4. **Retry Policy**: A recursive function `RetryPolicy` is implemented to manage retries. If all retries fail, it errors out with a message indicating that the maximum retry attempts were reached.
5. **Execution**: The transformation and retry logic are executed, ensuring that data is processed reliably.
