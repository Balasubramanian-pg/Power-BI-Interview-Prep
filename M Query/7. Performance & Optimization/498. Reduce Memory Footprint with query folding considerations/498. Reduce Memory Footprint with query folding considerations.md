# Reduce Memory Footprint with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be used to reduce memory footprint by leveraging query folding. Query folding allows Power Query to push data processing operations to the source, reducing the amount of data that needs to be transferred and processed in memory.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large datasets that need to be filtered or aggregated. For example, when loading sales data from a database, you might want to filter out irrelevant data and only load sales data for a specific region or time period.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`

### Code Source Execution
```powerquery
let
    Source = Sql.Database("server", "database"),
    SelectRows = Table.SelectRows(Source, each [Region] = "North")
    in
    SelectRows
```

### Example Execution Logic Step-By-Step
The transformation works by first connecting to a data source using a function like Sql.Database. Then, it uses Table.SelectRows to filter the data based on a condition, in this case, selecting only rows where the Region column equals 'North'. Power Query then folds this operation back to the source, allowing the database to perform the filtering and reducing the amount of data transferred.
