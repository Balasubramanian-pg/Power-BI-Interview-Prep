# Optimize for Incremental Refresh with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to optimize data refresh while adhering to GDPR and privacy constraints. It enables efficient data updates while ensuring sensitive information is handled appropriately.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with large datasets that are subject to GDPR and other privacy regulations. For instance, a company may need to regularly update customer data for analytics purposes while ensuring that personally identifiable information is protected and only accessed when necessary.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"CustomerID", "Name", "Email"},
        {"1", "John Doe", "john.doe@example.com"},
        {"2", "Jane Doe", "jane.doe@example.com"}
    }),
    // Apply GDPR/Privacy constraints
    ProtectedData = Table.SelectRows(Source, each "GDPR_Consent" = "Granted"),
    // Incremental Refresh setup
    IncrementalRefresh = Table.SelectRows(ProtectedData, each DateTime.ToDateTime(_[LastUpdated]) >= Date.AddDays(DateTime.LocalNow(), -30))
    in
    IncrementalRefresh
```

### Example Execution Logic Step-By-Step
This transformation works by first applying GDPR and privacy constraints to the data source, typically by selecting rows where consent has been granted. Then, it implements incremental refresh by selecting only the rows that have been updated within a certain timeframe (e.g., the last 30 days). This approach minimizes data load and processing while ensuring compliance with regulations.
