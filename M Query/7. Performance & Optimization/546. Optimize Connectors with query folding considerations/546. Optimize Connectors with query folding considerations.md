# Optimize Connectors with Query Folding Considerations

**Description:**
This use case demonstrates how Power Query (M) can be optimized with query folding considerations to improve performance. Query folding allows the data source to handle filtering, sorting, and other operations, reducing the amount of data transferred. This transformation is useful when working with large datasets and slow network connections.

### Typical Scenario
In a real data pipeline, this transformation would be used when connecting to a slow data source, such as an on-premises database or a cloud-based API with rate limits. For example, when extracting data from a large database, query folding can help reduce the amount of data transferred by pushing filtering and sorting operations to the source.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`, `Table.FilterRows`

### Code Source Execution
```powerquery
let
    Source = Sql.Database("server", "database"),
    #"Filtered Rows" = Table.SelectRows(Source, each [column] > 0),
    #"Selected Columns" = Table.SelectColumns(#"Filtered Rows", {"column1", "column2"})
in
    #"Selected Columns"
```

### Example Execution Logic Step-By-Step
This transformation works by applying query folding techniques to the M query. Here's a step-by-step explanation:
    1. The `Source` variable connects to the data source using `Sql.Database`.
    2. The `Table.SelectRows` function filters the rows in the source table, but because of query folding, this operation is pushed to the source, reducing the amount of data transferred.
    3. The `Table.SelectColumns` function selects only the required columns, which further reduces the amount of data transferred.
    4. The final result is returned, which contains only the filtered and selected data.
