# Push Transformations to Source optimized for incremental refresh

**Description:**
This use case demonstrates how Power Query (M) can be used to push transformations to the source, optimizing it for incremental refresh. It allows for efficient data processing by minimizing the amount of data that needs to be refreshed.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large datasets that are frequently updated. For instance, in a sales data pipeline, daily sales data is constantly being added. By pushing transformations to the source optimized for incremental refresh, the pipeline can efficiently handle new data without reprocessing the entire dataset.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`, `Table.AddColumn`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Age"},
        {1, "John", 25},
        {2, "Jane", 30},
        {3, "Bob", 35}
    }),
    IncrementalRefresh = Table.SelectRows(Source, each [Age] > 25)
    in
    IncrementalRefresh
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Source Table Creation**: A table named 'Source' is created with columns 'ID', 'Name', and 'Age'.
    2. **Incremental Refresh Logic**: The Table.SelectRows function is used to filter rows from the 'Source' table where 'Age' is greater than 25.
    3. **Result**: The resulting table, 'IncrementalRefresh', contains only the rows where 'Age' is greater than 25, demonstrating how transformations can be pushed to the source for incremental refresh.
