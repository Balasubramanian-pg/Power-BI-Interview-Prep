# Push Transformations to Source as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used to push transformations back to the source, enabling more efficient data processing and reducing the amount of data that needs to be transferred. By doing so, it allows for reusable functions that can be applied at the source, minimizing data manipulation needs downstream.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large datasets where filtering or data cleansing can be performed at the source, reducing the volume of data that needs to be processed and transferred. For instance, when connecting to a database, applying filters at the SQL level before the data is extracted can significantly improve performance.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"Peter", 42, "USA"}
    }, {"Name", "Age", "Country"}),
    FilteredRows = Table.SelectRows(Source, each [Age] > 30)
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works by first defining a table with sample data. Then, it uses the `Table.SelectRows` function to filter the rows where the 'Age' is greater than 30. This step effectively pushes the transformation logic back to the source, in this case, the M code itself, demonstrating how such a function can be reusable across different data sources and queries.
