# Enable Query Folding for large datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to enable query folding for large datasets, improving performance by reducing the amount of data transferred and processed.

### Typical Scenario
In a real data pipeline, query folding is particularly useful when dealing with large datasets stored in databases or data warehouses. For instance, when extracting data from a SQL Server database, folding allows Power Query to push filtering and sorting operations to the database, reducing the amount of data that needs to be transferred over the network.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Sql.Database("server", "database"),
    // Folding will occur for the following operations
    FilteredRows = Table.SelectRows(Source, each "column" > 100),
    // Additional operations that benefit from folding
    SortedRows = Table.Sort(FilteredRows, {{"column", Order.Ascending}})
in
    SortedRows
```

### Example Execution Logic Step-By-Step
The transformation works by using M functions that support query folding. When operations like filtering (Table.SelectRows) and sorting (Table.Sort) are applied, Power Query attempts to 'fold' these operations back to the source, in this case, the SQL database. This means that instead of retrieving all data and then filtering and sorting it in Power Query, the database performs these operations, returning only the relevant data. This process significantly reduces the volume of data transferred and processed, enhancing performance.
