# Parallelize File Processing without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to parallelize file processing without relying on query folding. It allows for efficient processing of large files by dividing the workload across multiple threads or processes.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large volumes of files that need to be processed simultaneously, such as log files from multiple servers, financial transactions from various branches, or social media data from different regions.

### Core M Functions Used
`Table.SelectRows`, `List.Transform`, `List.Zip`

### Code Source Execution
```powerquery
let
    // Source files
    files = {"file1.csv", "file2.csv", "file3.csv"},
    // Function to process a single file
    processFile = (file as text) =>
        let
            data = Csv.Document(File.Contents(file), [Delimiter=",", Encoding=1252]),
            processedData = Table.SelectRows(data, "Column1")
        in
            processedData,
    // Parallelize file processing
    processedFiles = List.Transform(files, processFile),
    // Combine results
    combinedData = Table.Combine(processedFiles)
    in
        combinedData
```

### Example Execution Logic Step-By-Step
The script works as follows:
1. List of files: Define a list of file paths to be processed.
2. Process function: Define a function `processFile` that takes a file path, reads the file using `Csv.Document`, and processes the data using `Table.SelectRows`.
3. Parallel processing: Use `List.Transform` to apply the `processFile` function to each file in parallel.
4. Combine results: Use `Table.Combine` to combine the processed data from all files into a single table.
