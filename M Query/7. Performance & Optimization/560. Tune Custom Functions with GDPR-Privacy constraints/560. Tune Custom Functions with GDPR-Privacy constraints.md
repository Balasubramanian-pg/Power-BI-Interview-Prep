# Tune Custom Functions with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to tune custom functions while adhering to GDPR/privacy constraints. It enables data transformations that respect sensitive information and ensure compliance with data protection regulations.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when handling sensitive customer data. For instance, a company may need to process customer information for analytics purposes while ensuring that personally identifiable information (PII) is protected and anonymized.

### Core M Functions Used
`Table.SelectRows`, `List.SelectMany`, `Text.Replace`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "Doe", "johndoe@example.com"},
        {"Jane", "Doe", "janedoe@example.com"}
    }, {"FirstName", "LastName", "Email"}),
    AnonymizedData = Table.SelectRows(Source, each Text.Replace([Email], "@example.com", "@ anonymized.com"))
    in
    AnonymizedData
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. **Data Source**: The process begins with a table containing sensitive information, such as customer names and email addresses.
2. **Anonymization**: The `Table.SelectRows` function is used in conjunction with `Text.Replace` to create a new table where email addresses are anonymized by replacing the domain part of the email with "@anonymized.com".
3. **Result**: The resulting table contains the same customer names but with anonymized email addresses, ensuring that PII is protected while still allowing for data analysis.
