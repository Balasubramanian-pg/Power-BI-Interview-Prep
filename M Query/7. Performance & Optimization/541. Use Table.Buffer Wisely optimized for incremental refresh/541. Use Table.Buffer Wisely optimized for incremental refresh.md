# Optimizing Incremental Refresh with Table.Buffer in Power Query

**Description:**
This use case demonstrates how to optimize incremental refresh in Power Query using Table.Buffer wisely. It explains the benefits and scenarios where this optimization is crucial for efficient data processing.

### Typical Scenario
In a real-world data pipeline, incremental refresh is often used to update data at regular intervals. However, this process can be slow if the data source is large or complex. By wisely using Table.Buffer, you can significantly improve performance by buffering only the necessary data for the incremental refresh, reducing the load on the data source and speeding up the refresh process.

### Core M Functions Used
`Table.Buffer`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Age"},
        {1, "John", 25},
        {2, "Jane", 30},
        {3, "Bob", 35}
    }),
    BufferedTable = Table.Buffer(Source, 1000)
    in
    BufferedTable
```

### Example Execution Logic Step-By-Step
The transformation works by first loading the data into a table. Then, by applying Table.Buffer, it caches the data in memory up to a specified number of rows (in this case, 1000). This buffering is particularly useful in incremental refresh scenarios where only a subset of data needs to be processed. By buffering the table, Power Query can quickly access and process the data without having to reload it from the source, significantly improving performance.
