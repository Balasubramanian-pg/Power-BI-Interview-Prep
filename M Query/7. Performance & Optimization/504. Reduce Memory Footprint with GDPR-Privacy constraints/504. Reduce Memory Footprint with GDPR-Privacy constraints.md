# Reduce Memory Footprint with GDPR/Privacy constraints

**Description:**
This transformation reduces the memory footprint of data while adhering to GDPR/privacy constraints by filtering and selecting only necessary columns and rows.

### Typical Scenario
In a real-world data pipeline, this transformation is used when dealing with large datasets that contain sensitive information. For instance, when processing customer data, it's essential to anonymize or remove sensitive columns to comply with GDPR regulations while also reducing memory usage to improve performance.

`Table.SelectRows`, `Table.SelectColumns`

```powerquery
    // Select specific columns to reduce memory footprint
    SelectedColumns = Table.SelectColumns(Source, {"FirstName", "LastName"}),
    // Apply GDPR/privacy constraints by removing sensitive data
    AnonymizedData = Table.SelectRows(SelectedColumns, each [LastName] = "Doe")
in
    AnonymizedData
```

4. **Output**: The resulting table contains only the required data, optimized for memory usage and compliant with privacy regulations.
