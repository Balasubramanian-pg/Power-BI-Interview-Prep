# Parallelize File Processing as a Reusable Function

**Description:**
This use case demonstrates how Power Query (M) can be used to parallelize file processing as a reusable function, improving data processing efficiency by leveraging multiple threads or processes.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to process large volumes of files concurrently, such as when ingesting data from multiple sources, data migration, or ETL processes where files need to be processed in parallel to meet tight deadlines.

### Core M Functions Used
`Table.SelectRows`, `List.Transform`, `List.Zip`

### Code Source Execution
```powerquery
let
    Source = Folder.Contents("C:\SourceFolder"),
    Files = Table.SelectRows(Source, each [Kind = "File"]),
    FileContents = List.Transform(Files, each [Content = File.Contents([Name])]),
    CombinedContent = List.Zip(FileContents)
    in
    CombinedContent
```

### Example Execution Logic Step-By-Step
The transformation works by first listing all files in a specified folder. It then selects only the files (not folders) and processes their contents in parallel using the List.Transform function. The results are then combined using List.Zip. This approach allows for efficient parallel processing of files, significantly speeding up data ingestion and processing tasks.
