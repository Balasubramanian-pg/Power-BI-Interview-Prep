# Minimize Column Scans with Error Handling and Retries

**Description:**
This transformation minimizes column scans by optimizing data processing with error handling and retries, improving data pipeline efficiency and reliability.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with large datasets that may contain inconsistent or missing data, ensuring that the pipeline can handle errors gracefully and continue processing.

### Core M Functions Used
`Table.SelectRows`, `try`, `catch`, `retry`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", 25, "USA"},
        {"Mary", 31, "Canada"},
        {"David", 42, "Mexico"}
    }, {"Name", "Age", "Country"}),
    OptimizedScan = Table.SelectRows(Source, each try
        let
            _Age = [Age],
            _Country = [Country]
        in
            _Age > 18 and _Country = "USA"
    catch
        (e) => false),
    RetryAttempt = (attempt as any) =>
        try
            attempt
        catch
            (e) =>
                if Error.Message(e) = "Timeout" then
                    RetryAttempt(attempt)
                else
                    error e
in
    RetryAttempt(OptimizedScan)
```

1. **Data Source**: A table with customer data is created.
2. **Optimized Scan**: The `Table.SelectRows` function is used with a try-catch block to filter rows where the age is greater than 18 and the country is 'USA'.
3. **Error Handling**: If an error occurs during the filtering process, the catch block returns false, effectively excluding that row.
4. **Retry Mechanism**: A retry function is implemented to re-attempt the operation if a timeout error occurs, ensuring that transient errors do not halt the pipeline.
