# Minimize Column Scans for Multi-Tenant Scenarios

**Description:**
This transformation optimizes data processing by reducing the number of column scans in multi-tenant environments, improving performance and efficiency. It achieves this by selectively accessing only the required columns, thereby minimizing unnecessary data processing.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with large datasets from multiple tenants. For instance, in a SaaS application, each tenant's data is stored in a shared database but distinguished by a tenant ID. When processing data for a specific tenant, using this transformation can significantly reduce the amount of data scanned, leading to faster processing times and reduced costs.

### Core M Functions Used
`Table.SelectColumns`, `Table.SelectRows`

```powerquery
let
    Source = Table.FromRows({

        {"Tenant C", "2022-01-04", 400}
    }, {"Tenant", "Date", "Value"}),
    FilteredRows = Table.SelectRows(Source, each [Tenant] = "Tenant A"),
    SelectedColumns = Table.SelectColumns(FilteredRows, {"Date", "Value"})
    in
    SelectedColumns
```

### Example Execution Logic Step-By-Step
This transformation works by first selecting rows that match a specific condition (e.g., a particular tenant) using the Table.SelectRows function. Then, it applies Table.SelectColumns to choose only the necessary columns from the filtered rows. This two-step process ensures that only the required data for the specific tenant and columns are processed, significantly reducing the volume of data scanned.
