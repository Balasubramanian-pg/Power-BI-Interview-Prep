# Optimize for Incremental Refresh for large datasets

**Description:**
This use case demonstrates how Power Query (M) can be used to optimize large datasets for incremental refresh, improving performance and reducing data load times.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with massive datasets that need to be updated periodically, such as daily sales data or hourly IoT sensor readings. For instance, a company might have a large dataset of sales data that is updated daily, and they want to optimize the refresh process to only load new or updated data.

`Table.SelectRows`, `Table.AddIndexColumn`, `Table.Skip`

### Example Execution Logic Step-By-Step
The transformation works as follows: 
    1. The source data is loaded into a table.
    2. An index column is added to the table to track the row numbers.
    3. The Table.Skip function is used to skip the specified number of rows from the top of the table, effectively implementing incremental refresh.
    For example, if the source data has 100 rows and we want to implement incremental refresh by skipping the first 50 rows, the Table.Skip function would skip the first 50 rows and return the remaining 50 rows.
