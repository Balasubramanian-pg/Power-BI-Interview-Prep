# Optimize Connectors for Incremental Refresh

**Description:**
This use case demonstrates how Power Query (M) can be optimized for incremental refresh, allowing for more efficient data updates by only refreshing changed data.

### Typical Scenario
In a real data pipeline, this transformation is crucial when dealing with large datasets that are updated frequently. For instance, a company with daily sales data might only need to update the new sales records rather than the entire dataset.

### Core M Functions Used
`Table.SelectRows`, `Table.Skip`, `Table.FirstN`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Date"},
        {1, "John", "2022-01-01"},
        {2, "Jane", "2022-01-02"},
        {3, "Bob", "2022-01-03"}
    }),
    IncrementalRefresh = Table.SelectRows(Source, each [Date] >= Date.AddDays(DateTime.LocalNow(), -1))
    in
    IncrementalRefresh
```

### Example Execution Logic Step-By-Step
The transformation works by selecting only the rows that have been updated within a specific timeframe. For example, if we only want data from the last day, we use `Table.SelectRows` to filter the table based on the date column. This approach significantly reduces the amount of data that needs to be processed and loaded, making the refresh much faster and more efficient.
