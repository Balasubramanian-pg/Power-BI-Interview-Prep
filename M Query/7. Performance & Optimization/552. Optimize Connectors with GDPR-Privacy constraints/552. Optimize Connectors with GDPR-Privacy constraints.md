# Optimize Connectors with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to optimize connectors while adhering to GDPR and privacy constraints. It enables data engineers to filter and transform data in a way that protects sensitive information. By applying specific transformations, data can be made compliant with privacy regulations. This approach ensures that personal data is handled securely.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to connect to various data sources while ensuring that the data fetched complies with GDPR and other privacy laws. For instance, a business might need to extract customer data from a CRM system but only allow pseudonymized or aggregated data to be processed further. This transformation would be applied at the connector level to filter out sensitive information before it is processed or stored.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"CustomerID", "Name", "Email"},
        {1, "John Doe", "john.doe@example.com"},
        {2, "Jane Doe", "jane.doe@example.com"}
    }),
    // Apply GDPR/Privacy constraints by selecting only non-sensitive data
    FilteredData = Table.SelectRows(Source, each [_"Email"] = null or [_"Email"] = "")
    in
    FilteredData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Source**: The process starts with a data source that contains potentially sensitive information.
    2. **Filtering**: The `Table.SelectRows` function is used to filter out rows that contain sensitive data. For example, in the context of GDPR, this might mean selecting only rows where the email address is null or an empty string, effectively pseudonymizing the data.
    3. **Transformation**: The selected data is then transformed or processed further, ensuring that it complies with privacy regulations.
    4. **Output**: The final output is a dataset that has been optimized for use within the constraints of GDPR and other privacy laws.
