# Optimize for Incremental Refresh without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used to optimize for incremental refresh without query folding. It allows for efficient data refresh by only loading new or changed data.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large datasets that are frequently updated. For instance, a daily sales report that needs to be refreshed every hour to reflect the latest sales data. By optimizing for incremental refresh, the pipeline can avoid reloading the entire dataset, reducing processing time and improving performance.

### Core M Functions Used
`Table.SelectRows`, `Table.Skip`, `Table.FirstN`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Date"},
        {1, "John", #date(2022,1,1)},
        {2, "Jane", #date(2022,1,2)},
        {3, "Bob", #date(2022,1,3)}
    }),
    IncrementalRefresh = Table.SelectRows(Source, each [Date] >= Date.AddDays(DateTime.LocalNow(), -1))
    in
        IncrementalRefresh
```

### Example Execution Logic Step-By-Step
The transformation works by first loading the entire dataset. Then, it uses the Table.SelectRows function to filter the data and only keep the rows where the date is within the last day. This approach allows for incremental refresh without relying on query folding, which can be useful when working with data sources that do not support query folding.
