# Minimize Column Scans without Query Folding

**Description:**
This transformation optimizes data processing by reducing the number of column scans, improving performance. It achieves this by selectively accessing only the required columns, thus minimizing unnecessary data reads.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with large datasets where not all columns are necessary for analysis. For instance, when loading data from a database into a data warehouse, you might only need a subset of columns for your reporting needs. By minimizing column scans, you can significantly reduce the amount of data transferred and processed, leading to faster data ingestion and analysis.

The result is a table with fewer columns, which reduces the amount of data that needs to be scanned in subsequent operations.
