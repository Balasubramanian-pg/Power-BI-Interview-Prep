# Enable Query Folding for Multi-Tenant Scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to enable query folding for multi-tenant scenarios, allowing for more efficient data retrieval and processing.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when dealing with multi-tenant databases where data needs to be filtered and aggregated for each tenant separately, improving performance by reducing the amount of data transferred and processed.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "Data1"},
        {"Tenant2", "Data2"},
        {"Tenant3", "Data3"}
    }, {"Tenant", "Data"}),
    FilteredRows = Table.SelectRows(Source, each [Tenant] = "Tenant1" or [Tenant] = "Tenant2")
    in
    FilteredRows
```

### Example Execution Logic Step-By-Step
The transformation works by first loading the data from a source. Then, it uses the Table.SelectRows function to filter the rows based on the tenant. This step enables query folding, which allows the filtering to occur at the source, reducing the amount of data that needs to be transferred and processed. The result is a table containing only the rows for the specified tenants.
