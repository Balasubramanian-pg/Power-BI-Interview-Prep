# Reduce Memory Footprint for Multi-Tenant Scenarios

**Description:**
This transformation optimizes memory usage in Power Query for multi-tenant scenarios by selectively loading data. It achieves this by filtering and aggregating data to reduce the overall memory footprint. This is particularly useful when dealing with large datasets from multiple tenants.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when ingesting data from multiple tenants, where each tenant has a large dataset. For instance, a SaaS application with thousands of customers, each having a massive dataset, would benefit from this transformation to reduce memory usage and improve performance.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`, `Table.Group`

```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "Data1"},
        {"Tenant1", "Data2"},
        {"Tenant2", "Data3"}
    }, {"Tenant", "Data"}),
    SelectTenantData = Table.SelectRows(Source, each [Tenant] = "Tenant1")
    in
    SelectTenantData
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
    1. **Data Ingestion**: The pipeline starts by ingesting data from various sources.
    2. **Filtering**: The `Table.SelectRows` function is used to filter data for a specific tenant, reducing the dataset size.
    3. **Aggregation**: Optionally, `Table.Group` can be used to aggregate data, further reducing memory usage.
    4. **Loading**: The filtered and aggregated data is then loaded into the next stage of the pipeline, significantly reducing the memory footprint.
