# Optimize Merge Strategies with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used to optimize merge strategies while adhering to GDPR/privacy constraints. It enables efficient data merging and filtering, ensuring sensitive information is protected. By applying specific transformations, users can manage data privacy and compliance requirements effectively.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when merging datasets from different sources while ensuring that personally identifiable information (PII) is handled according to GDPR guidelines. For instance, a company may need to combine customer data from various departments, applying filters to exclude sensitive details.

### Core M Functions Used
`Table.SelectRows`, `Table.Join`

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. **Data Source Definition**: Two sample tables, `Table1` and `Table2`, are defined. `Table1` contains personal data, while `Table2` has non-sensitive information.
2. **Applying GDPR/Privacy Constraints**: The `Table.SelectRows` function is used on `Table1` to create `ProtectedTable1`, which only includes the "ID" and "Name" columns, effectively anonymizing the data by excluding sensitive information like email addresses.
3. **Merging Tables**: The `Table.Join` function is then used to merge `ProtectedTable1` and `Table2` based on the "ID" column. This step combines the data while maintaining privacy constraints.
4. **Result**: The final merged table, `MergedTable`, contains the required information without exposing sensitive data.
