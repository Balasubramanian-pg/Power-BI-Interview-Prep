# Optimize for Incremental Refresh for multi-tenant scenarios

**Description:**
This use case demonstrates how Power Query (M) can be used to optimize data refresh for multi-tenant scenarios, enabling efficient data updates and reducing refresh times.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large datasets across multiple tenants, where data is updated incrementally and efficiently. For instance, a SaaS application with multiple clients, each having their own set of data, would benefit from incremental refresh to ensure timely updates without overwhelming the system.

### Core M Functions Used
`Table.SelectRows`, `Table.SelectColumns`, `Table.AddFilterRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "2022-01-01", "Data1"},
        {"Tenant2", "2022-01-02", "Data2"},
        {"Tenant1", "2022-01-03", "Data3"}
    }, {"Tenant", "Date", "Data"}),
    IncrementalRefresh = Table.SelectRows(Source, each [Date] >= Date.AddDays(DateTime.LocalNow(), -7))
    in
    IncrementalRefresh
```

The transformation works as follows:
    1. **Data Source**: Start with a table containing data for multiple tenants, including a date field.
    2. **Incremental Logic**: Use `Table.SelectRows` to filter the data, selecting only rows where the date is within a certain range (e.g., last 7 days).
    3. **Execution**: The query executes, returning only the relevant data for the specified period, thus optimizing the refresh process for multi-tenant scenarios.
