# Push Transformations to Source for Multi-Tenant Scenarios

**Description:**
This transformation enables pushing data transformations back to the source in a multi-tenant environment, optimizing data processing and reducing the need for intermediate data storage.

### Typical Scenario
In a multi-tenant data pipeline, different tenants may have varying data transformation requirements. This transformation is used when a data source needs to be dynamically updated based on tenant-specific rules, ensuring that each tenant's data is processed according to their specific needs.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"Tenant1", "Data1"},
        {"Tenant2", "Data2"}
    }, {"Tenant", "Data"}),
    TransformData = Table.TransformRows(Source, each {
        _[Tenant],
        Text.Upper(_[Data])
    }),
    PushToSource = TransformData
in
    PushToSource
```

### Example Execution Logic Step-By-Step
The transformation works as follows:
1. **Data Source**: A table with tenant data is created.
2. **Transformation**: The `Table.TransformRows` function is used to apply a transformation to each row of the table. In this example, it converts the 'Data' column to uppercase.
3. **Push to Source**: The transformed data is then pushed back to the source, updating it with the transformed values.
4. **Result**: The source now contains the transformed data, optimized for further processing or reporting.
