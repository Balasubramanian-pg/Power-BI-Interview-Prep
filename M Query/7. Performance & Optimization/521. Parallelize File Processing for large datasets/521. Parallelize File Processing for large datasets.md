# Parallelize File Processing for Large Datasets

**Description:**
This use case demonstrates how to utilize Power Query (M) for parallelizing file processing, significantly improving the efficiency of handling large datasets. By leveraging parallel processing, data can be processed in multiple streams simultaneously, reducing overall processing time.

### Typical Scenario
In a real-world data pipeline, this transformation is particularly useful when dealing with large volumes of data stored in multiple files across different directories. For instance, when aggregating sales data from various regional stores, parallelizing file processing can help in quickly combining and analyzing data from hundreds of files, making it easier to generate timely reports.

### Core M Functions Used
`Table.SelectRows`, `List.Transform`, `List.Zip`

### Code Source Execution
```powerquery
let
    // Source folder path
    sourceFolder = "C:\SourceData\",
    // Get list of files
    files = Directory.GetFiles(sourceFolder, "*.csv"),
    // Transform each file in parallel
    transformedFiles = List.Transform(files, each Table.FromRows(File.Contents(_)));
    // Combine the tables
    combinedTable = Table.Combine(transformedFiles)
    in
    combinedTable
```

    3. **Combine Tables**: Once all files are processed into tables, use `Table.Combine` to merge these tables into a single table.
    This approach allows for efficient processing of large datasets by distributing the workload across multiple streams.
