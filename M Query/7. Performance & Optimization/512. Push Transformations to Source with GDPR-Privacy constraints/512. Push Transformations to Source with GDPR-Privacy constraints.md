# Push Transformations to Source with GDPR/Privacy constraints

**Description:**
This use case demonstrates how Power Query (M) can be used for pushing transformations to the source while adhering to GDPR/privacy constraints. It enables data engineers to apply data transformations at the source level, ensuring that sensitive data is protected and handled according to privacy regulations.

### Typical Scenario
In a real-world data pipeline, this transformation would be used when a company needs to extract data from various sources, apply GDPR/privacy constraints, and then push these transformations back to the source. For instance, a company might need to anonymize personal data before processing it further. By applying transformations at the source, they can minimize the exposure of sensitive data during data processing.

### Core M Functions Used
`Table.SelectRows`, `Table.TransformColumns`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"John", "Doe", 25},
        {"Jane", "Doe", 30}
    }, {"FirstName", "LastName", "Age"}),
    // Apply GDPR/privacy transformation
    AnonymizedData = Table.TransformColumns(Source, {{"FirstName", each "Anonymous"}}),
    // Push transformation to source
    PushedData = AnonymizedData
in
    PushedData
```

### Example Execution Logic Step-By-Step
The transformation works as follows: 
1. **Data Source**: Start with a data table containing personal information.
2. **Apply Transformation**: Use M functions like `Table.TransformColumns` to apply GDPR/privacy constraints. For example, replace personal identifiable information (PII) like names with anonymized values.
3. **Push to Source**: The transformed data is then pushed back to the source, ensuring that the source data is updated with privacy-protected values.
4. **Further Processing**: Any further data processing or analysis is done on the anonymized data, ensuring compliance with GDPR/privacy regulations.
