# Parallelize File Processing with Error Handling and Retries

**Description:**

It enables efficient data processing by leveraging multiple threads for file operations, handling potential errors, and retrying failed operations. This approach is particularly useful for large-scale data processing tasks where files may be processed in parallel to save time. By implementing error handling and retries, it ensures robustness and reliability in data pipelines.

### Typical Scenario
In a real data pipeline, this transformation would be used when there is a need to process a large number of files in a short amount of time. For example, in a data ingestion pipeline where multiple files are being uploaded daily, parallelizing file processing can significantly reduce the overall processing time. This approach can also be applied in scenarios where files are being processed in a distributed environment, and there is a need to handle errors and retries efficiently.

    files = {"file1.csv", "file2.csv", "file3.csv"},

    processFile = (file as text) =>

            // Simulate file processing
            [File = file, Result = "Processed" + file]
        catch
            [File = file, Error = Error.Message],
    // Parallel processing with retries
    maxRetries = 3,
    processFiles = List.Transform(files, each
        let
            retryCount = 0,
            process = () => processFile(_),
            retry = () as any =>
                try
                    process()
                catch
                    if retryCount < maxRetries then
                        retryCount + 1; retry()
                    else
                        Error.Record("Max retries exceeded")
        in
            retry()
    ),
    // Convert to table
    resultTable = Table.FromList(processFiles, Splitter.SplitByNothing(), {"Result"})
in
    resultTable
