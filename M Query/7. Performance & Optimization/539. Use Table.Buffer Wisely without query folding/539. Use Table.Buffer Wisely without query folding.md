# Use Table.Buffer Wisely without query folding

**Description:**
This use case demonstrates how Power Query (M) can be used for buffering tables without relying on query folding. It helps in understanding when and how to apply buffering to improve performance in Power Query pipelines. Buffering tables can prevent multiple scans of the same data source, enhancing efficiency.

### Typical Scenario
In a real data pipeline, you might have a large data source that you need to scan multiple times to apply different transformations. Instead of re-scanning the source each time, you can buffer the table into memory after the first scan. This approach is particularly useful when dealing with slow data sources or when you need to perform multiple operations on the same data.

### Core M Functions Used
`Table.Buffer`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({{"John", 25}, {"Mary", 31}, {"Bob", 42}}, {"Name", "Age"}),
    BufferedTable = Table.Buffer(Source, 1000)
    in
    BufferedTable
```

### Example Execution Logic Step-By-Step
The transformation works by first loading the data into a table. The Table.Buffer function is then applied to this table, buffering it into memory. This buffered table can then be used for subsequent operations without having to re-scan the original data source. For example, you can filter, sort, or aggregate the buffered table without incurring the cost of multiple scans of the source.
