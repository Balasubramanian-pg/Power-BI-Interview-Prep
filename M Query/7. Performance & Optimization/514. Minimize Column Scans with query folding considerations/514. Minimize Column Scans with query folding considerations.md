# Minimize Column Scans with Query Folding Considerations

**Description:**
This transformation optimizes data retrieval by minimizing column scans, enhancing performance when dealing with large datasets. It achieves this by leveraging query folding, which allows the data source to perform operations, reducing the amount of data transferred. This approach is particularly beneficial in scenarios where data is filtered or projected. By applying query folding, the data processing load is shifted from the Power Query engine to the source database, leading to more efficient data processing.

### Typical Scenario
In a real-world data pipeline, this transformation is crucial when dealing with large datasets stored in databases. For instance, when extracting specific columns from a massive table for analysis, applying filters and projections directly at the database level (through query folding) can significantly reduce the volume of data that needs to be transferred and processed. This not only speeds up the data extraction process but also conserves network bandwidth and reduces the load on the database.
