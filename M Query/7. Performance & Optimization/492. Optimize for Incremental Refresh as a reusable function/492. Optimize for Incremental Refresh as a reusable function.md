# Optimize for Incremental Refresh as a reusable function

**Description:**
This use case demonstrates how Power Query (M) can be used to optimize for incremental refresh as a reusable function. It allows for efficient data refresh by only loading new or changed data.

### Typical Scenario
In a real data pipeline, this transformation would be used when dealing with large datasets that are updated periodically. For instance, a daily sales report that only needs to be updated with new sales data each day, rather than reloading the entire dataset.

### Core M Functions Used
`Table.SelectRows`, `Table.AddIndexColumn`, `Table.Column`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Date"},
        {1, "John", #date(2022,1,1)},
        {2, "Jane", #date(2022,1,2)}
    }),
    IncrementalRefresh = (SourceTable as table) =>
        let
            // Define the date column
            DateColumn = "Date",
            // Get the max date from previous load
            PreviousMaxDate = try DateTimeZone.ToLocal( DateTimeZone.FromText( Environment.CurrentFile.Contents ) ) otherwise #date(2022,1,1),
            // Filter new rows
            NewRows = Table.SelectRows(SourceTable, each [Date] > PreviousMaxDate)
        in
            NewRows
in
    IncrementalRefresh(Source)
```

### Example Execution Logic Step-By-Step
This transformation works by first defining a function `IncrementalRefresh` that takes a table as input. It then identifies a date column and determines the maximum date from a previous load. New rows are filtered based on having a date greater than the previous maximum date. The function returns a table with only the new rows.
