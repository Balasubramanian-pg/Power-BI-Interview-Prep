# Push Transformations to Source without Query Folding

**Description:**
This use case demonstrates how Power Query (M) can be used to push transformations to the source without query folding, enabling more efficient data processing and reducing the amount of data that needs to be transferred.

### Typical Scenario
In a real-world data pipeline, this transformation is useful when dealing with large datasets where filtering or data selection needs to occur at the source level, such as when working with databases or data warehouses that support query folding but the specific transformation logic cannot be folded.

### Core M Functions Used
`Table.SelectRows`

### Code Source Execution
```powerquery
let
    Source = Table.FromRows({
        {"ID", "Name", "Age"},
        {1, "John", 25},
        {2, "Jane", 30},
        {3, "Bob", 35}
    }),
    // Applying transformation that can't be folded
    TransformedData = Table.SelectRows(Source, each [Age] > 25)
    in
    TransformedData
```

### Example Execution Logic Step-By-Step
The transformation works by first creating a table from a set of rows. Then, it applies a transformation using Table.SelectRows, which filters the table to include only rows where the 'Age' is greater than 25. This step cannot be 'pushed' to the source if the source system doesn't natively support such filtering (or 'query folding'). The result is a new table with only the rows that meet the specified condition.
