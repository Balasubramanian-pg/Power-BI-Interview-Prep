# 4. Fabric Capacity Throttling And CU Monitoring

## Overview

Fabric capacity throttling and CU (Compute Unit) monitoring are critical concepts in Power BI that ensure efficient data processing and prevent overloading of system resources. Misunderstanding these concepts can lead to performance issues, slow report rendering, and even crashes. The process involves several stages, as illustrated in the following flow diagram:
```
Query → [Data Retrieval] → [Data Processing] → [Result Rendering] → Result
           ↓             ↓            ↓
      [Data Validation]    [Data Transformation]   [Result Caching]
```

## The Core Components

### Data Retrieval Engine

The data retrieval engine is responsible for fetching data from various sources, such as databases, files, and web services. Its primary function is to extract the required data and pass it to the next stage for processing.

**Characteristics:**
- Supports multiple data sources
- Handles data encryption and authentication
- Optimizes data retrieval for performance

### Data Processing Engine

The data processing engine is responsible for transforming and aggregating the retrieved data. It performs calculations, applies filters, and sorts the data according to the query requirements.

**Characteristics:**
- Supports various data transformation functions
- Handles data caching and buffering
- Optimizes data processing for performance

### Result Rendering Engine

The result rendering engine is responsible for rendering the final result, such as a report or a visualization. It takes the processed data and formats it according to the user's preferences.

**Characteristics:**
- Supports multiple visualization types
- Handles report layout and formatting
- Optimizes result rendering for performance

## The Processing Flow

The processing flow involves several stages, as illustrated in the following diagram:
```
[Data Retrieval] → [Data Processing] → [Result Rendering] → [Result Caching] → [Final Output]
    ↓            ↓            ↓            ↓
[Data Validation]    [Data Transformation]   [Result Formatting]     [Cache Management]
```

### How Data Flows

When a query is executed, the data retrieval engine fetches the required data from the source. The data is then passed to the data processing engine, which performs the necessary transformations and calculations. The processed data is then rendered by the result rendering engine, and the final result is cached for future use.

### When Throttling Occurs

Throttling occurs when the system resources are overwhelmed, and the processing flow needs to be slowed down to prevent overloading. The following steps occur when throttling is triggered:
1. **Resource Monitoring**: The system monitors the available resources, such as CPU, memory, and disk space.
2. **Throttling Detection**: The system detects when the resources are approaching their limits.
3. **Throttling Application**: The system applies throttling to slow down the processing flow.
4. **Resource Recovery**: The system recovers the resources and resumes normal processing.

### What is DAX?

DAX (Data Analysis Expressions) is a language used to define calculations and data models in Power BI. It is used to create measures, calculate columns, and define relationships between tables.
