# 10. Sensitivity Labels And Encryption In PBIX

## Overview

Sensitivity labels and encryption in Power BI (PBIX) are crucial for protecting sensitive data and ensuring compliance with organizational security policies. Misunderstanding or misconfiguring these features can lead to data breaches or unauthorized access, compromising the integrity of the data and the trust of stakeholders. The process of applying sensitivity labels and encryption involves several stages, as illustrated in the following flow diagram:
```
Data Ingestion → [Data Classification] → [Sensitivity Label Application] → [Encryption] → Secure Data Storage
           ↓                   ↓                   ↓             ↓
      [Data Source]    [Data Profiling]    [Label Assignment]   [Key Management]
```

## The Core Security Components

### Data Classification

Data classification is the process of categorizing data based on its sensitivity and importance. This step is essential for determining the appropriate sensitivity labels and encryption levels. The characteristics of data classification include:
**Characteristics:**
- **Data Discovery**: Identifying the data sources and types of data to be classified.
- **Data Profiling**: Analyzing the data to determine its sensitivity and importance.
- **Classification Rules**: Defining the rules and criteria for classifying data.

### Sensitivity Labels

Sensitivity labels are used to categorize data based on its sensitivity and importance. These labels determine the level of access control and encryption applied to the data. The characteristics of sensitivity labels include:
**Characteristics:**
- **Label Definition**: Defining the sensitivity labels and their corresponding access controls.
- **Label Assignment**: Assigning sensitivity labels to the classified data.
- **Label Enforcement**: Enforcing the access controls and encryption based on the assigned labels.

### Encryption

Encryption is the process of converting plaintext data into unreadable ciphertext to protect it from unauthorized access. The characteristics of encryption include:
**Characteristics:**
- **Encryption Algorithm**: Selecting the encryption algorithm and key management strategy.
- **Key Generation**: Generating the encryption keys and managing their lifecycle.
- **Encryption Process**: Encrypting the data using the selected algorithm and keys.

## The Security Process

### How Data Flows Through the System

The data flow process involves several stages, from data ingestion to secure data storage:
```
Data Ingestion → Data Classification → Sensitivity Label Application → Encryption → Secure Data Storage
    ↓                   ↓                   ↓             ↓
[Data Source]    [Data Profiling]    [Label Assignment]   [Key Management]
```
When the data is ingested, it is classified and assigned a sensitivity label. The label determines the level of encryption and access control applied to the data.

### When Data is Accessed

When data is accessed, the system checks the sensitivity label and enforces the corresponding access controls and encryption. The steps involved in this process are:
1. **Authentication**: Verifying the user's identity and permissions.
2. **Authorization**: Checking the user's access rights based on the sensitivity label.
3. **Decryption**: Decrypting the data using the corresponding encryption key.
4. **Data Retrieval**: Retrieving the decrypted data for the authorized user.

## Internal Encryption Mechanism

The internal encryption mechanism used in Power BI is based on the Advanced Encryption Standard (AES) algorithm. This algorithm is widely used and considered secure for protecting sensitive data.
```sql
-- Comment explaining the encryption mechanism
CREATE TABLE [SecureTable] (
    [Id] INT,
    [SensitiveData] VARBINARY(MAX) ENCRYPTED WITH (COLUMN_ENCRYPTION_KEY = [CEK], ENCRYPTION_TYPE = DETERMINISTIC, ALGORITHM = 'AEAD_AES_256_CBC_HMAC_SHA_256')
)
```
### Example: Encrypting Sensitive Data

To encrypt sensitive data, you can use the following DAX code:
```dax
-- DAX Query to encrypt sensitive data
EVALUATE
    'SecureTable' =
        ADDCOLUMNS (
            'SensitiveData',
            "EncryptedData", ENCRYPT ('SensitiveData'[SensitiveData], "CEK")
        )
```
This code encrypts the sensitive data using the specified column encryption key (CEK).

## When Sensitivity Labels and Encryption are Efficient

### Scenario 1: Classifying and Encrypting Sensitive Data

```dax
-- DAX Query to classify and encrypt sensitive data
EVALUATE
    'SecureTable' =
        ADDCOLUMNS (
            'SensitiveData',
            "SensitivityLabel", IF ('SensitiveData'[SensitiveData] = "High", "High", "Low"),
            "EncryptedData", IF ('SensitiveData'[SensitiveData] = "High", ENCRYPT ('SensitiveData'[SensitiveData], "CEK"), 'SensitiveData'[SensitiveData])
        )
```
**Result:**
- The sensitive data is classified and assigned a sensitivity label.
- The data is encrypted using the corresponding encryption key.
- The encrypted data is stored securely.

### Scenario 2: Accessing Encrypted Data

```dax
-- DAX Query to access encrypted data
EVALUATE
    'SecureTable' =
        FILTER (
            'SecureTable',
            'SecureTable'[SensitivityLabel] = "High"
        )
```
**Access Control:**
1. The system checks the user's access rights based on the sensitivity label.
2. The system decrypts the data using the corresponding encryption key.
3. The decrypted data is retrieved for the authorized user.

## When Sensitivity Labels and Encryption Become Problematic

### Scenario 1: Incorrect Sensitivity Label Assignment

```dax
-- DAX Query with incorrect sensitivity label assignment
EVALUATE
    'SecureTable' =
        ADDCOLUMNS (
            'SensitiveData',
            "SensitivityLabel", "Low",
            "EncryptedData", ENCRYPT ('SensitiveData'[SensitiveData], "CEK")
        )
```
**Problem:**
- The sensitive data is not properly classified and assigned a sensitivity label.
- The data is encrypted, but the encryption key may not be suitable for the assigned label.
- The encrypted data may be accessible to unauthorized users.

### Scenario 2: Insufficient Encryption Key Management

```dax
-- DAX Query with insufficient encryption key management
EVALUATE
    'SecureTable' =
        ADDCOLUMNS (
            'SensitiveData',
            "SensitivityLabel", "High",
            "EncryptedData", ENCRYPT ('SensitiveData'[SensitiveData], "WeakCEK")
        )
```
**Problem:**
- The encryption key is weak or not properly managed.
- The encrypted data may be vulnerable to unauthorized access.
- The system may not be able to decrypt the data properly.
