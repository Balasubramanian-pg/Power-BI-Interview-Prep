# 9. Break Glass Admin Access And Audit

## Overview

Break Glass Admin Access And Audit is a critical security feature in Power BI that allows administrators to access and audit sensitive data in emergency situations. This feature is essential in Power BI as it enables administrators to respond quickly to security incidents, while also providing a clear audit trail to ensure accountability. Misunderstanding this feature can lead to security breaches or unauthorized access to sensitive data. The process of Break Glass Admin Access And Audit can be represented by the following flow diagram:
```
Request → [Authentication] → [Authorization] → [Audit] → Result
           ↓             ↓            ↓
      [User Verification]    [Role-Based Access]   [Activity Logging]
```

## The Core Security Components

### Authentication

Authentication is the process of verifying the identity of users who request access to sensitive data. This component is responsible for ensuring that only authorized users can access the data. 
**Characteristics:**
- **Multi-Factor Authentication**: requires users to provide multiple forms of verification, such as passwords and biometric data.
- **Single Sign-On**: allows users to access multiple applications with a single set of credentials.
- **Password Policies**: enforces strong password requirements, such as password length and complexity.

### Authorization

Authorization is the process of determining what actions users can perform on sensitive data. This component is responsible for ensuring that users can only access data that they are authorized to access. 
**Characteristics:**
- **Role-Based Access Control**: assigns users to roles, which define the actions they can perform on data.
- **Attribute-Based Access Control**: grants access to data based on user attributes, such as department or job function.
- **Mandatory Access Control**: enforces a set of rules that restrict access to data based on user clearance and data classification.

### Audit

Audit is the process of tracking and logging all access to sensitive data. This component is responsible for providing a clear audit trail of all activities related to sensitive data. 
**Characteristics:**
- **Activity Logging**: records all access to sensitive data, including user ID, timestamp, and actions performed.
- **Audit Trail**: provides a chronological record of all activities related to sensitive data.
- **Compliance Reporting**: generates reports that demonstrate compliance with regulatory requirements.

## The Access And Audit Process

The Break Glass Admin Access And Audit process involves several stages, including request, authentication, authorization, and audit. The following diagram illustrates the flow of this process:
```
Request → [Authentication] → [Authorization] → [Audit] → [Access]
    ↓             ↓            ↓            ↓
[User Verification]    [Role-Based Access]   [Activity Logging]    [Data Access]
```

### How Access Flows

When a user requests access to sensitive data, the system first authenticates the user's identity. If the user is authenticated, the system then authorizes the user to access the data based on their role and attributes. Once the user is authorized, the system logs all access to the data and provides a clear audit trail.

### When The Emergency Occurs

When an emergency occurs, the Break Glass Admin Access And Audit feature is triggered, allowing administrators to access sensitive data quickly. The following steps occur:
1. **Emergency Declaration**: the administrator declares an emergency and requests access to sensitive data.
2. **Authentication Bypass**: the system bypasses normal authentication procedures to allow the administrator to access the data quickly.
3. **Authorization Override**: the system overrides normal authorization rules to grant the administrator access to the data.
4. **Audit Trail Creation**: the system creates an audit trail of all activities related to the emergency access.
