# Dynamic Source Switching for Dev/Test/Prod Environments in Power Query

## Overview

Dynamic source switching allows Power Query to connect to different data sources based on the environment (Development, Testing, Production) without manually changing connection strings. This is essential for maintaining separate environments, testing changes safely, and promoting solutions through deployment pipelines.

```
Environment Detection → Configuration Lookup → Dynamic Connection → Data Load
         ↓                      ↓                       ↓
   Desktop/Service        Dev/Test/Prod           Appropriate Source
```

## Core Concepts

### Why Dynamic Source Switching Matters

| Environment | Purpose | Data Characteristics |
|-------------|---------|----------------------|
| **Development** | Build and test transformations | Subset of data, sample databases, local files |
| **Testing** | Validate with realistic data | Full structure, anonymized or limited production data |
| **Production** | Live reports and dashboards | Complete data, production systems |

### Pattern 3: Environment-Specific Connection Functions

```m
// Base connection function
let
    fn_ConnectToDatabase = (Server as text, Database as text, optional Options as record) as table =>
        let
            DefaultOptions = [
                CommandTimeout = 60,
                CreateNavigationProperties = false
            ],
            
            MergeOptions = 
                if Options <> null then 
                    Record.Combine({DefaultOptions, Options})
                else 
                    DefaultOptions,
            
            Source = Sql.Database(Server, Database, MergeOptions)
        in
            Source,
    document = "Base database connection function"
in
    fn_ConnectToDatabase

// Development connection
let
    fn_ConnectDev = () as table =>
        let
            // Use local database with sample data
            Source = fn_ConnectToDatabase(
                "localhost\SQLEXPRESS", 
                "SalesDev_Sample",
                [CommandTimeout = 30]
            ),
            
            // Add dev-specific logging
            WithLog = Table.AddColumn(
                Source{[Schema="dbo",Item="FactSales"]}[Data],

                type text
            )
        in
            WithLog,
    document = "Development environment connection"
in
    fn_ConnectDev

// Test connection
let
    fn_ConnectTest = () as table =>
        let
            Source = fn_ConnectToDatabase(
                "test-sql-01",

        in
            Source{[Schema="dbo",Item="FactSales"]}[Data],
    document = "Test environment connection"

    fn_ConnectTest

// Production connection
let
    fn_ConnectProd = () as table =>
        let
            Source = fn_ConnectToDatabase(
                "prod-sql-01.database.windows.net",

                    ApplicationName = "PowerBI_Production"
                ]
            )
        in
            Source{[Schema="dbo",Item="FactSales"]}[Data],
    document = "Production environment connection"

    fn_ConnectProd

// Dynamic dispatcher
let
    fn_GetEnvironmentConnection = (Environment as text) as table =>
        let

                    fn_ConnectProd()
                else if Environment = "Testing" then
                    fn_ConnectTest()
                else
                    fn_ConnectDev()
        in
            Connection,

    fn_GetEnvironmentConnection
```

## Complete Implementation Example

### Step 1: Environment Detection Module

```m
// Module: Environment.pqm
let
    // Detect current runtime environment
    DetectEnvironment = () as text =>
        let
            // Check if running in Power BI Service
            InService = 
                try 
                    not Diagnostics.TraceLevel() is null
                otherwise false,
            
            // Check for environment parameter
            HasParameter = 
                try 
                    #"Environment Parameter" <> null
                otherwise false,
            
            // Determine environment
            Environment = 
                if InService then
                    // In Power BI Service, use parameter or default to Prod
                    try 
                        #"Environment Parameter" 
                    otherwise "Production"
                else if HasParameter then
                    // In Desktop with parameter
                    #"Environment Parameter"
                else
                    // In Desktop, check machine name
                    let
                        Machine = Environment.MachineName(),
                        DevMachines = {"DEV-LAPTOP", "USER-DESKTOP", "LOCALHOST"}
                    in
                        if List.Contains(DevMachines, Machine) then
                            "Development"
                        else
                            "Unknown",
            
            // Validate
            ValidEnvironment = 
                if List.Contains({"Development", "Testing", "Production"}, Environment) then
                    Environment
                else
                    "Development"  // Safe default
        in
            ValidEnvironment,
    
    // Get environment-specific configuration
    GetEnvironmentConfig = (Environment as text) as record =>
        let
            BaseConfigs = [
                Development = [
                    ConnectionType = "Database",
                    Server = "localhost\SQLEXPRESS",
                    Database = "SalesDev",
                    Authentication = "Windows",
                    SampleSize = 10000,
                    CacheResults = false,
                    LogLevel = "Debug",
                    Timeout = 30
                ],
                Testing = [
                    ConnectionType = "Database",
                    Server = "test-sql-01",
                    Database = "SalesTest",
                    Authentication = "SQL",
                    Username = "powerbi_user",
                    SampleSize = 100000,
                    CacheResults = true,
                    LogLevel = "Info",
                    Timeout = 60
                ],
                Production = [
                    ConnectionType = "Database",
                    Server = "prod-sql-01.database.windows.net",
                    Database = "SalesProd",
                    Authentication = "AzureAD",
                    SampleSize = null,  // No sampling
                    CacheResults = true,
                    LogLevel = "Error",
                    Timeout = 120
                ]
            ],
            
            Config = Record.Field(BaseConfigs, Environment)
        in
            Config,
    
    document = "Environment detection and configuration module"
in
    [
        DetectEnvironment = DetectEnvironment,
        GetEnvironmentConfig = GetEnvironmentConfig
    ]
```

### Step 2: Dynamic Source Function

```m
// Module: DynamicSource.pqm
let
    // Connect with environment-aware logic
    GetData = (QueryName as text) as table =>
        let
            // Detect environment
            EnvModule = Environment(),
            CurrentEnv = EnvModule[DetectEnvironment](),
            Config = EnvModule[GetEnvironmentConfig](CurrentEnv),
            
            // Build connection
            Source = 
                if Config[ConnectionType] = "Database" then
                    GetDatabaseData(Config, QueryName)
                else if Config[ConnectionType] = "File" then
                    GetFileData(Config, QueryName)
                else
                    error "Unsupported connection type: " & Config[ConnectionType],
            
            // Apply environment-specific transformations
            WithEnvHandling = ApplyEnvironmentRules(Source, CurrentEnv, Config)
        in
            WithEnvHandling,
    
    // Database connection handler
    GetDatabaseData = (Config as record, QueryName as text) as table =>
        let
            // Build connection string
            Connection = 
                if Config[Authentication] = "Windows" then
                    Sql.Database(Config[Server], Config[Database])
                else if Config[Authentication] = "SQL" then
                    Sql.Database(
                        Config[Server], 
                        Config[Database], 
                        [
                            UID = Config[Username],
                            PWD = GetPassword(Config[Username]),  // From credential store
                            CommandTimeout = Config[Timeout]
                        ]
                    )
                else if Config[Authentication] = "AzureAD" then
                    Sql.Database(
                        Config[Server], 
                        Config[Database], 
                        [
                            CommandTimeout = Config[Timeout],
                            ApplicationIntent = "ReadOnly"
                        ]
                    )
                else
                    error "Unsupported authentication",
            
            // Get specific query
            QueryMap = [
                FactSales = "SELECT * FROM dbo.FactSales",
                DimProduct = "SELECT * FROM dbo.DimProduct",
                DimCustomer = "SELECT * FROM dbo.DimCustomer"
            ],
            
            SqlQuery = Record.Field(QueryMap, QueryName),
            Result = Value.NativeQuery(Connection, SqlQuery)
        in
            Result,
    
    // File connection handler
    GetFileData = (Config as record, QueryName as text) as table =>
        let
            BasePath = 
                if CurrentEnv = "Development" then
                    "C:\data\dev\"
                else if CurrentEnv = "Testing" then
                    "\\test-server\share\data\"
                else
                    "\\prod-server\share\data\",
            
            FileMap = [
                FactSales = "sales.parquet",
                DimProduct = "products.csv",
                DimCustomer = "customers.parquet"
            ],
            
            FileName = Record.Field(FileMap, QueryName),
            FullPath = BasePath & FileName,
            
            Result = 
                if Text.EndsWith(FileName, ".parquet") then
                    Parquet.Document(File.Contents(FullPath))
                else
                    Csv.Document(File.Contents(FullPath))
        in
            Result,
    
    // Apply environment-specific rules
    ApplyEnvironmentRules = (Input as table, Environment as text, Config as record) as table =>
        let
            // Sampling for development
            WithSampling = 
                if Environment = "Development" and Config[SampleSize] <> null then
                    Table.FirstN(Input, Config[SampleSize])
                else
                    Input,
            
            // Add environment marker
            WithMarker = Table.AddColumn(
                WithSampling,
                "_Environment",
                each Environment,
                type text
            ),
            
            // Add audit columns
            WithAudit = Table.AddColumn(
                WithMarker,
                "_LoadedAt",
                each DateTime.LocalNow(),
                type datetime
            )
        in
            WithAudit,
    
    // Secure password retrieval (conceptual)
    GetPassword = (Username as text) as text =>
        let
            // In production, use credential store
            // This is a placeholder - never hardcode passwords
            Credential = 
                try 
                    #"Credential Store"[Username] 
                otherwise 
                    error "Password not found for " & Username
        in
            Credential
in
    GetData
```

### Step 3: Main Query Implementation

```m
// Main Sales Query
let
    // Get dynamic source
    DynamicSource = DynamicSource(),
    
    // Load fact table with environment awareness
    FactSales = DynamicSource("FactSales"),
    
    // Apply consistent transformations
    CleanedData = Table.SelectRows(FactSales, each [SalesAmount] <> null),
    
    // Add calculated columns
    WithCalculations = Table.AddColumn(
        CleanedData,
        "Profit",
        each [SalesAmount] - [Cost],
        type number
    ),
    
    // Group by relevant dimensions
    Result = Table.Group(
        WithCalculations,
        {"ProductID", "Date"},
        {
            {"TotalSales", each List.Sum([SalesAmount]), type number},
            {"TotalProfit", each List.Sum([Profit]), type number}
        }
    )
in
    Result
```

### Pattern 5: Hybrid Environment Support

```m
// Support for mixed environments (e.g., Dev + Prod data)
let
    fn_GetHybridSource = (SourceType as text, optional Environment as text) as table =>
        let
            // Determine environment for each source
            ActualEnv = 
                if Environment <> null then
                    Environment
                else
                    let
                        EnvModule = Environment(),
                        Current = EnvModule[DetectEnvironment]()
                    in
                        Current,
            
            // Get data based on source type
            SourceData = 
                if SourceType = "Sales" then
                    // Sales always from production
                    fn_GetProdData("Sales")
                else if SourceType = "Budget" then
                    // Budget from current environment
                    fn_GetEnvData(ActualEnv, "Budget")
                else if SourceType = "Targets" then
                    // Targets from test if available, else dev
                    try 
                        fn_GetTestData("Targets")
                    otherwise
                        fn_GetDevData("Targets")
                else
                    error "Unknown source type",
            
            // Add source tracking
            WithMetadata = Table.AddColumn(
                SourceData,
                "_Source",
                each SourceType & " (" & ActualEnv & ")",
                type text
            )
        in
            WithMetadata
in
    fn_GetHybridSource
```

## Error Handling for Dynamic Sources

### Robust Error Handling Module

```m
// Module: SafeSource.pqm
let
    // Safe connection with fallback
    SafeConnect = (Environment as text, QueryName as text) as table =>
        let
            // Try primary source
            PrimaryResult = 
                try
                    GetPrimarySource(Environment, QueryName)
                otherwise
                    null,
            
            // Fallback to secondary if primary fails
            Result = 
                if PrimaryResult <> null then
                    PrimaryResult
                else
                    try
                        GetFallbackSource(Environment, QueryName)
                    otherwise
                        GetEmergencySource(QueryName),
            
            // Log connection result
            LogEntry = 
                try
                    LogConnection(Environment, QueryName, Result <> null)
                otherwise
                    null
        in
            Result,
    
    // Primary source attempt
    GetPrimarySource = (Environment as text, QueryName as text) as table =>
        let
            Config = GetEnvironmentConfig(Environment),
            Source = Sql.Database(Config[Server], Config[Database]),
            Table = Source{[Schema="dbo",Item=QueryName]}[Data]
        in
            Table,
    
    // Fallback to read-only replica
    GetFallbackSource = (Environment as text, QueryName as text) as table =>
        let
            Config = GetEnvironmentConfig(Environment),
            // Use replica server
            ReplicaServer = Config[Server] & "-replica",
            Source = Sql.Database(ReplicaServer, Config[Database]),
            Table = Source{[Schema="dbo",Item=QueryName]}[Data]
        in
            Table,
    
    // Emergency local cache
    GetEmergencySource = (QueryName as text) as table =>
        let
            // Load from local CSV backup
            CachePath = "C:\data\cache\" & QueryName & ".csv",
            Source = Csv.Document(
                File.Contents(CachePath),
                [Delimiter = ",", Columns = 10, Encoding = 65001]
            )
        in
            Source,
    
    // Logging function
    LogConnection = (Environment as text, QueryName as text, Success as logical) as null =>
        let
            LogEntry = [
                Timestamp = DateTime.LocalNow(),
                Environment = Environment,
                Query = QueryName,
                Success = Success,
                Machine = Environment.MachineName()
            ],
            
            // Append to log table (conceptual)
            _ = 
                try 
                    // Write to log file
                    LogToFile(LogEntry)
                otherwise 
                    null
        in
            null
in
    SafeConnect
```

## Best Practices Summary

### Do's
✓ **Use parameters** - Define environment parameters in Power BI Desktop
✓ **Detect context** - Check if running in Desktop vs Service
✓ **Layer configurations** - Separate detection from connection logic
✓ **Fail gracefully** - Provide fallbacks for each environment
✓ **Document environment behavior** - Clear comments on switching logic
✓ **Test all paths** - Verify Dev, Test, and Prod connections work
✓ **Use credential store** - Never hardcode passwords

### Don'ts
✗ **Hardcode server names** - Use parameters or configuration
✗ **Assume Desktop-only** - Code must work in Service
✗ **Mix environment data** - Keep environments separate
✗ **Ignore error cases** - Handle connection failures
✗ **Complex nested logic** - Keep switching logic readable
✗ **Forget about gateways** - Test with data gateway connections

## Deployment Checklist

```
Before Deploying to Production:

□ Environment parameters defined in Power BI Desktop
□ All server names in configuration (not hardcoded)
□ Tested in Desktop with Dev connection
□ Tested in Desktop with Test connection (if possible)
□ Published to Test workspace
□ Verified Test workspace connections
□ Data gateway configured for Test
□ Tested with "View as" different roles
□ Deployed to Production workspace
□ Verified Production connections
□ Data gateway configured for Production
□ Monitored initial refresh
□ Documented environment setup
```

## Summary

**Key Points for Dynamic Source Switching:**

1. **Detect environment** - Use ActivityId, parameters, or machine name
2. **Centralize configuration** - Keep environment settings in one place
3. **Layer the logic** - Detection → Configuration → Connection
4. **Handle failures** - Each environment needs fallback options
5. **Test thoroughly** - Verify all environment paths
6. **Document behavior** - Make switching logic clear
7. **Secure credentials** - Use Power BI credential store

**The Golden Rule:**
> The same Power Query code should work in all environments by changing only the configuration, not the logic.
