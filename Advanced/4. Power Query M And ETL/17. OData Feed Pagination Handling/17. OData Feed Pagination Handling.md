# OData Feed Pagination Handling in Power Query

## Overview

Pagination is the process of retrieving large datasets through multiple API requests when the server returns data in "pages" rather than all at once. OData services typically implement server-driven pagination using `odata.nextLink` annotations in the response payload.

```
Initial Request → Page 1 (with nextLink) → Page 2 (with nextLink) → ... → Final Page (no nextLink)
                        ↓                          ↓
                 Contains URL for              Contains URL for
                  next page                      next page
```

## OData Pagination Fundamentals

### How OData Pagination Works

OData pagination is driven by **nextLink annotations** contained within the response payload . The `nextLink` value contains the URL to the next page of data:

```json
{
  "@odata.context": "https://services.odata.org/V4/TripPinService/$metadata#People",
  "@odata.count": 37,
  "value": [
    { "UserName": "russellwhyte", "FirstName": "Russell", "LastName": "Whyte" },
    { "UserName": "scottketchum", "FirstName": "Scott", "LastName": "Ketchum" },
    { "UserName": "ronaldmundy", "FirstName": "Ronald", "LastName": "Mundy" }
  ],
  "@odata.nextLink": "https://services.odata.org/v4/TripPinService/People?%24skiptoken=8"
}
```

**Key fields** :
- **`value`**: Contains the current page of data as an array of records
- **`@odata.nextLink`**: URL for the next page (absent on last page)
- **`@odata.count`**: Optional total record count

### Server-Driven vs. Client-Driven Pagination

| Type | How It Works | Example |
|------|--------------|---------|
| **Server-Driven** | Server provides next URL in response | OData `@odata.nextLink` |
| **Client-Driven** | Client calculates next URL using `$skip`/`$top` | REST APIs with offset pagination |

The OData standard defines server-driven pagination, where the server controls the page size and provides the next URL .

## The Easy Way: OData.Feed Built-in Pagination

**Important:** If you're using `OData.Feed` directly, you don't need to implement custom pagination—it's handled automatically .

```m
// Simple OData.Feed - pagination handled automatically
let
    Source = OData.Feed("https://services.odata.org/V4/TripPinService/People"),
    #"Converted to Table" = Table.FromList(Source, Splitter.SplitByNothing())
in
    #"Converted to Table"
```

`OData.Feed` automatically:
- Detects `@odata.nextLink` in responses
- Fetches all pages sequentially
- Combines results into a single table

### OData.Feed Options

```m
OData.Feed(
    "https://services.odata.org/V4/TripPinService/People",
    null,  // Headers
    [
        Query = [          // Add query parameters
            $top = "100",
            $select = "UserName,FirstName,LastName"
        ],
        Concurrent = true,  // Enable concurrent requests
        Timeout = #duration(0,0,5,0),  // 5-minute timeout
        EnableBatch = true  // Use $batch for multiple requests
    ]
)
```

**Key options** :
| Option | Description |
|--------|-------------|
| `Query` | Programmatically add query parameters with automatic escaping |
| `Concurrent` | When true, requests made concurrently (if service supports) |
| `Timeout` | Change request timeout (default 600 seconds) |
| `EnableBatch` | Allow OData $batch requests if MaxUriLength exceeded |
| `MaxUriLength` | Max URI length before batching (default 2048) |

## Custom Pagination with Table.GenerateByPage

When building custom connectors or handling non-OData pagination, you need explicit pagination logic.

### The Table.GenerateByPage Pattern

`Table.GenerateByPage` is a helper function (not part of standard library) that repeatedly calls a function until it returns `null` :

```m
// Source code for Table.GenerateByPage (copy into your .pq file)
let
    Table.GenerateByPage = (getNextPage as function) as table =>
        let
            listOfPages = List.Generate(
                () => getNextPage(null),            // First page
                (lastPage) => lastPage <> null,      // Stop when null
                (lastPage) => getNextPage(lastPage), // Next page from last
                (lastPage) => lastPage                // Return page
            )
        in
            Table.Combine(listOfPages)
in
    Table.GenerateByPage
```

### Complete OData Pagination Implementation

Based on the TripPin sample connector :

```m
let
    // ========== HELPER FUNCTIONS ==========
    
    // GetNextLink: Extract next URL from OData response
    GetNextLink = (response as record) as nullable text =>
        Record.FieldOrDefault(response, "@odata.nextLink"),
    
    // GetPage: Fetch single page and return with metadata
    GetPage = (url as text, optional headers as record) as table =>
        let
            response = Web.Contents(url, [Headers = headers]),
            body = Json.Document(response),
            nextLink = GetNextLink(body),
            data = Table.FromRecords(body[value])
        in
            data meta [NextLink = nextLink],
    
    // ========== PAGINATION FUNCTION ==========
    
    // GetAllPagesByNextLink: Fetch all pages using nextLink
    GetAllPagesByNextLink = (initialUrl as text, optional headers as record) as table =>
        let
            // Set default headers if not provided
            actualHeaders = if headers = null then [] else headers,
            
            // Generate all pages
            allPages = Table.GenerateByPage((previous) =>
                let
                    // First page: use initialUrl, subsequent: use NextLink from metadata
                    nextLink = if (previous = null) then initialUrl 
                              else Value.Metadata(previous)[NextLink]?,
                    
                    // Fetch page if nextLink exists
                    page = if (nextLink <> null) then GetPage(nextLink, actualHeaders) 
                          else null
                in
                    page
            )
        in
            allPages,
    
    // ========== USAGE ==========
    
    // Get all people with pagination
    Source = GetAllPagesByNextLink(
        "https://services.odata.org/V4/TripPinService/People",
        [Accept = "application/json"]
    )
in
    Source
```

### Step-by-Step Breakdown

**Step 1: GetNextLink Function** 
```m
GetNextLink = (response) => Record.FieldOrDefault(response, "@odata.nextLink")
```
- Extracts the next URL from the parsed JSON response
- Returns `null` if no next page

**Step 2: GetPage Function** 
```m
GetPage = (url, headers) =>
    let
        response = Web.Contents(url, [Headers = headers]),
        body = Json.Document(response),
        nextLink = GetNextLink(body),
        data = Table.FromRecords(body[value])
    in
        data meta [NextLink = nextLink]
```
- Fetches one page using `Web.Contents`
- Parses JSON response
- Extracts nextLink for subsequent pages
- Stores nextLink in the table's **metadata** using `meta`

**Step 3: GetAllPagesByNextLink** 
```m
GetAllPagesByNextLink = (initialUrl, headers) =>
    Table.GenerateByPage((previous) =>
        let
            nextLink = if previous = null then initialUrl 
                      else Value.Metadata(previous)[NextLink]?,
            page = if nextLink <> null then GetPage(nextLink, headers) else null
        in
            page
    )
```
- Uses `Table.GenerateByPage` to loop through pages
- First call: uses `initialUrl`
- Subsequent calls: retrieves `NextLink` from previous page's metadata
- Stops when `nextLink` is null

### Value.Metadata for State Management

The `Value.Metadata` function retrieves metadata attached to a value with `meta` :

```m
// Attach metadata when returning a page
data meta [NextLink = nextLink]

// Retrieve metadata later
nextLink = Value.Metadata(previous)[NextLink]?
```

This is how pagination state is passed between iterations.

## Alternative: Skip/Top Pagination Pattern

Some APIs use client-driven pagination with `$skip` and `$top` parameters :

```m
let
    // Function to fetch all pages using $skip/$top
    GetAllPagesSkipTop = (
        baseUrl as text, 
        pageSize as number,
        optional headers as record
    ) as table =>
        let
            // First, get total count (if available)
            countUrl = baseUrl & "/$count",
            countResponse = Web.Contents(countUrl, [Headers = headers]),
            totalRows = try Number.FromText(Text.FromBinary(countResponse)) otherwise null,
            
            // Calculate number of pages
            actualPageSize = pageSize,
            pages = if totalRows = null then null 
                   else Number.RoundUp(totalRows / actualPageSize),
            
            // Function to fetch one page
            GetPage = (skip as number) =>
                let
                    url = baseUrl & "?$skip=" & Text.From(skip) & "&$top=" & Text.From(actualPageSize),
                    response = Web.Contents(url, [Headers = headers]),
                    json = Json.Document(response),
                    table = Table.FromRecords(json[value])
                in
                    table,
            
            // Generate all pages
            pageNumbers = if pages = null then {0} else List.Numbers(0, pages),
            pageData = List.Transform(pageNumbers, each GetPage(_ * actualPageSize)),
            combined = Table.Combine(pageData)
        in
            combined,

    // Usage
    Source = GetAllPagesSkipTop(
        "https://api.example.com/odata/People",
        100,
        [Accept = "application/json"]
    )
in
    Source
```

## Real-World Example: Complete OData Connector

```m
// Custom OData connector with pagination, error handling, and options
let
    // Configuration
    BaseUrl = "https://services.odata.org/V4/TripPinService",
    EntitySet = "People",
    
    // ========== CORE FUNCTIONS ==========
    
    // GetNextLink from OData response
    GetNextLink = (response as record) as nullable text =>
        Record.FieldOrDefault(response, "@odata.nextLink"),
    
    // GetPage with error handling
    GetPage = (url as text, options as record) as table =>
        let
            // Attempt request
            response = try
                Web.Contents(url, [
                    Headers = options[Headers],
                    Timeout = options[Timeout],
                    IsRetry = options[IsRetry]? or false
                ])
            otherwise
                if options[RetryCount]? > 0 then
                    // Retry logic
                    @GetPage(url, options & [IsRetry = true, RetryCount = options[RetryCount] - 1])
                else
                    error "Failed to fetch page after retries",
            
            // Parse response
            body = try Json.Document(response) otherwise 
                error "Invalid JSON response",
            
            // Check for OData error
            ErrorCheck = if Record.HasFields(body, {"error"}) then
                error "OData error: " & body[error][message]?
            else
                null,
            
            // Extract nextLink and data
            nextLink = GetNextLink(body),
            data = if body[value]? = null then 
                #table({}, {})  // Empty table
            else
                Table.FromRecords(body[value])
        in
            data meta [NextLink = nextLink],
    
    // ========== PAGINATION FUNCTION ==========
    
    // Table.GenerateByPage (if not already defined)
    Table.GenerateByPage = (getNextPage as function) as table =>
        let
            listOfPages = List.Generate(
                () => getNextPage(null),
                (lastPage) => lastPage <> null,
                (lastPage) => getNextPage(lastPage),
                (lastPage) => lastPage
            )
        in
            Table.Combine(listOfPages),
    
    // Fetch all pages
    GetAllPages = (initialUrl as text, options as record) as table =>
        Table.GenerateByPage((previous) =>
            let
                nextLink = if previous = null then initialUrl 
                          else Value.Metadata(previous)[NextLink]?,
                page = if nextLink <> null then 
                    GetPage(nextLink, options)
                else
                    null
            in
                page
        ),
    
    // ========== MAIN FUNCTION ==========
    
    // Public function
    GetODataEntity = (
        serviceRoot as text,
        entityName as text,
        optional query as record,
        optional pageSize as number,
        optional timeoutSeconds as number
    ) as table =>
        let
            // Build URL
            baseUrl = Text.TrimEnd(serviceRoot, "/"),
            
            // Add query parameters
            baseQuery = [
                $top = if pageSize = null then null else Text.From(pageSize)
            ],
            mergedQuery = if query = null then baseQuery 
                         else Record.Combine({baseQuery, query}),
            
            // Build entity URL
            entityUrl = baseUrl & "/" & entityName,
            fullUrl = if mergedQuery = null or Record.IsEmpty(mergedQuery) then
                entityUrl
            else
                entityUrl & "?" & Uri.BuildQueryString(mergedQuery),
            
            // Configure options
            options = [
                Headers = [
                    Accept = "application/json",
                    #"User-Agent" = "PowerBI-CustomConnector"
                ],
                Timeout = #duration(0,0,0,timeoutSeconds ?? 120),
                RetryCount = 3
            ],
            
            // Fetch all pages
            Result = GetAllPages(fullUrl, options)
        in
            Result,
    
    // ========== USAGE ==========
    
    // Get all people with custom options
    Source = GetODataEntity(
        "https://services.odata.org/V4/TripPinService",
        "People",
        [$select = "UserName,FirstName,LastName,Emails"],
        50,    // Page size
        60     // Timeout in seconds
    )
in
    Source
```

## Performance Considerations

### Concurrent Requests

For APIs that support it, concurrent requests can improve performance :

```m
// Enable concurrent requests in OData.Feed
Source = OData.Feed(
    "https://services.odata.org/V4/TripPinService/People",
    null,
    [Concurrent = true]
)
```

For custom pagination, implement parallel requests cautiously:

```m
// Parallel page fetching (advanced)
GetAllPagesParallel = (initialUrl as text, maxConcurrent as number) as table =>
    let
        // Get first page to determine total pages (if possible)
        firstPage = GetPage(initialUrl, []),
        totalPages = ... // Extract from response metadata
        
        // Fetch pages in parallel batches
        pageBatches = List.Split({0..totalPages-1}, maxConcurrent),
        
        FetchBatch = (batch as list) =>
            Table.Combine(
                List.Transform(batch, each GetPage(initialUrl & "?$skip=" & Text.From(_ * 100)))
            ),
        
        AllBatches = List.Transform(pageBatches, FetchBatch),
        Combined = Table.Combine(AllBatches)
    in
        Combined
```

### Caching

```m
// Add cache control headers
GetPageWithCache = (url as text) as table =>
    let
        response = Web.Contents(url, [
            Headers = [
                #"Cache-Control" = "max-age=3600",
                #"If-None-Match" = previousETag  // Conditional request

```

## Error Handling

### Comprehensive Error Handling

```m
GetPageWithErrorHandling = (url as text, retryCount as number) as table =>
    let
        Result = try
            let
                response = Web.Contents(url, [
                    Headers = [Accept = "application/json"],
                    Timeout = #duration(0,0,1,0)  // 1 minute
                ]),
                body = Json.Document(response),
                
                // Check for OData error structure
                HasError = Record.HasFields(body, {"error"}),
                
                Result = if HasError then
                    error [
                        Reason = "ODataError",
                        Message = body[error][message]?,
                        Code = body[error][code]?
                    ]
                else
                    body
            in
                Result
        otherwise
            let
                // Log error
                _ = Diagnostics.LogValue("PageError", [
                    Url = url,
                    Error = Diagnostics.Trace(),
                    RetryCount = retryCount
                ]),
                
                // Retry logic
                Result = if retryCount > 0 then
                    @GetPageWithErrorHandling(url, retryCount - 1)
                else
                    error "Max retries exceeded"
            in
                Result
    in
        Result
```

### Common Error Patterns

| Error | Cause | Solution |
|-------|-------|----------|
| **400 Bad Request** | Invalid URL or parameters | Check URL encoding |
| **401 Unauthorized** | Authentication expired | Refresh token |
| **429 Too Many Requests** | Rate limited | Add exponential backoff |
| **500 Internal Server Error** | Server issue | Retry with delay |
| **Missing value field** | Not OData JSON | Check response format |

## Best Practices Summary

### Do's
✓ **Use OData.Feed when possible** - Automatic pagination handling   
✓ **Store nextLink in metadata** - Pass state between iterations   
✓ **Implement retry logic** - Handle transient failures  
✓ **Set appropriate timeouts** - Default 600 seconds may be too long   
✓ **Test with Fiddler** - Verify pagination behavior   
✓ **Handle empty pages** - Return empty table, not null  

### Don'ts
✗ **Don't assume page size** - Server may ignore `$top` preference   
✗ **Don't ignore nextLink** - Always follow server-provided URLs  
✗ **Don't hardcode page limits** - Support any size response  
✗ **Don't forget error handling** - Network and server failures happen  
✗ **Don't use custom pagination with OData.Feed** - Redundant   

## Summary

**Key Points for OData Pagination Handling:**

| Aspect | Recommendation |
|--------|----------------|
| **Simple OData** | Use `OData.Feed` - handles pagination automatically  |
| **Custom Connectors** | Implement `Table.GenerateByPage` pattern  |
| **NextLink Storage** | Use `meta` and `Value.Metadata`  |
| **Error Handling** | Add retries and exponential backoff |
| **Testing** | Use Fiddler to verify requests  |
| **Performance** | Consider `Concurrent = true` for supported services  |

**The Golden Rule:**
> For OData services, `OData.Feed` handles pagination automatically—only implement custom pagination when building connectors for non-OData APIs or when you need explicit control over the process.
