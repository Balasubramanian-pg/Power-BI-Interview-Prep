# Binary Data Processing from SharePoint in Power Query

## Overview

Processing binary data from SharePoint involves retrieving files stored in SharePoint document libraries and extracting meaningful information from their binary content. Power Query provides several functions and patterns for this, ranging from simple file listing to complex content extraction.

```
SharePoint Site → Document Library → Binary Content → Transformation → Structured Data
       ↓                  ↓               ↓                 ↓
  SharePoint.     Filter by       Extract via       Combine,
  Functions       Extension       Format-Specific   Clean, Load
                                  Functions
```

## Core SharePoint Functions for Binary Access

### SharePoint.Files vs SharePoint.Contents

| Function | Behavior | Best For |
|----------|----------|----------|
| **SharePoint.Files** | Returns flat list of all files with metadata and binary content | Small sites, simple file lists, quick combines |
| **SharePoint.Contents** | Hierarchical navigation through folders | Large sites, performance optimization, structured access |

### Critical API Version Parameter

The `ApiVersion` parameter significantly impacts performance and stability:

```m
// Default (slow) - ApiVersion=14
Source = SharePoint.Contents("https://tenant.sharepoint.com/sites/SiteName")

// Optimized - ApiVersion=15 (RECOMMENDED)
Source = SharePoint.Contents(
    "https://tenant.sharepoint.com/sites/SiteName", 
    [ApiVersion=15]
)
```

**Performance Impact** :
| Function | API Version | 7k Files | 26k Files |
|----------|-------------|----------|------------|
| SharePoint.Contents | 14 (Default) | 2:00 | 5:03 |
| SharePoint.Files | 15 | 1:00 | 1:30 |
| SharePoint.Contents | 15 | **0:53** | **0:50** |

## Accessing Binary Content

### Pattern 1: Basic File Listing with Binary

```m
let
    // Connect to SharePoint with optimized API
    Source = SharePoint.Contents(
        "https://tenant.sharepoint.com/sites/Sales", 
        [ApiVersion=15]
    ),
    
    // Navigate to specific document library
    Documents = Source{[Name="Shared Documents"]}[Content],
    
    // Filter for Excel files only
    ExcelFiles = Table.SelectRows(Documents, each [Extension] = ".xlsx"),
    
    // Binary content is in the [Content] column
    // Each row contains metadata + binary
    Result = ExcelFiles
in
    Result
```

### Pattern 2: Accessing Specific File by Path

```m
let
    Source = SharePoint.Contents(
        "https://tenant.sharepoint.com/sites/Sales", 
        [ApiVersion=15]
    ),
    
    // Navigate folder structure
    Documents = Source{[Name="Shared Documents"]}[Content],
    Reports = Documents{[Name="Monthly Reports"]}[Content],
    Q1Reports = Reports{[Name="Q1"]}[Content],
    
    // Get specific file
    TargetFile = Q1Reports{[Name="Sales_January.xlsx"]}[Content],
    
    // Process binary content
    Workbook = Excel.Workbook(TargetFile, null, true),
    Data = Workbook{[Item="SalesData",Kind="Table"]}[Data]
in
    Data
```

### Pattern 3: Handling Binary Access Errors

The common 500 error when accessing binary content can be resolved by specifying API version :

```m
// Error-prone approach
BadSource = SharePoint.Files("https://examplesite.sharepoint.com/sites/example/")
// Clicking [Binary] returns 500 Internal Server Error

// Correct approach
GoodSource = SharePoint.Files(
    "https://examplesite.sharepoint.com/sites/example/", 
    [ApiVersion=15]
)
// Binary access works correctly
```

```m

    "https://tenant.sharepoint.com/sites/Sales", 
    [ApiVersion=15]
),
Documents = Source{[Name="Shared Documents"]}[Content],

// Step 2: Filter relevant files
ExcelFiles = Table.SelectRows(Documents, each [Extension] = ".xlsx"),

// Step 3: Select Content column and use "Combine Files"
// This automatically generates:
// - Transform Sample File query
// - Helper function
// - Combined output

// The generated helper function looks like:
(Parameter1 as binary) as table =>
    let
        Source = Excel.Workbook(Parameter1, null, true),
        Data = Source{[Item="Sheet1",Kind="Sheet"]}[Data],
        #"Promoted Headers" = Table.PromoteHeaders(Data, [PromoteAllScalars=true])
    in
        #"Promoted Headers"
```

### Manual Combine with Custom Function

For more control, create a custom function :

```m
// Custom function: GetCellContent
// Extracts specific cell from Excel binary
(binaryParameter as binary, col as number, row as number) as any =>
let
    Source = Excel.Workbook(binaryParameter, null, false),
    
    // Get first unhidden sheet
    UnhiddenSheets = Table.SelectRows(
        Source, 
        each [Hidden] = false and [Kind] = "Sheet"
    ),
    Sheet = UnhiddenSheets{0}[Data],
    
    // Get specific column and row
    ColumnName = "Column" & Number.ToText(col),
    Column = Table.SelectColumns(Sheet, {ColumnName}),
    Cell = Record.Field(Column{row-1}, ColumnName)
in
    Cell

// Main query using the function
let
    Source = SharePoint.Contents(
        "https://tenant.sharepoint.com/sites/Sales",
        [ApiVersion=15]
    ),
    Documents = Source{[Name="Shared Documents"]}[Content],
    ExcelFiles = Table.SelectRows(Documents, each [Extension] = ".xlsx"),
    
    // Extract cell D5 from each file
    WithCellData = Table.AddColumn(
        ExcelFiles, 
        "CellD5", 
        each GetCellContent([Content], 4, 5)
    )
in
    WithCellData
```

## Advanced Binary Processing

### Processing ZIP Files

Extracting information from ZIP archives requires binary parsing :

```m
// Function to get ZIP contents metadata
(SourceBinary as binary) as table =>
let
    // Parse ZIP central directory structure
    ushort = BinaryFormat.ByteOrder(BinaryFormat.UnsignedInteger16, ByteOrder.LittleEndian),
    uint = BinaryFormat.ByteOrder(BinaryFormat.UnsignedInteger32, ByteOrder.LittleEndian),
    
    // Read end of central directory record
    EOCD = BinaryFormat.Record([
        Magic = uint,
        DiskNum = ushort,
        CDirectoryDiskId = ushort,
        CDirectoryRecordCountOnDisk = ushort,
        CDirectoryRecordCount = ushort,
        SizeOfCentralDirectory = uint,
        CentralDirectoryOffset = uint,
        CommendLength = ushort
    ]),
    
    // Parse central directory entries
    EOCDRecord = EOCD(SourceBinary),
    CentralDir = BinaryFormat.Record([
        Items = BinaryFormat.List(
            BinaryFormat.Record([
                Magic = uint,
                CurrentVersion = ushort,
                MinVersion = ushort,
                Flags = ushort,
                CompressionMethod = ushort,
                FileModificationTime = ushort,
                FileModificationDate = ushort,
                CRC32 = uint,
                CompressedSize = uint,
                UncompressedSize = uint,
                FileNameLength = ushort,
                ExtraFieldLength = ushort,
                FileCommentLength = ushort,
                // Extract filename
                FileName = BinaryFormat.Transform(
                    BinaryFormat.Binary(FileNameLength),
                    each Binary.ToText(_, TextEncoding.Ascii)
                )
            ]),
            EOCDRecord[CDirectoryRecordCount]
        )
    ]),
    
    // Get items
    Directory = CentralDir(
        Binary.Range(
            SourceBinary, 
            EOCDRecord[CentralDirectoryOffset], 
            EOCDRecord[SizeOfCentralDirectory]
        )
    ),
    
    // Convert to table
    FilesTable = Table.FromRecords(Directory[Items])
in
    FilesTable
```

### Processing PDF Metadata

```m
// Extract basic PDF info (conceptual example)
(SourceBinary as binary) as record =>
let
    // Simple PDF header check
    Header = Binary.Range(SourceBinary, 0, 5),
    IsPDF = Binary.ToText(Header) = "%PDF-",
    
    // Look for Info dictionary (simplified)
    // Real implementation would need full PDF parsing
    Text = Binary.ToText(SourceBinary, TextEncoding.Ascii),
    
    // Extract creation date using regex pattern
    // Pattern: /CreationDate\(D:[0-9]{14}\)
    CreationDate = 
        try
            let
                Start = Text.PositionOf(Text, "/CreationDate(D:"),
                DateStr = Text.Range(Text, Start + 15, 14)
            in
                DateStr
        otherwise
            null
in
    [
        IsValidPDF = IsPDF,
        CreationDate = CreationDate
    ]
```

## Performance Optimization

### Filter Early, Filter Often

```m
// BAD - Gets ALL files then filters
PoorPerformance = 
let
    AllFiles = SharePoint.Files("https://tenant.sharepoint.com/sites/Sales"),
    Filtered = Table.SelectRows(AllFiles, each [Extension] = ".xlsx")
in
    Filtered

// GOOD - Navigate directly to target folder
GoodPerformance = 
let
    Source = SharePoint.Contents("https://tenant.sharepoint.com/sites/Sales", [ApiVersion=15]),
    Docs = Source{[Name="Shared Documents"]}[Content],
    TargetFolder = Docs{[Name="Monthly Reports"]}[Content],
    Files = Table.SelectRows(TargetFolder, each [Extension] = ".xlsx")
in
    Files
```

### Binary Processing Optimization

```m
// Process binaries in batches to manage memory
let
    Source = SharePoint.Contents("https://tenant.sharepoint.com/sites/Sales", [ApiVersion=15]),
    Docs = Source{[Name="Shared Documents"]}[Content],
    
    // Get only recent files first
    RecentFiles = Table.SelectRows(Docs, each [Date modified] > #date(2024,1,1)),
    
    // Process in chunks to avoid memory pressure
    ChunkSize = 10,
    Chunks = List.Split(Table.ToRecords(RecentFiles), ChunkSize),
    
    ProcessChunk = (chunk as list) as table =>
        let
            AsTable = Table.FromRecords(chunk),
            WithData = Table.AddColumn(AsTable, "ExtractedData", 
                each ExtractFromBinary([Content]),
                type table
            ),
            Expanded = Table.ExpandTableColumn(WithData, "ExtractedData", 
                Table.ColumnNames(ExtractFromBinary(AsTable[Content]{0}))
            )
        in
            Expanded,
    
    Processed = List.Transform(Chunks, ProcessChunk),
    Combined = Table.Combine(Processed)
in
    Combined
```

## Hybrid Pattern: File Content + Custom Metadata

SharePoint functions don't automatically include custom library columns. Use a hybrid approach :

```m
let
    SiteUrl = "https://tenant.sharepoint.com/sites/Sales",
    
    // Part 1: Get file binaries via SharePoint.Contents
    FileSource = SharePoint.Contents(SiteUrl, [ApiVersion=15]),
    Docs = FileSource{[Name="Shared Documents"]}[Content],
    Files = Table.SelectRows(Docs, each [Extension] = ".xlsx"),
    
    // Add file URL for joining
    WithUrl = Table.AddColumn(Files, "FileRef", 
        each SiteUrl & "/Shared Documents/" & [Name], 
        type text
    ),
    
    // Part 2: Get custom metadata via OData
    MetadataSource = OData.Feed(
        SiteUrl & "/_api/web/lists/getbytitle('Shared Documents')/items",
        null,
        [Implementation="2.0"]
    ),
    
    // Select relevant metadata columns
    Metadata = Table.SelectColumns(
        MetadataSource,
        {"FileLeafRef", "CustomColumn1", "CustomColumn2", "ApprovalStatus"}
    ),
    
    // Part 3: Join binaries with metadata
    Joined = Table.NestedJoin(
        WithUrl,
        {"Name"},
        Metadata,
        {"FileLeafRef"},
        "Metadata",
        JoinKind.LeftOuter
    ),
    
    // Expand metadata
    Expanded = Table.ExpandTableColumn(
        Joined,
        "Metadata",
        {"CustomColumn1", "CustomColumn2", "ApprovalStatus"},
        {"Category", "Owner", "Status"}
    )
in
    Expanded
```

## Error Handling

### Handling Missing Files

```m
let
    Source = SharePoint.Contents(
        "https://tenant.sharepoint.com/sites/Sales", 
        [ApiVersion=15]
    ),
    
    // Try to navigate with error handling
    GetFolder = try
        let
            Docs = Source{[Name="Shared Documents"]}[Content],
            Reports = Docs{[Name="Monthly Reports"]}[Content]
        in
            Reports
    otherwise
        // Return empty table with expected structure
        #table({"Name", "Extension", "Content"}, {}),
    
    // If folder exists, get files
    Result = if GetFolder is table and Table.RowCount(GetFolder) > 0 then
        Table.SelectRows(GetFolder, each [Extension] = ".xlsx")
    else
        #table({"Name", "Extension", "Content"}, {})
in
    Result
```

### Handling Binary Processing Errors

```m
// Safe binary extraction
SafeExtract = (binary as binary, fileType as text) as table =>
let
    Result = try
        if fileType = ".xlsx" then
            Excel.Workbook(binary, null, true)
        else if fileType = ".csv" then
            Csv.Document(binary)
        else
            error "Unsupported file type"
    otherwise
        // Return error row instead of failing entire query
        #table(
            {"Error", "FileType"},
            {{"Extraction failed", fileType}}
        )
in
    Result

// Usage in main query
let
    Source = SharePoint.Contents("https://tenant.sharepoint.com/sites/Sales", [ApiVersion=15]),
    Docs = Source{[Name="Shared Documents"]}[Content],
    Files = Table.SelectRows(Docs, each List.Contains({".xlsx", ".csv"}, [Extension])),
    
    // Safe extraction for each file
    WithData = Table.AddColumn(Files, "FileData", 
        each SafeExtract([Content], [Extension]),
        type table
    )
in
    WithData
```

## Practical Example: Complete Sales Report Consolidation

```m
// Comprehensive solution for monthly sales reports
let
    // Configuration
    SiteUrl = "https://tenant.sharepoint.com/sites/Sales",
    LibraryName = "Shared Documents",
    ReportsFolder = "Monthly Reports",
    TargetYear = 2025,
    
    // Step 1: Connect to SharePoint with optimized API
    Source = SharePoint.Contents(SiteUrl, [ApiVersion=15]),
    
    // Step 2: Navigate to target folder
    DocumentLibrary = try Source{[Name=LibraryName]}[Content] 
        otherwise error "Document library not found",
    
    ReportsFolderTable = try DocumentLibrary{[Name=ReportsFolder]}[Content] 
        otherwise error "Reports folder not found",
    
    // Step 3: Get all Excel files for target year
    AllFiles = Table.SelectRows(
        ReportsFolderTable, 
        each [Extension] = ".xlsx" and 
              [Name] <> null and
              Text.Contains([Name], Text.From(TargetYear))
    ),
    
    // Step 4: Sort by date (newest first)
    SortedFiles = Table.Sort(AllFiles, {{"Date modified", Order.Descending}}),
    
    // Step 5: Create extraction function
    ExtractSalesData = (binary as binary) as table =>
        let
            Workbook = Excel.Workbook(binary, null, true),
            Sheet = try Workbook{[Item="SalesData",Kind="Table"]}[Data]
                otherwise Workbook{[Kind="Sheet"]}[0][Data],
            Promoted = Table.PromoteHeaders(Sheet, [PromoteAllScalars=true])
        in
            Promoted,
    
    // Step 6: Apply to all files
    WithData = Table.AddColumn(
        SortedFiles, 
        "SalesData", 
        each ExtractSalesData([Content]),
        type table
    ),
    
    // Step 7: Remove binary column to save memory
    WithoutBinary = Table.RemoveColumns(WithData, {"Content"}),
    
    // Step 8: Expand all data
    Expanded = Table.ExpandTableColumn(
        WithoutBinary,
        "SalesData",
        {"Date", "Product", "Quantity", "Amount", "Region"},
        {"Date", "Product", "Quantity", "Amount", "Region"}
    ),
    
    // Step 9: Add file metadata
    WithMetadata = Table.AddColumn(Expanded, "SourceFile", each [Name], type text),
    
    // Step 10: Final clean up
    FinalColumns = Table.SelectColumns(
        WithMetadata,
        {"Date", "Product", "Quantity", "Amount", "Region", "SourceFile"}
    ),
    
    // Step 11: Data type conversion
    TypedTable = Table.TransformColumnTypes(
        FinalColumns,
        {
            {"Date", type date},
            {"Product", type text},
            {"Quantity", Int64.Type},
            {"Amount", type number},
            {"Region", type text},
            {"SourceFile", type text}
        }
    )
in
    TypedTable
```

## Summary

**Key Points for Binary Processing from SharePoint:**

| Aspect | Recommendation |
|--------|----------------|
| **Function** | `SharePoint.Contents` with `[ApiVersion=15]` for performance  |
| **Navigation** | Drill down through folders, don't flatten everything |
| **Filtering** | Filter by extension and folder path as early as possible  |
| **Binary Access** | Use `[ApiVersion=15]` to avoid 500 errors  |
| **File Combination** | Use built-in combine or custom function based on needs  |
| **Custom Metadata** | Hybrid approach with OData feed  |
| **Error Handling** | Try-otherwise patterns for missing files/folders |
| **Performance** | Process in chunks for large file sets |

**The Golden Rule:**
> Always navigate to the most specific folder possible, filter by extension immediately, and use ApiVersion=15. The binary content is just the starting point—what matters is how efficiently you extract and transform it.
