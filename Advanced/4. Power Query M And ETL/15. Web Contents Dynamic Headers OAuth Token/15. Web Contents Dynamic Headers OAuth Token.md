# Web.Contents Dynamic Headers and OAuth Tokens in Power Query

## Overview

Using `Web.Contents` with dynamically generated OAuth tokens requires a two-step process: obtaining a token from an authentication endpoint, then using that token in subsequent API requests. This is essential for APIs that use OAuth 2.0, which is the industry standard for authorization.

```
Authentication Endpoint ← POST credentials → Access Token
         ↓
    API Endpoint ← GET with Bearer token → Data
```

## Core Pattern: Two-Step Token Flow

### Basic Token Generation

```m
let
    // Step 1: Get access token
    GetToken = () =>
        let
            url = "https://api.example.com/oauth/token",
            body = "grant_type=client_credentials&client_id=YOUR_ID&client_secret=YOUR_SECRET",
            headers = [#"Content-Type" = "application/x-www-form-urlencoded"],
            response = Web.Contents(url, [
                Headers = headers,
                Content = Text.ToBinary(body)
            ]),
            jsonResponse = Json.Document(response),
            accessToken = jsonResponse[access_token]
        in
            accessToken,
    
    // Step 2: Use token for API call
    token = GetToken(),
    apiUrl = "https://api.example.com/v1/data",
    apiHeaders = [#"Authorization" = "Bearer " & token],
    apiResponse = Web.Contents(apiUrl, [Headers = apiHeaders]),
    data = Json.Document(apiResponse)
in
    data
```

### Critical Requirement for Power BI Service

**Important:** If you plan to publish to Power BI service, all code must be in **a single query** . Do not call the access token from a separate query or scheduled refreshes will fail.

## Complete Implementation Patterns

### Pattern 1: Client Credentials Flow (Standard)

```m
let
    // Configuration parameters (use Power BI parameters)
    BaseUrl = "https://api.example.com",
    ClientId = "your-client-id",
    ClientSecret = "your-client-secret",
    
    // Get token
    tokenResponse = Json.Document(
        Web.Contents(
            BaseUrl,
            [
                RelativePath = "/oauth/token",
                Headers = [#"Content-Type" = "application/x-www-form-urlencoded"],
                Content = Text.ToBinary(
                    Uri.BuildQueryString([
                        grant_type = "client_credentials",
                        client_id = ClientId,
                        client_secret = ClientSecret
                    ])
                )
            ]
        )
    ),
    
    accessToken = tokenResponse[access_token],
    
    // Call API with token
    apiData = Json.Document(
        Web.Contents(
            BaseUrl,
            [
                RelativePath = "/v1/data",
                Headers = [
                    #"Authorization" = "Bearer " & accessToken,
                    #"Content-Type" = "application/json"
                ],
                Query = [limit = "1000", page = "1"]
            ]
        )
    ),
    
    // Convert to table
    #"Converted to Table" = Table.FromRecords(apiData)
in
    #"Converted to Table"
```

### Pattern 2: Password Grant Flow (Resource Owner)

```m
let
    BaseUrl = "https://api.example.com",
    Username = "user@example.com",
    Password = "user-password",
    ClientId = "your-client-id",

    // Get token with username/password
    tokenResponse = Json.Document(

                RelativePath = "/oauth/token",
                Headers = [#"Content-Type" = "application/x-www-form-urlencoded"],
                Content = Text.ToBinary(
                    Uri.BuildQueryString([
                        grant_type = "password",
                        username = Username,
                        password = Password,
                        client_id = ClientId,
                        client_secret = ClientSecret
                    ])
                )
            ]
        )
    ),
    
    accessToken = tokenResponse[access_token],
    
    // API call with token
    apiData = Json.Document(
        Web.Contents(
            BaseUrl,
            [
                RelativePath = "/v1/users/me",
                Headers = [#"Authorization" = "Bearer " & accessToken]
            ]
        )
    )
in
    apiData
```

### Pattern 3: Handling Token Expiry with Refresh Tokens

For APIs that provide refresh tokens, you need to handle token renewal :

```m
let
    BaseUrl = "https://api.example.com",
    ClientId = "your-client-id",
    ClientSecret = "your-client-secret",
    
    // Function to get new token with refresh token
    RefreshToken = (refreshToken as text) as record =>
        let
            response = Json.Document(
                Web.Contents(
                    BaseUrl,
                    [
                        RelativePath = "/oauth/token",
                        Headers = [#"Content-Type" = "application/x-www-form-urlencoded"],
                        Content = Text.ToBinary(
                            Uri.BuildQueryString([
                                grant_type = "refresh_token",
                                refresh_token = refreshToken,
                                client_id = ClientId,
                                client_secret = ClientSecret
                            ])
                        )
                    ]
                )
            )
        in
            response,
    
    // Initial token (first run only)
    initialToken = Json.Document(
        Web.Contents(
            BaseUrl,
            [
                RelativePath = "/oauth/token",
                Headers = [#"Content-Type" = "application/x-www-form-urlencoded"],
                Content = Text.ToBinary(
                    Uri.BuildQueryString([
                        grant_type = "client_credentials",
                        client_id = ClientId,
                        client_secret = ClientSecret
                    ])
                )
            ]
        )
    ),
    
    // For subsequent runs, you'd need to store the refresh token
    // This is simplified - actual implementation requires token storage
    currentToken = initialToken[access_token],
    currentRefreshToken = initialToken[refresh_token],
    
    // API call with current token
    apiData = try
        Json.Document(
            Web.Contents(
                BaseUrl,
                [
                    RelativePath = "/v1/data",
                    Headers = [#"Authorization" = "Bearer " & currentToken]
                ]
            )
        )
    otherwise
        // Token expired - refresh and retry
        let
            newTokens = RefreshToken(currentRefreshToken),
            retryData = Json.Document(
                Web.Contents(
                    BaseUrl,
                    [
                        RelativePath = "/v1/data",
                        Headers = [#"Authorization" = "Bearer " & newTokens[access_token]]
                    ]
                )
            )
        in
            retryData
in
    apiData
```

## Advanced Patterns

### Pattern 4: Pagination with Token

```m
let
    BaseUrl = "https://api.example.com",
    ClientId = "your-client-id",
    ClientSecret = "your-client-secret",
    
    // Get token
    tokenResponse = Json.Document(
        Web.Contents(
            BaseUrl,
            [
                RelativePath = "/oauth/token",
                Headers = [#"Content-Type" = "application/x-www-form-urlencoded"],
                Content = Text.ToBinary(
                    Uri.BuildQueryString([
                        grant_type = "client_credentials",
                        client_id = ClientId,
                        client_secret = ClientSecret
                    ])
                )
            ]
        )
    ),
    
    accessToken = tokenResponse[access_token],
    
    // Pagination function
    FetchPage = (page as number) as table =>
        let
            response = Json.Document(
                Web.Contents(
                    BaseUrl,
                    [
                        RelativePath = "/v1/data",
                        Headers = [#"Authorization" = "Bearer " & accessToken],
                        Query = [
                            page = Text.From(page),
                            per_page = "100"
                        ]
                    ]
                )
            ),
            items = response[items],
            table = if items = null then #table({}, {}) else Table.FromRecords(items)
        in
            table,
    
    // Get first page to determine total pages
    firstPage = FetchPage(1),
    totalPages = 5, // Extract from response headers or metadata
    
    // Fetch all pages
    AllPages = List.Generate(
        () => [Page = 1, Data = firstPage],
        each [Page] <= totalPages,
        each [Page = [Page] + 1, Data = FetchPage([Page] + 1)],
        each [Data]
    ),
    
    Combined = Table.Combine(AllPages)
in
    Combined
```

### Pattern 5: Secure API Key Storage (Alternative to OAuth)

For APIs that use API keys, use the `ApiKeyName` parameter for secure storage :

```m
let
    // ApiKeyName tells Power BI to prompt for the key value securely
    // The actual key is stored in credentials, not in code
    Source = Web.Contents(
        "https://api.example.com/v1/data",
        [
            ApiKeyName = "api_key",  // Name of query parameter
            Query = [
                format = "json",
                limit = "1000"
            ]
        ]
    ),
    json = Json.Document(Source),
    #"Converted to Table" = Table.FromRecords(json)
in
    #"Converted to Table"
```

When using `ApiKeyName`, Power BI will:
- Prompt for the API key value when setting up the data source
- Store it securely in credentials
- Automatically append `?api_key=******` to requests
- Never expose the key in M code 

## Handling POST Requests for Token Endpoints

### Making Web.Contents Perform POST

`Web.Contents` defaults to GET. To force a POST, include the `Content` parameter :

```m
// POST with form data
tokenResponse = Web.Contents(
    "https://api.example.com/token",
    [
        Headers = [#"Content-Type" = "application/x-www-form-urlencoded"],
        Content = Text.ToBinary("grant_type=client_credentials&client_id=123")
    ]
)

// POST with JSON body
tokenResponse = Web.Contents(
    "https://api.example.com/token",
    [
        Headers = [#"Content-Type" = "application/json"],
        Content = Json.FromValue([
            grant_type = "client_credentials",
            client_id = "123",
            client_secret = "abc"
        ])
    ]
)
```

### Using Uri.BuildQueryString

For cleaner parameter handling:

```m
tokenResponse = Web.Contents(
    "https://api.example.com/token",
    [
        Headers = [#"Content-Type" = "application/x-www-form-urlencoded"],
        Content = Text.ToBinary(
            Uri.BuildQueryString([
                grant_type = "client_credentials",
                client_id = ClientId,
                client_secret = ClientSecret,
                scope = "read write"
            ])
        )
    ]
)
```

## Troubleshooting Common Issues

### 400 Bad Request Errors

When token generation fails with 400 error :

```m
// Check your implementation against this working example
let
    GenerateToken = () =>
    let
        url = "https://login.example.com/oauth/token",
        body = "grant_type=client_credentials&client_id=YOUR_ID&client_secret=YOUR_SECRET",
        headers = [#"Content-Type" = "application/x-www-form-urlencoded"],
        response = Web.Contents(url, [
            Headers = headers,
            Content = Text.ToBinary(body)
        ]),
        jsonResponse = Json.Document(response),
        accessToken = jsonResponse[access_token]
    in
        accessToken
in
    GenerateToken()
```

**Common causes** :
- Incorrect content type header
- Missing or malformed body
- Invalid client credentials
- Wrong token endpoint URL

### Token Refresh Circular References

For APIs that return new refresh tokens with each request, you need to handle token updates :

```m
// This pattern avoids circular references by using parameters
let
    // Store tokens as parameters in Power BI
    CurrentRefreshToken = #"Refresh Token Parameter",
    
    // Function to get new tokens
    GetNewTokens = (refreshToken) =>
        let
            response = Json.Document(
                Web.Contents(
                    "https://api.example.com/token",
                    [
                        Content = Text.ToBinary(
                            "grant_type=refresh_token&refresh_token=" & refreshToken
                        )
                    ]
                )
            )
        in
            response,
    
    // Get data with token refresh
    GetData = () =>
        let
            newTokens = GetNewTokens(CurrentRefreshToken),
            data = Json.Document(
                Web.Contents(
                    "https://api.example.com/data",
                    [Headers = [#"Authorization" = "Bearer " & newTokens[access_token]]]
                )
            ),
            // Note: You'll need to manually update the refresh token parameter
            _ = #"Update Refresh Token"(newTokens[refresh_token])
        in
            data
in
    GetData()
```

### Service Refresh Failures

Common causes of refresh failures in Power BI service :

| Issue | Solution |
|-------|----------|
| **Separate token query** | Combine token and data in one query  |
| **Dynamic URLs with Text.From** | Use `Query` parameter instead of string concatenation  |
| **Rate limiting** | Implement pagination with delays |
| **Timeout** | Add `Timeout` parameter: `[Timeout = #duration(0,0,5,0)]`  |
| **POST in scheduled refresh** | Ensure POST is only for token, data should use GET |

## Best Practices Summary

### Do's
✓ **Combine in one query** - All steps in single query for service compatibility   
✓ **Use RelativePath and Query** - Instead of building URLs with string concatenation   
✓ **Store secrets in parameters** - Never hardcode client secrets   
✓ **Handle token expiry** - Implement retry logic with refresh tokens   
✓ **Test in Power BI Desktop first** - Verify before publishing   
✓ **Use ApiKeyName for API keys** - Secure storage of simple keys   

### Don'ts
✗ **Don't split token and data queries** - Causes service refresh failures   
✗ **Don't hardcode credentials** - Use parameters and data source settings  
✗ **Don't build URLs with & and ?** - Use Query parameter   
✗ **Don't ignore token expiry** - APIs typically expire tokens after 1 hour   
✗ **Don't use POST for data retrieval** - Only for token endpoints  

## Summary

**Key Points for Dynamic Headers and OAuth Tokens:**

1. **Two-step process** - Get token, then use it in Authorization header
2. **Single query required** - All steps combined for Power BI service 
3. **POST for tokens** - Use `Content` parameter to force POST 
4. **Bearer token format** - `"Bearer " & accessToken` in Authorization header
5. **Query parameters** - Use `Query` record, not string concatenation 
6. **Secure storage** - Use `ApiKeyName` or parameters for secrets 
7. **Error handling** - Implement retry logic for expired tokens 

**The Golden Rule:**
> Always keep token acquisition and API calls in the same M query, use the `Query` parameter for dynamic values, and never hardcode secrets. For Power BI service, single-query patterns are mandatory for scheduled refresh to work correctly .
