# Paginated API Responses in Power Query

## Overview

Paginated APIs return data in chunks (pages) rather than all at once. Handling pagination in Power Query is essential for retrieving complete datasets from REST APIs that limit response sizes. Different APIs use different pagination strategies, requiring tailored approaches.

```
Initial Request → Page 1 → Parse Response → Extract Next Token/URL → Page 2 → ... → Complete Data
                      ↓                         ↓
               Data for Page 1            Link/Token for Page 2
```

## Types of Pagination

| Type | Mechanism | Example |
|------|-----------|---------|
| **Offset/Limit** | `?offset=0&limit=100` | Traditional REST APIs |
| **Page/Per Page** | `?page=1&per_page=100` | Common in many APIs |
| **Cursor/Token** | `?cursor=abc123` | Modern APIs (Twitter, Slack) |
| **Next Link** | Link header or response field | GitHub, OData |
| **Keyset** | `?last_id=12345` | Efficient for large datasets |

## Helper Function: Table.GenerateByPage

The foundation for custom pagination is the `Table.GenerateByPage` helper function:

```m
// Table.GenerateByPage - Copy this into your query
let
    Table.GenerateByPage = (getNextPage as function) as table =>
        let
            listOfPages = List.Generate(
                () => getNextPage(null),            // First page
                (lastPage) => lastPage <> null,      // Stop when null
                (lastPage) => getNextPage(lastPage), // Get next page
                (lastPage) => lastPage                // Return page
            )
        in
            Table.Combine(listOfPages)
in
    Table.GenerateByPage
```

## Pattern 1: Offset/Limit Pagination

### Basic Implementation

```m
let
    // Configuration
    BaseUrl = "https://api.example.com/items",
    PageSize = 100,
    
    // Function to fetch a single page
    GetPage = (offset as number) as table =>
        let
            // Build URL with offset
            url = BaseUrl & "?offset=" & Text.From(offset) & "&limit=" & Text.From(PageSize),
            
            // Make request
            response = Web.Contents(url, [
                Headers = [Accept = "application/json"],
                Timeout = #duration(0,0,2,0)
            ]),
            
            // Parse JSON
            json = Json.Document(response),
            
            // Extract data array
            data = json[items]? ?? {},
            
            // Convert to table
            table = if List.IsEmpty(data) then null else Table.FromRecords(data)
        in
            table,
    
    // Get first page to determine total (if available)
    FirstPage = GetPage(0),
    
    // Get total count from response headers or metadata
    TotalCount = 1000,  // Extract from response or calculate
    
    // Calculate number of pages
    TotalPages = Number.RoundUp(TotalCount / PageSize),
    
    // Generate all pages
    PageNumbers = List.Numbers(0, TotalPages),
    AllPages = List.Transform(PageNumbers, each GetPage(_ * PageSize)),
    Combined = Table.Combine(AllPages)
in
    Combined
```

### Complete Offset/Limit Pattern

```m
let
    // Configuration
    BaseUrl = "https://api.example.com/orders",
    PageSize = 100,
    MaxPages = 50,  // Safety limit
    
    // Function to fetch a page with error handling
    FetchPage = (offset as number) as nullable table =>
        let
            url = BaseUrl & "?offset=" & Text.From(offset) & "&limit=" & Text.From(PageSize),
            
            response = try
                Web.Contents(url, [
                    Headers = [
                        Accept = "application/json",
                        #"User-Agent" = "Power-BI"
                    ],
                    Timeout = #duration(0,0,3,0)
                ])
            otherwise
                error "Failed to fetch page at offset " & Text.From(offset),
            
            json = try Json.Document(response) otherwise 
                error "Invalid JSON response",
            
            // Handle different response structures
            data = 
                if json[data]? <> null then json[data]
                else if json[items]? <> null then json[items]
                else if json[results]? <> null then json[results]
                else if json is list then json
                else {},
            
            table = if List.IsEmpty(data) then null else Table.FromRecords(data)
        in
            table,
    
    // Function to get total count (if available)
    GetTotalCount = () as number =>
        let
            url = BaseUrl & "?offset=0&limit=1",
            response = Web.Contents(url, [Headers = [Accept = "application/json"]]),
            json = Json.Document(response),
            total = 
                if json[total]? <> null then json[total]
                else if json[count]? <> null then json[count]
                else if json[pagination]?[total]? <> null then json[pagination][total]
                else 999999  // Unknown, will rely on empty page detection
        in
            total,
    
    // Try to get total (may fail)
    TotalRecords = try GetTotalCount() otherwise null,
    
    // Use Table.GenerateByPage for dynamic pagination
    Table.GenerateByPage = (getNextPage as function) as table =>
        let
            listOfPages = List.Generate(
                () => getNextPage(null),
                (lastPage) => lastPage <> null,
                (lastPage) => getNextPage(lastPage),
                (lastPage) => lastPage
            )
        in
            Table.Combine(listOfPages),
    
    // Main pagination logic
    AllData = Table.GenerateByPage((previous) =>
        let
            // Calculate next offset
            previousOffset = 
                if previous = null then 0
                else Value.Metadata(previous)[NextOffset]? ?? 0,
            
            // Stop if we've hit max pages
            currentPage = previousOffset / PageSize,
            shouldContinue = currentPage < MaxPages,
            
            // Fetch next page if continuing
            nextPage = if shouldContinue then
                let
                    pageData = FetchPage(previousOffset),
                    nextOffset = previousOffset + PageSize,
                    
                    // Check if we have data
                    result = if pageData = null or Table.IsEmpty(pageData) then
                        null
                    else
                        pageData meta [NextOffset = nextOffset]
                in
                    result
            else
                null
        in
            nextPage
    )
in
    AllData
```

## Pattern 2: Page/Per Page Pagination

```m
let
    BaseUrl = "https://api.example.com/products",
    PerPage = 100,
    
    // Fetch a single page
    FetchPage = (page as number) as table =>
        let
            url = BaseUrl & "?page=" & Text.From(page) & "&per_page=" & Text.From(PerPage),
            response = Web.Contents(url, [Headers = [Accept = "application/json"]]),
            json = Json.Document(response),
            
            // Extract data and pagination info
            data = json[data]? ?? json,
            
            // Get total pages from headers or response
            totalPages = try Number.FromText(Text.FromBinary(Value.Metadata(response)[#"x-total-pages"]?)) otherwise null,
            
            table = Table.FromRecords(data) meta [TotalPages = totalPages]
        in
            table,
    
    // Get first page to determine total pages
    FirstPage = FetchPage(1),
    TotalPages = Value.Metadata(FirstPage)[TotalPages]? ?? 1,
    
    // Fetch remaining pages
    PageNumbers = if TotalPages = null then {1} else {2..TotalPages},
    RemainingPages = List.Transform(PageNumbers, each FetchPage(_)),
    AllPages = {FirstPage} & RemainingPages,
    Combined = Table.Combine(AllPages)
in
    Combined
```

## Pattern 3: Cursor/Token Pagination

### Implementation with Continuation Tokens

```m
let
    BaseUrl = "https://api.example.com/messages",
    PageSize = 100,
    
    // Function to fetch page with cursor
    FetchPage = (cursor as nullable text) as table =>
        let
            // Build URL with cursor
            url = BaseUrl & "?limit=" & Text.From(PageSize) & 
                  (if cursor = null then "" else "&cursor=" & cursor),
            
            response = Web.Contents(url, [Headers = [Accept = "application/json"]]),
            json = Json.Document(response),
            
            // Extract data and next cursor
            data = json[data]? ?? json,
            nextCursor = json[next_cursor]? ?? json[cursor]? ?? json[paging]?[next]?,
            
            table = Table.FromRecords(data) meta [NextCursor = nextCursor]
        in
            table,
    
    // Table.GenerateByPage helper
    Table.GenerateByPage = (getNextPage as function) as table =>
        let
            listOfPages = List.Generate(
                () => getNextPage(null),
                (lastPage) => lastPage <> null,
                (lastPage) => getNextPage(lastPage),
                (lastPage) => lastPage
            )
        in
            Table.Combine(listOfPages),
    
    // Main pagination
    AllMessages = Table.GenerateByPage((previous) =>
        let
            // Get cursor from previous page metadata
            previousCursor = if previous = null then null 
                            else Value.Metadata(previous)[NextCursor]?,
            
            // Fetch next page (stop if cursor is null/empty)
            nextPage = if previousCursor = null and previous <> null then
                null  // No more pages
            else
                let
                    pageData = FetchPage(previousCursor),
                    // Check if we got data
                    result = if Table.IsEmpty(pageData) then null else pageData
                in
                    result
        in
            nextPage
    )
in
    AllMessages
```

### Twitter/X API v2 Example

```m
let
    BearerToken = "YOUR_BEARER_TOKEN",
    BaseUrl = "https://api.twitter.com/2/users/123456789/tweets",
    
    // Fetch page with pagination token
    FetchPage = (nextToken as nullable text) as table =>
        let
            // Build query
            query = [
                max_results = "100",
                "tweet.fields" = "created_at,public_metrics"
            ],
            
            // Add pagination token if present
            fullQuery = if nextToken = null then query
                       else Record.Combine({query, [pagination_token = nextToken]}),
            
            // Make request
            response = Web.Contents(
                BaseUrl,
                [
                    Headers = [
                        Authorization = "Bearer " & BearerToken,
                        Accept = "application/json"
                    ],
                    Query = fullQuery
                ]
            ),
            
            json = Json.Document(response),
            
            // Extract data and next token
            data = json[data],
            meta = json[meta],
            nextToken = meta[next_token]?,
            
            table = Table.FromRecords(data) meta [NextToken = nextToken]
        in
            table,
    
    // Use Table.GenerateByPage
    Table.GenerateByPage = ...,
    
    AllTweets = Table.GenerateByPage((previous) =>
        let
            prevToken = if previous = null then null 
                       else Value.Metadata(previous)[NextToken]?,
            nextPage = if prevToken = null and previous <> null then
                null
            else
                FetchPage(prevToken)
        in
            nextPage
    )
in
    AllTweets
```

## Pattern 4: Next Link Pagination (GitHub-style)

```m
let
    BaseUrl = "https://api.github.com/repos/microsoft/powerbi-desktop/issues",
    
    // Function to fetch page and extract next link from headers
    FetchPage = (url as text) as table =>
        let
            response = Web.Contents(url, [
                Headers = [
                    Accept = "application/vnd.github.v3+json",
                    #"User-Agent" = "Power-BI"
                ]
            ]),
            
            // Parse JSON data
            json = Json.Document(response),
            data = Table.FromRecords(json),
            
            // Extract Link header for pagination
            metadata = Value.Metadata(response),
            linkHeader = metadata[#"Link"]?,
            
            // Parse Link header to find next URL
            nextUrl = 
                if linkHeader = null then null
                else
                    let
                        // Parse format: <https://api.github.com/issues?page=2>; rel="next"
                        parts = Text.Split(linkHeader, ", "),
                        nextPart = List.Select(parts, each Text.Contains(_, "rel=\"next\"")){0}?,
                        urlPart = if nextPart = null then null
                                  else Text.BetweenDelimiters(nextPart, "<", ">")
                    in
                        urlPart,
            
            // Store next URL in metadata
            result = data meta [NextUrl = nextUrl]
        in
            result,
    
    // Use Table.GenerateByPage
    Table.GenerateByPage = ...,
    
    AllIssues = Table.GenerateByPage((previous) =>
        let
            // First page uses BaseUrl, subsequent use NextUrl
            currentUrl = if previous = null then BaseUrl
                        else Value.Metadata(previous)[NextUrl]?,
            
            nextPage = if currentUrl = null then null
                      else FetchPage(currentUrl)
        in
            nextPage
    )
in
    AllIssues
```

## Pattern 5: Keyset Pagination (Last ID)

```m
let
    BaseUrl = "https://api.example.com/events",
    PageSize = 1000,
    
    // Fetch page after last ID
    FetchPage = (lastId as nullable number) as table =>
        let
            query = [limit = Text.From(PageSize)],
            fullQuery = if lastId = null then query
                       else Record.Combine({query, [after = Text.From(lastId)]}),
            
            response = Web.Contents(BaseUrl, [
                Headers = [Accept = "application/json"],
                Query = fullQuery
            ]),
            
            json = Json.Document(response),
            data = json[data]? ?? json,
            
            // Get last ID from this batch
            table = Table.FromRecords(data),
            lastRow = Table.Last(table),
            nextId = if Table.IsEmpty(table) then null
                    else Record.Field(lastRow, "id"),
            
            result = table meta [NextId = nextId]
        in
            result,
    
    // Use Table.GenerateByPage
    Table.GenerateByPage = ...,
    
    AllEvents = Table.GenerateByPage((previous) =>
        let
            prevId = if previous = null then null
                    else Value.Metadata(previous)[NextId]?,
            nextPage = if prevId = null and previous <> null then
                null
            else
                FetchPage(prevId)
        in
            nextPage
    )
in
    AllEvents
```

## Advanced: Concurrent Pagination

For APIs that support concurrent requests, you can fetch pages in parallel:

```m
let
    BaseUrl = "https://api.example.com/items",
    PageSize = 100,
    
    // Function to fetch a single page
    FetchPage = (page as number) as table =>
        let
            url = BaseUrl & "?page=" & Text.From(page) & "&per_page=" & Text.From(PageSize),
            response = Web.Contents(url),
            json = Json.Document(response),
            data = json[items]? ?? json
        in
            Table.FromRecords(data),
    
    // Get first page to determine total pages
    FirstPage = FetchPage(1),
    // Assume we got total pages from metadata
    TotalPages = 10,
    
    // Create page list
    PageNumbers = {2..TotalPages},
    
    // Fetch remaining pages in parallel (if supported)
    // Note: Not all APIs support concurrent requests
    RemainingPages = List.Transform(PageNumbers, each FetchPage(_)),
    
    // Combine all pages
    AllPages = {FirstPage} & RemainingPages,
    Combined = Table.Combine(AllPages)
in
    Combined
```

## Error Handling and Retries

```m
let
    // Robust fetch with retries
    FetchWithRetry = (url as text, retryCount as number) as table =>
        let
            Result = try
                let
                    response = Web.Contents(url, [
                        Headers = [Accept = "application/json"],
                        Timeout = #duration(0,0,3,0)
                    ]),
                    json = Json.Document(response)
                in
                    Table.FromRecords(json)
            otherwise
                if retryCount > 0 then
                    // Exponential backoff
                    Duration.Sleep(#duration(0,0,0,10 * (3 - retryCount))),
                    @FetchWithRetry(url, retryCount - 1)
                else
                    error "Failed after retries: " & url
        in
            Result,
    
    // Usage in pagination
    FetchPage = (cursor) =>
        let
            url = BaseUrl & "?cursor=" & (cursor ?? ""),
            data = FetchWithRetry(url, 3) meta [NextCursor = ExtractCursor(url)]
        in
            data
in
    ...
```

## Common Pagination Patterns by API

| API | Pattern | Implementation |
|-----|---------|----------------|
| **GitHub** | Link header with next URL | Parse `Link` header, extract next URL |
| **Twitter/X** | Cursor token in response | Extract `next_token` from meta |
| **Salesforce** | Next URL in response | Parse `nextRecordsUrl` field |
| **Stripe** | Pagination with `has_more` flag | Continue while `has_more` is true |
| **Google APIs** | Page token in response | Use `nextPageToken` |
| **OData** | `@odata.nextLink` in JSON | Follow nextLink URL |

## Best Practices Summary

### Do's
✓ **Use Table.GenerateByPage** - Standard pattern for pagination
✓ **Store state in metadata** - Use `meta` to track pagination tokens
✓ **Implement timeout** - Prevent hanging on slow responses
✓ **Add retry logic** - Handle transient failures
✓ **Respect rate limits** - Add delays between requests if needed
✓ **Test with small page sizes** - Verify pagination works
✓ **Handle empty responses** - Stop when no data returned

### Don'ts
✗ **Don't hardcode page counts** - APIs may change limits
✗ **Don't ignore rate limits** - Can get your IP banned
✗ **Don't assume all APIs use same pattern** - Check documentation
✗ **Don't fetch unlimited pages** - Add max page safety limit
✗ **Don't ignore response headers** - May contain pagination info

## Summary

**Key Points for Paginated API Responses:**

| Aspect | Recommendation |
|--------|----------------|
| **Foundation** | Use `Table.GenerateByPage` helper function |
| **State Tracking** | Store pagination tokens in `meta` |
| **Stopping Condition** | Stop when no data or next token is null |
| **Error Handling** | Implement retries with exponential backoff |
| **Rate Limiting** | Add delays and respect `Retry-After` headers |
| **Testing** | Verify with small page sizes first |
| **Documentation** | Always check API pagination docs |

**The Golden Rule:**
> Every paginated API is different. Understand the specific pattern (offset, cursor, next link) and implement accordingly. Store pagination state in metadata and always include stopping conditions to prevent infinite loops.
