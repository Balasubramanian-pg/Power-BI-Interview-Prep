# SELECTEDVALUE with Multiple Blanks in Power BI

## Overview

`SELECTEDVALUE` is a DAX function that returns the value when a column has exactly one distinct value; otherwise, it returns an alternate result (or BLANK by default).

```
SELECTEDVALUE(<column>[, <alternateResult>])
```

- Returns the **single value** from the column when filtered to one distinct value
- Returns **alternateResult** (or BLANK) when zero or multiple values are selected
- Evaluates in the current filter context

## The Multiple Blanks Problem

### What Are Multiple Blanks?

In Power BI, BLANK is a special value representing:
- **Null** in source data
- **Missing values** from outer joins
- **Calculation errors** (division by zero)
- **Rows with no data**

The problem: **Multiple BLANK rows are treated as a single distinct value** for most DAX functions, but `SELECTEDVALUE` counts them as ONE distinct value.

### The Behavioral Quirk

```
Scenario: A column with:
- 100 BLANK values
- No other values

DISTINCTCOUNT(Column) = 1 (the BLANK value)
SELECTEDVALUE(Column) = BLANK (single value found)
Result: ✓ Intuitive

Scenario: A column with:
- 100 BLANK values
- 50 "Red" values
- 50 "Blue" values

DISTINCTCOUNT(Column) = 3 (BLANK, Red, Blue)
SELECTEDVALUE(Column) = alternateResult (multiple distinct values)
Result: ✓ Intuitive

Scenario: A column with:
- 100 BLANK values
- 50 "Red" values

DISTINCTCOUNT(Column) = 2 (BLANK, Red)
SELECTEDVALUE(Column) = alternateResult (multiple distinct values)
Result: ✓ Intuitive

Scenario: A column with:
- 100 BLANK values
- 100 empty strings ("")

DISTINCTCOUNT(Column) = 2 (BLANK, "")
SELECTEDVALUE(Column) = alternateResult (multiple distinct values)
Result: ✓ Intuitive

Scenario: A slicer with:
- BLANK selected (from list containing only BLANK and Red)
- Selection includes BLANK + Red

SELECTEDVALUE(Column) = alternateResult
Result: ✓ Intuitive
```

So what's the actual problem? The issue arises when **BLANK appears in both the data and as a representation of "no selection"** .

## Common Scenarios Where Issues Arise

### Scenario 1: Slicer with BLANK Option

```
Slicer shows: [All], [BLANK], [Red], [Blue]
User selects: [BLANK] (meaning show rows with nulls)
```

```dax
Selected Color = SELECTEDVALUE(Product[Color])
// Returns: BLANK
// Problem: Can't distinguish between "BLANK selected" and "nothing selected"
```

### Scenario 2: Multiple Sources of BLANK

```dax
Color Selection = 
VAR SelectedColor = SELECTEDVALUE(DimProduct[Color])
// Is BLANK because:
// a) User selected the BLANK value in slicer
// b) User made no selection
// c) Multiple colors are selected (including BLANK)
// d) All visible rows have BLANK color
```

### Scenario 3: Dynamic Titles with BLANK

```dax
Dynamic Title = 
"Showing: " & 
SELECTEDVALUE(DimProduct[Color], "Multiple Colors")
// Returns "Showing: Multiple Colors" when BLANK selected
// User expected "Showing: (Blank)"
```

## Detecting BLANK Selection

### Pattern 1: HASONEVALUE + VALUES

```dax
Color Selection with Blank Detection = 
IF(
    HASONEVALUE(DimProduct[Color]),
    // Single value exists (including BLANK)
    IF(
        ISBLANK(VALUES(DimProduct[Color])),
        "Blank Selected",
        VALUES(DimProduct[Color])
    ),
    "Multiple or No Selection"
)
```

### Pattern 2: ISFILTERED + ISBLANK

```dax
Blank-Aware Selection = 
VAR ColumnFiltered = ISFILTERED(DimProduct[Color])
VAR CurrentValue = SELECTEDVALUE(DimProduct[Color])
VAR BlankSelected = 
    ColumnFiltered && 
    ISBLANK(CurrentValue) && 
    CALCULATE(COUNTROWS(DimProduct), DimProduct[Color] = BLANK()) > 0
RETURN
    SWITCH(
        TRUE(),
        BlankSelected, "Blank Selected",
        ColumnFiltered && NOT ISBLANK(CurrentValue), CurrentValue,
        ColumnFiltered && ISBLANK(CurrentValue), "No Value (All Blank)",
        "No Filter"
    )
```

### Pattern 3: Count Visible BLANKs

```dax
Blank Detection = 
VAR VisibleColors = VALUES(DimProduct[Color])
VAR BlankCount = 
    CALCULATE(
        COUNTROWS(DimProduct),
        DimProduct[Color] = BLANK()
    )
VAR SelectedCount = COUNTROWS(VisibleColors)
RETURN
    IF(
        SelectedCount = 1 && 
        BLANK() IN VisibleColors &&
        BlankCount > 0,
        "BLANK explicitly selected",
        "Other"
    )
```

## Advanced Patterns

### Pattern 4: Three-State Selection Handling

```dax
Color Selection State = 
VAR SelectedColor = SELECTEDVALUE(DimProduct[Color])
VAR HasSelection = ISFILTERED(DimProduct[Color])
VAR VisibleValues = VALUES(DimProduct[Color])
VAR ValueCount = COUNTROWS(VisibleValues)
VAR BlankExists = BLANK() IN VisibleValues

RETURN
    SWITCH(
        TRUE(),
        // Case 1: No filter
        NOT HasSelection, "No Filter Applied",
        
        // Case 2: Single value (including BLANK)
        ValueCount = 1,
            IF(
                BlankExists,
                "Single: BLANK",
                "Single: " & SelectedColor
            ),
        
        // Case 3: Multiple values
        ValueCount > 1,
            IF(
                BlankExists && ValueCount = 2,
                "BLANK + One Value",
                "Multiple Values" & IF(BlankExists, " (includes BLANK)", "")
            ),
        
        // Default
        "Unknown"
    )
```

### Pattern 5: Safe Value Extraction

```dax
Safe Color Value = 
// Returns: 
// - Actual color if single color selected (excluding BLANK)
// - "Blank" if single BLANK selected
// - "Multiple" otherwise
VAR VisibleColors = VALUES(DimProduct[Color])
VAR ColorCount = COUNTROWS(VisibleColors)
VAR BlankPresent = BLANK() IN VisibleColors
VAR NonBlankCount = 
    COUNTROWS(
        FILTER(
            VisibleColors,
            NOT ISBLANK([Color])
        )
    )

RETURN
    IF(
        ColorCount = 1 && NonBlankCount = 1,
        // Exactly one non-blank value
        FORMAT(SELECTEDVALUE(DimProduct[Color]), "0"),
        IF(
            ColorCount = 1 && BlankPresent && NonBlankCount = 0,
            // Only BLANK selected
            "Blank",
            IF(
                ColorCount > 1 && BlankPresent && NonBlankCount = 1,
                // BLANK + one color selected
                "BLANK + " & 
                    FORMAT(
                        MINX(
                            FILTER(VisibleColors, NOT ISBLANK([Color])),
                            [Color]
                        ),
                        "0"
                    ),
                // Multiple or other
                "Multiple (" & ColorCount & " values)"
            )
        )
    )
```

## Matrix Visual Considerations

### Scenario: Row and Column Contexts

In matrix visuals, `SELECTEDVALUE` behaves differently depending on context:

```dax
Matrix Measure = 
VAR CurrentColor = SELECTEDVALUE(DimProduct[Color])
VAR RowColor = SELECTEDVALUE(DimProduct[Color])  // Same!
RETURN
    IF(
        ISBLANK(CurrentColor),
        "No Color",
        CurrentColor
    )
```

### Pattern 6: Context-Aware Blank Handling

```dax
Matrix Color Display = 
// Distinguishes between:
// - Row has BLANK color
// - Row has no data
// - No color selected in slicer
VAR CurrentColor = SELECTEDVALUE(DimProduct[Color])
VAR HasData = NOT ISBLANK([Total Sales])
VAR ColorInScope = 
    COUNTROWS(
        FILTER(
            DimProduct,
            DimProduct[Color] = CurrentColor
        )
    ) > 0

RETURN
    IF(
        HasData,
        IF(
            ISBLANK(CurrentColor),
            "Uncategorized",
            CurrentColor
        ),
        BLANK()
    )
```

## Performance Implications

### BLANK Cardinality Impact

BLANK values affect cardinality estimates:

```dax
// BLANK is treated as one distinct value
Cardinality = 
VAR DistinctColors = DISTINCTCOUNT(DimProduct[Color])
VAR BlankCount = 
    CALCULATE(
        COUNTROWS(DimProduct),
        DimProduct[Color] = BLANK()
    )
RETURN
    "Distinct: " & DistinctColors & 
    " (including BLANK), " &
    "Blank rows: " & BlankCount
```

### SELECTEDVALUE Performance

- `SELECTEDVALUE` is **optimized** - similar to `HASONEVALUE + VALUES`
- No significant performance cost
- Main overhead comes from BLANK handling logic you add

## Best Practices Summary

### Do's
✓ **Check ISFILTERED first** - Know if column is actually filtered
✓ **Use HASONEVALUE** - Confirm single value before extracting
✓ **Handle BLANK explicitly** - Don't assume BLANK means "no selection"
✓ **Document behavior** - Team knows how blanks are handled
✓ **Test with slicers** - Verify behavior with BLANK selected

### Don'ts
✗ **Don't assume BLANK = no selection** - Could be explicit BLANK
✗ **Don't use in RLS without testing** - BLANK behaves differently
✗ **Don't ignore cardinality** - Multiple blanks count as one value
✗ **Don't forget empty strings** - "" ≠ BLANK, affects counts
✗ **Don't use for validation alone** - Combine with other functions

## Decision Matrix

```
Situation: Need to get selected value with blanks
        ↓
Is column filtered?
    ↓                       ↓
   No                      Yes
    ↓                       ↓
Return default         How many distinct values?
    ↓                    ↓           ↓        ↓
  Done                 1          >1       Unknown
                        ↓           ↓        ↓
                Is it BLANK?   Return     Check if
                    ↓          default    BLANK in
                Yes      No               selection
                 ↓        ↓                  ↓
          Return     Return            Handle
          "Blank"    Value            accordingly
```

## Complete Example

```dax
Robust Color Selection = 
// Production-ready pattern for BLANK handling
VAR ColumnName = "Color"
VAR ColumnFiltered = ISFILTERED(DimProduct[Color])
VAR VisibleValues = VALUES(DimProduct[Color])
VAR ValueCount = COUNTROWS(VisibleValues)
VAR BlankInSelection = BLANK() IN VisibleValues
VAR NonBlankValues = 
    FILTER(
        VisibleValues,
        NOT ISBLANK(DimProduct[Color])
    )
VAR NonBlankCount = COUNTROWS(NonBlankValues)

RETURN
    SWITCH(
        TRUE(),
        // No filter at all
        NOT ColumnFiltered, 
            "No filter on " & ColumnName,
        
        // Exactly one value (could be BLANK)
        ValueCount = 1,
            IF(
                BlankInSelection,
                "BLANK (explicitly selected)",
                FORMAT(SELECTEDVALUE(DimProduct[Color]), "0")
            ),
        
        // Multiple values including BLANK
        ValueCount > 1 && BlankInSelection,
            "Multiple (" & NonBlankCount & " colors + BLANK)",
        
        // Multiple values no BLANK
        ValueCount > 1,
            "Multiple (" & ValueCount & " colors)",
        
        // Should never happen
        "Unexpected state"
    )
```

## Summary

**Key Points for SELECTEDVALUE with Multiple Blanks:**

1. **BLANK is a distinct value** - Counts as one in DISTINCTCOUNT
2. **SELECTEDVALUE returns BLANK** when single BLANK selected
3. **Can't distinguish** between "BLANK selected" and "no selection" without additional checks
4. **Use ISFILTERED** to know if column is actually filtered
5. **Check HASONEVALUE** before extracting with VALUES
6. **Test all scenarios** - Slicer selections, visual interactions, cross-filtering
7. **Document behavior** - Make it clear how blanks are interpreted

**The Golden Rule:**
> BLANK is both a valid data value AND a representation of "no value". Always add explicit checks to distinguish between these cases when the distinction matters.
