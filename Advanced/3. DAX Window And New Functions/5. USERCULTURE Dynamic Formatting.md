# USERCULTURE and Dynamic Formatting in Power BI

## Overview

`USERCULTURE` is a DAX function that returns the locale (language code-country code) for the current user, determined by the operating system, browser settings, or Power BI service settings .

```
USERCULTURE()
```

- Takes no parameters
- Returns locale as a string (e.g., "en-US", "de-DE", "fr-FR")
- **Currently supported only in Power BI Premium per capacity, Power BI Premium per user, and Power BI Embedded** 

## How User Culture Is Determined

### Power BI Service
- Locale is determined by **Settings > Language > Language Settings**
- Default is determined by the user's browser language setting 

### Power BI Desktop
- Locale is determined by the operating system language settings 

## Dynamic Formatting with USERCULTURE

### Pattern 1: Format Function with Locale Parameter

The `FORMAT` function accepts an optional third parameter for locale :

```dax
FORMAT(TODAY(), "dddd", USERCULTURE())
```

This returns the current day name in the user's language:

| Locale | Formatted Weekday |
|--------|-------------------|
| de-DE | Dienstag |
| en-US | Tuesday |
| es-ES_tradnl | martes |
| fr-FR | mardi |
| it-IT | martedì |
| nl-NL | dinsdag |
| pl-PL | wtorek |
| pt-BR | terça-feira |
| ru-RU | вторник |
| zh-CN | 星期二 |

### Pattern 2: Number Formatting

```dax
Formatted Sales = 
FORMAT(
    SUM(Sales[Amount]), 
    "Currency", 
    USERCULTURE()
)
```

This formats currency based on user locale:
- **en-US**: $1,234.56
- **de-DE**: 1.234,56 €
- **fr-FR**: 1 234,56 €
- **ja-JP**: ¥1,234

### Pattern 3: Conditional Logic with SWITCH

For custom translations not covered by FORMAT's built-in locale support :

```dax
Dynamic Title = 
SWITCH(
    USERCULTURE(),
    "de-DE", "Umsatz nach Produkt",
    "fr-FR", "Ventes par produit",
    "es-ES", "Ventas por producto",
    "Sales by Product"  // Default
)
```

### Pattern 4: Lookup Table Approach

For managing many translations, use a lookup table :

```
Translation Table:
| Culture | ObjectID | TranslatedText |
|---------|----------|----------------|
| en-US   | Title1   | Sales by Product |
| de-DE   | Title1   | Umsatz nach Produkt |
| fr-FR   | Title1   | Ventes par produit |
```

```dax
Translated Title = 
CALCULATE(
    SELECTEDVALUE(Translations[TranslatedText]),
    Translations[ObjectID] = "Title1",
    Translations[Culture] = USERCULTURE()
)
```

### Pattern 5: Date Formatting

```dax
Formatted Date = 
VAR SelectedDate = SELECTEDVALUE('Date'[Date])
RETURN
    SelectedDate & " - " &
    FORMAT(SelectedDate, "dddd, MMMM d, yyyy", USERCULTURE())
```

## Dynamic Visual Titles with USERCULTURE

### Step 1: Create a Title Measure

```dax
Dynamic Chart Title = 
SWITCH(
    USERCULTURE(),
    "de-DE", "Verkaufstrend nach Monat",
    "fr-FR", "Tendance des ventes par mois",
    "es-ES", "Tendencia de ventas por mes",
    "Sales Trend by Month"  // Default (English)
)
```

### Step 2: Apply to Visual Title

1. Select the visual
2. In **Format** pane > **General** > **Title**
3. Turn Title **On**
4. Click the **fx** (conditional formatting) button next to Title text 
5. Select the measure as the title field
6. Click **OK** 

### Important Requirements

- The field used for title must be **string data type** 
- Measures returning numbers or dates are **not supported** for dynamic titles 
- For DirectQuery, the measure must be defined in the model, not created locally in the report 

## Considerations and Limitations

### Import Mode Behavior

When used in calculated tables and calculated columns in **Import mode**, the result is statically determined during refresh and **will not vary at query time** .

For managed refreshes (scheduled or interactive), locale is not based on the user's browser but uses an **invariant locale** (can be overridden via XMLA endpoint) .

### DirectQuery Mode Behavior

In **DirectQuery mode**, the result is determined by the language specified in Language Settings in the Power BI service .

The same calculated table/column can return different results depending on each user's browser language settings .

### Live Connect Reports

In Live Connect reports, `USERCULTURE` **may not return the correct user locale** when called from a report measure expression .

### Scope Limitations

`USERCULTURE` returns the correct user locale when used in object expressions called **from within the model**:
- Measures
- Row-level security (RLS) expressions
- Calculation items 

It may **not** return the correct user locale when used in expressions from **outside the model**:
- Queries
- Live-connect report measures 

### Field Parameters Integration

When combined with Field Parameters, `USERCULTURE` can reliably translate dynamic visualization titles and captions when used in measure and RLS expressions within the same model .

## Practical Examples

### Example 1: Multi-Language Measure

```dax
Sales KPI = 
VAR CurrentCulture = USERCULTURE()
VAR SalesValue = [Total Sales]
VAR TargetValue = [Sales Target]
VAR Variance = SalesValue - TargetValue
VAR PercentVariance = DIVIDE(Variance, TargetValue)

RETURN
SWITCH(
    TRUE(),
    CurrentCulture = "de-DE",
        "Umsatz: " & FORMAT(SalesValue, "Currency", "de-DE") &
        " | Ziel: " & FORMAT(TargetValue, "Currency", "de-DE") &
        " | Abweichung: " & FORMAT(PercentVariance, "Percent", "de-DE"),
    
    CurrentCulture = "fr-FR",
        "Ventes: " & FORMAT(SalesValue, "Currency", "fr-FR") &
        " | Objectif: " & FORMAT(TargetValue, "Currency", "fr-FR") &
        " | Écart: " & FORMAT(PercentVariance, "Percent", "fr-FR"),
    
    // Default English
    "Sales: " & FORMAT(SalesValue, "Currency", "en-US") &
    " | Target: " & FORMAT(TargetValue, "Currency", "en-US") &
    " | Variance: " & FORMAT(PercentVariance, "Percent", "en-US")
)
```

### Example 2: Dynamic Axis Labels

```dax
Month Name Localized = 
SWITCH(
    USERCULTURE(),
    "de-DE", FORMAT('Date'[Date], "MMMM", "de-DE"),
    "fr-FR", FORMAT('Date'[Date], "MMMM", "fr-FR"),
    "es-ES", FORMAT('Date'[Date], "MMMM", "es-ES"),
    FORMAT('Date'[Date], "MMMM")  // Default
)
```

### Example 3: RLS with Culture Awareness

```dax
[Product Language] = 
SWITCH(
    USERCULTURE(),
    "de-DE", "German",
    "fr-FR", "French",
    "es-ES", "Spanish",
    "English"
)
```

## Summary

**Key Points for USERCULTURE Dynamic Formatting:**

1. **Returns user locale** as string based on OS, browser, or service settings
2. **Premium capacity required** - not available in shared capacity
3. **Use with FORMAT** - pass USERCULTURE() as third parameter for locale-aware formatting
4. **Dynamic titles** - create string measures and apply via conditional formatting (fx)
5. **SWITCH for custom translations** - when built-in locale support isn't sufficient
6. **Lookup tables for scale** - manage translations in model for larger implementations
7. **Import mode limitation** - static during refresh, not query-time dynamic
8. **Live Connect caution** - may not return correct locale

**Best Practice Checklist:**
- [ ] Verify Premium capacity/licensing
- [ ] Use in measures (not calculated columns) for dynamic results
- [ ] Test with multiple user personas/locales
- [ ] Consider lookup tables for extensive translations
- [ ] Validate in target deployment environment (Import vs. DirectQuery)
