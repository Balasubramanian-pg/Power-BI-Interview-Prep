# Error Handling in Folder Loops in Power Query

## Overview

Processing multiple files from folders is common in Power Query, but it introduces significant error-handling challenges. A single corrupt file, permission issue, or unexpected format can break the entire refresh. Robust error handling in folder loops ensures that individual file failures don't stop the entire process.

```
Folder Contents → Loop Through Files → Process Each File → Combine Results
       ↓                  ↓                    ↓                ↓
  Many files        Try/Catch per        Success or        Continue despite
                     file                Error record       individual failures
```

## The Problem: One Bad File Breaks Everything

### Typical Vulnerable Pattern

```m
// VULNERABLE - One bad file breaks the whole refresh
let
    Source = Folder.Files("C:\Data\MonthlyReports"),
    ExcelFiles = Table.SelectRows(Source, each [Extension] = ".xlsx"),
    
    // If ANY file fails, entire query fails
    ProcessFile = (fileBinary as binary) as table =>
        let
            Workbook = Excel.Workbook(fileBinary, null, true),
            Sheet = Workbook{[Item="Sheet1",Kind="Sheet"]}[Data],
            Promoted = Table.PromoteHeaders(Sheet)
        in
            Promoted,
    
    WithData = Table.AddColumn(ExcelFiles, "Data", each ProcessFile([Content]), type table),
    Combined = Table.Combine(WithData[Data])
in
    Combined
```

## Pattern 1: Try-Otherwise per File

### Basic Error Isolation

```m
let
    Source = Folder.Files("C:\Data\MonthlyReports"),
    ExcelFiles = Table.SelectRows(Source, each [Extension] = ".xlsx"),
    
    // Safe file processor with error handling
    SafeProcessFile = (fileBinary as binary, fileName as text) as table =>
        try
            let
                Workbook = Excel.Workbook(fileBinary, null, true),
                Sheet = Workbook{[Item="Sheet1",Kind="Sheet"]}[Data],
                Promoted = Table.PromoteHeaders(Sheet)
            in
                Promoted
        otherwise
            // Return error table instead of failing
            #table(
                {"Error", "FileName", "ErrorMessage"},
                {{"Processing failed", fileName, "Could not read Excel file"}}
            ),
    
    // Apply safely
    WithData = Table.AddColumn(
        ExcelFiles, 
        "Data", 
        each SafeProcessFile([Content], [Name]),
        type table
    ),
    
    // Filter out error tables if desired, or keep them
    GoodData = Table.SelectRows(WithData, each Table.ColumnNames([Data]) <> {"Error"}),
    Combined = if Table.IsEmpty(GoodData) then 
        #table({}, {})  // Empty table if all failed
    else
        Table.Combine(GoodData[Data])
in
    Combined
```

### Detailed Error Capture

```m
let
    Source = Folder.Files("C:\Data\MonthlyReports"),
    ExcelFiles = Table.SelectRows(Source, each [Extension] = ".xlsx"),
    
    // Comprehensive error handling
    ProcessWithDetails = (fileBinary as binary, fileRecord as record) as table =>
        let
            result = try
                let
                    Workbook = Excel.Workbook(fileBinary, null, true),
                    Sheet = try 
                        Workbook{[Item="Sheet1",Kind="Sheet"]}[Data] 
                    otherwise
                        Workbook{[Kind="Sheet"]}[0][Data],  // Fallback to first sheet
                    
                    Promoted = Table.PromoteHeaders(Sheet, [PromoteAllScalars=true])
                in
                    [
                        Success = true,
                        Data = Promoted,
                        Error = null,
                        ErrorType = null
                    ]
            otherwise
                let
                    errorDetails = Diagnostics.TraceValue()  // Capture error details
                in
                    [
                        Success = false,
                        Data = null,
                        Error = "Processing failed",
                        ErrorType = "Excel format error",
                        FileName = fileRecord[Name],
                        FilePath = fileRecord[Folder Path] & fileRecord[Name],
                        FileSize = fileRecord[Size],
                        Timestamp = DateTime.LocalNow()
                    ]
        in
            result,
    
    // Apply with detailed capture
    Results = Table.AddColumn(ExcelFiles, "ProcessResult", each ProcessWithDetails([Content], _)),
    
    // Split success and failures
    Successes = Table.SelectRows(Results, each [ProcessResult][Success] = true),
    Failures = Table.SelectRows(Results, each [ProcessResult][Success] = false),
    
    // Extract data from successes
    SuccessData = if Table.IsEmpty(Successes) then
        #table({}, {})
    else
        let
            WithData = Table.AddColumn(Successes, "FileData", each [ProcessResult][Data]),
            Combined = Table.Combine(WithData[FileData])
        in
            Combined,
    
    // Log failures (can write to a table or file)
    FailureLog = if Table.IsEmpty(Failures) then
        "No failures"
    else
        let
            LogTable = Table.FromRecords(Failures[ProcessResult])
        in
            LogTable  // This could be loaded as a separate table in your model
in
    SuccessData
```

## Pattern 2: List.Generate for Granular Control

### Controlled Loop with Error Handling

```m
let
    Source = Folder.Files("C:\Data\MonthlyReports"),
    ExcelFiles = Table.SelectRows(Source, each [Extension] = ".xlsx"),
    
    // Convert to list of records for List.Generate
    FileRecords = Table.ToRecords(ExcelFiles),
    FileCount = List.Count(FileRecords),
    
    // Process with List.Generate for maximum control
    ProcessFiles = List.Generate(
        () => [Index = 0, Results = {}, Errors = {}],
        each [Index] < FileCount,
        each 
            let
                currentFile = FileRecords{[Index]},
                
                // Process file with try
                fileResult = try
                    let
                        Workbook = Excel.Workbook(currentFile[Content], null, true),
                        Sheet = Workbook{[Item="Sheet1",Kind="Sheet"]}[Data],
                        Promoted = Table.PromoteHeaders(Sheet)
                    in
                        [
                            Success = true,
                            Data = Promoted,
                            FileName = currentFile[Name]
                        ]
                otherwise
                    [
                        Success = false,
                        Data = null,
                        FileName = currentFile[Name],
                        Error = "Processing failed"
                    ],
                
                // Append to appropriate list
                newResults = if fileResult[Success] then 
                    [Results] & {fileResult[Data]} 
                else 
                    [Results],
                    
                newErrors = if not fileResult[Success] then
                    [Errors] & {fileResult}
                else
                    [Errors]
            in
                [Index = [Index] + 1, Results = newResults, Errors = newErrors],
        each [Results, Errors]
    ),
    
    // Get final results
    FinalState = List.Last(ProcessFiles),
    AllData = if List.IsEmpty(FinalState[Results]) then
        #table({}, {})
    else
        Table.Combine(FinalState[Results]),
    
    // Error summary
    ErrorSummary = if List.IsEmpty(FinalState[Errors]) then
        "No errors"
    else
        Table.FromRecords(FinalState[Errors])
in
    AllData
```

## Pattern 3: Batch Processing with Progress Tracking

### Process in Batches with Error Tolerance

```m
let
    Source = Folder.Files("C:\Data\MonthlyReports"),
    ExcelFiles = Table.SelectRows(Source, each [Extension] = ".xlsx"),
    
    // Configuration
    BatchSize = 10,
    MaxErrors = 5,
    
    // Convert to list
    FileList = Table.ToRecords(ExcelFiles),
    TotalFiles = List.Count(FileList),
    
    // Split into batches
    Batches = List.Split(FileList, BatchSize),
    
    // Process a single batch
    ProcessBatch = (batch as list, batchIndex as number) as record =>
        let
            results = List.Transform(
                batch,
                each 
                    let
                        file = _,
                        result = try
                            let
                                Workbook = Excel.Workbook(file[Content], null, true),
                                Sheet = Workbook{[Item="Sheet1",Kind="Sheet"]}[Data],
                                Promoted = Table.PromoteHeaders(Sheet)
                            in
                                [Success = true, Data = Promoted, File = file[Name]]
                        otherwise
                            [Success = false, Data = null, File = file[Name], Error = "Failed"]
                    in
                        result
            ),
            
            successes = List.Select(results, each [Success]),
            failures = List.Select(results, each not [Success]),
            
            combinedData = if List.IsEmpty(successes) then
                #table({}, {})
            else
                Table.Combine(List.Transform(successes, each [Data]))
        in
            [
                BatchIndex = batchIndex,
                SuccessCount = List.Count(successes),
                FailureCount = List.Count(failures),
                Data = combinedData,
                Failures = failures
            ],
    
    // Process all batches
    BatchResults = List.Transform(
        {0..List.Count(Batches)-1},
        each ProcessBatch(Batches{_}, _)
    ),
    
    // Combine successful data
    AllData = Table.Combine(
        List.Select(
            List.Transform(BatchResults, each [Data]),
            each not Table.IsEmpty(_)
        )
    ),
    
    // Collect all failures
    AllFailures = List.Combine(
        List.Transform(
            List.Select(BatchResults, each [FailureCount] > 0),
            each [Failures]
        )
    ),
    
    // Check error threshold
    TotalFailures = List.Count(AllFailures),
    ThresholdExceeded = TotalFailures > MaxErrors,
    
    // Log warning if threshold exceeded
    Warning = if ThresholdExceeded then
        "Warning: " & Text.From(TotalFailures) & " files failed (threshold: " & Text.From(MaxErrors) & ")"
    else
        null
in
    AllData
```

## Pattern 4: Retry Logic for Transient Errors

### Retry Failed Files

```m
let
    Source = Folder.Files("C:\Data\MonthlyReports"),
    ExcelFiles = Table.SelectRows(Source, each [Extension] = ".xlsx"),
    
    // Function with retry
    ProcessWithRetry = (fileBinary as binary, fileName as text, maxRetries as number) as table =>
        let
            RetryLoop = List.Generate(
                () => [Attempt = 1, Result = null, Error = null],
                each [Attempt] <= maxRetries and [Result] = null,
                each 
                    let
                        nextTry = try
                            let
                                Workbook = Excel.Workbook(fileBinary, null, true),
                                Sheet = Workbook{[Item="Sheet1",Kind="Sheet"]}[Data],
                                Promoted = Table.PromoteHeaders(Sheet)
                            in
                                [Result = Promoted, Error = null]
                        otherwise
                            [Result = null, Error = "Attempt " & Text.From([Attempt]) & " failed"]
                    in
                        [Attempt = [Attempt] + 1, Result = nextTry[Result], Error = nextTry[Error]],
                each [Result]
            ),
            
            // Get first successful result or null
            SuccessfulResult = List.Select(RetryLoop, each _ <> null),
            
            FinalResult = if List.IsEmpty(SuccessfulResult) then
                // All retries failed
                #table(
                    {"Error", "FileName"},
                    {{"All retries failed", fileName}}
                )
            else
                SuccessfulResult{0}
        in
            FinalResult,
    
    // Apply with retry
    WithRetry = Table.AddColumn(
        ExcelFiles,
        "Data",
        each ProcessWithRetry([Content], [Name], 3),
        type table
    ),
    
    // Separate good and bad
    GoodFiles = Table.SelectRows(
        WithRetry, 
        each not Table.HasColumns([Data], {"Error"})
    ),
    
    BadFiles = Table.SelectRows(
        WithRetry, 
        each Table.HasColumns([Data], {"Error"})
    ),
    
    // Combine good data
    Combined = if Table.IsEmpty(GoodFiles) then
        #table({}, {})
    else
        Table.Combine(GoodFiles[Data])
in
    Combined
```

## Pattern 5: Custom Error Table with Detailed Information

### Comprehensive Error Logging

```m
let
    Source = Folder.Files("C:\Data\MonthlyReports"),
    ExcelFiles = Table.SelectRows(Source, each [Extension] = ".xlsx"),
    
    // Function to process with detailed error capture
    ProcessFile = (file as record) as record =>
        let
            startTime = DateTime.LocalNow(),
            
            result = try
                let
                    Workbook = Excel.Workbook(file[Content], null, true),
                    
                    // Check if expected sheet exists
                    SheetExists = List.Contains(
                        Table.Column(Workbook, "Name"),
                        "Sheet1"
                    ),
                    
                    Sheet = if SheetExists then
                        Workbook{[Item="Sheet1",Kind="Sheet"]}[Data]
                    else
                        Workbook{[Kind="Sheet"]}[0][Data],  // Fallback
                    
                    Promoted = Table.PromoteHeaders(Sheet),
                    
                    // Get row count
                    RowCount = Table.RowCount(Promoted)
                in
                    [
                        Success = true,
                        Data = Promoted,
                        RowCount = RowCount,
                        ErrorMessage = null
                    ]
            otherwise
                let
                    errorMsg = Diagnostics.TraceValue()
                in
                    [
                        Success = false,
                        Data = null,
                        RowCount = 0,
                        ErrorMessage = errorMsg
                    ],
            
            endTime = DateTime.LocalNow(),
            duration = Duration.TotalSeconds(endTime - startTime)
        in
            [
                FileName = file[Name],
                FilePath = file[Folder Path] & file[Name],
                FileSize = file[Size],
                DateModified = file[Date modified],
                ProcessingStart = startTime,
                ProcessingEnd = endTime,
                DurationSeconds = duration,
                Success = result[Success],
                RowCount = result[RowCount],
                ErrorMessage = result[ErrorMessage]
            ],
    
    // Process all files
    Results = Table.AddColumn(ExcelFiles, "ProcessingResult", each ProcessFile(_)),
    ResultRecords = Results[ProcessingResult],
    
    // Split into success and failure
    SuccessRecords = List.Select(ResultRecords, each [Success]),
    FailureRecords = List.Select(ResultRecords, each not [Success]),
    
    // Create success data table
    SuccessData = if List.IsEmpty(SuccessRecords) then
        #table({}, {})
    else
        let
            // Extract data from successful files
            DataTables = List.Transform(
                SuccessRecords,
                (r) => r[Data]
            ),
            Combined = Table.Combine(DataTables)
        in
            Combined,
    
    // Create error log table
    ErrorLog = if List.IsEmpty(FailureRecords) then
        #table(
            {"FileName", "FilePath", "FileSize", "ErrorMessage", "Timestamp"},
            {}
        )
    else
        Table.FromRecords(
            List.Transform(
                FailureRecords,
                (r) => [
                    FileName = r[FileName],
                    FilePath = r[FilePath],
                    FileSize = r[FileSize],
                    ErrorMessage = r[ErrorMessage],
                    Timestamp = DateTime.LocalNow()
                ]
            )
        ),
    
    // Create processing summary
    Summary = [
        TotalFiles = List.Count(ResultRecords),
        Successful = List.Count(SuccessRecords),
        Failed = List.Count(FailureRecords),
        SuccessRate = List.Count(SuccessRecords) / List.Count(ResultRecords),
        TotalRows = List.Sum(List.Transform(SuccessRecords, each [RowCount]))
    ]
in
    SuccessData  // Main output
    // ErrorLog could be loaded as a separate table
    // Summary could be used for monitoring
```

## Pattern 6: Folder Structure with Error Isolation

### Process Subfolders Independently

```m
let
    RootFolder = "C:\Data\RegionalReports",
    
    // Get all subfolders
    GetSubfolders = (folderPath as text) as list =>
        let
            folder = Folder.Contents(folderPath),
            subfolders = Table.SelectRows(folder, each [Attributes]?[Kind]? = "Folder"),
            folderPaths = Table.TransformColumns(subfolders, {{"Folder Path", each _, type text}})[Folder Path]
        in
            folderPaths,
    
    Subfolders = GetSubfolders(RootFolder),
    
    // Function to process a single folder
    ProcessFolder = (folderPath as text) as record =>
        let
            // Get Excel files in this folder
            folder = Folder.Files(folderPath),
            excelFiles = Table.SelectRows(folder, each [Extension] = ".xlsx"),
            
            // Process each file in this folder
            ProcessFile = (fileBinary as binary, fileName as text) as table =>
                try
                    let
                        Workbook = Excel.Workbook(fileBinary, null, true),
                        Sheet = Workbook{[Item="Sheet1",Kind="Sheet"]}[Data],
                        Promoted = Table.PromoteHeaders(Sheet)
                    in
                        Promoted
                otherwise
                    #table({"Error"}, {{"Failed: " & fileName}}),
            
            // Apply to all files
            withData = Table.AddColumn(
                excelFiles,
                "Data",
                each ProcessFile([Content], [Name]),
                type table
            ),
            
            // Separate good and bad
            goodData = Table.SelectRows(
                withData,
                each not Table.ColumnNames([Data]) = {"Error"}
            ),
            
            badFiles = Table.SelectRows(
                withData,
                each Table.ColumnNames([Data]) = {"Error"}
            ),
            
            // Combine good data
            combined = if Table.IsEmpty(goodData) then
                #table({}, {})
            else
                Table.Combine(goodData[Data])
        in
            [
                Folder = folderPath,
                Success = not Table.IsEmpty(goodData),
                FileCount = Table.RowCount(excelFiles),
                SuccessCount = Table.RowCount(goodData),
                FailureCount = Table.RowCount(badFiles),
                Data = combined,
                Failures = badFiles
            ],
    
    // Process all folders
    FolderResults = List.Transform(Subfolders, each ProcessFolder(_)),
    
    // Combine all successful data
    AllData = Table.Combine(
        List.Select(
            List.Transform(FolderResults, each [Data]),
            each not Table.IsEmpty(_)
        )
    ),
    
    // Summary by folder
    FolderSummary = Table.FromRecords(
        List.Transform(
            FolderResults,
            each [
                Folder = [Folder],
                Files = [FileCount],
                Successful = [SuccessCount],
                Failed = [FailureCount]
            ]
        )
    )
in
    AllData
```

## Pattern 7: Configuration-Driven Error Handling

### Flexible Error Handling Based on File Types

```m
let
    // Configuration table
    FileTypeConfig = #table(
        {"Extension", "SheetName", "RequiredColumns", "Critical"},
        {
            {".xlsx", "SalesData", {"Date", "Amount"}, true},
            {".xls", "Sheet1", {"ID", "Value"}, false},
            {".csv", null, {"Timestamp", "Reading"}, false}
        }
    ),
    
    Source = Folder.Files("C:\Data\SensorData"),
    
    // Process with config-based error handling
    ProcessFile = (file as record) as table =>
        let
            extension = file[Extension],
            
            // Get config for this file type
            config = Table.SelectRows(
                FileTypeConfig,
                each [Extension] = extension
            ),
            
            // If no config, skip with warning
            hasConfig = not Table.IsEmpty(config),
            
            result = if not hasConfig then
                #table(
                    {"Warning", "File"},
                    {{"Unknown file type", file[Name]}}
                )
            else
                let
                    configRow = config{0},
                    
                    // Try to process
                    processAttempt = try
                        let
                            Workbook = Excel.Workbook(file[Content], null, true),
                            
                            // Find sheet
                            SheetName = configRow[SheetName],
                            Sheet = if SheetName <> null then
                                Workbook{[Item=SheetName,Kind="Sheet"]}[Data]
                            else
                                Workbook{[Kind="Sheet"]}[0][Data],
                            
                            Promoted = Table.PromoteHeaders(Sheet),
                            
                            // Check required columns
                            required = configRow[RequiredColumns],
                            actual = Table.ColumnNames(Promoted),
                            missing = List.RemoveItems(required, actual),
                            
                            hasRequired = List.IsEmpty(missing)
                        in
                            if hasRequired or not configRow[Critical] then
                                Promoted
                            else
                                #table(
                                    {"Error", "File", "MissingColumns"},
                                    {{"Missing required columns", file[Name], Text.Combine(missing, ", ")}}
                                )
                    otherwise
                        #table(
                            {"Error", "File"},
                            {{"Processing failed", file[Name]}}
                        )
                in
                    processAttempt
        in
            result,
    
    // Apply to all files
    Results = Table.AddColumn(
        Source,
        "Data",
        each ProcessFile(_),
        type table
    ),
    
    // Separate by outcome
    Warnings = Table.SelectRows(
        Results,
        each Table.ColumnNames([Data]) = {"Warning", "File"}
    ),
    
    Errors = Table.SelectRows(
        Results,
        each Table.ColumnNames([Data]) = {"Error", "File"} or 
             Table.ColumnNames([Data]) = {"Error", "File", "MissingColumns"}
    ),
    
    GoodData = Table.SelectRows(
        Results,
        each not (Table.ColumnNames([Data]) = {"Warning", "File"} or
                 Table.ColumnNames([Data]) = {"Error", "File"} or
                 Table.ColumnNames([Data]) = {"Error", "File", "MissingColumns"})
    ),
    
    // Combine good data
    Combined = if Table.IsEmpty(GoodData) then
        #table({}, {})
    else
        Table.Combine(GoodData[Data])
in
    Combined
```

## Performance Considerations

### Error Handling Overhead

```m
// Balance error handling with performance
let
    Source = Folder.Files("C:\Data\LargeFolder"),
    
    // Quick validation before full processing
    ValidateFile = (fileBinary as binary) as logical =>
        try
            let
                // Just check if it's a valid Excel file (first few bytes)
                header = Binary.Range(fileBinary, 0, 2),
                headerList = Binary.ToList(header)
            in
                headerList = {0xD0, 0xCF} or headerList = {0x50, 0x4B}  // XLS or XLSX signature
        otherwise
            false,
    
    // Quick validation pass
    WithValidation = Table.AddColumn(
        Source,
        "IsValid",
        each ValidateFile([Content]),
        type logical
    ),
    
    // Filter invalid files first (fast)
    ValidFiles = Table.SelectRows(WithValidation, each [IsValid]),
    InvalidFiles = Table.SelectRows(WithValidation, each not [IsValid]),
    
    // Only process valid files with full error handling
    ProcessValidFile = (fileBinary as binary, fileName as text) as table =>
        try
            Excel.Workbook(fileBinary, null, true){[Kind="Sheet"]}[0][Data]
        otherwise
            #table({"Error", "File"}, {{"Unexpected error", fileName}}),
    
    // Apply to valid files only
    Processed = Table.AddColumn(
        ValidFiles,
        "Data",
        each ProcessValidFile([Content], [Name]),
        type table
    ),
    
    // Combine results
    Result = if Table.IsEmpty(Processed) then
        #table({}, {})
    else
        Table.Combine(Processed[Data])
in
    Result
```

## Best Practices Summary

### Do's
✓ **Isolate per-file errors** - One file failure shouldn't stop others
✓ **Log detailed error info** - File name, path, error type, timestamp
✓ **Use try-otherwise per file** - Not around the entire loop
✓ **Separate successes and failures** - Keep good data flowing
✓ **Set error thresholds** - Stop if too many files fail
✓ **Provide fallback data** - Empty tables or error markers
✓ **Monitor error patterns** - Track which files consistently fail

### Don'ts
✗ **Don't wrap entire loop in try** - Loses granular error info
✗ **Don't ignore errors silently** - Know what failed and why
✗ **Don't assume all files same format** - Handle variations
✗ **Don't let one bad file block refresh** - Critical for scheduled refreshes
✗ **Don't forget to log** - Errors need investigation

## Summary

**Key Points for Error Handling in Folder Loops:**

| Strategy | Implementation |
|----------|----------------|
| **Per-file isolation** | Try-otherwise per file, not per loop |
| **Detailed logging** | Capture file name, path, error, timestamp |
| **Error tables** | Return error structure instead of failing |
| **Batch processing** | Process in groups with thresholds |
| **Retry logic** | Handle transient failures |
| **Validation first** | Quick checks before full processing |
| **Separate outputs** | Keep good data, log bad files |

**The Golden Rule:**
> In folder loops, the failure of one file should never cause the failure of all files. Always process each file in isolation, capture detailed error information, and ensure your refresh completes even when some files fail.
